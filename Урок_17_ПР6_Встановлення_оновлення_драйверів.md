## 1. Мета роботи

- Навчитися відкривати та працювати з Диспетчером пристроїв (Device Manager).
- Переглядати інформацію про встановлені пристрої та їхні драйвери.
- Оновлювати драйвери через Диспетчер пристроїв та Windows Update.
- Знаходити драйвери на офіційних сайтах виробників обладнання.
- Використовувати команди `driverquery` та `pnputil` у командному рядку.
- Створювати точку відновлення системи перед оновленням драйверів.
- Ідентифікувати невідомі пристрої за апаратним ID (VEN/DEV).

---

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Доступ до мережі Інтернет (для пошуку драйверів на сайтах виробників).
- Командний рядок (cmd.exe) з правами адміністратора (для окремих завдань).

---

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не встановлюйте та не видаляйте драйвери без дозволу викладача** — це може призвести до непрацездатності обладнання.
4. Не завантажуйте файли з невідомих та підозрілих сайтів.
5. Не вимикайте та не перемикайте критичні пристрої (мережевий адаптер, дисковий контролер) без вказівки викладача.
6. Після завершення роботи приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Що таке драйвер

**Драйвер (driver)** — це спеціальна програма, яка забезпечує взаємодію операційної системи з апаратним забезпеченням комп'ютера. Без драйвера ОС "не знає", як керувати конкретним пристроєм.

**Приклади пристроїв, яким потрібні драйвери:**

| Пристрій | Приклад драйвера |
|----------|-----------------|
| Відеокарта (GPU) | NVIDIA GeForce Driver, AMD Radeon Software, Intel Graphics Driver |
| Мережевий адаптер | Realtek Ethernet Driver, Intel Wi-Fi Driver |
| Звукова карта | Realtek HD Audio Driver |
| Принтер | HP Universal Print Driver |
| Веб-камера | Драйвер від виробника або вбудований у Windows |

**Звідки беруться драйвери:**
- **Вбудовані у Windows** — при встановленні ОС автоматично встановлює базові драйвери для більшості пристроїв.
- **Windows Update** — Microsoft розповсюджує оновлені драйвери через центр оновлень.
- **Сайт виробника обладнання** — найсвіжіші версії драйверів (Intel, NVIDIA, AMD, Realtek тощо).
- **Диск або флешка з драйверами** — часто додається до материнської плати, відеокарти та інших пристроїв.

### 4.2. Де зберігаються драйвери у Windows

| Розташування | Опис |
|-------------|------|
| `C:\Windows\System32\drivers\` | Основна папка з файлами драйверів (формат `.sys`). |
| `C:\Windows\System32\DriverStore\` | Сховище драйверів (Driver Store) — тут Windows зберігає всі доступні пакети драйверів. |
| `C:\Windows\INF\` | Файли `.inf` — інформаційні файли, що описують, як встановлювати драйвери. |

### 4.3. Диспетчер пристроїв (Device Manager)

**Диспетчер пристроїв** — це вбудований інструмент Windows для перегляду та управління апаратними пристроями комп'ютера та їхніми драйверами.

**Як відкрити Диспетчер пристроїв:**

| Спосіб | Дія |
|--------|-----|
| Команда "Виконати" | `Win + R` → введіть `devmgmt.msc` → Enter |
| Пошук | `Win + S` → введіть "Диспетчер пристроїв" → вибрати результат |
| Контекстне меню "Пуск" | Натисніть правою кнопкою на кнопку "Пуск" (або `Win + X`) → "Диспетчер пристроїв" |

**Що можна побачити в Диспетчері пристроїв:**
- Усі апаратні пристрої комп'ютера, згруповані за категоріями (відеоадаптери, мережеві адаптери, звукові пристрої тощо).
- Стан кожного пристрою: працює нормально, вимкнений, має проблеми.
- Версію, дату та постачальника драйвера для кожного пристрою.

**Позначення стану пристроїв:**

| Значок | Значення |
|--------|----------|
| Жовтий трикутник з `!` | Пристрій має проблему (драйвер відсутній, конфлікт, помилка). |
| Червоний хрест `X` | Пристрій вимкнений. |
| Стрілка вниз | Пристрій вимкнений користувачем. |
| Знак питання `?` | Невідомий пристрій (Windows не може визначити тип). |
| Без позначок | Пристрій працює нормально. |

### 4.4. Команди CMD для роботи з драйверами

Для виконання деяких команд потрібно запустити командний рядок **від імені адміністратора**: `Win + S` → введіть "cmd" → натисніть "Запуск від імені адміністратора".

#### Команда `driverquery`

Виводить список усіх встановлених драйверів у системі.

| Команда | Опис |
|---------|------|
| `driverquery` | Показати список усіх драйверів (стислий вигляд). |
| `driverquery /v` | Показати детальну інформацію (версія, стан, шлях до файлу). |
| `driverquery /fo list` | Показати у форматі списку (зручніше для читання). |
| `driverquery /fo csv > C:\drivers.csv` | Експортувати список драйверів у CSV-файл. |
| `driverquery /fo table` | Показати у табличному форматі (за замовчуванням). |
| `driverquery /si` | Показати інформацію про підписані драйвери. |

#### Команда `pnputil`

Розширений інструмент для управління драйверами (потребує прав адміністратора).

| Команда | Опис |
|---------|------|
| `pnputil /enum-drivers` | Показати список усіх сторонніх (не Microsoft) драйверів. |
| `pnputil /enum-devices` | Показати список усіх пристроїв у системі. |
| `pnputil /enum-devices /problem` | Показати пристрої з проблемами. |
| `pnputil /scan-devices` | Сканувати систему на нові пристрої (аналог "Оновити конфігурацію обладнання"). |
| `pnputil /export-driver oem*.inf C:\backup_drivers` | Експортувати пакет драйверів у вказану папку. |
| `pnputil /add-driver C:\driver\driver.inf` | Додати драйвер до сховища драйверів. |
| `pnputil /add-driver C:\driver\driver.inf /install` | Додати та встановити драйвер. |

### 4.5. Безпека при встановленні драйверів

**Правила безпечного встановлення драйверів:**

1. **Завантажуйте драйвери тільки з офіційних джерел:**
   - Сайт виробника пристрою (Intel, NVIDIA, AMD, Realtek, HP, Dell, Lenovo тощо).
   - Центр оновлень Windows (Windows Update).
   - Каталог оновлень Microsoft: [catalog.update.microsoft.com](https://catalog.update.microsoft.com/).
2. **Ніколи не завантажуйте драйвери з:**
   - Невідомих сайтів-агрегаторів драйверів.
   - Торентів та файлообмінників.
   - Сайтів, що пропонують "універсальні" програми для оновлення драйверів (часто містять рекламне та шкідливе ПЗ).
3. **Перед оновленням драйвера створіть точку відновлення** — це дозволить повернути систему до попереднього стану, якщо новий драйвер спричинить проблеми.
4. **Перевіряйте цифровий підпис драйвера** — підписані драйвери пройшли перевірку Microsoft і є безпечнішими.
5. **Не оновлюйте драйвер без потреби** — якщо все працює стабільно, оновлення драйвера може спричинити нові проблеми. Оновлюйте, якщо: є конкретна проблема з пристроєм; вийшло оновлення безпеки; потрібна підтримка нових функцій.

## 5. Практичні завдання

### Завдання 1. Відкрити Диспетчер пристроїв та переглянути категорії пристроїв

1. Натисніть комбінацію клавіш `Win + R`.
2. У вікні "Виконати" введіть команду `devmgmt.msc` та натисніть **Enter**.
3. Відкриється **Диспетчер пристроїв** з переліком категорій.
4. Розгорніть послідовно такі категорії та запишіть у зошит назви пристроїв, які ви бачите:
   - **Відеоадаптери** (Display adapters)
   - **Мережеві адаптери** (Network adapters)
   - **Звукові, ігрові та відеопристрої** (Sound, video and game controllers)
   - **Дискові пристрої** (Disk drives)
   - **Процесори** (Processors)
   - **Контролери USB** (Universal Serial Bus controllers)
5. Зафіксуйте: скільки всього категорій є у вашому Диспетчері пристроїв.

> **Результат:** у зошиті записано назви пристроїв із шести категорій та загальну кількість категорій.

---

### Завдання 2. Дослідити інформацію про відеоадаптер, мережевий адаптер та звукову карту

Для кожного з трьох пристроїв виконайте такі кроки:

1. У Диспетчері пристроїв розгорніть відповідну категорію.
2. Двічі клацніть на назві пристрою, щоб відкрити його **Властивості**.
3. На вкладці **"Загальне" (General)** запишіть:
   - Назву пристрою.
   - Виробника.
   - Стан пристрою (має бути "Пристрій працює нормально").
4. Заповніть таблицю:

| Пристрій | Повна назва | Виробник | Стан |
|----------|-------------|----------|------|
| Відеоадаптер | | | |
| Мережевий адаптер | | | |
| Звукова карта | | | |

> **Результат:** заповнена таблиця з інформацією про три пристрої.

---

### Завдання 3. Переглянути властивості драйвера

Для кожного з трьох пристроїв із завдання 2:

1. У вікні Властивостей пристрою перейдіть на вкладку **"Драйвер" (Driver)**.
2. Запишіть таку інформацію:
   - **Постачальник драйвера** (Driver Provider) — хто розробив драйвер (наприклад, Microsoft, Intel, NVIDIA).
   - **Дата драйвера** (Driver Date).
   - **Версія драйвера** (Driver Version).
   - **Цифровий підпис** (Digital Signer).
3. Зверніть увагу на кнопки, доступні на цій вкладці:
   - **"Відомості про драйвер" (Driver Details)** — показує файли, з яких складається драйвер.
   - **"Оновити драйвер" (Update Driver)** — запускає процес оновлення.
   - **"Відкотити драйвер" (Roll Back Driver)** — повертає попередню версію драйвера (може бути неактивною).
   - **"Вимкнути пристрій" (Disable Device)** — тимчасово вимикає пристрій.
   - **"Видалити пристрій" (Uninstall Device)** — видаляє драйвер.
4. Заповніть таблицю:

| Пристрій | Постачальник | Дата | Версія | Цифровий підпис |
|----------|-------------|------|--------|----------------|
| Відеоадаптер | | | | |
| Мережевий адаптер | | | | |
| Звукова карта | | | | |

> **Результат:** заповнена таблиця з інформацією про драйвери трьох пристроїв.

---

### Завдання 4. Перевірити наявність оновлень драйверів через Windows Update

1. Натисніть `Win + I`, щоб відкрити **Параметри (Settings)**.
2. Перейдіть до розділу **"Windows Update"** (у Windows 10: "Оновлення та безпека" → "Windows Update").
3. Натисніть **"Перевірити наявність оновлень" (Check for updates)**.
4. Після завершення перевірки перейдіть до **"Додаткові параметри" (Advanced options)** → **"Необов'язкові оновлення" (Optional updates)**.
5. Якщо є розділ **"Оновлення драйверів" (Driver updates)** — розгорніть його та запишіть у зошит:
   - Які драйвери пропонуються до оновлення.
   - Для яких пристроїв вони призначені.
6. **Не встановлюйте оновлення без дозволу викладача!** Лише перегляньте та запишіть.

> **Результат:** у зошиті зафіксовано список доступних оновлень драйверів (або запис "оновлення не знайдено").

---

### Завдання 5. Оновити драйвер через Диспетчер пристроїв

> **Увага!** Виконуйте це завдання лише з дозволу та під наглядом викладача.

1. Відкрийте **Диспетчер пристроїв** (`Win + R` → `devmgmt.msc`).
2. Розгорніть категорію, вказану викладачем (наприклад, "Мережеві адаптери").
3. Натисніть правою кнопкою миші на пристрої та оберіть **"Оновити драйвер" (Update driver)**.
4. У вікні, що з'явилося, оберіть **"Автоматичний пошук драйверів" (Search automatically for drivers)**.
5. Дочекайтеся завершення пошуку. Можливі результати:
   - **"Найкращі драйвери для цього пристрою вже встановлені"** — оновлення не потрібне.
   - **Знайдено оновлення** — Windows запропонує встановити новий драйвер.
6. Запишіть у зошит результат пошуку.

> **Результат:** у зошиті записано результат спроби оновлення драйвера.

---

### Завдання 6. Знайти драйвер на сайті виробника

У цьому завданні ви **не завантажуєте** драйвер, а лише знаходите потрібну сторінку завантаження.

1. На основі інформації з Завдання 2 визначте виробника вашого відеоадаптера (Intel, NVIDIA або AMD).
2. Відкрийте у браузері офіційний сайт виробника:
   - **Intel:** [https://www.intel.com/content/www/us/en/support.html](https://www.intel.com/content/www/us/en/support.html)
   - **NVIDIA:** [https://www.nvidia.com/download/index.aspx](https://www.nvidia.com/download/index.aspx)
   - **AMD:** [https://www.amd.com/en/support](https://www.amd.com/en/support)
   - **Realtek** (звук, мережа): [https://www.realtek.com/downloads](https://www.realtek.com/downloads)
3. Використовуючи назву вашого пристрою та версію Windows, знайдіть сторінку завантаження відповідного драйвера.
4. Запишіть у зошит:
   - URL-адресу сторінки завантаження.
   - Версію драйвера, яку пропонує сайт.
   - Розмір файлу.
   - Дату публікації.
5. Порівняйте версію драйвера на сайті з тією, що встановлена у вашій системі (дані з Завдання 3).

> **Результат:** у зошиті записано інформацію про знайдений драйвер та порівняння версій.

---

### Завдання 7. Експортувати список драйверів через командний рядок

1. Натисніть `Win + S`, введіть **cmd**.
2. Натисніть **"Запуск від імені адміністратора" (Run as administrator)** та підтвердіть запит UAC.
3. Виконайте послідовно такі команди та зафіксуйте результати:

**Крок 1.** Перегляд списку драйверів у табличному форматі:
```
driverquery
```
Запишіть: скільки приблизно рядків з'явилося (прокрутіть список).

**Крок 2.** Перегляд детальної інформації:
```
driverquery /v /fo list | more
```
Знайдіть у списку будь-який знайомий драйвер і запишіть його назву, тип, стан (Running/Stopped) та шлях.

**Крок 3.** Експорт у CSV-файл:
```
driverquery /fo csv > C:\Users\%USERNAME%\Desktop\drivers_list.csv
```
Перевірте, що файл `drivers_list.csv` з'явився на робочому столі. Відкрийте його подвійним клацанням (відкриється в Excel або Блокноті).

**Крок 4.** Перегляд сторонніх драйверів за допомогою `pnputil`:
```
pnputil /enum-drivers
```
Запишіть кількість знайдених сторонніх (OEM) драйверів.

> **Результат:** на робочому столі створено файл `drivers_list.csv`, у зошиті записано кількість драйверів та приклад детальної інформації.

---

### Завдання 8. Створити точку відновлення системи

1. Натисніть `Win + S`, введіть **"Створення точки відновлення"** (або "Create a restore point") та оберіть відповідний результат.
2. Відкриється вікно **"Властивості системи" (System Properties)** на вкладці **"Захист системи" (System Protection)**.
3. Перевірте стовпець **"Захист"** для диска C: — він має бути **"Увімкнено" (On)**.
   - Якщо вимкнено — натисніть **"Налаштувати" (Configure)** → оберіть **"Увімкнути захист системи"** → встановіть повзунок на 5-10% → **OK**.
4. Натисніть кнопку **"Створити..." (Create...)**.
5. Введіть опис: `Перед оновленням драйверів — ПР6` та натисніть **"Створити"**.
6. Дочекайтеся повідомлення **"Точку відновлення створено успішно"**.
7. Запишіть у зошит дату та час створення точки відновлення.

> **Результат:** створено точку відновлення системи, у зошиті зафіксовано дату та час.

---

### Завдання 9. Вимкнути та увімкнути пристрій через Диспетчер пристроїв

> **Увага!** Виконуйте лише з тим пристроєм, який вкаже викладач (наприклад, другий мережевий адаптер або USB-контролер). **Не вимикайте** основний мережевий адаптер, відеоадаптер або дисковий контролер!

1. Відкрийте **Диспетчер пристроїв**.
2. Знайдіть пристрій, вказаний викладачем.
3. Натисніть правою кнопкою миші на пристрої → оберіть **"Вимкнути пристрій" (Disable device)**.
4. У вікні підтвердження натисніть **"Так" (Yes)**.
5. Зверніть увагу: біля назви пристрою з'явилася **стрілка вниз** — пристрій вимкнено.
6. Запишіть у зошит, який значок з'явився біля вимкненого пристрою.
7. Тепер натисніть правою кнопкою миші на цьому ж пристрої → оберіть **"Увімкнути пристрій" (Enable device)**.
8. Пристрій знову має працювати — стрілка вниз зникне.
9. Запишіть у зошит: які зміни відбулися у значку пристрою після вимкнення та увімкнення.

> **Результат:** у зошиті описано процес та візуальні зміни при вимкненні/увімкненні пристрою.

---

### Завдання 10. Ідентифікувати "невідомий пристрій" за апаратним ID

> Якщо у вашому Диспетчері пристроїв немає невідомих пристроїв, виконайте це завдання для будь-якого пристрою (наприклад, відеоадаптера).

1. У Диспетчері пристроїв знайдіть пристрій зі значком **жовтий трикутник** або **знак питання** (категорія "Інші пристрої"). Якщо таких немає — оберіть будь-який пристрій за вказівкою викладача.
2. Двічі клацніть на пристрої, щоб відкрити **Властивості**.
3. Перейдіть на вкладку **"Відомості" (Details)**.
4. У випадаючому списку **"Властивість" (Property)** оберіть **"Ідентифікатори обладнання" (Hardware Ids)**.
5. У полі **"Значення" (Value)** ви побачите рядки вигляду:
   ```
   PCI\VEN_8086&DEV_1916&SUBSYS_22161043&REV_07
   ```
6. Запишіть у зошит значення **VEN_** (Vendor ID) та **DEV_** (Device ID). У цьому прикладі: VEN = `8086`, DEV = `1916`.
7. Відкрийте у браузері сайт **[https://devicehunt.com](https://devicehunt.com)** або **[https://pcilookup.com](https://pcilookup.com)**.
8. Введіть Vendor ID та Device ID у відповідні поля пошуку.
9. Запишіть у зошит:
   - Назву виробника (за Vendor ID).
   - Назву пристрою (за Device ID).
10. Порівняйте знайдену інформацію з назвою пристрою у Диспетчері пристроїв.

**Довідка з найпоширеніших Vendor ID:**

| VEN ID | Виробник |
|--------|----------|
| `8086` | Intel |
| `10DE` | NVIDIA |
| `1002` | AMD / ATI |
| `10EC` | Realtek |
| `8087` | Intel (бездротові модулі) |
| `14E4` | Broadcom |

> **Результат:** у зошиті записано апаратний ID пристрою, знайдено виробника та назву за допомогою онлайн-бази.

## 6. Контрольні запитання та завдання для самостійного опрацювання

1. Що таке драйвер і навіщо він потрібен операційній системі? Наведіть приклад ситуації, коли відсутність драйвера призводить до проблем.

2. Назвіть три способи відкрити Диспетчер пристроїв у Windows. Який із них, на вашу думку, найшвидший?

3. Поясніть, що означають такі позначки у Диспетчері пристроїв: жовтий трикутник з `!`, стрілка вниз, знак питання. Як виправити кожну з цих ситуацій?

4. Яку інформацію можна знайти на вкладці "Драйвер" у властивостях пристрою? Чому важливо знати версію та дату драйвера?

5. Чим відрізняється автоматичний пошук драйверів через Диспетчер пристроїв від пошуку оновлень через Windows Update? В яких ситуаціях краще шукати драйвер на сайті виробника?

6. Опишіть крок за кроком, як знайти та встановити драйвер для невідомого пристрою, використовуючи його апаратний ID (VEN/DEV).

7. Поясніть різницю між командами `driverquery` та `pnputil`. Для яких завдань краще використовувати кожну з них?

8. Що таке точка відновлення системи? Чому рекомендується створювати її перед оновленням драйверів? Опишіть порядок дій для відновлення системи, якщо новий драйвер спричинив проблеми.

9. Чому не рекомендується завантажувати драйвери з невідомих сайтів та використовувати "програми для автоматичного оновлення драйверів"? Які ризики це несе?

10. Що таке цифровий підпис драйвера? Як перевірити, чи підписаний драйвер, і навіщо Microsoft вимагає підписування драйверів?

11. Уявіть ситуацію: після оновлення драйвера відеокарти екран став чорним і комп'ютер не завантажується нормально. Опишіть ваші дії для вирішення цієї проблеми (мінімум два способи).

12. Виконайте у командному рядку команду `pnputil /enum-devices /problem` та опишіть результат. Якщо проблемних пристроїв не знайдено — поясніть, що це означає та яка команда використовується для перевірки.
