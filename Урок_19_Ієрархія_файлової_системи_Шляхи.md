## 1. Деревоподібна (ієрархічна) структура файлової системи

### Що таке ієрархічна структура?

Файлова система організована у вигляді **дерева** — структури, де є один головний елемент (корінь), від якого відходять гілки (каталоги), а на кінцях гілок знаходяться листки (файли).

### Аналогія з деревом

Уявіть справжнє дерево, перевернуте догори коренем:

```
        [КОРІНЬ]            <-- корінь дерева (вершина ієрархії)
        /      \
    [гілка1]  [гілка2]      <-- каталоги (папки)
    /    \        \
 [лист] [лист]  [лист]      <-- файли
```

- **Корінь (root)** — найвища точка, звідки починається вся структура. У Windows це диск (`C:\`), у Linux — символ `/`.
- **Гілки** — каталоги (папки), які містять інші каталоги або файли.
- **Листки** — кінцеві файли, які більше нічого не містять всередині себе.
- **Батьківський каталог** — каталог, який містить інший каталог або файл (тобто знаходиться на рівень вище).
- **Дочірній каталог** (підкаталог) — каталог, що знаходиться всередині іншого каталогу.

### Чому саме дерево?

Деревоподібна структура дозволяє:

- **Організовувати** тисячі файлів у логічні групи.
- **Швидко знаходити** потрібну інформацію завдяки чіткій ієрархії.
- **Уникати конфліктів імен** — два файли з однаковою назвою можуть існувати в різних каталогах.
- **Контролювати доступ** — можна встановлювати різні права на різні каталоги.

## 2. Структура файлової системи Windows

### Диски як кореневі елементи

У Windows файлова система починається не з єдиного кореня, а з **логічних дисків**, кожному з яких присвоюється літера латинського алфавіту:

| Диск | Типове призначення |
|------|-------------------|
| `A:\`, `B:\` | Історично — дисководи для дискет (сьогодні практично не використовуються) |
| `C:\` | Системний диск — тут встановлена операційна система Windows |
| `D:\`, `E:\` та інші | Додаткові розділи для даних, оптичні диски, USB-накопичувачі |

### Стандартні системні каталоги

| Каталог | Призначення |
|---------|-------------|
| `C:\Windows` | Основні файли операційної системи: ядро, драйвери, системні бібліотеки |
| `C:\Windows\System32` | Критично важливі системні файли: DLL-бібліотеки, виконувані файли, драйвери |
| `C:\Program Files` | Програми, встановлені для 64-розрядних систем |
| `C:\Program Files (x86)` | Програми для 32-розрядних систем (на 64-розрядній ОС) |
| `C:\ProgramData` | Спільні дані програм для всіх користувачів (прихований каталог) |
| `C:\Users` | Профілі всіх користувачів системи |
| `C:\Windows\Temp` | Тимчасові файли операційної системи |

### Профіль користувача (`C:\Users\Ім'я`)

Кожен користувач має власну папку з особистими каталогами:

| Каталог | Призначення |
|---------|-------------|
| `Desktop` | Файли та ярлики на робочому столі |
| `Documents` | Документи користувача |
| `Downloads` | Завантажені файли з Інтернету |
| `Pictures` | Зображення та фотографії |
| `Music` | Музичні файли |
| `Videos` | Відеофайли |
| `AppData` | Налаштування та дані програм (прихований каталог) |

Каталог `AppData` містить три підкаталоги:
- **Roaming** — дані, які можуть синхронізуватися між комп'ютерами в мережі.
- **Local** — дані, що зберігаються лише на цьому комп'ютері.
- **LocalLow** — дані програм з обмеженими правами доступу.

### Текстова схема дерева Windows

```
C:\
├── Windows\
│   ├── System32\
│   │   ├── drivers\
│   │   │   └── etc\
│   │   │       └── hosts
│   │   ├── cmd.exe
│   │   └── notepad.exe
│   ├── Temp\
│   └── Fonts\
├── Program Files\
│   ├── Google\
│   │   └── Chrome\
│   └── Microsoft Office\
├── Program Files (x86)\
│   └── ...
├── ProgramData\
├── Users\
│   ├── Student\
│   │   ├── Desktop\
│   │   ├── Documents\
│   │   │   ├── реферат.docx
│   │   │   └── завдання.pdf
│   │   ├── Downloads\
│   │   ├── Pictures\
│   │   ├── Music\
│   │   ├── Videos\
│   │   └── AppData\
│   │       ├── Roaming\
│   │       ├── Local\
│   │       └── LocalLow\
│   └── Public\
└── Temp\
```

## 3. Структура файлової системи Linux (FHS)

### Що таке FHS?

**FHS (Filesystem Hierarchy Standard)** — це стандарт ієрархії файлової системи, який визначає структуру каталогів в Unix-подібних операційних системах (Linux, macOS тощо). Поточна версія — FHS 3.0.

На відміну від Windows, у Linux існує **єдиний корінь** файлової системи — символ `/` (косою рискою). Усі файли та пристрої знаходяться всередині цього кореня, навіть якщо фізично розташовані на різних дисках.

### Основні каталоги Linux

| Каталог | Призначення |
|---------|-------------|
| `/` | Кореневий каталог — вершина всієї файлової системи |
| `/home` | Домашні каталоги звичайних користувачів (аналог `C:\Users`) |
| `/root` | Домашній каталог суперкористувача (адміністратора root) |
| `/etc` | Файли конфігурації системи та програм |
| `/var` | Змінювані дані: логи, кеші, черги друку, бази даних |
| `/tmp` | Тимчасові файли (очищуються при перезавантаженні) |
| `/usr` | Програми та дані користувача (утиліти, бібліотеки, документація) |
| `/usr/bin` | Виконувані файли звичайних програм |
| `/usr/local` | Програми, встановлені вручну адміністратором |
| `/bin` | Основні системні команди (ls, cp, mv, cat) |
| `/sbin` | Системні команди для адміністрування (fdisk, mkfs, iptables) |
| `/dev` | Файли пристроїв (жорсткі диски, USB, клавіатура, миша) |
| `/mnt` | Точка монтування для постійно підключених дисків |
| `/media` | Точка монтування для знімних носіїв (USB-флешки, CD/DVD) |
| `/opt` | Додаткове (опціональне) програмне забезпечення |
| `/boot` | Файли завантажувача системи (ядро Linux, GRUB) |
| `/proc` | Віртуальна файлова система з інформацією про процеси |
| `/lib` | Системні бібліотеки, необхідні для роботи програм із /bin та /sbin |

### Текстова схема дерева Linux

```
/ (корінь)
├── home/
│   ├── student/
│   │   ├── Documents/
│   │   │   ├── referat.odt
│   │   │   └── zavdannya.pdf
│   │   ├── Downloads/
│   │   ├── Pictures/
│   │   ├── Music/
│   │   ├── Videos/
│   │   └── Desktop/
│   └── teacher/
│       └── ...
├── etc/
│   ├── passwd
│   ├── hostname
│   └── fstab
├── var/
│   ├── log/
│   │   └── syslog
│   └── tmp/
├── tmp/
├── usr/
│   ├── bin/
│   ├── lib/
│   ├── local/
│   └── share/
├── bin/
│   ├── ls
│   ├── cp
│   └── mv
├── sbin/
├── dev/
│   ├── sda        (жорсткий диск)
│   └── usb0
├── mnt/
├── media/
│   └── usb-flash/
├── opt/
├── boot/
│   ├── vmlinuz    (ядро Linux)
│   └── grub/
├── root/
├── proc/
└── lib/
```

## 4. Абсолютний шлях

### Визначення

**Абсолютний шлях** (absolute path) — це повна адреса файлу або каталогу, яка починається від кореня файлової системи. Абсолютний шлях завжди однозначно вказує на розташування файлу, незалежно від того, в якому каталозі ви зараз перебуваєте.

### Ознаки абсолютного шляху

- У **Windows**: починається з літери диска та зворотної косої риски — `C:\`
- У **Linux**: починається з прямої косої риски — `/`

### Приклади

**Windows:**
```
C:\Users\Student\Documents\file.txt
C:\Program Files\Google\Chrome\chrome.exe
D:\Photos\vacation\photo001.jpg
C:\Windows\System32\cmd.exe
```

**Linux:**
```
/home/student/documents/file.txt
/etc/passwd
/var/log/syslog
/usr/bin/firefox
```

### Коли використовувати?

Абсолютний шлях використовується, коли потрібна **точність і однозначність**:
- У системних скриптах та конфігураціях
- У програмуванні для вказівки на критичні ресурси
- Коли ви не знаєте, в якому каталозі перебуваєте

## 5. Відносний шлях

### Визначення

**Відносний шлях** (relative path) — це шлях до файлу або каталогу відносно **поточного робочого каталогу**. Він не починається з кореня файлової системи.

### Спеціальні позначення

| Позначення | Значення | Приклад |
|-----------|----------|---------|
| `.` | Поточний каталог | `./file.txt` — файл у поточному каталозі |
| `..` | Батьківський каталог (на рівень вище) | `../file.txt` — файл у каталозі на рівень вище |
| `../..` | На два рівні вище | `../../data/file.txt` |

### Приклади

Припустімо, ви знаходитесь у каталозі `C:\Users\Student\Documents` (Windows) або `/home/student/documents` (Linux):

**Перехід до файлу в поточному каталозі:**
```
.\referat.docx          (Windows)
./referat.odt           (Linux)
```

**Перехід до файлу в сусідньому каталозі (Downloads):**
```
..\Downloads\setup.exe       (Windows)
../Downloads/setup.deb       (Linux)
```
Пояснення: `..` — піднятися на рівень вище (до `Student`), потім зайти в `Downloads`.

**Перехід на два рівні вгору та в інший каталог:**
```
..\..\Public\shared.txt      (Windows)
../../teacher/notes.txt      (Linux)
```

### Візуальний приклад

```
C:\Users\Student\
├── Documents\        <-- ви тут (поточний каталог)
│   ├── referat.docx       шлях: .\referat.docx
│   └── homework\
│       └── task1.txt      шлях: .\homework\task1.txt
├── Downloads\
│   └── setup.exe          шлях: ..\Downloads\setup.exe
└── Desktop\
    └── shortcut.lnk       шлях: ..\Desktop\shortcut.lnk
```

### Коли використовувати?

- Для швидкої навігації в командному рядку
- У веб-розробці (посилання на файли всередині проєкту)
- Коли проєкт може переміщуватися між комп'ютерами

## 6. Роздільник шляхів: Windows vs Linux

### Зворотна коса риска `\` (backslash) — Windows

Windows використовує символ `\` для розділення каталогів у шляху:

```
C:\Users\Student\Documents\file.txt
```

Це історична особливість, що склалася з часів MS-DOS (1981 р.), коли символ `/` вже використовувався для параметрів команд.

### Пряма коса риска `/` (forward slash) — Linux

Linux та інші Unix-подібні системи використовують символ `/`:

```
/home/student/documents/file.txt
```

Цей символ був обраний ще в оригінальній системі UNIX (1969 р.).

### Порівняння

| Характеристика | Windows | Linux |
|---------------|---------|-------|
| Роздільник | `\` (зворотна коса) | `/` (пряма коса) |
| Назва англійською | backslash | forward slash |
| На клавіатурі | праворуч від правого Shift | Shift + клавіша біля правого Shift (на українській розкладці — `.`) |

> **Зверніть увагу:** сучасні версії Windows також розуміють пряму косу риску `/` у багатьох контекстах (наприклад, у PowerShell, браузерах), але офіційним роздільником залишається `\`.

## 7. Порівняльна таблиця Windows vs Linux

| Характеристика | Windows | Linux |
|---------------|---------|-------|
| Кореневий елемент | Літера диска (`C:\`, `D:\`) | Єдиний корінь `/` |
| Роздільник шляхів | `\` (зворотна коса) | `/` (пряма коса) |
| Регістр імен файлів | Не розрізняє (`File.txt` = `file.txt`) | Розрізняє (`File.txt` та `file.txt` — різні файли) |
| Домашній каталог | `C:\Users\Ім'я` | `/home/ім'я` |
| Системні файли | `C:\Windows\System32` | `/bin`, `/sbin`, `/usr/bin` |
| Конфігурації | Реєстр Windows + файли в `AppData` | Текстові файли в `/etc` |
| Тимчасові файли | `C:\Windows\Temp`, `%TEMP%` | `/tmp`, `/var/tmp` |
| Програми | `C:\Program Files` | `/usr/bin`, `/opt`, `/usr/local` |
| Файловий менеджер | Провідник (File Explorer) | Nautilus, Dolphin, Thunar та ін. |
| Командний рядок | CMD, PowerShell | Bash, Zsh, Fish |
| Список файлів | `dir` | `ls` |
| Поточний каталог | `cd` (без аргументів) | `pwd` |
| Стандарт структури | Немає єдиного стандарту | FHS (Filesystem Hierarchy Standard) |
| Монтування дисків | Автоматичне (літери дисків) | Вручну або автоматично (точки монтування) |

## 8. Навігація через командний рядок

### Основні команди

| Дія | Windows (CMD) | Linux (Bash) |
|-----|---------------|-------------|
| Показати поточний каталог | `cd` (без аргументів) | `pwd` |
| Перейти в каталог | `cd шлях` | `cd шлях` |
| Піднятися на рівень вище | `cd ..` | `cd ..` |
| Перейти в корінь диска | `cd \` | `cd /` |
| Перейти в домашній каталог | `cd %HOMEPATH%` | `cd ~` або просто `cd` |
| Показати вміст каталогу | `dir` | `ls` |
| Показати детальний вміст | `dir /a` | `ls -la` |
| Створити каталог | `mkdir назва` | `mkdir назва` |
| Створити вкладені каталоги | `mkdir шлях\до\каталогу` | `mkdir -p шлях/до/каталогу` |
| Видалити порожній каталог | `rmdir назва` | `rmdir назва` |
| Очистити екран | `cls` | `clear` |

### Приклади використання

**Windows (CMD):**
```cmd
C:\> cd Users\Student\Documents
C:\Users\Student\Documents> dir
C:\Users\Student\Documents> mkdir Проєкт
C:\Users\Student\Documents> cd Проєкт
C:\Users\Student\Documents\Проєкт> cd ..
C:\Users\Student\Documents> cd \
C:\>
```

**Linux (Bash):**
```bash
$ pwd
/home/student
$ cd documents
$ ls
referat.odt  zavdannya.pdf
$ mkdir project
$ cd project
$ pwd
/home/student/documents/project
$ cd ../..
$ pwd
/home/student
$ cd /
$ pwd
/
```

### Корисні поради

1. **Автодоповнення (Tab)** — натисніть клавішу `Tab`, щоб автоматично дописати назву каталогу або файлу. Це працює і в Windows, і в Linux.
2. **Історія команд** — натискайте стрілки "вгору"/"вниз" для перегляду попередніх команд.
3. **Перетягування файлу** — у Windows можна перетягнути файл або папку в вікно командного рядка, і повний шлях вставиться автоматично.

## 9. Символічні та жорсткі посилання

### Символічне посилання (Symbolic Link / Symlink)

**Символічне посилання** — це спеціальний файл, який містить шлях до іншого файлу або каталогу. Воно діє як "покажчик" або "ярлик".

**Властивості:**
- Вказує на шлях до цільового файлу (а не на самі дані).
- Може посилатися на файли та каталоги.
- Може працювати між різними дисками та файловими системами.
- Якщо оригінальний файл видалити, посилання стає "битим" (не працює).

**Аналогія:** дорожній знак "Аеропорт -->". Знак вказує напрямок, але якщо аеропорт знесуть, знак залишиться, хоча й стане марним.

**Створення:**
```bash
# Linux
ln -s /home/student/documents/referat.odt ~/Desktop/referat_link.odt

# Windows (CMD з правами адміністратора)
mklink C:\Users\Student\Desktop\link.txt C:\Users\Student\Documents\original.txt
```

### Жорстке посилання (Hard Link)

**Жорстке посилання** — це додаткове ім'я (альтернативна назва) для того самого файлу на диску. Обидва імені вказують безпосередньо на одні й ті самі дані.

**Властивості:**
- Вказує безпосередньо на дані файлу (на його inode у Linux).
- Може створюватися лише для файлів (не для каталогів).
- Працює лише в межах однієї файлової системи (одного диска).
- Якщо оригінальний файл видалити, жорстке посилання продовжує працювати — дані залишаються, доки існує хоча б одне посилання.

**Аналогія:** одна людина з двома номерами телефону. Незалежно від того, на який номер зателефонувати, відповість та сама людина.

**Створення:**
```bash
# Linux
ln /home/student/documents/referat.odt /home/student/backup/referat_copy.odt

# Windows (CMD)
mklink /H C:\Users\Student\backup\link.txt C:\Users\Student\Documents\original.txt
```

### Порівняння

| Характеристика | Символічне посилання | Жорстке посилання |
|---------------|---------------------|-------------------|
| На що вказує | На шлях (адресу) файлу | На дані (inode) файлу |
| Для каталогів | Так | Ні |
| Між різними дисками | Так | Ні |
| Оригінал видалено | Посилання "битий" | Посилання працює |
| Розмір | Мінімальний (лише шлях) | Не займає додаткового місця |

## 11. Глосарій

| Термін | Визначення |
|--------|-----------|
| **Файлова система** | Спосіб організації, зберігання та доступу до файлів на носії інформації |
| **Ієрархія** | Структура з чітким підпорядкуванням елементів від вищого рівня до нижчого |
| **Корінь (root)** | Найвищий рівень файлової системи; у Windows — диск (`C:\`), у Linux — `/` |
| **Каталог (директорія)** | Контейнер для файлів та інших каталогів; синонім слова "папка" |
| **Підкаталог** | Каталог, що знаходиться всередині іншого каталогу |
| **Батьківський каталог** | Каталог, який містить даний каталог або файл (на рівень вище) |
| **Абсолютний шлях** | Повний шлях до файлу від кореня файлової системи |
| **Відносний шлях** | Шлях до файлу відносно поточного робочого каталогу |
| **Поточний каталог (`.`)** | Каталог, у якому користувач перебуває в даний момент |
| **`..` (дві крапки)** | Спеціальне позначення батьківського каталогу |
| **Роздільник шляхів** | Символ, що розділяє назви каталогів у шляху (`\` або `/`) |
| **FHS** | Filesystem Hierarchy Standard — стандарт ієрархії файлової системи Linux |
| **Символічне посилання (symlink)** | Файл-покажчик, що містить шлях до іншого файлу або каталогу |
| **Жорстке посилання (hard link)** | Додаткове ім'я файлу, що вказує безпосередньо на ті самі дані на диску |
| **Inode** | Структура даних у файловій системі Linux, що описує файл (метадані, розташування на диску) |
| **Точка монтування** | Каталог, до якого приєднується файлова система зовнішнього пристрою |
| **CMD (Command Prompt)** | Командний рядок Windows — текстовий інтерфейс для введення команд |
| **Bash** | Командна оболонка Linux — текстовий інтерфейс для введення команд |
| **pwd** | Команда Linux для виведення поточного робочого каталогу (Print Working Directory) |
| **cd** | Команда для зміни поточного каталогу (Change Directory) |
| **ls / dir** | Команди для відображення вмісту каталогу (ls — Linux, dir — Windows) |
| **mkdir** | Команда для створення нового каталогу (Make Directory) |
| **Регістр** | Розрізнення великих та малих літер (Linux розрізняє, Windows — ні) |
