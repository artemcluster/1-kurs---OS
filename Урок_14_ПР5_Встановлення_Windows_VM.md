## 1. Мета роботи

- Зрозуміти поняття віртуалізації та призначення віртуальних машин.
- Навчитися створювати віртуальну машину в Oracle VirtualBox.
- Самостійно встановити операційну систему Windows 10/11 на віртуальну машину.
- Встановити Guest Additions для покращення продуктивності гостьової ОС.
- Навчитися створювати та використовувати знімки (snapshots) стану VM.

---

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з ОС Windows 10/11 (хост-машина).
- Процесор з підтримкою апаратної віртуалізації (Intel VT-x або AMD-V).
- Не менше 8 GB оперативної пам'яті на хост-машині.
- Не менше 60 GB вільного місця на жорсткому диску.
- Встановлена програма Oracle VirtualBox (версія 7.x).
- ISO-образ Windows 10 або Windows 11.

---

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. Не змінюйте налаштування хост-машини (основного комп'ютера) без дозволу викладача.
4. Працюйте лише з віртуальною машиною, яку створюєте самостійно.
5. Не завантажуйте файли з невідомих джерел — використовуйте лише офіційні сайти.
6. Після завершення роботи коректно вимкніть віртуальну машину та приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Що таке віртуальна машина

**Віртуальна машина (Virtual Machine, VM)** — це програмна емуляція фізичного комп'ютера, яка працює всередині вашого реального комп'ютера. По суті, це "комп'ютер у комп'ютері".

Основні поняття:

| Термін | Пояснення |
|--------|-----------|
| **Хост (Host)** | Ваш реальний (фізичний) комп'ютер, на якому запущена програма віртуалізації. |
| **Гість (Guest)** | Операційна система, що працює всередині віртуальної машини. |
| **Гіпервізор** | Програмне забезпечення, що створює та керує віртуальними машинами. Саме гіпервізор розподіляє ресурси хост-машини між віртуальними машинами. |
| **ISO-образ** | Файл-копія оптичного диска (CD/DVD). Містить усі файли, необхідні для встановлення операційної системи. Розширення файлу — `.iso`. |
| **Віртуальний жорсткий диск** | Файл на диску хост-машини, який VM сприймає як свій власний жорсткий диск. |

Віртуальна машина повністю ізольована від хост-системи. Усе, що відбувається всередині VM (встановлення програм, вірусне зараження, випадкове видалення системних файлів), **жодним чином не впливає на основну операційну систему**. Саме це робить VM ідеальним інструментом для безпечного навчання.

### 4.2. Навіщо використовувати VM для навчання

Віртуальні машини мають численні переваги для навчального процесу:

1. **Безпека** — можна експериментувати з ОС без ризику пошкодити основну систему. Навіть якщо ви зламаєте систему на VM, хост-машина залишиться неушкодженою.
2. **Знімки (Snapshots)** — можна зберегти стан VM у будь-який момент та миттєво повернутися до нього. Це як "збереження гри" для операційної системи.
3. **Кілька ОС одночасно** — на одному комп'ютері можна запустити Windows, Linux та інші системи без перезавантаження.
4. **Тренування встановлення ОС** — можна багаторазово встановлювати та перевстановлювати операційну систему для відпрацювання навичок.
5. **Економія обладнання** — не потрібен окремий фізичний комп'ютер для кожної ОС.
6. **Портативність** — файли VM можна копіювати на інший комп'ютер і запускати там.

### 4.3. Популярні гіпервізори

| Гіпервізор | Тип | Ліцензія | Особливості |
|------------|-----|----------|-------------|
| **Oracle VirtualBox** | Тип 2 (на базі ОС) | Безкоштовний (GPLv3) | Кросплатформний (Windows, macOS, Linux). Простий інтерфейс. Ідеальний для навчання. Поточна версія — 7.2.6. |
| **VMware Workstation Player** | Тип 2 | Безкоштовний для особистого використання | Висока продуктивність. Підтримка DirectX 11. Зручний для роботи з Windows-гостями. |
| **VMware Workstation Pro** | Тип 2 | Платний (безкоштовний для персонального використання з 2024 р.) | Розширена функціональність: клонування, групи VM, мережеві конфігурації. |
| **Microsoft Hyper-V** | Тип 1 (bare-metal) | Вбудований у Windows 10/11 Pro та Enterprise | Працює безпосередньо з апаратним забезпеченням (без посередництва хост-ОС). Висока продуктивність, але складніший у налаштуванні. |
| **KVM** | Тип 1 | Безкоштовний (вбудований у ядро Linux) | Стандарт віртуалізації на Linux-серверах. |

**Тип 1 (bare-metal)** — гіпервізор працює безпосередньо на апаратному забезпеченні, без основної ОС. Приклади: Hyper-V, VMware ESXi.

**Тип 2 (hosted)** — гіпервізор працює як звичайна програма поверх основної ОС. Приклади: VirtualBox, VMware Workstation Player.

> У цій практичній роботі ми використовуємо **Oracle VirtualBox**, оскільки він безкоштовний, простий у використанні та доступний на всіх популярних платформах.

### 4.4. Системні вимоги для роботи з VirtualBox

**Мінімальні вимоги до хост-машини:**

| Параметр | Мінімум | Рекомендовано |
|----------|---------|---------------|
| **Процесор** | 64-bit із підтримкою VT-x/AMD-V | 4+ ядра, 2 GHz+ |
| **Оперативна пам'ять** | 8 GB (4 GB для хоста + 4 GB для VM) | 16 GB |
| **Вільне місце на диску** | 50 GB | 100 GB (SSD бажано) |
| **Операційна система хоста** | Windows 10 64-bit і вище | Windows 10/11 64-bit |

**Вимоги гостьової ОС Windows 10:**

| Параметр | Значення |
|----------|----------|
| Процесор | 1 GHz+, 2 ядра |
| RAM | 2 GB (мінімум), 4 GB (рекомендовано) |
| Диск | 32 GB+ |
| Графіка | DirectX 12, WDDM 2.0 |

**Вимоги гостьової ОС Windows 11:**

| Параметр | Значення |
|----------|----------|
| Процесор | 1 GHz+, 2 ядра, 64-bit |
| RAM | 4 GB |
| Диск | 64 GB+ |
| TPM | Версія 2.0 (VirtualBox емулює TPM) |
| Firmware | UEFI з Secure Boot |

> **Примітка:** Підтримка Windows 10 від Microsoft завершилася 14 жовтня 2025 року. Для навчальних цілей ви можете використовувати як Windows 10, так і Windows 11.

## 5. Підготовка до роботи

### 5.1. Завантаження та встановлення VirtualBox

1. Відкрийте браузер та перейдіть на офіційний сайт: **https://www.virtualbox.org**
2. Натисніть **"Downloads"** у верхньому меню.
3. У розділі **"VirtualBox platform packages"** оберіть версію для вашої ОС:
   - **Windows hosts** — якщо ваш основний комп'ютер працює на Windows.
4. Завантажте інсталятор (файл приблизно 100 MB).
5. Запустіть завантажений файл (`VirtualBox-7.x.x-xxxxx-Win.exe`).
6. У вікні інсталятора натискайте **"Next"** на кожному кроці, залишаючи параметри за замовчуванням.
7. Під час встановлення може з'явитися попередження про тимчасове відключення мережі — натисніть **"Yes"**.
8. Натисніть **"Install"**, дочекайтеся завершення, натисніть **"Finish"**.

> **Додатково:** На сторінці Downloads також доступний **Oracle VM VirtualBox Extension Pack**. Він додає підтримку USB 2.0/3.0, шифрування дисків та інші функції. Для базової роботи Extension Pack не є обов'язковим, але рекомендується його встановити: завантажте файл, двічі клікніть на ньому — VirtualBox автоматично запропонує встановлення.

### 5.2. Отримання ISO-образу Windows

**Спосіб 1 — Media Creation Tool (для Windows 10):**
1. Перейдіть на сторінку: **https://www.microsoft.com/software-download/windows10**
2. Натисніть **"Завантажити засіб зараз"** (Download tool now).
3. Запустіть завантажений файл `MediaCreationTool.exe`.
4. Прийміть ліцензійну угоду.
5. Оберіть **"Створити інсталяційний носій для іншого ПК"**.
6. Оберіть мову (Українська або English), випуск (Windows 10) та архітектуру (64-розрядна).
7. Оберіть **"ISO-файл"** та вкажіть місце збереження.
8. Дочекайтеся завершення завантаження (файл ~5-6 GB).

**Спосіб 2 — Пряме завантаження ISO (для Windows 11):**
1. Перейдіть на сторінку: **https://www.microsoft.com/software-download/windows11**
2. У розділі **"Завантажити образ диска Windows 11 (ISO)"** оберіть **"Windows 11 (multi-edition ISO)"**.
3. Натисніть **"Завантажити"**, оберіть мову, натисніть **"Підтвердити"**.
4. Натисніть **"64-bit Download"** та дочекайтеся завершення (файл ~6 GB).

> **Важливо:** Завантажуйте ISO-образи **тільки** з офіційного сайту Microsoft. Образи з торрентів або невідомих сайтів можуть містити шкідливе програмне забезпечення.

## 6. Покрокова інструкція — Створення віртуальної машини

### Крок 1. Запуск VirtualBox та створення нової VM

1. Запустіть **Oracle VirtualBox** (значок на робочому столі або через меню "Пуск").
2. У головному вікні VirtualBox Manager натисніть кнопку **"Створити"** (New) на панелі інструментів.

### Крок 2. Ім'я та операційна система

1. У полі **"Ім'я"** (Name) введіть зрозумілу назву, наприклад: `Windows 10 Навчальна` або `Windows 11 Test`.
2. **"Папка"** (Folder) — залиште за замовчуванням або оберіть папку на диску з достатнім вільним місцем.
3. **"ISO Image"** — натисніть на випадаючий список та оберіть **"Інше..."** (Other). Вкажіть шлях до завантаженого ISO-файлу Windows.
4. VirtualBox автоматично визначить **"Тип"** (Type) — **Microsoft Windows** та **"Версію"** (Version) — **Windows 10 (64-bit)** або **Windows 11 (64-bit)**. Якщо ні — оберіть вручну.
5. Зніміть галочку **"Skip Unattended Installation"** якщо хочете пройти встановлення вручну (рекомендується для навчання).

> **Порада:** Якщо ви хочете побачити всі етапи встановлення вручну — поставте галочку **"Skip Unattended Installation"**. Автоматичне встановлення пропускає багато кроків, які важливо побачити для навчання.

### Крок 3. Виділення оперативної пам'яті та процесора

1. Натисніть **"Далі"** (Next) або перейдіть на вкладку **"Hardware"**.
2. **"Основна пам'ять"** (Base Memory) — за допомогою повзунка виділіть оперативну пам'ять для VM:
   - **Мінімум:** 2048 MB (2 GB) для Windows 10 / 4096 MB (4 GB) для Windows 11.
   - **Рекомендовано:** 4096 MB (4 GB).
   - **Важливо:** Не виділяйте більше половини RAM хост-машини! Якщо у вас 8 GB — виділіть максимум 4 GB.
3. **"Процесори"** (Processors) — виділіть **2 ядра** (мінімум). Якщо процесор хоста має 4+ ядра, можна виділити 2-3.
   - Повзунок не повинен заходити в червону зону!

### Крок 4. Створення віртуального жорсткого диска

1. Натисніть **"Далі"** або перейдіть на вкладку **"Hard Disk"**.
2. Оберіть **"Створити віртуальний жорсткий диск зараз"** (Create a Virtual Hard Disk Now).
3. Вкажіть розмір диска:
   - **Рекомендовано:** 50-60 GB.
   - **Мінімум:** 40 GB для Windows 10, 64 GB для Windows 11.
4. Залиште тип диска за замовчуванням — **VDI (VirtualBox Disk Image)**.
5. Варіант розміщення — **"Динамічний"** (Pre-allocate Full Size — знято):
   - **Динамічний** — файл диска буде зростати поступово, займаючи місце на хості лише в міру заповнення. Якщо ви вказали 50 GB, а Windows займає лише 20 GB, файл буде ~20 GB.
   - **Фіксований** — файл одразу займе 50 GB на хості. Працює трохи швидше, але займає більше місця.

> **Для навчання обирайте динамічний** — він економить місце на диску хост-машини.

### Крок 5. Перевірка налаштувань та завершення створення

1. На сторінці **"Підсумок"** (Summary) перевірте всі параметри:
   - Ім'я VM
   - Тип та версія ОС
   - RAM: 4096 MB
   - Процесори: 2
   - Диск: 50 GB, динамічний, VDI
2. Натисніть **"Готово"** (Finish).
3. Нова VM з'явиться у списку зліва у вікні VirtualBox Manager.

### Крок 6. Додаткові налаштування VM (перед першим запуском)

Виділіть створену VM у списку та натисніть **"Налаштування"** (Settings):

**Система (System) > Материнська плата:**
- Порядок завантаження: Оптичний диск (Optical) повинен бути першим, Жорсткий диск — другим.
- Якщо встановлюєте Windows 11, переконайтеся, що увімкнено **EFI**.

**Система (System) > Процесор:**
- Переконайтеся, що виділено 2+ ядра.

**Дисплей (Display):**
- **Відеопам'ять:** збільшіть до **128 MB** (максимум).
- **Графічний контролер:** залиште **VBoxSVGA** (для Windows-гостей).

**Носії (Storage):**
- Переконайтеся, що ISO-образ Windows підключений до віртуального CD/DVD-приводу. Якщо ні:
  - Натисніть на значок порожнього диска під контролером IDE.
  - Праворуч натисніть значок диска > **"Обрати файл диска..."** (Choose a disk file).
  - Вкажіть шлях до ISO-файлу Windows.

**Мережа (Network):**
- **Адаптер 1:** увімкнений, тип підключення — **NAT** (за замовчуванням). Цього достатньо для доступу до інтернету з VM.

Натисніть **"OK"** для збереження налаштувань.

## 7. Покрокова інструкція — Встановлення Windows на VM

### Крок 1. Запуск віртуальної машини

1. Виділіть VM у списку VirtualBox Manager.
2. Натисніть кнопку **"Запустити"** (Start) — зелена стрілка на панелі інструментів.
3. Відкриється нове вікно — це екран вашої віртуальної машини.
4. VM завантажиться з ISO-образу. Ви побачите напис **"Press any key to boot from CD or DVD..."** — натисніть будь-яку клавішу.

> **Підказка:** Щоб VM "захопила" вашу мишу та клавіатуру, клікніть всередині вікна VM. Щоб "звільнити" мишу та повернутися до хосту, натисніть **праву клавішу Ctrl** (це так звана Host Key, що відображається в нижньому правому куті вікна VM).

### Крок 2. Вибір мови та регіональних параметрів

1. У першому вікні інсталятора оберіть:
   - **Мова встановлення:** Українська (або English).
   - **Формат часу та валюти:** Українська (Україна).
   - **Розкладка клавіатури:** Українська.
2. Натисніть **"Далі"** (Next).

### Крок 3. Початок встановлення

1. Натисніть кнопку **"Інсталювати"** (Install now).
2. Зачекайте, поки програма встановлення підготується.

### Крок 4. Ключ продукту

1. З'явиться вікно **"Активація Windows"** з проханням ввести ключ продукту.
2. Натисніть **"У мене немає ключа продукту"** (I don't have a product key).

> **Примітка:** Для навчальних цілей Windows можна встановити без ключа. Система буде повністю функціональною, але з деякими обмеженнями персоналізації (наприклад, не можна змінити шпалери через Параметри) та водяним знаком "Активуйте Windows" у правому нижньому куті.

### Крок 5. Вибір випуску Windows

1. Оберіть випуск ОС: **Windows 10/11 Pro** (рекомендовано для навчання — має більше функцій, включаючи Hyper-V, шифрування BitLocker, груповые політики).
2. Натисніть **"Далі"** (Next).
3. Прийміть ліцензійну угоду (поставте галочку) та натисніть **"Далі"**.

### Крок 6. Тип встановлення

1. Оберіть **"Вибіркова: тільки встановлення Windows (для досвідчених)"** (Custom: Install Windows only (advanced)).
   - Варіант "Оновлення" призначений для оновлення наявної ОС, але оскільки наш віртуальний диск порожній, він не підходить.

### Крок 7. Розмітка диска

1. Ви побачите **"Нерозподілений простір диска 0"** (Drive 0 Unallocated Space) — це наш віртуальний жорсткий диск (50-60 GB).
2. **Для простого варіанту:** просто виділіть цей рядок та натисніть **"Далі"** (Next). Windows автоматично створить необхідні розділи.
3. **Для розуміння розмітки (необов'язково):** натисніть **"Створити"** (New), вкажіть розмір та натисніть **"Застосувати"** (Apply). Windows створить:
   - Розділ відновлення (~500 MB)
   - Системний розділ EFI (~100 MB)
   - Зарезервований розділ MSR (~16 MB)
   - Основний розділ (решта простору) — сюди встановлюється Windows

### Крок 8. Копіювання файлів та встановлення

1. Натисніть **"Далі"**. Почнеться процес встановлення:
   - Копіювання файлів Windows
   - Підготовка файлів
   - Встановлення компонентів
   - Встановлення оновлень
   - Завершення
2. **Цей процес займає 10-30 хвилин** залежно від швидкості хост-машини.
3. VM автоматично перезавантажиться (можливо, кілька разів).

> **Увага:** Після першого перезавантаження **НЕ натискайте** жодну клавішу, коли з'явиться "Press any key to boot from CD or DVD". Система повинна завантажитися з жорсткого диска, а не з ISO.

### Крок 9. Початкове налаштування (OOBE — Out-Of-Box Experience)

Після завершення копіювання файлів система запросить налаштування:

**Для Windows 10:**
1. **Регіон:** оберіть **"Україна"** > "Так".
2. **Розкладка клавіатури:** оберіть **"Українська"** > "Так". Додаткову розкладку (англійська) можна додати або пропустити.
3. **Мережа:** для VM з NAT мережа зазвичай підключається автоматично. Якщо з'являється пропозиція підключитися — оберіть мережу або натисніть "У мене немає інтернету" > "Продовжити обмежене налаштування".
4. **Обліковий запис:** для навчальних цілей оберіть **локальний обліковий запис** (Offline account > Limited experience). Введіть ім'я користувача (наприклад, `Student`).
5. **Пароль:** можете задати простий пароль або залишити порожнім (натисніть "Далі" без введення пароля).
6. **Параметри конфіденційності:** вимкніть усі перемикачі (для навчальної VM це не критично) > "Прийняти".
7. Дочекайтеся завершення налаштування. З'явиться робочий стіл Windows.

**Для Windows 11:**
1. **Регіон:** оберіть **"Україна"** > "Так".
2. **Розкладка клавіатури:** оберіть **"Українська"** > "Так". Пропустіть додаткову розкладку.
3. **Ім'я ПК:** введіть довільне ім'я (наприклад, `VM-Student`) > "Далі".
4. **Обліковий запис Microsoft:** Windows 11 наполегливо вимагає обліковий запис Microsoft. Для навчання можна:
   - Ввести будь-яку e-mail адресу та натиснути "Далі", потім ввести неправильний пароль кілька разів — з'явиться опція створити локальний обліковий запис.
   - Або: відключити мережу у VM (Пристрої > Мережа > зняти галочку "Підключити мережевий адаптер"), після чого з'явиться опція локального облікового запису.
5. **Ім'я користувача та пароль:** введіть ім'я (наприклад, `Student`), пароль можна залишити порожнім.
6. **Параметри конфіденційності:** вимкніть усе > "Прийняти".
7. Дочекайтеся завершення. З'явиться робочий стіл Windows 11.

## 8. Встановлення Guest Additions

**Guest Additions** — це набір драйверів та утиліт від VirtualBox, які значно покращують роботу гостьової ОС.

**Що дає Guest Additions:**

| Функція | Опис |
|---------|------|
| **Інтеграція миші** | Курсор миші плавно переходить між хостом та VM без необхідності "захоплення" та "звільнення". |
| **Динамічна зміна роздільної здатності** | Роздільна здатність екрана VM автоматично підлаштовується під розмір вікна. |
| **Спільний буфер обміну** | Можливість копіювати текст між хостом і VM (Ctrl+C на хості > Ctrl+V у VM). |
| **Спільні папки** | Можливість створити папку, доступну і з хосту, і з VM. |
| **Покращена графіка** | Апаратне прискорення графіки, підтримка більшої роздільної здатності. |
| **Синхронізація часу** | Годинник VM автоматично синхронізується з хостом. |
| **Seamless Mode** | Вікна гостьової ОС можуть відображатися на робочому столі хоста. |

**Покрокова інструкція встановлення Guest Additions:**

1. Переконайтеся, що Windows на VM повністю завантажений та ви бачите робочий стіл.
2. У меню вікна VirtualBox (зверху) оберіть: **Пристрої (Devices)** > **"Підключити образ диска Guest Additions..."** (Insert Guest Additions CD image).
3. У VM з'явиться сповіщення про новий диск або вікно автозапуску.
4. Відкрийте **Провідник** у VM > **Цей ПК** > двічі клікніть на диску **"VBox_GAs_7.x.x"** (CD-привід).
5. Знайдіть та запустіть файл **`VBoxWindowsAdditions.exe`** (або `VBoxWindowsAdditions-amd64.exe` для 64-bit).
6. Натисніть **"Yes"** у вікні UAC (запит адміністратора).
7. У вікні інсталятора натискайте **"Next"** > **"Next"** > **"Install"**.
8. Під час встановлення може з'явитися запит на встановлення драйверів — натисніть **"Встановити"** (Install) для кожного.
9. Після завершення оберіть **"Reboot now"** та натисніть **"Finish"**.
10. VM перезавантажиться. Після перезавантаження Guest Additions будуть активні.

**Перевірка роботи Guest Additions:**
- Спробуйте змінити розмір вікна VM — роздільна здатність екрана повинна змінюватися автоматично.
- Увімкніть спільний буфер обміну: меню **Пристрої** > **Спільний буфер обміну** > **Двонапрямлений** (Bidirectional).

## 9. Перевірка роботи — базові дії після встановлення

Після встановлення Windows та Guest Additions виконайте наступні перевірки:

### 9.1. Перевірка базової функціональності

1. **Меню "Пуск"** — натисніть кнопку "Пуск" або клавішу Win. Переконайтеся, що меню відкривається.
2. **Провідник** — натисніть **Win + E**. Перевірте, що відкривається Провідник та відображається віртуальний жорсткий диск (C:).
3. **Перегляньте інформацію про систему:**
   - Правий клік на **"Цей ПК"** > **"Властивості"** (або **Параметри > Система > Про систему**).
   - Зверніть увагу на: версію Windows, обсяг RAM, ім'я комп'ютера.
4. **Інтернет** — відкрийте браузер (Edge) та спробуйте завантажити будь-яку веб-сторінку (наприклад, google.com). Якщо мережа налаштована на NAT, інтернет повинен працювати.

### 9.2. Перевірка параметрів VM зсередини

1. Відкрийте **Диспетчер задач** (Task Manager): натисніть **Ctrl + Shift + Esc**.
2. Перейдіть на вкладку **"Продуктивність"** (Performance):
   - **ЦП (CPU):** перевірте кількість ядер (має відповідати налаштуванням VM).
   - **Пам'ять (Memory):** перевірте обсяг RAM (має відповідати налаштуванням VM).
   - **Диск (Disk):** перевірте наявність віртуального диска.
3. Перейдіть на вкладку **"Процеси"** — подивіться, скільки RAM займає система після чистого встановлення.

### 9.3. Базові налаштування після встановлення

1. **Перевірте Windows Update:** Параметри > Оновлення та безпека > Windows Update > "Перевірити наявність оновлень". Для навчальної VM оновлення не обов'язкові.
2. **Налаштуйте роздільну здатність екрана** (якщо Guest Additions не встановлені): Параметри > Система > Дисплей > Роздільна здатність.

## 10. Знімки (Snapshots)

### Що таке знімок?

**Знімок (Snapshot)** — це збережений стан віртуальної машини в конкретний момент часу. Він включає:
- Стан усіх віртуальних дисків
- Стан оперативної пам'яті (якщо VM працювала)
- Налаштування VM

Знімок дозволяє **миттєво повернути VM до збереженого стану**, незалежно від того, що з нею сталося після створення знімка. Це як "машина часу" для віртуальної машини.

### Коли створювати знімки?

- **Після чистого встановлення ОС** — щоб мати "точку нуль" без програм та налаштувань.
- **Перед встановленням нового ПЗ** — якщо програма пошкодить систему, можна відкотитися.
- **Перед експериментами** — зміна реєстру, налаштувань системи, видалення файлів.
- **Після важливих налаштувань** — встановлення драйверів, оновлень, програм.

### Як створити знімок

**Спосіб 1 — З вікна VM (VM працює):**
1. У меню вікна VM оберіть: **Машина (Machine)** > **"Зробити знімок..."** (Take Snapshot).
2. Введіть назву знімка (наприклад, `Чиста Windows після встановлення`).
3. Додайте опис (необов'язково, але корисно): наприклад, `Windows 10 Pro, Guest Additions встановлені, без додаткових програм`.
4. Натисніть **"OK"**. Знімок буде створений за кілька секунд.

**Спосіб 2 — З VirtualBox Manager (VM вимкнена або працює):**
1. Виділіть VM у списку.
2. Натисніть значок **"Знімки"** (Snapshots) — зазвичай це іконка фотоапарата поряд із назвою VM або у правій панелі.
3. Натисніть **"Зробити"** (Take) на панелі інструментів.
4. Введіть назву та опис > "OK".

### Як відновити знімок

1. **Вимкніть або збережіть стан VM** (якщо вона працює).
2. Перейдіть до списку знімків VM у VirtualBox Manager.
3. Виділіть потрібний знімок.
4. Натисніть **"Відновити"** (Restore).
5. VirtualBox запропонує створити знімок поточного стану — оберіть за потребою.
6. Натисніть **"Відновити"**. VM повернеться до стану знімка.

### Практичне завдання зі знімками

1. Створіть знімок **"Чиста система"** одразу після встановлення ОС та Guest Additions.
2. Встановіть у VM будь-яку програму (наприклад, Блокнот++ або 7-Zip).
3. Відновіть знімок **"Чиста система"**.
4. Переконайтеся, що встановлена програма зникла — VM повернулася до стану до встановлення.

## 12. Завдання для самостійного опрацювання

1. Дослідіть налаштування мережі у VirtualBox. Поясніть, чим відрізняються режими підключення: **NAT**, **Мережевий міст (Bridged Adapter)**, **Внутрішня мережа (Internal Network)** та **Тільки хост (Host-Only Adapter)**. У яких випадках який режим варто використовувати?

2. Створіть другу віртуальну машину з іншою версією Windows (якщо перша була Windows 10 — створіть з Windows 11, і навпаки). Порівняйте процес встановлення: які кроки відрізняються? Які вимоги до ресурсів?

3. Дослідіть функцію **"Спільні папки" (Shared Folders)** у VirtualBox. Створіть спільну папку між хостом та VM. Опишіть покроково, як це зробити. Створіть файл на хості та переконайтеся, що він доступний у VM.

4. Знайдіть у VM інформацію про обладнання (Диспетчер пристроїв). Які пристрої відображаються у віртуальній машині? Чим вони відрізняються від пристроїв реального комп'ютера? Знайдіть у списку пристрої VirtualBox.

5. Дослідіть, що відбудеться, якщо виділити VM недостатньо ресурсів: змініть обсяг RAM до 1 GB та кількість ядер до 1 (попередньо створивши знімок!). Запустіть VM. Як зміниться швидкість роботи? Виконайте відновлення знімка після експерименту.

6. Дослідіть функцію **"Клонування" (Clone)** віртуальної машини. Яка різниця між повним клоном (Full Clone) та зв'язаним клоном (Linked Clone)? Створіть клон вашої VM та перевірте, чи працює він.

7. Експортуйте вашу віртуальну машину у формат **OVA** (Файл > Експорт конфігурації). Який розмір отриманого файлу? Для чого використовується формат OVA? Чи можна імпортувати OVA-файл на іншому комп'ютері з VirtualBox?

8. Увімкніть у VM **Seamless Mode** (Вигляд > Режим інтеграції) — для цього мають бути встановлені Guest Additions. Опишіть, як виглядає цей режим. Чим він відрізняється від повноекранного режиму (Fullscreen) та масштабованого режиму (Scaled Mode)?

9. Дослідіть, скільки дискового простору займає ваша VM на хост-машині. Знайдіть папку VM на диску хоста (зазвичай у `C:\Users\<ім'я>\VirtualBox VMs\`). Які файли там знаходяться? Яке розширення мають файли віртуального диска? Який їхній розмір порівняно з розміром, вказаним у налаштуваннях?

10. Порівняйте VirtualBox з Hyper-V (якщо у вас Windows 10/11 Pro). Увімкніть Hyper-V (Панель керування > Програми > Увімкнення або вимкнення компонентів Windows). Створіть просту VM у Hyper-V. Які переваги та недоліки кожного гіпервізора ви помітили? **Увага:** Hyper-V та VirtualBox можуть конфліктувати при одночасній роботі.

11. Створіть **ланцюжок знімків**: зробіть знімок "Стан 1" (чиста система), встановіть програму, зробіть "Стан 2", змініть шпалери робочого столу, зробіть "Стан 3". Поверніться до "Стану 1". Потім — до "Стану 2". Потім — до "Стану 3". Опишіть, як VirtualBox керує деревом знімків.

12. Дослідіть, яку кількість віртуальних машин можна запустити **одночасно** на вашому комп'ютері. Спробуйте запустити дві VM паралельно (за наявності достатніх ресурсів). Як це впливає на продуктивність хост-машини? Скористайтеся Диспетчером задач хоста для моніторингу навантаження на CPU та RAM.
