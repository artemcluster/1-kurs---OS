# КОНСПЕКТ УРОКУ №40

## Тема: Практична робота №15 — Диспетчер пристроїв. Діагностика обладнання

---

## Мета роботи

Навчитися працювати з Диспетчером пристроїв Windows, аналізувати стан апаратного забезпечення, діагностувати проблеми обладнання за допомогою вбудованих інструментів операційної системи.

---

## Теоретичний вступ

### 1. Диспетчер пристроїв (Device Manager)

**Диспетчер пристроїв** — це системна оснастка Windows, що дозволяє переглядати та керувати апаратним забезпеченням комп'ютера. Запускається командою `devmgmt.msc` або через **Пуск → Диспетчер пристроїв**.

Основні можливості Диспетчера пристроїв:

- перегляд усіх встановлених пристроїв, згрупованих за категоріями;
- оновлення, відкат та видалення драйверів;
- увімкнення та вимкнення пристроїв;
- перегляд властивостей пристрою: ресурси (IRQ, DMA, I/O), версія драйвера, ідентифікатор обладнання (Hardware ID);
- діагностика проблем — виявлення конфліктів ресурсів та помилок.

**Способи відкриття Диспетчера пристроїв:**

| Спосіб | Дія |
|--------|-----|
| Комбінація клавіш | `Win + X` → Диспетчер пристроїв |
| Команда «Виконати» | `Win + R` → ввести `devmgmt.msc` → Enter |
| Панель керування | Панель керування → Система → Диспетчер пристроїв |
| Пошук | Натиснути `Win`, ввести «Диспетчер пристроїв» |
| Через Мій комп'ютер | ПКМ на «Цей ПК» → Властивості → Диспетчер пристроїв |

---

### 2. Значки стану пристроїв

Диспетчер пристроїв використовує спеціальні значки для візуального відображення стану кожного пристрою:

| Значок | Опис | Що означає |
|--------|------|------------|
| Звичайний значок (без позначок) | Пристрій працює нормально | Драйвер встановлено, конфліктів немає |
| ⚠ Жовтий трикутник з `!` | Проблема з пристроєм | Драйвер відсутній, пошкоджений або несумісний; конфлікт ресурсів |
| ❌ Червоний хрест (X) | Пристрій вимкнено або має критичну помилку | У старих версіях Windows — вимкнений пристрій |
| ⬇ Стрілка вниз | Пристрій вимкнено | Пристрій вимкнено користувачем або системою (Windows Vista та новіші) |
| ❓ Жовтий знак питання | Пристрій не розпізнано | Пристрій виявлено, але драйвер не встановлено |
| ℹ Синій значок «i» | Ресурси налаштовано вручну | Автоматичне налаштування вимкнено, ресурси задані вручну |

> **Важливо:** Якщо поруч із пристроєм відображається жовтий трикутник із знаком оклику — це означає, що Windows зафіксувала помилку. Подвійний клік на пристрої покаже код помилки та її опис.

---

### 3. Коди помилок Диспетчера пристроїв

Windows використовує спеціальні коди помилок для ідентифікації проблем з пристроями:

| Код | Повідомлення | Причина | Рішення |
|-----|-------------|---------|---------|
| **Code 1** | «Цей пристрій неправильно налаштовано» | Драйвер не встановлено або встановлено неправильно | Оновити драйвер через Диспетчер пристроїв або завантажити з сайту виробника |
| **Code 10** | «Запуск цього пристрою неможливий» | Драйвер застарілий, пошкоджений або несумісний | Оновити або перевстановити драйвер; можлива апаратна несправність |
| **Code 22** | «Цей пристрій вимкнено» | Пристрій вимкнено користувачем | ПКМ на пристрої → «Увімкнути пристрій» |
| **Code 28** | «Драйвери для цього пристрою не встановлено» | Відсутній драйвер | Встановити драйвер із сайту виробника або через Windows Update |
| **Code 31** | «Пристрій працює неправильно» | Windows не може завантажити потрібні драйвери | Перевстановити драйвер; перевірити реєстр Windows |
| **Code 43** | «Windows зупинив цей пристрій, оскільки він повідомив про неполадки» | Пристрій повідомив про помилку; часто апаратна проблема | Перезавантажити ПК; оновити драйвер; перевірити підключення; можлива заміна пристрою |

> **Порада:** Щоб побачити код помилки, двічі клацніть на проблемному пристрої та відкрийте вкладку **«Загальне»** — у полі «Стан пристрою» буде вказано код та опис проблеми.

---

### 4. Режими перегляду Диспетчера пристроїв

Через меню **Вид** доступні різні режими перегляду:

- **Пристрої за типом** (стандартний) — групування за категоріями: дисплей, мережа, звук тощо.
- **Пристрої за підключенням** — ієрархічне дерево шин: PCI, USB, ACPI. Показує, через яку шину підключено кожний пристрій.
- **Ресурси за типом** — перегляд розподілу системних ресурсів:
  - **IRQ** (Interrupt Request) — запити переривань;
  - **DMA** (Direct Memory Access) — канали прямого доступу до пам'яті;
  - **I/O** (Input/Output) — порти введення/виведення;
  - **Memory** — діапазони адрес пам'яті.
- **Ресурси за підключенням** — ресурси, розподілені відповідно до підключення пристроїв.
- **Показати приховані пристрої** — відображає раніше підключені, але відсутні зараз пристрої, а також драйвери режиму ядра.

---

### 5. Вкладки властивостей пристрою

При подвійному клацанні на будь-якому пристрої відкривається вікно властивостей із кількома вкладками:

| Вкладка | Інформація |
|---------|------------|
| **Загальне** | Назва пристрою, тип, розташування, стан пристрою (код помилки, якщо є) |
| **Драйвер** | Постачальник драйвера, дата, версія, цифровий підпис. Кнопки: оновити, відкотити, вимкнути, видалити |
| **Відомості** | Детальна інформація: Hardware ID, Compatible IDs, шлях до INF-файлу, клас пристрою тощо |
| **Ресурси** | IRQ, діапазони I/O, адреси пам'яті. Інформація про конфлікти ресурсів |
| **Події** | Журнал подій, пов'язаних із цим пристроєм |

> **Hardware ID** (Ідентифікатор обладнання) — це унікальний рядок, що ідентифікує пристрій. Має формат `VEN_XXXX&DEV_YYYY` (для PCI) або `VID_XXXX&PID_YYYY` (для USB). За цим ідентифікатором можна знайти потрібний драйвер в Інтернеті.

### 6. Додаткові інструменти діагностики

#### msinfo32 — Відомості про систему

Команда `msinfo32` відкриває утиліту **«Відомості про систему»**, яка надає детальну інформацію про апаратне та програмне забезпечення.

**Основні розділи msinfo32:**

| Розділ | Вміст |
|--------|-------|
| **Відомості про систему** | Версія Windows, модель комп'ютера, процесор, обсяг RAM, версія BIOS/UEFI |
| **Апаратні ресурси** | Конфлікти/Спільне використання, DMA, IRQ, I/O порти, адреси пам'яті |
| **Компоненти** | Мультимедіа, дисплей, звук, мережа, порти, USB, диски, друк, проблемні пристрої |
| **Програмне середовище** | Драйвери, змінні середовища, завдання, мережеві підключення, служби |

> Розділ **Компоненти → Проблемні пристрої** показує список усіх пристроїв, які мають проблеми — аналогічно до жовтих трикутників у Диспетчері пристроїв.

Для збереження звіту: **Файл → Зберегти** (формат .nfo) або **Файл → Експорт** (текстовий файл .txt).

#### dxdiag — Засіб діагностики DirectX

Команда `dxdiag` відкриває **Засіб діагностики DirectX**, який надає інформацію про компоненти та драйвери DirectX.

**Вкладки dxdiag:**

| Вкладка | Інформація |
|---------|------------|
| **Система** | Версія Windows, модель ПК, процесор, RAM, версія DirectX |
| **Екран (Display)** | Відеокарта: назва, виробник, тип мікросхеми, обсяг відеопам'яті, версія драйвера, дата драйвера; стан прискорення DirectDraw та Direct3D |
| **Звук (Sound)** | Звукова карта: назва, виробник, драйвер; стан апаратного прискорення звуку |
| **Введення (Input)** | Підключені пристрої введення: клавіатура, миша, ігрові контролери, їх стан та драйвери |

> Кнопка **«Зберегти всі відомості»** дозволяє зберегти повний звіт у текстовий файл — це зручно для передачі інформації технічному спеціалісту.

#### systeminfo — Команда командного рядка

Команда `systeminfo` у **Командному рядку (CMD)** виводить повну інформацію про систему:

```
systeminfo
```

Відображає: назву ОС, версію, тип системи, виробника, модель, процесор, обсяг фізичної та віртуальної пам'яті, мережеві адаптери, виправлення (Hotfix), дату встановлення Windows та багато іншого.

### 7. Діагностика проблем

#### Засіб усунення неполадок Windows (Troubleshooter)

Windows має вбудований інструмент для автоматичного пошуку та виправлення неполадок:

**Параметри → Оновлення та безпека → Усунення неполадок → Додаткові засоби усунення неполадок**

Доступні засоби для: звуку, мережі, принтера, Bluetooth, відеовідтворення тощо.

#### Журнал подій (Event Viewer)

Команда `eventvwr.msc` відкриває **Переглядач подій** — інструмент для аналізу системних журналів:

- **Журнали Windows → Система** — містить помилки та попередження, пов'язані з обладнанням та драйверами.
- **Журнали Windows → Програма** — помилки програмного забезпечення.

**Рівні важливості подій:**

| Рівень | Значення |
|--------|----------|
| ℹ Відомості | Інформаційна подія, все працює нормально |
| ⚠ Попередження | Потенційна проблема, яка може стати критичною |
| ❌ Помилка | Значна проблема, що може вплинути на роботу системи |
| 🔴 Критична | Серйозний збій, що потребує негайного втручання |

> Для пошуку помилок обладнання: відкрийте **Журнали Windows → Система**, відфільтруйте за рівнем «Помилка» та «Критична», зверніть увагу на джерела, пов'язані з драйверами та пристроями.

#### Конфлікти ресурсів

Конфлікти виникають, коли два або більше пристроїв намагаються використовувати один і той самий ресурс (IRQ, I/O порт, адресу пам'яті). Для перевірки:

- У Диспетчері пристроїв: **Вид → Ресурси за типом** → перегляд розподілу IRQ.
- В msinfo32: **Апаратні ресурси → Конфлікти/Спільне використання**.

### 8. PowerShell — діагностика Plug and Play пристроїв

PowerShell надає потужні командлети для роботи з пристроями:

#### Get-PnpDevice — перегляд усіх PnP-пристроїв

```powershell
# Список усіх пристроїв
Get-PnpDevice

# Тільки підключені зараз пристрої
Get-PnpDevice -PresentOnly

# Фільтр за назвою (із символом підстановки)
Get-PnpDevice -FriendlyName "Intel*"

# Знайти пристрої зі статусом помилки
Get-PnpDevice -Status "ERROR"

# Знайти пристрої зі статусом OK
Get-PnpDevice -Status "OK"

# Пристрої певного класу (наприклад, мережеві)
Get-PnpDevice -Class "Net"

# Вивести у вигляді таблиці: статус, клас, назва
Get-PnpDevice -PresentOnly | Format-Table Status, Class, FriendlyName -AutoSize
```

**Можливі значення параметра `-Status`:** `OK`, `ERROR`, `UNKNOWN`, `DEGRADED`.

#### Get-PnpDeviceProperty — властивості пристрою

```powershell
# Отримати InstanceId пристрою
$device = Get-PnpDevice -FriendlyName "*Realtek*"
$device.InstanceId

# Переглянути всі властивості пристрою за InstanceId
Get-PnpDeviceProperty -InstanceId "PCI\VEN_8086&DEV_1234&..."

# Отримати конкретні властивості
Get-PnpDeviceProperty -InstanceId $device.InstanceId -KeyName 'DEVPKEY_Device_DeviceDesc', 'DEVPKEY_Device_HardwareIds', 'DEVPKEY_Device_ProblemCode'

# Передати пристрій через конвеєр
Get-PnpDevice -FriendlyName "*Audio*" | Get-PnpDeviceProperty
```

#### Додаткові корисні команди PowerShell

```powershell
# Увімкнути пристрій
Enable-PnpDevice -InstanceId "PCI\VEN_XXXX&DEV_YYYY\..."

# Вимкнути пристрій (з підтвердженням)
Disable-PnpDevice -InstanceId "PCI\VEN_XXXX&DEV_YYYY\..."

# Вимкнути без підтвердження
Disable-PnpDevice -InstanceId "..." -Confirm:$false
```

## Практичні завдання

### Завдання 1. Відкриття Диспетчера пристроїв та перегляд категорій

1. Натисніть `Win + R`, введіть `devmgmt.msc`, натисніть **Enter**.
2. Перегляньте всі категорії пристроїв, розгорнувши кожну з них (клікніть на стрілку ▸ зліва).
3. Запишіть кількість категорій та загальну кількість пристроїв у кожній категорії.
4. Зробіть знімок екрана Диспетчера пристроїв із розгорнутими категоріями.

---

### Завдання 2. Перегляд дерева шин — Пристрої за підключенням

1. У Диспетчері пристроїв оберіть меню **Вид → Пристрої за підключенням**.
2. Розгорніть дерево та знайдіть основні шини: **ACPI**, **PCI Express**, **USB**.
3. Визначте, які пристрої підключені через кожну шину.
4. Запишіть ієрархію підключення для відеоадаптера та мережевого адаптера.
5. Поверніть стандартний вигляд: **Вид → Пристрої за типом**.

---

### Завдання 3. Ресурси за типом — IRQ, DMA, I/O, Memory

1. Оберіть **Вид → Ресурси за типом**.
2. Розгорніть кожну категорію ресурсів:
   - **Запит на переривання (IRQ)** — запишіть, яким пристроям призначено IRQ 0, 1, 8, 12, 14.
   - **Канал прямого доступу до пам'яті (DMA)** — запишіть, які пристрої використовують DMA.
   - **Порт введення-виведення (I/O)** — знайдіть діапазон I/O для клавіатури.
   - **Адреса пам'яті (Memory)** — знайдіть діапазон адрес для відеоадаптера.
3. Зробіть знімок екрана розділу IRQ.

---

### Завдання 4. Властивості відеоадаптера

1. У Диспетчері пристроїв розгорніть категорію **«Відеоадаптери»**.
2. Двічі клацніть на вашому відеоадаптері. Дослідіть кожну вкладку:
   - **Загальне:** запишіть назву пристрою, розташування та стан пристрою.
   - **Драйвер:** запишіть постачальника, дату, версію драйвера та чи має драйвер цифровий підпис.
   - **Відомості:** у випадаючому списку оберіть «Ідентифікатори обладнання» (Hardware IDs) — запишіть перший рядок.
   - **Ресурси:** запишіть виділені ресурси (діапазон пам'яті, IRQ, порти I/O).
3. Заповніть таблицю:

| Параметр | Значення |
|----------|----------|
| Назва пристрою | |
| Постачальник драйвера | |
| Версія драйвера | |
| Дата драйвера | |
| Hardware ID | |
| Діапазон пам'яті | |
| IRQ | |

---

### Завдання 5. Пошук Hardware ID для пристроїв

1. Знайдіть **Hardware ID** для трьох різних пристроїв:
   - мережевий адаптер;
   - звуковий пристрій;
   - USB-контролер.
2. Для кожного пристрою: ПКМ → **Властивості → Відомості** → у випадаючому списку оберіть **«Ідентифікатори обладнання»**.
3. Заповніть таблицю:

| Пристрій | Hardware ID | Тип шини (PCI/USB) |
|----------|------------|-------------------|
| Мережевий адаптер | | |
| Звуковий пристрій | | |
| USB-контролер | | |

4. Поясніть, з яких частин складається Hardware ID (VEN, DEV, SUBSYS або VID, PID).

---

### Завдання 6. Вимкнення та увімкнення мережевого адаптера

1. Розгорніть категорію **«Мережеві адаптери»**.
2. ПКМ на мережевому адаптері → **«Вимкнути пристрій»** → підтвердити.
3. Зверніть увагу на зміни:
   - Значок пристрою змінився (з'явилася стрілка вниз ⬇).
   - Перевірте з'єднання з Інтернетом (спробуйте відкрити будь-який сайт).
4. ПКМ на мережевому адаптері → **«Увімкнути пристрій»**.
5. Перевірте, що з'єднання з Інтернетом відновилося.
6. Запишіть свої спостереження: який значок відображався, скільки часу зайняло відновлення з'єднання.

> ⚠ **Увага:** Після вимкнення мережевого адаптера ви тимчасово втратите доступ до мережі. Не забудьте увімкнути його назад!

---

### Завдання 7. Приховані пристрої

1. У Диспетчері пристроїв оберіть **Вид → Показати приховані пристрої**.
2. Зверніть увагу на нові категорії та пристрої, що з'явилися (вони відображаються напівпрозорими значками).
3. Запишіть:
   - Скільки нових категорій з'явилося?
   - Знайдіть категорію **«Драйвери, що не підтримують Plug and Play»** — які драйвери там перелічені?
   - Чи є серед прихованих пристроїв раніше підключені USB-пристрої?
4. Поясніть, навіщо потрібна функція відображення прихованих пристроїв.

### Завдання 8. msinfo32 — Компоненти та апаратні ресурси

1. Натисніть `Win + R`, введіть `msinfo32`, натисніть **Enter**.
2. У лівій панелі перегляньте розділи:
   - **Відомості про систему** — запишіть: назву ОС, версію, виробника системи, модель, процесор, обсяг RAM.
   - **Апаратні ресурси → Конфлікти/Спільне використання** — чи є конфлікти? Запишіть будь-які знайдені.
   - **Апаратні ресурси → IRQ** — порівняйте з даними з Диспетчера пристроїв (завдання 3).
   - **Компоненти → Дисплей** — запишіть назву та роздільну здатність.
   - **Компоненти → Проблемні пристрої** — чи є записи?
3. Збережіть звіт: **Файл → Експорт** → збережіть як текстовий файл на робочий стіл.

### Завдання 9. dxdiag — Діагностика DirectX

### Завдання 10. CMD: systeminfo — інформація про систему

1. Відкрийте **Командний рядок** (CMD) від імені адміністратора.
2. Введіть команду:
   ```
   systeminfo
   ```
3. Дочекайтеся завершення збору інформації та запишіть:
   - Назву ОС та її версію.
   - Тип системи (x64 або x86).
   - Загальний обсяг фізичної пам'яті та доступний обсяг.
   - Кількість мережевих адаптерів та їх назви.

   - Дату першого встановлення Windows.
4. Збережіть результат у файл:
   ```
   systeminfo > C:\Users\%USERNAME%\Desktop\systeminfo.txt
   ```

## Діагностика проблем — додаткові інструменти

### Засіб усунення неполадок Windows

1. Відкрийте **Параметри** (`Win + I`) → **Оновлення та безпека** → **Усунення неполадок**.
2. Натисніть **«Додаткові засоби усунення неполадок»**.
3. Запустіть засіб для категорії **«Відтворення звуку»** або **«Мережевий адаптер»**.
4. Дочекайтеся завершення та запишіть результат діагностики.

### Журнал подій (eventvwr.msc)

1. Натисніть `Win + R`, введіть `eventvwr.msc`, натисніть **Enter**.
2. Перейдіть до **Журнали Windows → Система**.
3. У правій панелі натисніть **«Фільтр поточного журналу»**.
4. Встановіть прапорці: **Помилка** та **Критична**. Натисніть **OK**.
5. Перегляньте останні помилки — знайдіть ті, що пов'язані з обладнанням або драйверами.
6. Запишіть для однієї з помилок: **Джерело**, **Код події (Event ID)**, **Опис**.

### PowerShell — діагностика пристроїв

1. Відкрийте **PowerShell** від імені адміністратора.
2. Виконайте команди та запишіть результати:

```powershell
# Список усіх підключених пристроїв
Get-PnpDevice -PresentOnly | Format-Table Status, Class, FriendlyName -AutoSize

# Пошук пристроїв із помилками
Get-PnpDevice -Status "ERROR"

# Переглянути властивості конкретного пристрою (замініть назву на свою)
Get-PnpDevice -FriendlyName "*Audio*" | Get-PnpDeviceProperty | Format-Table KeyName, Data -AutoSize
```

3. Зробіть знімок екрана результатів.
