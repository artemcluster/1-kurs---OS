## 1. Мета роботи

- Зрозуміти принцип Plug and Play та процес автоматичного розпізнавання периферійних пристроїв у Windows 10/11.
- Навчитися підключати пристрої через USB, Bluetooth та інші інтерфейси.
- Оволодіти навичками роботи з Диспетчером пристроїв (Device Manager).
- Навчитися налаштовувати принтери, сканери та інші периферійні пристрої.
- Ознайомитися з командою `pnputil` для перегляду PnP-пристроїв у командному рядку.
- Засвоїти правила безпечного від'єднання USB-пристроїв.

---

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Клавіатура та миша.
- USB-флешка (або інший USB-накопичувач).
- Bluetooth-пристрій (навушники, миша, клавіатура) — за наявності.
- Командний рядок (CMD), запущений від імені адміністратора (для команди `pnputil`).

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не вставляйте та не виймайте USB-пристрої силою** — роз'єми мають вставлятися легко. Якщо USB-штекер не входить — переверніть його або перевірте тип порту.
4. **Завжди виконуйте безпечне вилучення** USB-пристроїв перед від'єднанням (через значок у треї або через Провідник).
5. **Не встановлюйте драйвери з невідомих джерел.** Використовуйте тільки драйвери з Windows Update або з офіційного сайту виробника.
6. **Не змінюйте налаштування мережевих адаптерів** без дозволу викладача.
7. Не вимикайте та не видаляйте пристрої в Диспетчері пристроїв без дозволу викладача.
8. Після завершення роботи приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Plug and Play — автоматичне розпізнавання пристроїв

**Plug and Play (PnP)** — це технологія, яка дозволяє операційній системі автоматично розпізнавати підключені пристрої, підбирати та встановлювати для них драйвери без участі користувача.

**Як працює Plug and Play у Windows 10/11:**

1. Ви підключаєте пристрій (наприклад, USB-флешку) до комп'ютера.
2. Операційна система виявляє новий пристрій та зчитує його ідентифікатор (Vendor ID, Product ID).
3. Windows шукає відповідний драйвер: спочатку у локальному сховищі драйверів, потім — через Windows Update.
4. Драйвер встановлюється автоматично, і пристрій готовий до роботи.
5. У системному треї (область сповіщень) з'являється повідомлення: «Пристрій готовий до використання».

**Служба Plug and Play** — це системна служба Windows, яка відповідає за весь цей процес. Перевірити її стан можна так: **Win + R** --> ввести `services.msc` --> знайти «Plug and Play» у списку. Служба має бути запущена і мати тип запуску «Автоматичний».

> **Історична довідка:** До появи Plug and Play (до Windows 95) кожен пристрій потрібно було налаштовувати вручну: встановлювати драйвер з дискети, задавати номер переривання (IRQ), адресу порту тощо. Це було складно навіть для досвідчених користувачів.

### 4.2. Типи підключення периферійних пристроїв

| Тип підключення | Інтерфейс | Типові пристрої | Швидкість передачі даних | Особливості |
|-----------------|-----------|-----------------|--------------------------|-------------|
| **USB 2.0** | Дротовий | Флешки, миші, клавіатури, вебкамери | До 480 Мбіт/с (High Speed) | Найпоширеніший тип, сумісний майже з усіма пристроями |
| **USB 3.0 / 3.1 / 3.2** | Дротовий | Зовнішні HDD/SSD, флешки, док-станції | 5–20 Гбіт/с (SuperSpeed) | Порти зазвичай позначені синім кольором або літерою «SS» |
| **USB-C** | Дротовий | Смартфони, монітори, зарядні пристрої, аудіо | До 40 Гбіт/с (USB4) | Роз'єм симетричний (вставляється будь-якою стороною); USB-C — це тип роз'єму, а не стандарт швидкості |
| **Bluetooth** | Бездротовий | Навушники, миші, клавіатури, колонки, геймпади | До 3 Мбіт/с (Bluetooth 5.0) | Радіус дії — до 10 метрів, не потребує прямої видимості |
| **Wi-Fi (бездротовий друк)** | Бездротовий | Принтери, сканери, МФУ | Залежить від стандарту Wi-Fi | Пристрій і комп'ютер мають бути в одній мережі |
| **HDMI** | Дротовий | Монітори, телевізори, проєктори | До 48 Гбіт/с (HDMI 2.1) | Передає відео та аудіо одним кабелем |
| **DisplayPort** | Дротовий | Монітори | До 77,4 Гбіт/с (DP 2.0) | Часто використовується для ігрових та професійних моніторів |
| **3.5 мм аудіороз'єм** | Дротовий | Навушники, мікрофони, колонки | Аналоговий сигнал | Може бути комбінований (навушники + мікрофон) або окремий |

**Як визначити версію USB-порту на комп'ютері:**

- **Візуально:** USB 2.0 — чорний або білий роз'єм; USB 3.0 — синій; USB 3.1/3.2 — червоний або бірюзовий.
- **Через Диспетчер пристроїв:** Якщо у розділі «Контролери USB» є запис «USB 3.x eXtensible Host Controller» або «USB Root Hub (USB 3.0)» — комп'ютер підтримує USB 3.0+.
- **Через властивості USB Root Hub:** Диспетчер пристроїв --> Контролери USB --> USB Root Hub --> Властивості --> вкладка «Додатково» — вказано максимальну швидкість порту.

**Довідка швидкостей USB:**

| Позначення | Версія USB | Швидкість |
|------------|-----------|-----------|
| Low Speed | USB 1.0 | 1,5 Мбіт/с |
| Full Speed | USB 1.1 | 12 Мбіт/с |
| High Speed | USB 2.0 | 480 Мбіт/с |
| SuperSpeed | USB 3.0 | 5 Гбіт/с |
| SuperSpeed+ | USB 3.1 Gen 2 | 10 Гбіт/с |
| SuperSpeed+ | USB 3.2 Gen 2x2 | 20 Гбіт/с |

### 4.3. Диспетчер пристроїв (Device Manager)

**Диспетчер пристроїв** — це вбудована утиліта Windows для перегляду та управління всіма апаратними компонентами комп'ютера.

**Як відкрити:**
- **Win + X** --> вибрати «Диспетчер пристроїв»
- **Win + R** --> ввести `devmgmt.msc` --> Enter
- Правий клік на «Пуск» --> «Диспетчер пристроїв»

**Що показує Диспетчер пристроїв:**

Усі пристрої організовані за категоріями (деревоподібний список):

| Категорія | Приклади пристроїв |
|-----------|-------------------|
| **Дисплейні адаптери (Display adapters)** | Відеокарти (NVIDIA, AMD, Intel) |
| **Звукові, відео- та ігрові пристрої** | Аудіокодеки (Realtek, Intel), мікрофони |
| **Контролери USB (Universal Serial Bus controllers)** | USB Root Hub, USB 3.0 Host Controller |
| **Мережеві адаптери (Network adapters)** | Ethernet-адаптер, Wi-Fi-адаптер, Bluetooth |
| **Черги друку (Print queues)** | Встановлені принтери |
| **Клавіатури** | HID-клавіатура |
| **Миші та інші вказівні пристрої** | HID-сумісна миша |
| **Дискові пристрої (Disk drives)** | HDD, SSD, USB-флешки |

**Значки стану пристрою:**

| Значок | Стан | Що робити |
|--------|------|-----------|
| Немає додаткових значків | Працює нормально | Нічого |
| Жовтий трикутник з **!** | Проблема з драйвером або конфлікт | Оновити або перевстановити драйвер |
| Червоний хрестик | Пристрій вимкнено | Правий клік --> «Увімкнути пристрій» |
| Стрілка вниз | Пристрій вимкнено вручну | Правий клік --> «Увімкнути пристрій» |

### 4.4. Віртуальні пристрої

**Віртуальний пристрій** — це програмний компонент, який імітує реальний фізичний пристрій. Операційна система працює з ним так, ніби це справжнє обладнання.

**Приклади віртуальних пристроїв у Windows:**

| Віртуальний пристрій | Опис | Де знайти |
|---------------------|------|-----------|
| **Microsoft Print to PDF** | Віртуальний принтер, який замість фізичного друку зберігає документ як PDF-файл | Параметри --> Bluetooth та пристрої --> Принтери та сканери |
| **Microsoft XPS Document Writer** | Віртуальний принтер для збереження у форматі XPS | Там само |
| **VirtualBox Host-Only Adapter** | Віртуальний мережевий адаптер для зв'язку з віртуальними машинами | Диспетчер пристроїв --> Мережеві адаптери (якщо встановлено VirtualBox) |
| **Loopback Adapter** | Віртуальний мережевий адаптер для тестування | Додається вручну через Диспетчер пристроїв |

**Microsoft Print to PDF** встановлений за замовчуванням у Windows 10/11. Якщо його немає, можна ввімкнути: **Win + R** --> `optionalfeatures.exe` --> поставити галочку «Microsoft Print to PDF» --> OK.

> **Навіщо потрібні віртуальні пристрої?** Вони дозволяють працювати з функціональністю без фізичного обладнання. Наприклад, можна «друкувати» документи у PDF, навіть якщо до комп'ютера не підключено жодного принтера.

### 4.5. Безпечне від'єднання пристроїв

**Чому потрібно безпечно вилучати USB-пристрої?**

Коли ви копіюєте файли на USB-флешку, Windows може використовувати **кеш запису (write cache)** — дані спочатку зберігаються у оперативній пам'яті, а потім поступово записуються на флешку. Якщо вийняти флешку, поки дані ще не записані — файли будуть пошкоджені або втрачені.

Крім того, якщо якась програма все ще працює з файлами на флешці (навіть у фоновому режимі), раптове від'єднання може призвести до пошкодження файлової системи на флешці.

**Як безпечно вилучити USB-пристрій (3 способи):**

**Спосіб 1: Через системний трей (найпоширеніший)**
1. У правій нижній частині екрана знайдіть значок «Безпечне вилучення пристрою та носія» (зображення USB-штекера зі стрілкою).
2. Якщо значка не видно — натисніть стрілку **^** («Показати приховані значки»).
3. Клацніть правою кнопкою миші на значку.
4. Виберіть пункт **«Вилучити <назва пристрою>»** (Eject).
5. Дочекайтеся повідомлення **«Обладнання може бути безпечно вилучене»**.
6. Тепер можна фізично від'єднати пристрій.

**Спосіб 2: Через Провідник**
1. Відкрийте **Провідник** (Win + E).
2. У лівій панелі знайдіть USB-пристрій.
3. Натисніть правою кнопкою миші --> **«Вилучити»** (Eject).

**Спосіб 3: Через Параметри**
1. Відкрийте **Параметри** (Win + I) --> **Bluetooth та пристрої** --> **Пристрої**.
2. Знайдіть USB-пристрій у списку.
3. Натисніть три крапки (**...**) біля нього --> **«Видалити пристрій»** (Remove device).

> **Примітка:** Починаючи з Windows 10 версії 1809, для USB-флешок за замовчуванням увімкнено політику «Швидке вилучення» (Quick Removal), яка вимикає кеш запису. Це означає, що у більшості випадків флешку можна від'єднати без безпечного вилучення. Проте для зовнішніх жорстких дисків та SSD безпечне вилучення **обов'язкове**, оскільки для них за замовчуванням увімкнено «Кращу продуктивність» (Better Performance) з кешуванням.

## 5. Практичні завдання

### Завдання 1. Підключити USB-флешку та спостерігати процес розпізнавання

1. Вставте USB-флешку у вільний USB-порт комп'ютера.
2. Спостерігайте за реакцією системи:
   - Чи з'явилося повідомлення у правому нижньому куті екрана? Якщо так — запишіть його текст.
   - Чи відтворився звук підключення?
3. Відкрийте **Провідник** (Win + E) і знайдіть флешку у розділі «Цей ПК» (This PC).
4. Запишіть у зошит:

| Параметр | Значення |
|----------|----------|
| Буква диска, яку присвоїла система | |
| Назва (мітка) пристрою | |
| Файлова система | |
| Загальний обсяг | |
| Вільне місце | |

5. Натисніть правою кнопкою миші на флешці --> **«Властивості»** --> вкладка **«Обладнання»** (Hardware). Знайдіть назву пристрою та запишіть її.

---

### Завдання 2. Безпечне вилучення USB-флешки

1. Переконайтеся, що жодна програма не використовує файли на флешці (закрийте всі відкриті файли та папки з флешки).
2. У правій нижній частині екрана (системний трей) знайдіть значок **«Безпечне вилучення пристрою»** (USB-значок зі стрілкою).
   - Якщо значка не видно — натисніть стрілку **^** для показу прихованих значків.
3. Клацніть на значку **правою кнопкою миші**.
4. Виберіть пункт **«Вилучити <назва вашої флешки>»** (Eject).
5. Дочекайтеся повідомлення **«Обладнання може бути безпечно вилучене»**.
6. Фізично від'єднайте флешку.
7. Запишіть у зошит:
   - Чи з'явилося повідомлення про безпечне вилучення?
   - Чи зникла флешка з Провідника після вилучення?
8. Підключіть флешку знову — вона знадобиться для наступних завдань.

---

### Завдання 3. Налаштування принтера: Microsoft Print to PDF

1. Відкрийте **Параметри** (Win + I) --> **Bluetooth та пристрої** --> **Принтери та сканери**.
2. Знайдіть у списку **«Microsoft Print to PDF»**. Якщо він є — натисніть на нього.
3. Натисніть кнопку **«Встановити за замовчуванням»** (Set as default) або увімкніть перемикач, якщо він доступний.
4. Натисніть **«Друк тестової сторінки»** (Print test page).
5. Відкриється вікно «Зберегти друк як» — виберіть **Робочий стіл** як місце збереження, назвіть файл `Тестова_сторінка.pdf`, натисніть **«Зберегти»**.
6. Знайдіть збережений файл на Робочому столі та відкрийте його. Переконайтеся, що тестова сторінка відображається коректно.
7. Запишіть у зошит:

| Параметр | Значення |
|----------|----------|
| Назва принтера за замовчуванням | |
| Тип принтера (фізичний / віртуальний) | |
| Чи вдалося надрукувати тестову сторінку? | |

8. Якщо «Microsoft Print to PDF» відсутній у списку — ввімкніть його: **Win + R** --> `optionalfeatures.exe` --> знайдіть «Microsoft Print to PDF» --> поставте галочку --> OK.

---

### Завдання 4. Переглянути підключені пристрої через Параметри

1. Відкрийте **Параметри** (Win + I) --> **Bluetooth та пристрої** --> **Пристрої**.
2. Перегляньте список усіх підключених пристроїв.
3. Заповніть таблицю у зошиті (запишіть усі пристрої, які бачите):

| № | Назва пристрою | Тип (миша/клавіатура/аудіо/інше) | Тип підключення (USB/Bluetooth/інше) |
|---|---------------|----------------------------------|--------------------------------------|
| 1 | | | |
| 2 | | | |
| 3 | | | |
| ... | | | |

4. Зверніть увагу: у цьому розділі відображаються **не всі** пристрої — лише ті, що Windows вважає «периферійними» (миші, клавіатури, аудіо, принтери тощо). Внутрішні компоненти (відеокарта, процесор, RAM) тут не показуються.

---

### Завдання 5. Bluetooth: увімкнути, переглянути доступні пристрої

1. Відкрийте **Параметри** (Win + I) --> **Bluetooth та пристрої**.
2. Переконайтеся, що перемикач **Bluetooth** увімкнено (On). Якщо Bluetooth недоступний — запишіть це і переходьте до пункту 7.
3. Натисніть **«Додати пристрій»** (Add device).
4. Виберіть **«Bluetooth»** у списку типів підключення.
5. Windows почне пошук доступних Bluetooth-пристроїв. Дочекайтеся завершення пошуку (10–30 секунд).
6. Запишіть у зошит:
   - Чи є Bluetooth на вашому комп'ютері?
   - Скільки пристроїв знайдено під час пошуку?
   - Назви знайдених пристроїв (якщо є).
7. **Якщо є Bluetooth-пристрій для з'єднання** (навушники, миша тощо):
   - Увімкніть на ньому режим сполучення (pairing mode).
   - У вікні пошуку виберіть його та натисніть **«Підключити»** (Connect).
   - Якщо з'явиться PIN-код — переконайтеся, що він збігається на обох пристроях, і підтвердіть.
   - Запишіть результат: чи вдалося підключити?
8. Натисніть **«Скасувати»** (Cancel), щоб закрити вікно пошуку.

> **Примітка:** Bluetooth ефективно працює на відстані до 10 метрів. USB-пристрої, підключені до порту USB 3.0, можуть створювати радіоперешкоди для Bluetooth-сигналу.

6. Розгорніть категорію **«Мережеві адаптери»** (Network adapters).

### Завдання 7. Перевірити версію USB-порту через властивості USB Root Hub

1. У **Диспетчері пристроїв** розгорніть категорію **«Контролери USB»**.
2. Знайдіть запис **«USB Root Hub»** або **«USB Root Hub (USB 3.0)»**.
3. Натисніть правою кнопкою миші --> **«Властивості»** (Properties).
4. Перейдіть на вкладку **«Загальні»** (General) — запишіть стан пристрою (має бути «Пристрій працює нормально»).
5. Перейдіть на вкладку **«Додатково»** (Advanced) або **«Відомості»** (Details).
   - Якщо є вкладка «Додатково» — знайдіть інформацію про швидкість порту.
   - Якщо є вкладка «Відомості» — у випадаючому списку виберіть **«Ідентифікатори обладнання»** (Hardware IDs) та запишіть перший рядок.
6. Перейдіть на вкладку **«Драйвер»** (Driver). Запишіть:

| Параметр | Значення |
|----------|----------|
| Постачальник драйвера | |
| Дата драйвера | |
| Версія драйвера | |

7. Перейдіть на вкладку **«Керування електроживленням»** (Power Management).
   - Запишіть, чи стоїть галочка **«Дозволити комп'ютеру вимикати цей пристрій для економії енергії»**.

> **Пояснення:** Якщо USB-пристрій раптово перестає працювати через деякий час, причина може бути саме в цій опції — Windows вимикає USB-порт для економії енергії. Зняття цієї галочки вирішує проблему.

### Завдання 8. Команда pnputil — список PnP-пристроїв

1. Відкрийте **Командний рядок від імені адміністратора**: **Win + R** --> ввести `cmd` --> натиснути **Ctrl + Shift + Enter**. Підтвердіть запит UAC.
2. Введіть команду:
   ```
   pnputil /enum-devices /connected
   ```
3. Дочекайтеся виведення списку. Команда покаже **всі підключені (активні) PnP-пристрої**.
4. Знайдіть у списку свою **USB-флешку**. Запишіть:
   - Instance ID (ідентифікатор екземпляра)
   - Device Description (опис пристрою)
   - Status (стан — має бути **Started**)
5. Тепер введіть команду для перегляду тільки USB-пристроїв:
   ```
   pnputil /enum-devices /connected /class USB
   ```
6. Запишіть, скільки USB-пристроїв показала команда.
7. Для перегляду пристроїв з проблемами введіть:
   ```
   pnputil /enum-devices /problem
   ```
8. Запишіть результат: чи є пристрої з проблемами? Якщо так — запишіть їх назви та коди проблем.

> **Довідка:** `pnputil` — це консольна утиліта для управління Plug and Play пристроями та драйверами. Вона входить до складу Windows і знаходиться у `C:\Windows\System32\pnputil.exe`. Для більшості команд потрібні права адміністратора.

### Завдання 9. Вирішення проблеми: «USB-пристрій не розпізнано»

**Ситуація:** Уявіть, що ви підключили USB-пристрій, але Windows показує повідомлення «USB-пристрій не розпізнано» (USB Device Not Recognized) або пристрій не з'являється у Провіднику.

**Виконайте послідовність дій з діагностики (запишіть кожен крок та результат):**

1. **Спробуйте інший USB-порт:**
   - Від'єднайте флешку і підключіть її до іншого USB-порту (бажано на задній панелі комп'ютера, якщо це десктоп).
   - Запишіть: чи допомогло?

2. **Перевірте в Диспетчері пристроїв:**
   - Відкрийте Диспетчер пристроїв (Win + X --> Device Manager).
   - Розгорніть «Контролери USB». Чи є пристрої з жовтим трикутником?
   - Якщо є — натисніть правою кнопкою миші --> **«Оновити драйвер»** --> **«Автоматичний пошук драйверів»**.
   - Запишіть результат.

3. **Перевірте у «Керуванні дисками»:**
   - Відкрийте **Win + R** --> `diskmgmt.msc`.
   - Чи відображається флешка у нижній панелі (навіть без букви диска)?
   - Якщо відображається без букви — натисніть правою кнопкою миші --> «Змінити букву диска або шлях» --> «Додати».
   - Запишіть результат.

4. **Засіб усунення несправностей:**
   - Відкрийте **Параметри** --> **Система** --> **Усунення несправностей** --> **Інші засоби усунення несправностей**.
   - Знайдіть і запустіть засіб для «Обладнання та пристрої» (Hardware and Devices), якщо доступний.
   - Запишіть результат діагностики.

5. Заповніть підсумкову таблицю:

| Крок діагностики | Що перевіряли | Результат |
|------------------|---------------|-----------|
| 1. Інший порт | Фізичне підключення | |
| 2. Диспетчер пристроїв | Драйвер, помилки | |
| 3. Керування дисками | Розпізнавання пристрою | |
| 4. Засіб усунення несправностей | Автоматична діагностика | |

### Завдання 10. Налаштування принтера: встановити за замовчуванням та тестовий друк

1. Відкрийте **Параметри** (Win + I) --> **Bluetooth та пристрої** --> **Принтери та сканери**.
2. Перегляньте список усіх встановлених принтерів та запишіть їх:

| № | Назва принтера | Тип (фізичний / віртуальний) | Стан (підключено / офлайн) |
|---|---------------|------------------------------|----------------------------|
| 1 | | | |
| 2 | | | |
| ... | | | |

3. Знайдіть внизу сторінки опцію **«Дозволити Windows керувати принтером за замовчуванням»** (Let Windows manage my default printer). Запишіть її поточний стан (увімкнено/вимкнено).
4. **Вимкніть** цю опцію (якщо вона увімкнена).

## 7. Завдання для самостійного опрацювання

1. На домашньому комп'ютері відкрийте Диспетчер пристроїв (Win + X --> Device Manager) та підрахуйте загальну кількість категорій пристроїв і загальну кількість окремих пристроїв. Які категорії мають найбільше пристроїв?

2. Підключіть до домашнього комп'ютера USB-флешку та виконайте команду `pnputil /enum-devices /connected /class USB` у CMD від імені адміністратора. Знайдіть свою флешку у списку та запишіть її Instance ID та Device Description.

3. Дослідіть, які Bluetooth-пристрої знаходяться поблизу вашого комп'ютера або смартфона. Чому деякі пристрої видимі для всіх, а деякі — ні? Що таке «режим видимості» (Discoverable Mode) у Bluetooth?

4. Поясніть, чому USB-C — це не стандарт швидкості, а лише тип роз'єму. Наведіть приклади: з якою швидкістю може працювати USB-C порт у різних пристроях (смартфон, ноутбук, зовнішній SSD)?

5. Знайдіть інформацію про технологію USB OTG (On-The-Go). Що це таке? Які пристрої її підтримують? Наведіть практичні приклади використання.

6. Уявіть ситуацію: ви підключили нову вебкамеру до комп'ютера, але вона не працює. Опишіть покроковий алгоритм діагностики та вирішення проблеми (не менше 5 кроків).

7. Дослідіть команду `pnputil /enum-drivers` у CMD. Що вона показує? Виконайте її на домашньому комп'ютері та знайдіть у списку драйвер для вашої відеокарти або мережевого адаптера. Запишіть його назву, версію та дату.

8. Порівняйте два способи підключення принтера: через USB-кабель та через Wi-Fi. Які переваги та недоліки кожного способу? Коли краще використовувати кожен з них?

9. Дослідіть, що таке «фантомні пристрої» (Ghost Devices) у Диспетчері пристроїв. Як їх побачити? (Підказка: Вигляд --> Показати приховані пристрої). Чому вони з'являються і чи потрібно їх видаляти?

10. Знайдіть інформацію про стандарт Thunderbolt 4. Чим він відрізняється від звичайного USB-C? Які пристрої підтримують Thunderbolt? Яку максимальну швидкість він забезпечує?

11. На домашньому комп'ютері виконайте команду `wmic path Win32_PnPEntity get Name,Status` у CMD. Ця команда покаже всі PnP-пристрої та їхній стан. Знайдіть у списку пристрої зі статусом, відмінним від «OK», та запишіть їх назви. Поясніть, що може означати такий стан.

12. Дослідіть, як підключити два монітори до одного комп'ютера (через HDMI, DisplayPort або VGA). Які режими відображення підтримує Windows 10/11 (дублювання, розширення, тільки другий екран)? Як перемикатися між режимами (підказка: Win + P)?

13. Поясніть різницю між драйвером класу (class driver) та драйвером від виробника (vendor driver). Чому іноді стандартний драйвер Windows працює, але не забезпечує повну функціональність пристрою? Наведіть приклад.

14. Дослідіть, як працює функція «Віддалений робочий стіл» (Remote Desktop) у Windows для перенаправлення периферійних пристроїв. Чи можна друкувати на локальному принтері, підключившись до віддаленого комп'ютера?

15. Створіть порівняльну таблицю сучасних бездротових технологій для підключення периферійних пристроїв: Bluetooth 5.0, Wi-Fi Direct, NFC, Zigbee, Z-Wave. Для кожної технології вкажіть: швидкість, радіус дії, типові пристрої, переваги та недоліки.
