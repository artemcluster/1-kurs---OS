## 1. Вступ

Встановлення операційної системи — це одна з базових навичок будь-якого IT-спеціаліста. Проте перш ніж розпочати інсталяцію, необхідно ретельно підготуватися: перевірити апаратне забезпечення, створити резервні копії, підготувати інсталяційний носій і правильно розмітити диск. Помилки на етапі підготовки можуть призвести до втрати даних або некоректної роботи системи.

У цьому уроці ми розглянемо всі кроки підготовки, порівняємо системні вимоги різних ОС, розберемо структуру розділів диска та дізнаємося, як BIOS/UEFI впливає на процес встановлення.

## 2. Підготовка до встановлення ОС

### 2.1. Перевірка сумісності апаратного забезпечення

Перед встановленням будь-якої ОС необхідно переконатися, що комп'ютер відповідає системним вимогам. Перевірити потрібно:

- **Процесор (CPU)** — тактова частота, кількість ядер, архітектура (32-біт чи 64-біт).
- **Оперативна пам'ять (RAM)** — обсяг у гігабайтах.
- **Накопичувач (HDD/SSD)** — вільний простір на диску.
- **Відеокарта (GPU)** — підтримка DirectX, OpenGL.
- **Мережевий адаптер** — для завантаження оновлень під час інсталяції.
- **Спеціальні вимоги** — наприклад, TPM 2.0 та Secure Boot для Windows 11.

**Як перевірити характеристики комп'ютера (у Windows):**

1. Натиснути `Win + R`, ввести `msinfo32` і натиснути Enter — відкриється вікно «Відомості про систему».
2. Або натиснути `Win + Pause/Break` для швидкого перегляду основних параметрів.
3. Для Windows 11 — скористатися офіційною утилітою **PC Health Check** від Microsoft.

### 2.2. Резервне копіювання даних

**Резервне копіювання (backup)** — це створення копії важливих файлів на окремому носії перед будь-якими маніпуляціями з диском.

**Що потрібно зберегти:**

| Тип даних | Розташування за замовчуванням |
|-----------|------------------------------|
| Документи | `C:\Users\Ім'я\Documents` |
| Фотографії | `C:\Users\Ім'я\Pictures` |
| Робочий стіл | `C:\Users\Ім'я\Desktop` |
| Закладки браузера | Експорт із налаштувань браузера |
| Ліцензійні ключі | Записати окремо |
| Драйвери | Завантажити заздалегідь |

**Куди зберігати:**

- Зовнішній жорсткий диск або SSD.
- USB-флешка (для невеликих обсягів).
- Хмарне сховище (Google Drive, OneDrive, Dropbox).
- Інший розділ диска (менш надійно, бо при помилці розмітки може бути втрачений).

> **Важливо!** Ніколи не зберігайте єдину копію важливих файлів на тому самому диску, на який буде встановлюватися ОС.

### 2.3. Підготовка інсталяційного носія

Для встановлення ОС потрібен інсталяційний носій — фізичний або віртуальний диск з файлами інсталятора.

**Варіанти інсталяційних носіїв:**

- **USB-флешка** — найпоширеніший спосіб. Потрібна флешка об'ємом не менше 8 ГБ.
- **DVD-диск** — застарілий спосіб, але іноді використовується.
- **ISO-образ** — файл-образ диска з розширенням `.iso`, який містить повну копію інсталяційного диска. Може використовуватися для запису на флешку або DVD, а також для встановлення у віртуальну машину.

**Де завантажити ISO-образи (офіційні джерела):**

- **Windows 10/11** — [microsoft.com/software-download](https://www.microsoft.com/software-download/)
- **Ubuntu** — [ubuntu.com/download](https://ubuntu.com/download)

### 2.4. Створення завантажувальної флешки

Завантажувальна флешка — це USB-накопичувач, підготовлений спеціальним чином, щоб комп'ютер міг з нього запуститися та розпочати встановлення ОС.

**Rufus (для Windows):**

Rufus — безкоштовна програма для створення завантажувальних USB-накопичувачів.

Порядок дій:
1. Завантажити Rufus з офіційного сайту [rufus.ie](https://rufus.ie).
2. Вставити USB-флешку (мінімум 8 ГБ).
3. Запустити Rufus.
4. У полі «Пристрій» обрати потрібну флешку.
5. У полі «Метод завантаження» натиснути «Обрати» і вказати ISO-файл.
6. Обрати схему розділу: **GPT** (для UEFI) або **MBR** (для Legacy BIOS).
7. Файлова система: **NTFS** для Windows, **FAT32** для UEFI-сумісних.
8. Натиснути «Старт» і дочекатися завершення.

> **Увага!** Всі дані на флешці будуть видалені!

**Media Creation Tool (тільки для Windows):**

Офіційна утиліта Microsoft, яка автоматично завантажує останню версію Windows і записує її на флешку.

Порядок дій:
1. Завантажити Media Creation Tool з сайту Microsoft.
2. Запустити програму, прийняти ліцензійну угоду.
3. Обрати «Створити інсталяційний носій для іншого ПК».
4. Обрати мову, випуск та архітектуру.
5. Обрати «USB-пристрій флеш-пам'яті».
6. Обрати потрібну флешку та дочекатися завершення.

**balenaEtcher (для Linux та macOS):**

Простий кросплатформний інструмент для запису ISO-образів на флешку. Порядок дій: обрати образ, обрати флешку, натиснути «Flash».

## 4. Розділи диска (Partitions)

### 4.1. Що таке розділ диска і навіщо ділити диск

**Розділ диска (partition)** — це логічно відокремлена частина фізичного накопичувача (HDD або SSD), яка функціонує як самостійний диск.

**Навіщо ділити диск на розділи:**

1. **Організація даних** — системні файли окремо від користувацьких. Наприклад, ОС на диску C:, а документи на диску D:.
2. **Безпека даних** — при перевстановленні ОС на системному розділі дані на інших розділах залишаються неушкодженими.
3. **Встановлення кількох ОС** — кожна ОС потребує свого окремого розділу (Dual Boot).
4. **Оптимізація продуктивності** — різні файлові системи для різних задач.
5. **Вимоги ОС** — деякі операційні системи потребують кількох розділів для коректної роботи (наприклад, розділ завантаження, розділ підкачки).

### 4.2. MBR vs GPT — таблиці розділів

**Таблиця розділів** — це структура даних на початку диска, яка описує, як диск поділений на розділи.

Існують два основних стандарти:

**MBR (Master Boot Record):**
- Створений у 1983 році.
- Розміщується в перших 512 байтах диска.
- Підтримує максимум **4 основних розділи** (або 3 основних + 1 розширений).
- Максимальний розмір диска — **2 ТБ**.
- Працює з **Legacy BIOS**.
- Менш стійкий до пошкоджень — інформація про розділи зберігається в одному місці.

**GPT (GUID Partition Table):**
- Сучасний стандарт, частина специфікації UEFI.
- Підтримує до **128 розділів** (без потреби у розширених розділах).
- Максимальний розмір диска — **9.4 ЗБ** (зеттабайт), тобто практично необмежений.
- Працює з **UEFI**.
- Зберігає **резервну копію** таблиці розділів у кінці диска, що підвищує надійність.
- Включає контрольні суми CRC32 для виявлення помилок.

**Порівняльна таблиця MBR та GPT:**

| Характеристика | MBR | GPT |
|----------------|-----|-----|
| Рік створення | 1983 | Кінець 1990-х |
| Максимум розділів | 4 основних | 128 |
| Максимальний розмір диска | 2 ТБ | 9.4 ЗБ |
| Сумісність з BIOS | Так | Ні |
| Сумісність з UEFI | Обмежена | Повна |
| Резервна копія таблиці | Ні | Так |
| Контрольні суми | Ні | Так (CRC32) |
| Рекомендація для нових ПК | Ні | Так |

> **Висновок:** Для сучасних комп'ютерів завжди обирайте **GPT + UEFI**. MBR використовується лише для старих комп'ютерів із Legacy BIOS.

### 4.3. Типи розділів

**Основний (Primary) розділ:**
- Розділ, на який можна встановити ОС та з якого можна завантажити комп'ютер.
- MBR підтримує до 4 основних розділів.
- GPT підтримує до 128 розділів, і всі вони є основними.

**Розширений (Extended) розділ:**
- Спеціальний тип розділу, який існує тільки в MBR.
- Це «контейнер» для логічних розділів.
- На диску може бути лише один розширений розділ.
- Дозволяє обійти обмеження MBR у 4 розділи.

**Логічний (Logical) розділ:**
- Розділ, створений всередині розширеного розділу.
- Може зберігати дані, але не завжди може бути завантажувальним.
- Кількість логічних розділів обмежена тільки розміром диска.

**Приклад структури MBR-диска:**

```
Диск 500 ГБ (MBR)
├── Основний розділ 1 (C:) — 100 ГБ — Windows
├── Основний розділ 2 (D:) — 150 ГБ — Дані
├── Основний розділ 3 — 50 ГБ — Linux
└── Розширений розділ — 200 ГБ
    ├── Логічний розділ 1 (E:) — 100 ГБ — Мультимедіа
    └── Логічний розділ 2 (F:) — 100 ГБ — Архіви
```

### 4.4. Спеціальні розділи

**Системний розділ (System Partition):**
- Містить файли, необхідні для завантаження ОС (завантажувач).
- У Windows це зазвичай невеликий прихований розділ (100-500 МБ).

**Завантажувальний розділ (Boot Partition):**
- Розділ, де розміщені файли самої операційної системи.
- У Windows це зазвичай диск `C:\`.
- Часто системний і завантажувальний розділ — це різні розділи.

**Розділ відновлення (Recovery Partition):**
- Прихований розділ, який містить засоби відновлення системи.
- Дозволяє повернути ОС до заводських налаштувань без інсталяційного диска.
- Зазвичай має розмір 500 МБ — 1 ГБ.
- Створюється виробником ноутбука або самою Windows при встановленні.

**EFI System Partition (ESP):**
- Спеціальний розділ, обов'язковий для систем із **UEFI**.
- Формат файлової системи — **FAT32**.
- Типовий розмір — **100-550 МБ**.
- Містить завантажувачі всіх встановлених ОС, драйвери UEFI та утиліти.
- Комп'ютер при увімкненні першим читає саме ESP, щоб знайти, яку ОС завантажувати.
- При встановленні кількох ОС (Dual Boot) усі їхні завантажувачі розміщуються на одному ESP.

## 5. Файлові системи для встановлення ОС

**Файлова система** — це спосіб організації та зберігання файлів на розділі диска. Різні ОС використовують різні файлові системи.

### NTFS (New Technology File System)

- Стандартна файлова система для **Windows** (починаючи з Windows NT).
- Підтримує файли розміром до **16 ТБ**.
- Максимальний розмір тому — **256 ТБ**.
- Підтримує **права доступу** до файлів і папок.
- Підтримує **журналювання** — у разі збою система може відновити цілісність даних.
- Підтримує **стиснення** та **шифрування** файлів (EFS).
- Linux може читати та писати на NTFS-розділи (через драйвер NTFS-3G або вбудований ntfs3).

### ext4 (Fourth Extended Filesystem)

- Стандартна файлова система для **Linux**.
- Максимальний розмір файлу — **16 ТБ**.
- Максимальний розмір тому — **1 ЕБ** (ексабайт).
- Підтримує журналювання.
- Висока продуктивність та надійність.
- Windows не може читати ext4 без спеціального програмного забезпечення.

### swap (розділ підкачки)

- Спеціальний розділ у **Linux**, який використовується як додаткова «віртуальна пам'ять».
- Коли оперативна пам'ять заповнюється, ОС переміщує частину даних із RAM на swap-розділ.
- Рекомендований розмір: дорівнює обсягу RAM або у 1.5-2 рази більше (для систем із малим обсягом RAM).
- Також використовується для **гібернації** (збереження стану системи при вимкненні).
- У сучасних Linux-системах замість окремого розділу може використовуватися **swap-файл**.

### FAT32

- Використовується для розділу **ESP (EFI System Partition)**.
- Проста файлова система, сумісна з більшістю ОС.
- Обмеження: максимальний розмір файлу — **4 ГБ**.

| Файлова система | Використання | Макс. розмір файлу | Журналювання |
|----------------|-------------|-------------------|-------------|
| NTFS | Windows | 16 ТБ | Так |
| ext4 | Linux | 16 ТБ | Так |
| swap | Linux (підкачка) | Не застосовується | Ні |
| FAT32 | ESP, флешки | 4 ГБ | Ні |

## 6. BIOS vs UEFI — вплив на встановлення

**BIOS (Basic Input/Output System)** та **UEFI (Unified Extensible Firmware Interface)** — це мікропрограми (прошивки), вбудовані в материнську плату, які запускаються першими при увімкненні комп'ютера.

### BIOS (Legacy BIOS)

- Розроблений у 1975 році компанією IBM.
- Текстовий інтерфейс, керування тільки клавіатурою.
- Працює в **16-бітному** режимі.
- Завантажується з **MBR**.
- Не підтримує диски більше **2 ТБ** як завантажувальні.
- Повільніше завантаження.
- Немає вбудованого захисту від шкідливого ПЗ при завантаженні.

### UEFI (Unified Extensible Firmware Interface)

- Сучасна заміна BIOS (з'явилася у 2005 році, масово — з 2012).
- Графічний інтерфейс з підтримкою миші.
- Працює в **32-бітному** або **64-бітному** режимі.
- Завантажується з **GPT**.
- Підтримує диски будь-якого розміру.
- Значно **швидше завантаження** (за рахунок паралельної ініціалізації пристроїв).
- Підтримує **Secure Boot** — захист від завантаження неавторизованого коду.
- Необхідний для Windows 11.

### Порівняльна таблиця

| Характеристика | BIOS | UEFI |
|----------------|------|------|
| Інтерфейс | Текстовий | Графічний |
| Розрядність | 16-біт | 32/64-біт |
| Таблиця розділів | MBR | GPT |
| Макс. розмір завантажувального диска | 2 ТБ | Необмежений |
| Secure Boot | Ні | Так |
| Швидкість завантаження | Повільна | Швидка |
| Підтримка Windows 11 | Ні | Так |

### Вплив на встановлення ОС

При створенні завантажувальної флешки потрібно обрати правильний режим:
- **Якщо комп'ютер має UEFI** — обирати GPT при створенні флешки в Rufus.
- **Якщо комп'ютер має тільки Legacy BIOS** — обирати MBR.
- У налаштуваннях UEFI перед встановленням потрібно перевірити, чи увімкнений **Secure Boot** (для Windows 11 — обов'язково), та налаштувати порядок завантаження (Boot Order), щоб комп'ютер першим завантажувався з USB.

## 7. Dual Boot — встановлення двох ОС на один комп'ютер

**Dual Boot (подвійне завантаження)** — це конфігурація, при якій на одному комп'ютері встановлено дві (або більше) операційні системи, і при кожному запуску користувач обирає, яку з них завантажити.

### Типові комбінації

- **Windows + Linux** — найпоширеніша комбінація.
- **Windows 10 + Windows 11** — для тестування або сумісності.
- **Linux + Linux** — наприклад, Ubuntu + Fedora.

### Правила встановлення Dual Boot

1. **Спочатку встановлювати Windows, потім Linux.** Windows при встановленні перезаписує завантажувач і не враховує інші ОС. Linux-інсталятор (з GRUB) виявить Windows і додасть її до меню завантаження.
2. **Кожна ОС — на окремому розділі.** Не можна встановити дві ОС на один розділ.
3. **ESP (EFI System Partition) — спільний.** Обидві ОС розміщують свої завантажувачі на одному ESP.
4. **Виділити достатньо місця** — для Windows мінімум 64 ГБ, для Linux мінімум 25 ГБ.

### Завантажувач GRUB

**GRUB (GRand Unified Bootloader)** — це завантажувач, який встановлюється разом із Linux. При запуску комп'ютера він показує меню з переліком усіх встановлених ОС, дозволяючи обрати потрібну.

### Ризики Dual Boot

- При оновленні Windows може пошкодити завантажувач GRUB.
- Невірна розмітка диска може призвести до втрати даних.
- Потрібне розуміння розділів та файлових систем.

### Варіант 1: Один диск, тільки Windows 11 (GPT + UEFI)

└──────────┴───────────┴───────────────────┴───────────────┘
```

```

│                      Диск 1 ТБ SSD (GPT)                          │
├────────┬────────┬──────────┬──────────┬─────────┬─────────────────┤

```

```

## 10. Глосарій

| Термін | Визначення |
|--------|-----------|
| **Backup (резервне копіювання)** | Створення копії даних на окремому носії для захисту від втрати |
| **BIOS (Basic Input/Output System)** | Мікропрограма материнської плати, яка ініціалізує апаратне забезпечення та запускає завантаження ОС (застарілий стандарт) |
| **Boot Partition (завантажувальний розділ)** | Розділ диска, що містить файли операційної системи |
| **Dual Boot (подвійне завантаження)** | Конфігурація з двома або більше ОС на одному комп'ютері з можливістю вибору при запуску |
| **ESP (EFI System Partition)** | Спеціальний розділ із файловою системою FAT32 для зберігання завантажувачів ОС у системах з UEFI |
| **ext4** | Стандартна файлова система Linux з підтримкою журналювання |
| **FAT32** | Проста файлова система, сумісна з більшістю ОС; обмеження розміру файлу — 4 ГБ |
| **GPT (GUID Partition Table)** | Сучасний стандарт таблиці розділів, підтримує до 128 розділів і диски понад 2 ТБ |
| **GRUB (GRand Unified Bootloader)** | Завантажувач Linux, який дозволяє обирати ОС при запуску комп'ютера |
| **ISO-образ** | Файл-образ оптичного диска з розширенням .iso, що містить повну копію даних |
| **MBR (Master Boot Record)** | Застарілий стандарт таблиці розділів; обмеження: 4 основних розділи, макс. 2 ТБ |
| **NTFS (New Technology File System)** | Стандартна файлова система Windows з підтримкою журналювання, прав доступу та шифрування |
| **Partition (розділ)** | Логічно відокремлена частина фізичного диска, що функціонує як самостійний том |
| **Rufus** | Безкоштовна програма для створення завантажувальних USB-накопичувачів |
| **Secure Boot** | Функція UEFI, що запобігає завантаженню неавторизованого програмного забезпечення |
| **Swap (підкачка)** | Розділ або файл у Linux, що використовується як розширення оперативної пам'яті |
| **TPM (Trusted Platform Module)** | Апаратний модуль безпеки для шифрування, захисту ключів та перевірки цілісності системи |
| **UEFI (Unified Extensible Firmware Interface)** | Сучасна заміна BIOS з графічним інтерфейсом, підтримкою GPT та Secure Boot |
| **WDDM (Windows Display Driver Model)** | Архітектура графічних драйверів Windows |
| **Журналювання** | Механізм файлової системи, що фіксує зміни перед їх записом, забезпечуючи відновлення після збоїв |
