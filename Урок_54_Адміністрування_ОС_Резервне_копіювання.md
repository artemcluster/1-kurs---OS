## 1. Що таке адміністрування ОС

**Адміністрування операційної системи** — це комплекс заходів із встановлення, налаштування, підтримки працездатності, оптимізації та захисту операційної системи та програмного забезпечення на комп'ютерах і серверах.

Адміністрування охоплює:

- встановлення та конфігурацію ОС;
- управління обліковими записами користувачів;
- налаштування прав доступу до файлів і ресурсів;
- моніторинг стану системи та її продуктивності;
- оновлення програмного забезпечення;
- забезпечення інформаційної безпеки;
- резервне копіювання та відновлення даних;
- усунення неполадок і збоїв.

Адміністрування необхідне як на окремих персональних комп'ютерах, так і в масштабних корпоративних мережах з десятками серверів і сотнями робочих станцій.

## 2. Хто такий системний адміністратор

**Системний адміністратор (сисадмін)** — це фахівець, відповідальний за безперебійну роботу комп'ютерної техніки, комп'ютерної мережі та програмного забезпечення в організації.

Системне адміністрування базується на глибокому знанні комп'ютерних систем та ефективних способів їх використання. Сисадмін повинен розуміти як операційні системи і застосунки, так і цілі, з якими співробітники організації використовують комп'ютери.

**Ключові навички системного адміністратора:**

| Навичка | Опис |
|---------|------|
| Операційні системи | Впевнене володіння Windows Server, Linux (Ubuntu, CentOS), macOS |
| Мережеві технології | Знання TCP/IP, DNS, DHCP, налаштування маршрутизаторів і комутаторів |
| Безпека | Захист від вірусів, налаштування брандмауерів, шифрування |
| Скриптування | Автоматизація за допомогою PowerShell, Bash, Python |
| Апаратне забезпечення | Діагностика та ремонт серверного та клієнтського обладнання |
| Резервне копіювання | Створення та перевірка резервних копій даних |

> **Цікавий факт:** Професія системного адміністратора активно трансформується. Сучасні сисадміни все частіше стають DevOps-інженерами або SRE-інженерами (Site Reliability Engineer), поєднуючи навички адміністрування з розробкою та автоматизацією.

## 3. Основні задачі адміністратора

### 3.1. Встановлення та оновлення ОС і ПЗ

Адміністратор відповідає за:

- **Встановлення ОС** — вибір відповідної версії, розмітку дисків, конфігурацію під конкретне обладнання.
- **Оновлення ОС** — регулярне встановлення оновлень безпеки (security patches), виправлень помилок та нових функцій. В Windows це Windows Update, в Linux — менеджери пакетів (`apt`, `yum`, `dnf`).
- **Встановлення прикладного ПЗ** — офісні пакети, антивіруси, спеціалізовані програми.
- **Контроль ліцензій** — відстеження легальності встановленого програмного забезпечення.

**Приклад:** Адміністратор налаштовує груповою політикою автоматичне оновлення Windows на 50 робочих станціях в офісі, щоб усі комп'ютери отримували патчі безпеки без участі користувачів.

### 3.2. Управління користувачами та правами доступу

- **Створення облікових записів** — для кожного працівника створюється окремий обліковий запис із унікальним логіном і паролем.
- **Призначення прав доступу** — визначення, хто має доступ до яких папок, файлів і програм. Використовується принцип **найменших привілеїв** (надавати лише ті права, які необхідні для виконання роботи).
- **Групи користувачів** — об'єднання користувачів у групи (наприклад, "Бухгалтерія", "Менеджери") для зручного управління правами.
- **Політика паролів** — встановлення вимог до складності паролів (мінімальна довжина, наявність спецсимволів, термін дії).

### 3.3. Моніторинг продуктивності та ресурсів

Адміністратор постійно контролює стан системи:

- **Завантаження процесора (CPU)** — якщо процесор постійно завантажений на 100%, це може означати проблему з програмою або вірусну активність.
- **Використання оперативної пам'яті (RAM)** — нестача пам'яті призводить до різкого уповільнення роботи.
- **Дисковий простір** — заповнений диск може спричинити збій системи.
- **Мережевий трафік** — аномальне зростання трафіку може вказувати на кібератаку.

**Інструменти моніторингу:**
- Windows: Диспетчер завдань (Task Manager), Монітор ресурсів (Resource Monitor), Монітор продуктивності (Performance Monitor)
- Linux: команди `top`, `htop`, `df`, `free`, `iotop`, `netstat`

### 3.4. Забезпечення безпеки

Системний адміністратор — перша лінія оборони організації від кіберзагроз:

- **Антивірусний захист** — встановлення та оновлення антивірусного ПЗ.
- **Брандмауер (Firewall)** — налаштування правил фільтрації мережевого трафіку.
- **Шифрування даних** — використання BitLocker (Windows) або LUKS (Linux) для захисту дисків.
- **Контроль доступу** — обмеження фізичного та віддаленого доступу до серверів.
- **Аудит подій** — відстеження підозрілої активності через журнали подій.
- **Навчання користувачів** — проведення інструктажів щодо безпечної роботи (не відкривати підозрілі листи, не використовувати прості паролі тощо).

### 3.5. Резервне копіювання та відновлення

Це одна з найважливіших задач адміністратора (детально розглянута в розділах 5-9):

- Регулярне створення резервних копій критичних даних.
- Перевірка цілісності резервних копій.
- Розробка плану відновлення після збою (Disaster Recovery Plan).
- Тестування процедур відновлення.

### 3.6. Усунення несправностей

Адміністратор повинен уміти швидко діагностувати та усувати проблеми:

- **Апаратні проблеми** — вихід з ладу жорсткого диска, оперативної пам'яті, блоку живлення.
- **Програмні проблеми** — конфлікти драйверів, помилки оновлень, збої програм.
- **Мережеві проблеми** — відсутність з'єднання, низька швидкість, збої DNS.

**Алгоритм усунення несправностей:**
1. Визначити проблему (збір інформації від користувача).
2. Відтворити проблему на своєму боці.
3. Перевірити журнали подій (Event Viewer / системні логи).
4. Виключити найпростіші причини (перезавантаження, перевірка кабелів).
5. Застосувати рішення.
6. Перевірити, що проблему вирішено.
7. Задокументувати рішення.

## 4. Інструменти адміністрування Windows

Windows надає набір вбудованих інструментів для адміністрування системи:

### Панель керування (Control Panel)

Класичний інтерфейс для налаштування системи. Дозволяє керувати обліковими записами, мережевими з'єднаннями, програмами, пристроями та іншими параметрами. Доступна через меню Пуск або команду `control` у вікні "Виконати" (Win+R).

### Параметри (Settings)

Сучасний інтерфейс налаштувань Windows 10/11, який поступово замінює Панель керування. Містить розділи: Система, Пристрої, Мережа, Облікові записи, Оновлення та безпека тощо.

### Командний рядок (CMD)

Текстовий інтерфейс для виконання команд. Корисні команди адміністратора:

| Команда | Призначення |
|---------|-------------|
| `ipconfig` | Показати мережеві налаштування |
| `ping` | Перевірити з'єднання з іншим комп'ютером |
| `sfc /scannow` | Перевірити цілісність системних файлів |
| `chkdsk` | Перевірити диск на помилки |
| `net user` | Управління обліковими записами |
| `tasklist` | Список запущених процесів |
| `shutdown /r /t 0` | Перезавантажити комп'ютер |

### PowerShell

Потужна оболонка командного рядка з мовою скриптування. Перевага над CMD — підтримка складних сценаріїв автоматизації, робота з об'єктами замість простого тексту, можливість віддаленого управління.

**Приклад:** команда `Get-Process | Sort-Object CPU -Descending | Select-Object -First 10` покаже 10 процесів, що найбільше навантажують процесор.

### Консоль управління Microsoft (MMC)

MMC (Microsoft Management Console) — це фреймворк, що надає адміністраторам єдиний інтерфейс для управління системою через набір модулів, які називаються **оснастками (snap-ins)**.

Запуск: Win+R → `mmc` → Enter.

Популярні оснастки:
- **Управління дисками** (Disk Management) — робота з розділами дисків
- **Локальні користувачі та групи** (Local Users and Groups) — управління обліковими записами
- **Перегляд подій** (Event Viewer) — перегляд журналів
- **Служби** (Services) — управління службами Windows
- **Диспетчер пристроїв** (Device Manager) — управління обладнанням

### Планувальник завдань (Task Scheduler)

Дозволяє автоматизувати виконання завдань за розкладом. Наприклад:
- запуск антивірусної перевірки щоночі о 2:00;
- автоматичне очищення тимчасових файлів щотижня;
- запуск скрипту резервного копіювання щодня.

Запуск: Win+R → `taskschd.msc` → Enter.

### Перегляд подій (Event Viewer)

Централізована система журналів Windows. Записує всі важливі події системи, безпеки та програм.

Основні журнали:
- **Система (System)** — події ОС, драйверів, служб
- **Безпека (Security)** — спроби входу, зміна прав доступу
- **Програма (Application)** — помилки та попередження програм

Запуск: Win+R → `eventvwr.msc` → Enter.

> **Порада:** При усуненні несправностей завжди починайте з перевірки Event Viewer — він часто містить точний опис проблеми та її причину.

## 5. Резервне копіювання (Backup)

### 5.1. Навіщо потрібне резервне копіювання

**Резервне копіювання (backup)** — це процес створення копій даних для їх відновлення у разі втрати або пошкодження оригіналу.

**Від чого захищає резервне копіювання:**

- **Апаратні збої** — вихід з ладу жорсткого диска, SSD (за статистикою, близько 5% жорстких дисків виходять з ладу протягом перших 4 років роботи).
- **Програмні збої** — пошкодження файлової системи, помилки оновлень ОС.
- **Кібератаки** — віруси-шифрувальники (ransomware) можуть зашифрувати всі файли і вимагати викуп.
- **Людський фактор** — випадкове видалення або перезапис файлів.
- **Стихійні лиха** — пожежа, повінь, крадіжка обладнання.

> **Важливо:** Питання не в тому, чи втратите ви дані, а в тому, КОЛИ це станеться. Резервне копіювання — це страховка ваших даних.

### 5.2. Типи резервного копіювання

#### Повне копіювання (Full Backup)

Створюється **повна копія всіх даних**, які потрібно захистити, незалежно від того, чи були внесені зміни з моменту останнього копіювання.

**Переваги:**
- Найпростіше відновлення — всі дані в одному архіві.
- Найшвидше відновлення.
- Не залежить від попередніх копій.

**Недоліки:**
- Потребує найбільше місця на носії.
- Найповільніше створення.
- Не підходить для щоденного використання при великих обсягах даних.

**Приклад:** Щоп'ятниці о 22:00 створюється повна копія всього сервера — 500 ГБ даних.

#### Інкрементне копіювання (Incremental Backup)

Копіюються **лише ті дані, що змінились з моменту останнього будь-якого копіювання** (повного або інкрементного).

**Переваги:**
- Найменший обсяг кожної окремої копії.
- Найшвидше створення.
- Мінімальне навантаження на систему.

**Недоліки:**
- Найповільніше відновлення — потрібно відновити повну копію + усі інкрементні копії по ланцюжку.
- Якщо одна інкрементна копія пошкоджена, всі наступні стають марними.

**Приклад:** Понеділок — повна копія (500 ГБ). Вівторок — інкрементна копія (тільки зміни за вівторок — 10 ГБ). Середа — інкрементна копія (тільки зміни за середу — 12 ГБ).

#### Диференційне копіювання (Differential Backup)

Копіюються **всі дані, що змінились з моменту останнього повного копіювання**.

**Переваги:**
- Швидше відновлення, ніж інкрементне — потрібні лише повна копія + остання диференційна.
- Надійніше, ніж інкрементне — не залежить від ланцюжка.

**Недоліки:**
- Кожна наступна диференційна копія стає більшою.
- Потребує більше місця, ніж інкрементне.

**Приклад:** Понеділок — повна копія (500 ГБ). Вівторок — диференційна (зміни за вівторок — 10 ГБ). Середа — диференційна (зміни за вівторок + середу — 22 ГБ). Четвер — диференційна (зміни за вівторок + середу + четвер — 35 ГБ).

### 5.3. Порівняльна таблиця типів резервного копіювання

| Характеристика | Повне (Full) | Інкрементне (Incremental) | Диференційне (Differential) |
|---|---|---|---|
| **Що копіюється** | Усі дані | Зміни з останнього будь-якого бекапу | Зміни з останнього повного бекапу |
| **Швидкість створення** | Найповільніша | Найшвидша | Середня |
| **Обсяг копії** | Найбільший | Найменший | Середній (зростає з часом) |
| **Швидкість відновлення** | Найшвидша | Найповільніша | Швидка (2 набори) |
| **Складність відновлення** | Проста (1 копія) | Складна (повна + весь ланцюжок) | Проста (повна + остання диференційна) |
| **Надійність** | Найвища | Найнижча (залежність від ланцюжка) | Висока |
| **Типове використання** | Раз на тиждень | Щодня | Щодня |

### 5.4. Правило 3-2-1

**Правило 3-2-1** — це золотий стандарт стратегії резервного копіювання, який рекомендує Агентство з кібербезпеки та захисту інфраструктури (CISA, США).

```
              ПРАВИЛО 3-2-1

  3 копії даних    2 типи носіїв    1 копія поза офісом
  ┌──────────┐     ┌──────────┐     ┌──────────────┐
  │ Оригінал │     │ Внутр.   │     │  Хмарне      │
  │ Копія 1  │     │  диск    │     │  сховище     │
  │ Копія 2  │     │ Зовнішній│     │  або інша    │
  │          │     │  диск    │     │  локація     │
  └──────────┘     └──────────┘     └──────────────┘
```

**Пояснення:**

- **3 копії** — оригінал даних + дві резервні копії. Якщо одна копія пошкоджена, залишається ще одна.
- **2 різних типи носіїв** — наприклад, внутрішній жорсткий диск + зовнішній USB-диск, або SSD + хмарне сховище. Це захищає від збою одного типу носія.
- **1 копія за межами основного розташування** — якщо в офісі сталася пожежа, повінь або крадіжка, віддалена копія залишиться неушкодженою. Це може бути хмарне сховище або фізичний носій в іншому місці.

> **Сучасне розширення — правило 3-2-1-1-0:** додається ще 1 незмінна (immutable) або офлайн-копія та 0 помилок під час тестового відновлення. Це посилений захист від вірусів-шифрувальників.

**Приклад для звичайного користувача:**
1. Оригінал файлів — на ноутбуці.
2. Копія 1 — на зовнішньому жорсткому диску.
3. Копія 2 — у Google Drive або OneDrive (хмара, інша локація).

## 6. Засоби резервного копіювання Windows

### Історія файлів (File History)

Вбудований інструмент Windows, який автоматично зберігає копії файлів із бібліотек користувача (Документи, Зображення, Відео, Музика, Робочий стіл) на зовнішній диск або мережеву папку.

**Особливості:**
- Зберігає попередні версії файлів — можна повернутися до будь-якої збереженої версії.
- Працює автоматично у фоновому режимі.
- Інтервал копіювання — від 10 хвилин до 1 разу на добу.
- Не створює копію системи — тільки файли користувача.

**Як увімкнути:** Параметри → Оновлення та безпека → Резервне копіювання → Додати диск.

### Резервне копіювання та відновлення (Windows 7 Backup)

Утиліта, успадкована від Windows 7, яка дозволяє створювати заплановані резервні копії файлів користувача та образи системи.

**Можливості:**
- Резервне копіювання за розкладом (щодня, щотижня, щомісяця).
- Вибір конкретних папок для копіювання.
- Створення образу системи.
- Створення диска відновлення системи.

**Як відкрити:** Панель керування → Резервне копіювання та відновлення (Windows 7).

> **Примітка:** Microsoft вважає цей інструмент застарілим, але він досі доступний у Windows 10 та 11.

### Образ системи (System Image)

Повний знімок (snapshot) системного розділу, що включає ОС, встановлені програми, драйвери, налаштування та файли користувача.

**Коли використовувати:**
- Перед великим оновленням Windows.
- Після повного налаштування нового комп'ютера.
- Для швидкого відновлення системи після збою.

**Важливо:** Образ системи відновлює диск цілком — не можна відновити окремий файл.

### Точки відновлення (System Restore)

Механізм збереження стану системних файлів, драйверів і реєстру Windows. Дозволяє "відкотити" систему до попереднього стану, якщо після встановлення програми або оновлення виникли проблеми.

**Особливості:**
- Не впливає на файли користувача (документи, фото тощо).
- Створюється автоматично перед встановленням програм та оновлень.
- Можна створити вручну перед ризикованими змінами.
- У Windows 10/11 може бути вимкнено за замовчуванням — потрібно ввімкнути вручну.

**Як увімкнути:** Панель керування → Система → Захист системи → Налаштувати → Увімкнути захист системи.

### OneDrive — хмарне копіювання

Хмарне сховище від Microsoft, інтегроване у Windows 10/11.

**Можливості для резервного копіювання:**
- Автоматична синхронізація папок Робочий стіл, Документи, Зображення.
- 5 ГБ безкоштовного простору (1 ТБ з підпискою Microsoft 365).
- Доступ до файлів з будь-якого пристрою.
- Кошик OneDrive — відновлення випадково видалених файлів (до 30 днів).
- Захист від шифрувальників — можливість відновити файли на момент до атаки.

## 7. Засоби резервного копіювання Linux

### rsync — інкрементна синхронізація файлів

**rsync** (remote sync) — потужна утиліта командного рядка для синхронізації файлів і каталогів між двома розташуваннями (локальними або віддаленими).

**Ключова перевага:** використовує алгоритм дельта-передачі — копіює лише змінені частини файлів, що значно економить час і трафік.

```bash
# Створити резервну копію домашньої папки на зовнішній диск
rsync -av --delete /home/user/ /media/backup/home/

# Параметри:
# -a (archive) — зберігає права доступу, часові мітки, посилання
# -v (verbose) — показує детальний хід виконання
# --delete — видаляє з копії файли, яких вже немає в оригіналі
```

### tar — архівування та стиснення

**tar** (tape archive) — класична Unix-утиліта для створення архівів. Існує з 1979 року.

```bash
# Створити стиснений архів домашньої папки
tar -czf /media/backup/home-backup.tar.gz /home/user

# Параметри:
# -c (create) — створити архів
# -z (gzip) — стиснути за допомогою gzip
# -f (file) — вказати ім'я файлу архіву
```

### dd — побітове копіювання дисків

**dd** (disk dump) — утиліта для побітового копіювання даних. Створює точну копію диска або розділу, включаючи завантажувач, таблицю розділів та всі дані.

```bash
# Створити образ диска
sudo dd if=/dev/sda of=/media/backup/disk.img bs=4M status=progress

# Параметри:
# if — вхідний файл (джерело)
# of — вихідний файл (копія)
# bs — розмір блоку (впливає на швидкість)
# status=progress — показувати прогрес
```

> **Увага!** Команда `dd` дуже потужна і не запитує підтвердження. Помилка у параметрах `if` та `of` може знищити дані. Будьте дуже уважні!

### Timeshift — точки відновлення для Linux

**Timeshift** — графічна програма, аналог System Restore з Windows. Створює знімки файлової системи для відновлення ОС до попереднього стану.

**Особливості:**
- Два режими: RSYNC (rsync + жорсткі посилання) та BTRFS (знімки файлової системи BTRFS).
- Підтримує заплановані знімки: щогодини, щодня, щотижня, щомісяця.
- Захищає лише системні файли — файли користувача за замовчуванням виключені.
- Можна відновити систему як із запущеної ОС, так і з LiveCD/USB.

### cron — автоматизація за розкладом

**cron** — системна служба Linux для виконання завдань за розкладом (аналог Task Scheduler у Windows).

```bash
# Відкрити таблицю завдань для редагування
crontab -e

# Приклад: щоденна резервна копія о 2:00 ночі
0 2 * * * /home/user/backup-script.sh

# Формат: хвилини години день_місяця місяць день_тижня команда
# *  означає "кожен"
```

**Приклад розкладу:**

| Запис cron | Означає |
|------------|---------|
| `0 2 * * *` | Щодня о 2:00 |
| `0 2 * * 1` | Щопонеділка о 2:00 |
| `0 2 1 * *` | 1-го числа кожного місяця о 2:00 |
| `*/30 * * * *` | Кожні 30 хвилин |

## 8. Хмарне резервне копіювання

Хмарне резервне копіювання — це збереження копій даних на віддалених серверах через Інтернет. Дані зберігаються у дата-центрах провайдера, доступ до них можливий з будь-якого пристрою.

### Порівняння популярних хмарних сервісів

| Сервіс | Безкоштовний обсяг | Платні тарифи | Інтеграція |
|--------|-------------------|---------------|------------|
| **Google Drive** | 15 ГБ | від 100 ГБ за ~$1.99/міс | Android, Chrome, Google Workspace |
| **OneDrive** | 5 ГБ | 1 ТБ з Microsoft 365 | Windows 10/11, Office |
| **Dropbox** | 2 ГБ | від 2 ТБ за ~$11.99/міс | Windows, macOS, Linux, мобільні |

### Переваги хмарного копіювання

- **Захист від локальних катастроф** — дані зберігаються у віддаленому дата-центрі.
- **Доступ з будь-якого пристрою** — потрібен лише Інтернет.
- **Автоматична синхронізація** — зміни зберігаються автоматично.
- **Масштабованість** — легко збільшити обсяг сховища.
- **Версіонування** — можливість повернутися до попередніх версій файлів.

### Недоліки хмарного копіювання

- **Залежність від Інтернету** — без з'єднання доступу до хмари немає.
- **Конфіденційність** — дані зберігаються на серверах сторонньої компанії.
- **Вартість** — великі обсяги даних потребують платної підписки.
- **Швидкість** — завантаження/вивантаження великих обсягів обмежене швидкістю Інтернету.

## 9. Відновлення після збою

### Середовище відновлення Windows (WinRE)

**Windows Recovery Environment (WinRE)** — це набір вбудованих інструментів Windows для усунення неполадок та відновлення системи, коли нормальний запуск неможливий.

**Як увійти в WinRE:**

1. **Автоматично** — Windows сама запускає WinRE, якщо система не змогла завантажитися 2-3 рази поспіль.
2. **Shift + Перезавантаження** — утримуйте Shift і натисніть "Перезавантаження" в меню живлення.
3. **Через Параметри** — Параметри → Система → Відновлення → Розширений запуск → Перезавантажити зараз.
4. **З інсталяційного USB** — завантажтеся з USB-флешки з Windows і виберіть "Відновити комп'ютер".

**Інструменти WinRE:**

| Інструмент | Призначення |
|------------|-------------|
| **Відновлення при запуску** (Startup Repair) | Автоматичне виправлення проблем завантаження |
| **Відновлення системи** (System Restore) | Повернення до попередньої точки відновлення |
| **Відновлення образу системи** (System Image Recovery) | Відновлення з раніше створеного образу |
| **Командний рядок** (Command Prompt) | Ручне виконання команд для діагностики |
| **Видалення оновлень** (Uninstall Updates) | Видалення проблемного оновлення |
| **Параметри UEFI** (UEFI Firmware Settings) | Доступ до налаштувань BIOS/UEFI |

### Безпечний режим (Safe Mode)

**Безпечний режим** — це спеціальний режим завантаження Windows з мінімальним набором драйверів і служб. Використовується для діагностики та усунення проблем.

**Три варіанти безпечного режиму:**

- **Безпечний режим** — мінімальний набір драйверів, без мережі.
- **Безпечний режим із завантаженням мережевих драйверів** — дозволяє використовувати Інтернет.
- **Безпечний режим із командним рядком** — графічний інтерфейс замінений командним рядком.

**Як увійти:** WinRE → Усунення неполадок → Додаткові параметри → Параметри завантаження → Перезавантажити → натисніть F4 (безпечний режим) або F5 (з мережею).

**Коли використовувати безпечний режим:**
- Видалення проблемного драйвера або програми.
- Перевірка на віруси, якщо шкідливе ПЗ блокує антивірус.
- Виправлення проблем із запуском Windows.
- Зміна системних налаштувань, які спричиняють збій.

### Порядок дій при відновленні після серйозного збою

```
Збій системи
     │
     ▼
Спроба звичайного перезавантаження
     │
     ▼ (не допомогло)
Запуск у безпечному режимі
     │
     ▼ (не допомогло)
Відновлення при запуску (Startup Repair)
     │
     ▼ (не допомогло)
Відновлення до точки відновлення (System Restore)
     │
     ▼ (не допомогло)
Відновлення з образу системи (System Image Recovery)
     │
     ▼ (немає образу)
Повне перевстановлення Windows
```
