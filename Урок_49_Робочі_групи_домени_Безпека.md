## 1. Вступ

Коли кілька комп'ютерів об'єднані в мережу, постає питання: як організувати спільну роботу користувачів? Хто має доступ до яких ресурсів? Хто керує обліковими записами? Як захистити мережу від зловмисників?

У Windows існують дві принципово різні моделі організації мережі: **робоча група (Workgroup)** та **домен (Domain)**. Перша — проста й підходить для маленьких мереж, друга — потужна й призначена для організацій із сотнями та тисячами комп'ютерів.

Окрім організації мережі, критично важливим є питання **мережевої безпеки** — захисту даних від перехоплення, несанкціонованого доступу, шкідливого програмного забезпечення та інших загроз. Сьогодні ми розглянемо обидві теми: як організувати мережу та як її захистити.

## 2. Робоча група (Workgroup)

### 2.1. Визначення та принцип роботи

**Робоча група (Workgroup)** — це логічне об'єднання комп'ютерів у локальній мережі, де всі комп'ютери рівноправні. Таку мережу називають **одноранговою (peer-to-peer, P2P)**, оскільки в ній немає виділеного сервера — кожен комп'ютер є одночасно і клієнтом, і сервером.

Уявіть квартиру, де живуть кілька людей. Кожен має свій замок на двері кімнати, свої речі і сам вирішує, що саме дозволити іншим використовувати. Немає «головного», який керує всіма — кожен сам собі господар. Саме так працює робоча група.

**Як це влаштовано технічно:**
- Кожен комп'ютер у робочій групі зберігає **власну базу облікових записів** — так звану **SAM-базу (Security Accounts Manager)**. Це означає, що облікові записи користувачів, паролі та права доступу існують лише на конкретному комп'ютері.
- Якщо вам потрібен доступ до ресурсів трьох різних комп'ютерів, вам потрібно мати три окремі облікові записи — по одному на кожному.
- Усі комп'ютери робочої групи зазвичай підключені до одної локальної мережі (LAN).

### 2.2. Характеристики робочої групи

| Характеристика | Опис |
|---|---|
| **Модель мережі** | Однорангова (P2P) — кожен ПК рівноправний |
| **Облікові записи** | Зберігаються локально на кожному ПК (SAM-база) |
| **Адміністрування** | Кожен користувач — сам собі адміністратор |
| **Обмеження розміру** | Рекомендовано до 10–20 комп'ютерів |
| **Виділений сервер** | Не потрібен |
| **Географія** | Зазвичай одна локальна мережа |
| **Ім'я за замовчуванням** | WORKGROUP (у Windows) |

### 2.3. Переваги та обмеження

**Переваги:**
- Простота налаштування — не потрібен виділений сервер, спеціальне програмне забезпечення чи кваліфікований адміністратор.
- Низька вартість — не потрібно купувати серверне обладнання та ліцензії.
- Підходить для невеликих офісів і домашніх мереж.

**Обмеження:**
- **Немає централізованого управління.** Якщо потрібно змінити пароль або налаштування для 15 комп'ютерів, доведеться робити це окремо на кожному.
- **Погана масштабованість.** Із зростанням кількості ПК адміністрування стає кошмаром — потрібно підтримувати десятки облікових записів на кожному комп'ютері.
- **Обмежена безпека.** Немає єдиної точки контролю доступу. Якщо звільняється працівник, потрібно вручну видалити його обліковий запис на кожному ПК.
- **Немає групових політик (GPO)** — неможливо централізовано задати однакові правила для всіх комп'ютерів.

## 3. Домен (Domain)

### 3.1. Визначення домену

**Домен (Domain)** — це логічна група мережевих об'єктів (комп'ютерів, користувачів, пристроїв), які спільно використовують єдину базу даних **Active Directory** і керуються централізовано.

Якщо робоча група — це квартира зі співмешканцями, то домен — це готель: є адміністратор на рецепції (контролер домену), який видає картки доступу (облікові записи), визначає, хто в який номер може зайти (права доступу), і встановлює загальні правила (групові політики). Гість зі своєю карткою може увійти в будь-які дозволені приміщення.

**Ключові особливості доменної моделі:**
- Мережа побудована за моделлю **«клієнт — сервер»** (client-server).
- Є виділений сервер — **контролер домену (Domain Controller, DC)**, який відповідає за аутентифікацію та авторизацію всіх користувачів.
- Усі облікові записи зберігаються **централізовано** в єдиній базі Active Directory, а не на кожному ПК окремо.
- Користувач із доменним обліковим записом може увійти на **будь-який комп'ютер** у домені зі своїм логіном і паролем.

### 3.2. Контролер домену (Domain Controller)

**Контролер домену (Domain Controller, DC)** — це сервер під управлінням Windows Server, на якому встановлена роль **Active Directory Domain Services (AD DS)**. Це «мозок» доменної мережі.

**Що робить контролер домену:**

| Функція | Опис |
|---|---|
| **Аутентифікація** | Перевіряє, чи справді користувач є тим, за кого себе видає (перевірка логіна і пароля) |
| **Авторизація** | Визначає, до яких ресурсів користувач має доступ і з якими правами |
| **Зберігання бази AD** | Містить усі облікові записи, групи, комп'ютери, налаштування |
| **Реплікація** | Синхронізує дані з іншими контролерами домену (для відмовостійкості) |
| **Застосування політик** | Розповсюджує групові політики (GPO) на всі комп'ютери та користувачів |

> **Важливо:** У реальних мережах завжди використовують щонайменше **два контролери домену**. Якщо один вийде з ладу, другий продовжить обслуговувати мережу. Це називається **відмовостійкість (fault tolerance)**.

### 3.3. Переваги доменної моделі

- **Централізоване управління.** Адміністратор із одного місця керує тисячами облікових записів, встановлює програми, задає політики безпеки.
- **Єдиний обліковий запис (Single Sign-On).** Користувач входить у систему один раз і отримує доступ до всіх дозволених мережевих ресурсів.
- **Масштабованість.** Домен може об'єднувати від кількох десятків до сотень тисяч комп'ютерів по всьому світу.
- **Посилена безпека.** Якщо працівник звільняється — достатньо заблокувати один обліковий запис у AD, і він миттєво втрачає доступ до всіх ресурсів.
- **Групові політики.** Можливість задати єдині правила для всіх (наприклад, вимагати складний пароль, заблокувати USB-порти, налаштувати шпалери робочого столу).

## 4. Порівняльна таблиця: робоча група vs домен

| Критерій | Робоча група (Workgroup) | Домен (Domain) |
|---|---|---|
| **Модель мережі** | Однорангова (P2P) | Клієнт-сервер |
| **Центральний сервер** | Немає | Контролер домену (DC) |
| **Облікові записи** | Локальні на кожному ПК (SAM) | Централізовані в Active Directory |
| **Масштаб** | До 10–20 комп'ютерів | До сотень тисяч комп'ютерів |
| **Географія** | Одна локальна мережа | Може охоплювати весь світ |
| **Групові політики (GPO)** | Не підтримуються | Повна підтримка |
| **Вхід у систему** | Тільки на конкретному ПК | На будь-якому ПК у домені |
| **Безпека** | Мінімальна, кожен ПК окремо | Централізовані політики, аудит, шифрування |
| **Вартість** | Безкоштовно (не потрібен сервер) | Потрібен Windows Server + ліцензії CAL |
| **Складність налаштування** | Мінімальна | Вимагає кваліфікованого адміністратора |

## 5. Active Directory (AD)

### 5.1. Що таке Active Directory

**Active Directory (AD)** — це служба каталогів, розроблена компанією Microsoft, яка входить до складу серверних операційних систем Windows Server. Active Directory являє собою централізовану базу даних, у якій зберігається інформація про всі мережеві ресурси: користувачів, комп'ютери, принтери, спільні папки, групи тощо.

Простими словами, Active Directory — це **«телефонна книга» корпоративної мережі**, в якій записано: хто є хто, що кому дозволено, і як усе організовано.

**Основні можливості AD:**
- Зберігання інформації про всі об'єкти мережі в єдиній базі.
- Аутентифікація та авторизація користувачів і комп'ютерів.
- Централізоване управління правами доступу.
- Розповсюдження групових політик (GPO).
- Організація мережевих ресурсів у зручну ієрархічну структуру.

### 5.2. Об'єкти Active Directory

Усе, що зберігається в Active Directory, називається **об'єктом**. Об'єкти бувають двох типів:
- **Контейнерні об'єкти (container objects)** — можуть містити інші об'єкти всередині себе (наприклад, організаційні підрозділи, групи).
- **Кінцеві об'єкти (leaf objects)** — не можуть містити інших об'єктів, вони лише представляють ресурси (наприклад, користувачі, комп'ютери, принтери).

**Основні типи об'єктів AD:**

| Об'єкт | Тип | Опис | Приклад |
|---|---|---|---|
| **Користувач (User)** | Кінцевий | Обліковий запис людини, описується атрибутами: ім'я, логін, email, посада | Іваненко Олена, логін: o.ivanenko |
| **Комп'ютер (Computer)** | Кінцевий | Комп'ютер або сервер, приєднаний до домену | PC-RECEPTION, SERVER-DB |
| **Група (Group)** | Контейнерний | Об'єднання користувачів / комп'ютерів для спрощення управління правами | Група «Бухгалтерія», група «Адміністратори» |
| **Організаційний підрозділ (OU — Organizational Unit)** | Контейнерний | Контейнер для організації об'єктів; до OU можна прив'язати групові політики | OU «Київський офіс», OU «Студенти» |
| **Принтер (Printer)** | Кінцевий | Мережевий принтер, опублікований у AD | Принтер на 3-му поверсі |

> **Різниця між групою та OU:** Група використовується для **призначення прав доступу** (наприклад, група «Бухгалтерія» має доступ до папки з фінансовими звітами). OU використовується для **організаційної структури і застосування GPO** (наприклад, OU «Київський офіс» має свої налаштування принтерів). Група має ідентифікатор безпеки (SID), а OU — ні.

### 5.3. Ієрархія AD: домен, дерево, ліс

Active Directory має багаторівневу ієрархічну структуру:

```
ЛІС (Forest)
└── ДЕРЕВО (Tree)
    ├── Домен (Domain): company.com
    │   ├── OU "Київ"
    │   │   ├── Користувач: Іваненко О.
    │   │   ├── Комп'ютер: PC-01
    │   │   └── Група: Відділ продажів
    │   └── OU "Львів"
    │       ├── Користувач: Петренко М.
    │       └── Комп'ютер: PC-02
    └── Дочірній домен: branch.company.com
```

- **Домен** — базова одиниця AD. Усі об'єкти належать якомусь домену.
- **Дерево (Tree)** — набір доменів, пов'язаних довірчими відносинами (trust) і спільним простором імен (наприклад, company.com та branch.company.com).
- **Ліс (Forest)** — найвищий рівень ієрархії, що об'єднує одне або кілька дерев. Усі дерева в лісі мають спільну схему каталогу та глобальний каталог.

### 5.4. Протокол LDAP

**LDAP (Lightweight Directory Access Protocol)** — це відкритий стандартний протокол для доступу до служб каталогів і керування ними. Active Directory використовує LDAP як один із основних протоколів для пошуку, зчитування та модифікації об'єктів у каталозі.

**Простими словами:** LDAP — це «мова», якою програми та адміністратори спілкуються з Active Directory, щоб знайти потрібну інформацію.

**Ключові поняття LDAP:**
- **DN (Distinguished Name)** — унікальна повна «адреса» об'єкта в каталозі, подібна до повного шляху файлу. Наприклад: `CN=Іваненко Олена,OU=Київ,DC=company,DC=com`
- **CN (Common Name)** — ім'я об'єкта (наприклад, «Іваненко Олена»).
- **OU (Organizational Unit)** — організаційний підрозділ.
- **DC (Domain Component)** — частина доменного імені.

> **Важливо розуміти:** LDAP — це **протокол** (набір правил комунікації), а Active Directory — це **служба** (конкретна реалізація каталогу від Microsoft). AD використовує LDAP, але LDAP існує і поза AD — наприклад, на серверах Linux з OpenLDAP.

## 6. Групові політики (Group Policy, GPO)

### 6.1. Що таке GPO

**Групова політика (Group Policy)** — це механізм централізованого управління налаштуваннями операційних систем, програм і параметрів безпеки для користувачів і комп'ютерів в Active Directory.

Набір конкретних налаштувань називається **об'єктом групової політики (Group Policy Object, GPO)**. GPO можна прив'язати до домену, сайту або організаційного підрозділу (OU), і всі комп'ютери та користувачі в цьому контейнері автоматично отримають задані налаштування.

**Аналогія:** GPO — це як наказ директора школи: «З понеділка всі учні носять форму, телефони на уроках забороняються.» Один наказ — і правила діють для всіх. Не потрібно підходити до кожного учня окремо.

### 6.2. Що можна налаштувати через GPO

GPO містить два великі розділи:
- **Конфігурація комп'ютера (Computer Configuration)** — налаштування, що стосуються самого ПК, незалежно від того, хто за ним працює (наприклад, правила брандмауера, налаштування оновлень).
- **Конфігурація користувача (User Configuration)** — налаштування, що стосуються конкретного користувача, незалежно від того, за яким ПК він сидить (наприклад, обмеження Панелі керування, налаштування меню Пуск).

**Приклади того, що можна зробити через GPO:**

| Категорія | Приклади налаштувань |
|---|---|
| **Безпека паролів** | Мінімальна довжина пароля — 8 символів, зміна кожні 90 днів, заборона повторення останніх 5 паролів |
| **Блокування облікового запису** | Після 5 невірних спроб — блокування на 30 хвилин |
| **Обмеження робочого столу** | Заборона зміни шпалер, приховання значків, блокування Панелі керування |
| **Встановлення ПЗ** | Автоматичне встановлення Microsoft Office на всі комп'ютери відділу |
| **Налаштування оновлень** | Автоматичне завантаження та встановлення оновлень Windows |
| **Скрипти** | Запуск певних скриптів при вході користувача або включенні ПК |
| **Брандмауер** | Увімкнення брандмауера Windows і налаштування правил |

### 6.3. Порядок застосування GPO

Коли до одного об'єкта (комп'ютера чи користувача) може застосовуватися кілька GPO, вони обробляються у визначеному порядку:

```
1. Локальна політика (Local GPO)   — пріоритет найнижчий
2. Політика сайту (Site GPO)
3. Політика домену (Domain GPO)
4. Політика OU (OU GPO)            — пріоритет найвищий
```

> **Правило:** Остання застосована політика (ближча до об'єкта) має **найвищий пріоритет** і перекриває попередні. Тобто налаштування OU GPO перекриють налаштування Domain GPO, якщо вони конфліктують.

За замовчуванням Windows оновлює групові політики кожні **90 хвилин** (з випадковим відхиленням до 30 хвилин). На контролерах домену — кожні **5 хвилин**.

Для управління GPO використовується **консоль управління груповими політиками (Group Policy Management Console, GPMC)** — графічний інструмент, що дозволяє створювати, редагувати, прив'язувати, резервувати та відновлювати об'єкти групових політик.

## 7. Мережева безпека

### 7.1. Основні загрози

Мережева безпека — це сукупність заходів для захисту мережевої інфраструктури та даних від несанкціонованого доступу, атак і руйнування. Розглянемо основні типи загроз:

**1. Перехоплення даних (Sniffing / Eavesdropping)**
Зловмисник «прослуховує» мережевий трафік і перехоплює дані, що передаються у незашифрованому вигляді: паролі, листи, файли. Особливо небезпечно у відкритих Wi-Fi мережах (кафе, аеропорти).

**2. Несанкціонований доступ (Unauthorized Access)**
Отримання доступу до системи, мережі або даних без дозволу. Причини: слабкі паролі, відсутність оновлень, неправильна конфігурація прав доступу.

**3. Шкідливе програмне забезпечення (Malware)**
Загальна назва для програм, створених із метою завдати шкоди:
- **Віруси** — прикріплюються до файлів і поширюються при їх відкритті.
- **Трояни** — маскуються під корисні програми, але виконують шкідливі дії.
- **Черв'яки (Worms)** — поширюються мережею самостійно, без дій користувача.
- **Програми-вимагачі (Ransomware)** — шифрують файли та вимагають викуп за розшифровку.
- **Шпигунське ПЗ (Spyware)** — таємно збирає інформацію про дії користувача.

**4. DDoS-атака (Distributed Denial of Service)**
Атака, спрямована на виведення з ладу сервера або мережі шляхом перевантаження величезною кількістю запитів. Зловмисник використовує **ботнет** — мережу заражених комп'ютерів (ботів), які одночасно надсилають запити до жертви.

```
Ботнет (тисячі заражених ПК)
    │  │  │  │  │
    ▼  ▼  ▼  ▼  ▼
  ┌───────────────┐
  │  Сервер-жертва │ ← перевантаження → відмова в обслуговуванні
  └───────────────┘
```

Типи DDoS-атак:
- **Volumetric (об'ємні)** — забивають канал зв'язку величезним обсягом трафіку (наприклад, UDP flood).
- **Protocol (протокольні)** — експлуатують вразливості мережевих протоколів (наприклад, SYN flood).
- **Application Layer (рівень додатків)** — імітують легітимні запити до веб-додатків, щоб вичерпати ресурси сервера.

**5. Фішинг (Phishing)**
Вид шахрайства, при якому зловмисник надсилає підроблені електронні листи або створює фальшиві веб-сайти, що імітують справжні (банк, пошту, соціальну мережу), з метою виманити конфіденційні дані: логіни, паролі, номери банківських карток.

**Приклад:** Ви отримуєте лист нібито від «ПриватБанку»: «Ваш рахунок заблоковано! Терміново перейдіть за посиланням.» Посилання веде на фальшивий сайт, який виглядає як справжній, але введені дані потрапляють до зловмисника.

**6. Атака «людина посередині» (Man-in-the-Middle, MITM)**
Зловмисник таємно перехоплює і, можливо, змінює дані, якими обмінюються дві сторони, які думають, що спілкуються безпосередньо одна з одною.

```
Алісa  ←──────→  Зловмисник  ←──────→  Сервер банку
         (думає, що                 (думає, що
         спілкується                спілкується
         з банком)                  з Алісою)
```

Типи MITM-атак:
- **ARP-спуфінг** — підробка ARP-повідомлень для перехоплення трафіку в локальній мережі.
- **DNS-спуфінг** — перенаправлення DNS-запитів на шкідливі сервери.
- **SSL/TLS-стріпінг** — примусове зниження з'єднання з HTTPS до HTTP.
- **Evil Twin (злий близнюк)** — створення підробної точки Wi-Fi з назвою, ідентичною легітимній.

### 7.2. Брандмауер (Firewall)

**Брандмауер (Firewall, міжмережевий екран)** — це система мережевої безпеки, яка контролює та фільтрує вхідний і вихідний мережевий трафік відповідно до заданих правил. Брандмауер є «воротарем» мережі — він вирішує, який трафік пропустити, а який заблокувати.

**Як працює брандмауер:**

```
Інтернет  →  [ БРАНДМАУЕР ]  →  Внутрішня мережа / ПК
                  │
         Перевіряє кожен
         пакет за правилами:
         ✓ Дозволити
         ✗ Заблокувати
```

Брандмауер аналізує кожен мережевий пакет і порівнює його з набором правил:

| Тип правил | Опис |
|---|---|
| **Правила вхідного трафіку (Inbound Rules)** | Контролюють трафік, що **надходить** до комп'ютера з мережі. За замовчуванням — більшість вхідних з'єднань **заблоковано** |
| **Правила вихідного трафіку (Outbound Rules)** | Контролюють трафік, що **йде** від комп'ютера в мережу. За замовчуванням — вихідний трафік **дозволено** |

**Що може перевіряти брандмауер:**
- IP-адресу відправника та одержувача.
- Номер порту (наприклад, порт 80 — HTTP, порт 443 — HTTPS, порт 3389 — RDP).
- Протокол (TCP, UDP, ICMP).
- Програму, яка намагається отримати доступ до мережі.

**Типи брандмауерів:**
- **Програмний** — програма, встановлена на комп'ютері (наприклад, Брандмауер Windows).
- **Апаратний** — окремий пристрій у мережі (наприклад, брандмауер в корпоративному роутері).

### 7.3. Брандмауер Windows — налаштування

**Брандмауер Windows (Windows Defender Firewall)** — це вбудований програмний брандмауер, який входить до складу Windows 10 та Windows 11.

**Як відкрити Брандмауер Windows:**
1. Натиснути **Win + R**, ввести `wf.msc`, натиснути Enter — відкриється вікно **«Брандмауер Windows у режимі підвищеної безпеки»** (Windows Defender Firewall with Advanced Security).
2. Або: **Панель керування → Система та безпека → Брандмауер Windows Defender**.

**Профілі мережі:**
Брандмауер Windows застосовує різні набори правил залежно від типу мережі, до якої підключений комп'ютер:

| Профіль | Коли використовується | Рівень захисту |
|---|---|---|
| **Доменний (Domain)** | Комп'ютер підключений до доменної мережі | Правила, задані адміністратором через GPO |
| **Приватний (Private)** | Домашня або офісна мережа | Середній (дозволено виявлення в мережі) |
| **Загальнодоступний (Public)** | Публічний Wi-Fi (кафе, аеропорт) | Максимальний (заблоковано виявлення в мережі) |

**Створення правила (приклад):**
1. Відкрити `wf.msc`.
2. Натиснути **«Правила для вхідних підключень» (Inbound Rules)** → **«Нове правило» (New Rule)**.
3. Обрати тип: Порт, Програма, Попередньо визначене, Настроюване.
4. Задати параметри (наприклад, заблокувати порт 23 для Telnet).
5. Обрати дію: Дозволити або Заблокувати.
6. Обрати профілі: Доменний / Приватний / Загальнодоступний.
7. Дати правилу зрозуміле ім'я та зберегти.

### 7.4. VPN — віртуальна приватна мережа

**VPN (Virtual Private Network)** — це технологія, що створює зашифрований «тунель» через публічну мережу (Інтернет), забезпечуючи безпечне та конфіденційне з'єднання між пристроєм користувача та віддаленою мережею.

**Простими словами:** VPN — це як секретний підземний тунель між двома будівлями. Навіть якщо хтось стоїть зверху і спостерігає за дорогою (Інтернетом), він не може побачити, хто і що передає цим тунелем, бо всі дані зашифровані.

**Як працює VPN:**

```
┌──────────┐    Зашифрований тунель     ┌──────────────┐
│  Ваш ПК  │ ═══════════════════════════│  VPN-сервер  │──→ Інтернет
│(VPN-клієнт)│    через Інтернет        │              │
└──────────┘                            └──────────────┘
```

1. Програма-клієнт VPN на вашому комп'ютері встановлює зашифроване з'єднання з VPN-сервером.
2. Усі ваші дані шифруються перед відправкою і передаються через «тунель».
3. VPN-сервер розшифровує дані і надсилає їх далі в Інтернет від свого імені.
4. Відповідь з Інтернету повертається на VPN-сервер, шифрується і передається назад вам.

**Навіщо потрібен VPN:**
- **Безпека у публічних мережах.** Захист даних від перехоплення в кафе, аеропорту, готелі.
- **Віддалений доступ до корпоративної мережі.** Працівник із дому може безпечно працювати з корпоративними ресурсами, ніби він фізично в офісі.
- **Конфіденційність.** VPN приховує вашу реальну IP-адресу від сайтів і сервісів.

**Основні протоколи VPN:**

| Протокол | Опис |
|---|---|
| **IPSec** | Працює на мережевому рівні, шифрує IP-пакети. Часто використовується для з'єднання офісів (site-to-site VPN) |
| **SSL/TLS VPN** | Працює через браузер на порту 443 (HTTPS). Не потрібен спеціальний клієнт. Зручний для віддаленого доступу |
| **WireGuard** | Сучасний протокол з мінімальним кодом, високою швидкістю і надійним шифруванням |

### 7.5. Шифрування: SSL/TLS, HTTPS vs HTTP

**Шифрування** — це процес перетворення даних у нечитабельний формат, щоб їх міг прочитати лише той, хто має відповідний ключ дешифрування.

**SSL (Secure Sockets Layer)** та **TLS (Transport Layer Security)** — це криптографічні протоколи, що забезпечують безпеку з'єднань в Інтернеті. SSL був розроблений першим, але у 2015 році його визнали застарілим і замінили на TLS. Сьогодні використовується TLS (версії 1.2 та 1.3), але за традицією часто вживають термін «SSL».

**Як працює TLS-з'єднання (спрощено):**
1. **Клієнт (браузер)** з'єднується з сервером і запитує захищене з'єднання.
2. **Сервер** надсилає свій **цифровий сертифікат**, що підтверджує його справжність.
3. Клієнт перевіряє сертифікат і генерує **сеансовий ключ шифрування**.
4. Далі всі дані передаються у **зашифрованому вигляді**.

**HTTPS vs HTTP:**

| Характеристика | HTTP | HTTPS |
|---|---|---|
| **Повна назва** | HyperText Transfer Protocol | HyperText Transfer Protocol **Secure** |
| **Шифрування** | Немає — дані передаються відкритим текстом | Є — дані зашифровані за допомогою TLS |
| **Порт** | 80 | 443 |
| **Перехоплення даних** | Легко — зловмисник бачить усе | Дуже складно — зловмисник бачить лише шифротекст |
| **Позначка в браузері** | «Не захищено» або знак оклику | Значок замка (🔒) |
| **Де використовується** | Застарілі сайти | Банки, пошта, соцмережі, інтернет-магазини — майже всі сучасні сайти |

> **Правило безпеки:** Ніколи не вводьте паролі, номери карток або іншу конфіденційну інформацію на сайтах без HTTPS (без замочка в адресному рядку браузера).

### 7.6. Антивірусний захист: Windows Defender

**Microsoft Defender Antivirus (Windows Defender)** — це вбудований антивірус, який входить до складу Windows 10 та Windows 11 безкоштовно. За останні роки Microsoft значно покращила його, і сьогодні він забезпечує конкурентний рівень захисту.

**Основні можливості Windows Defender:**

| Функція | Опис |
|---|---|
| **Захист у реальному часі** | Безперервне сканування файлів, програм і завантажень на наявність загроз |
| **Хмарний захист** | Використання хмарних баз Microsoft для виявлення найновіших загроз |
| **Брандмауер і захист мережі** | Вбудований брандмауер з контролем вхідного та вихідного трафіку |
| **SmartScreen** | Захист від шкідливих веб-сайтів, завантажень і програм |
| **Контрольований доступ до папок** | Захист від програм-вимагачів: заборона несанкціонованої зміни файлів у захищених папках |
| **Захист від експлойтів** | Запобігання використанню вразливостей програмного забезпечення |
| **Сканування офлайн** | Можливість сканувати систему навіть без підключення до Інтернету |

**Як відкрити Windows Defender:**
- **Пуск → Параметри → Оновлення та безпека → Безпека Windows** (Windows 10).
- **Пуск → Параметри → Конфіденційність і безпека → Безпека Windows** (Windows 11).
- Або натиснути на значок щита в області сповіщень (системний лоток).

**Що Windows Defender НЕ включає (безкоштовна версія):**
- Вбудований VPN.
- Менеджер паролів.
- Моніторинг витоків даних у Dark Web.
- Захист від крадіжки особистих даних.

> **Висновок:** Windows Defender — це надійний базовий антивірус, якого достатньо для більшості звичайних користувачів. Для організацій з підвищеними вимогами до безпеки можуть знадобитися додаткові комерційні рішення.

## 8. Як перевірити робочу групу або домен на своєму ПК

Існує кілька способів перевірити, до якої робочої групи або домену належить ваш комп'ютер.

### Спосіб 1: Через System Properties (sysdm.cpl)

1. Натиснути **Win + R** (відкриється вікно «Виконати»).
2. Ввести `sysdm.cpl` та натиснути **Enter**.
3. У вікні **«Властивості системи»** на вкладці **«Ім'я комп'ютера»** буде вказано:
   - **Робоча група** (наприклад: `WORKGROUP`) — якщо ПК не входить до домену.
   - **Домен** (наприклад: `corp.company.com`) — якщо ПК приєднаний до домену.

### Спосіб 2: Через командний рядок (CMD)

Відкрити **Командний рядок** (Win + R → `cmd` → Enter) і ввести одну з команд:

```cmd
systeminfo | find "Domain"
```

Ця команда виведе рядок з інформацією про домен. Якщо комп'ютер у робочій групі, значення буде `WORKGROUP`.

```cmd
net config workstation
```

Ця команда покаже ім'я комп'ютера, домен або робочу групу та іншу мережеву інформацію.

### Спосіб 3: Через Параметри Windows

**Параметри → Система → Відомості про пристрій (About)** — у розділі «Специфікації пристрою» буде вказано ім'я пристрою та домен або робочу групу.

Сьогодні ми розглянули два ключові блоки:

**Організація мережі:**
- **Робоча група** — проста однорангова мережа без централізованого управління, підходить для невеликих мереж (до 20 ПК).
- **Домен** — централізована клієнт-серверна модель на основі Active Directory з контролером домену, підходить для організацій будь-якого масштабу.
- **Active Directory** — служба каталогів Microsoft, яка зберігає інформацію про всі об'єкти мережі (користувачі, комп'ютери, групи, OU).
- **Групові політики (GPO)** — потужний інструмент централізованого управління налаштуваннями в домені.

**Мережева безпека:**
- Основні загрози: перехоплення даних, malware, DDoS, фішинг, MITM-атаки.
- **Брандмауер** — фільтрує мережевий трафік за правилами; у Windows — вбудований Брандмауер Windows Defender.
- **VPN** — створює зашифрований тунель через Інтернет для безпечного з'єднання.
- **SSL/TLS і HTTPS** — шифрування даних у веб-з'єднаннях.
- **Windows Defender** — вбудований антивірус Windows, що забезпечує базовий рівень захисту.

## 11. Глосарій ключових термінів

| Термін | Визначення |
|---|---|
| **Робоча група (Workgroup)** | Логічне об'єднання комп'ютерів у одноранговій мережі без централізованого управління |
| **Однорангова мережа (P2P)** | Мережа, в якій усі комп'ютери рівноправні, немає виділеного сервера |
| **SAM (Security Accounts Manager)** | Локальна база облікових записів на кожному комп'ютері Windows |
| **Домен (Domain)** | Логічна група мережевих об'єктів, що спільно використовують єдину базу Active Directory |
| **Контролер домену (Domain Controller, DC)** | Сервер, що керує аутентифікацією, авторизацією та зберіганням бази Active Directory |
| **Active Directory (AD)** | Служба каталогів Microsoft для централізованого зберігання та управління мережевими об'єктами |
| **Організаційний підрозділ (OU)** | Контейнерний об'єкт AD для логічної організації ресурсів і застосування GPO |
| **LDAP (Lightweight Directory Access Protocol)** | Відкритий протокол для доступу до служб каталогів і управління ними |
| **DN (Distinguished Name)** | Унікальна повна «адреса» об'єкта в каталозі Active Directory |
| **GPO (Group Policy Object)** | Об'єкт групової політики — набір централізованих налаштувань для комп'ютерів і користувачів |
| **GPMC (Group Policy Management Console)** | Консоль управління груповими політиками |
| **Брандмауер (Firewall)** | Система мережевої безпеки, що фільтрує вхідний і вихідний трафік за правилами |
| **DDoS (Distributed Denial of Service)** | Розподілена атака відмови в обслуговуванні — перевантаження сервера масою запитів |
| **Фішинг (Phishing)** | Шахрайство з використанням підроблених листів/сайтів для виманювання конфіденційних даних |
| **MITM (Man-in-the-Middle)** | Атака «людина посередині» — перехоплення та можлива зміна даних між двома сторонами |
| **Malware** | Загальна назва шкідливого програмного забезпечення (віруси, трояни, черв'яки тощо) |
| **VPN (Virtual Private Network)** | Технологія створення зашифрованого тунелю через публічну мережу |
| **SSL (Secure Sockets Layer)** | Застарілий криптографічний протокол захисту з'єднань (замінений на TLS) |
| **TLS (Transport Layer Security)** | Сучасний криптографічний протокол шифрування з'єднань в Інтернеті |
| **HTTPS** | HTTP із шифруванням через TLS; безпечна версія HTTP (порт 443) |
| **Windows Defender** | Вбудований безкоштовний антивірус у Windows 10 та Windows 11 |
| **SmartScreen** | Компонент Windows Defender для захисту від шкідливих сайтів і завантажень |
| **Ботнет (Botnet)** | Мережа заражених комп'ютерів, керованих зловмисником для проведення атак |
| **ARP-спуфінг** | Підробка ARP-повідомлень для перехоплення трафіку в локальній мережі |
| **DNS-спуфінг** | Підробка DNS-відповідей для перенаправлення трафіку на шкідливі сервери |
| **Ransomware** | Програма-вимагач, що шифрує файли і вимагає викуп за їх розшифровку |
| **sysdm.cpl** | Команда для швидкого відкриття «Властивостей системи» у Windows |
