## 1. Перші кроки після встановлення ОС

Після завершення встановлення Windows система ще не готова до повноцінної роботи. Потрібно виконати низку обов'язкових кроків, щоб забезпечити стабільність, безпеку та продуктивність комп'ютера.

### Послідовність дій після встановлення:

**Крок 1. Оновлення Windows (Windows Update)**
Перше, що потрібно зробити — перевірити та встановити всі доступні оновлення. Це закриває критичні вразливості безпеки та покращує сумісність з обладнанням.

**Крок 2. Встановлення драйверів пристроїв**
Переконатися, що для всіх апаратних компонентів встановлено правильні драйвери. Windows може встановити базові драйвери автоматично, але для повної функціональності часто потрібні драйвери від виробника.

**Крок 3. Налаштування параметрів безпеки**
- Переконатися, що вбудований антивірус Windows Defender (Microsoft Defender) увімкнений та оновлений.
- Увімкнути брандмауер Windows (Windows Firewall).
- Налаштувати параметри Контролю облікових записів (UAC — User Account Control).
- Створити пароль для облікового запису, якщо його ще немає.

**Крок 4. Створення точки відновлення**
Після того, як система повністю налаштована і працює стабільно, створити точку відновлення. Це дозволить повернути систему до робочого стану, якщо в майбутньому виникнуть проблеми.

**Крок 5. Встановлення необхідного програмного забезпечення**
- Веб-браузер (якщо потрібний інший, ніж вбудований Edge).
- Офісний пакет (Microsoft Office, LibreOffice тощо).
- Архіватор (7-Zip, WinRAR).
- Переглядач PDF (Adobe Acrobat Reader, Foxit Reader).
- Спеціалізовані програми за потребою.

> **Важливо!** Програми слід завантажувати лише з офіційних сайтів виробників або перевірених джерел (наприклад, Microsoft Store). Завантаження з неперевірених сайтів загрожує встановленням шкідливого ПЗ.

## 2. Оновлення Windows (Windows Update)

### Чому оновлення критично важливі?

Windows Update — це служба автоматичного оновлення операційної системи від Microsoft. Вона забезпечує:

1. **Безпеку** — виправлення вразливостей (security patches). Щомісяця Microsoft випускає оновлення безпеки, що закривають виявлені «діри» в системі. Без цих оновлень комп'ютер вразливий до вірусів, троянів та хакерських атак.
2. **Стабільність** — виправлення помилок (bug fixes). Оновлення усувають збої, зависання та інші проблеми в роботі системи.
3. **Драйвери** — Windows Update автоматично знаходить та встановлює драйвери для багатьох пристроїв.
4. **Нові функції** — великі оновлення (feature updates) додають нові можливості.

### Як запустити Windows Update:

1. Відкрити **Параметри** (Settings): натиснути `Win + I`.
2. Перейти до розділу **Оновлення та безпека** (Update & Security) у Windows 10 або **Windows Update** у Windows 11.
3. Натиснути **Перевірити наявність оновлень** (Check for updates).
4. Дочекатися завантаження та встановлення оновлень.
5. За потреби перезавантажити комп'ютер.

> **Рекомендація:** Після свіжого встановлення Windows виконуйте перевірку оновлень кілька разів поспіль, оскільки деякі оновлення залежать від попередніх і стають доступними лише після їх встановлення.

## 3. Драйвери пристроїв — поглиблений розгляд

### Що таке драйвер (нагадування)

**Драйвер (driver)** — це спеціальна програма, яка забезпечує взаємодію операційної системи з конкретним апаратним пристроєм. Драйвер виступає «перекладачем» між ОС та обладнанням.

### Як ОС взаємодіє з апаратурою через драйвери

Схема взаємодії виглядає так:

```
Прикладна програма
       |
       v
Операційна система (ядро)
       |
       v
    Драйвер
       |
       v
Апаратний пристрій
```

**Детальніше:**

1. Прикладна програма надсилає запит до ОС (наприклад, «відтворити звук»).
2. Ядро ОС приймає запит і передає його відповідному драйверу (аудіо-драйверу).
3. Драйвер «перекладає» стандартну команду ОС у специфічні інструкції для конкретного пристрою (наприклад, звукової карти Realtek ALC897).
4. Пристрій виконує команду (відтворює звук через динаміки).
5. Результат передається у зворотному напрямку.

**Навіщо це потрібно?** Існують тисячі різних моделей апаратних пристроїв. Завдяки драйверам ОС не повинна «знати» особливості кожного конкретного пристрою — вона працює через стандартний інтерфейс, а драйвер бере на себе всю специфіку обладнання. Це дозволяє одній ОС працювати на мільйонах різних комп'ютерів з різною конфігурацією.

## 4. Типи драйверів: kernel-mode та user-mode

Процесор комп'ютера працює у двох режимах, і драйвери також поділяються на два типи залежно від того, в якому режимі вони працюють.

### Драйвери режиму ядра (Kernel-mode drivers)

- Працюють у **привілейованому режимі процесора** (kernel mode).
- Мають **прямий необмежений доступ** до апаратного забезпечення: пам'яті, процесора, пристроїв введення-виведення.
- Працюють у тому ж адресному просторі, що й ядро ОС.
- **Переваги:** висока швидкість роботи, прямий доступ до обладнання.
- **Недоліки:** помилка в такому драйвері може призвести до краху всієї системи (синій екран смерті — BSOD).
- **Приклади:** драйвери відеокарт (GPU), мережевих адаптерів, контролерів дисків, антивірусне ПЗ рівня ядра.

> **Реальний приклад:** У липні 2024 року помилка в оновленні драйвера безпеки CrowdStrike, що працював у режимі ядра, призвела до масового збою мільйонів комп'ютерів з Windows по всьому світу. Це наочно демонструє, наскільки критичними є драйвери режиму ядра.

### Драйвери режиму користувача (User-mode drivers)

- Працюють у **непривілейованому режимі** (user mode).
- Мають **обмежений доступ** до апаратури — взаємодіють через системні API (програмні інтерфейси).
- Працюють в **ізольованому адресному просторі** — окремо від ядра ОС.
- **Переваги:** безпечніші — якщо такий драйвер зазнає збою, він не «обвалить» всю систему, а лише перестане працювати сам.
- **Недоліки:** нижча продуктивність через необхідність «перемикання» між режимами.
- **Приклади:** драйвери принтерів, сканерів, деяких USB-пристроїв.

### Порівняльна таблиця

| Характеристика | Kernel-mode (режим ядра) | User-mode (режим користувача) |
|---|---|---|
| Доступ до апаратури | Прямий, необмежений | Через системні API |
| Швидкість | Висока | Нижча |
| Стабільність системи | Ризик краху ОС при помилці | Збій ізольований |
| Адресний простір | Спільний з ядром ОС | Окремий, ізольований |
| Складність розробки | Висока | Нижча |
| Приклади пристроїв | Відеокарти, мережеві карти, диски | Принтери, сканери, USB |

## 5. Підписані драйвери (Driver Signing)

### Що таке підпис драйвера?

**Цифровий підпис драйвера (Driver Signature)** — це електронна «печатка», яка підтверджує:
1. **Автентичність** — драйвер справді створений заявленим виробником.
2. **Цілісність** — файл драйвера не був змінений або пошкоджений після підписання.

### Як це працює?

Виробник драйвера надсилає його до Microsoft для тестування та підписання. Microsoft перевіряє драйвер на сумісність та безпеку, після чого додає цифровий підпис. Операційна система перед встановленням будь-якого драйвера перевіряє наявність цього підпису.

### Навіщо потрібне підписання драйверів?

**Driver Signature Enforcement (DSE)** — примусова перевірка підпису драйверів — є критично важливою функцією безпеки Windows:

1. **Захист від шкідливого ПЗ.** Без підпису зловмисники могли б замаскувати вірус під драйвер і отримати доступ до ядра системи. Підписання блокує такі спроби.
2. **Захист від руткітів.** Руткіти (rootkits) — це шкідливі програми, що ховаються глибоко в системі. Вони часто намагаються діяти як драйвери, щоб отримати доступ до ядра. DSE суттєво ускладнює це.
3. **Запобігання встановленню пошкоджених драйверів.** Якщо файл драйвера був пошкоджений при завантаженні або змінений вірусом, підпис не пройде перевірку, і Windows заблокує встановлення.
4. **Зменшення людського фактору.** Раніше система лише показувала попередження «Ви хочете встановити цей файл?», і користувач часто натискав «Так», не розуміючи ризику. Тепер система автоматично блокує небезпечні драйвери.

### Що відбувається при спробі встановити непідписаний драйвер?

- У **64-бітних версіях** Windows 10/11 непідписані драйвери режиму ядра **не можуть бути завантажені** за замовчуванням.
- Система показує повідомлення: «Windows не може перевірити видавця цього драйверного ПЗ».
- Вимкнення DSE можливе, але **категорично не рекомендується** — це робить систему вразливою.

> **Для учня:** Ніколи не вимикайте перевірку підпису драйверів. Якщо драйвер не має підпису — це привід не встановлювати його, а шукати офіційний підписаний драйвер від виробника.

## 6. Generic та vendor-specific драйвери

### Generic (стандартні) драйвери

- **Що це:** Універсальні драйвери, що входять до складу Windows або завантажуються через Windows Update.
- **Створені:** Microsoft або спільнотою розробників.
- **Переваги:** Встановлюються автоматично, забезпечують базову функціональність.
- **Недоліки:** Можуть не підтримувати всі функції пристрою, продуктивність може бути нижчою.
- **Приклад:** Після встановлення Windows відеокарта працює зі стандартним драйвером «Microsoft Basic Display Adapter» — екран працює, але роздільна здатність обмежена, немає 3D-прискорення, відео гальмує.

### Vendor-specific (від виробника) драйвери

- **Що це:** Драйвери, розроблені конкретним виробником пристрою для конкретної моделі.
- **Створені:** Виробником обладнання (NVIDIA, AMD, Intel, Realtek, Broadcom тощо).
- **Переваги:** Повна підтримка всіх функцій пристрою, оптимальна продуктивність, додаткові утиліти налаштування.
- **Недоліки:** Потрібно завантажувати та встановлювати вручну; необхідно стежити за оновленнями.
- **Приклад:** Драйвер NVIDIA GeForce Game Ready Driver для відеокарти — підтримує повну роздільну здатність, 3D-прискорення, технології DLSS, Ray Tracing, а також включає панель налаштувань NVIDIA Control Panel.

### Порівняння на прикладі

| Ситуація | Generic драйвер | Vendor-specific драйвер |
|---|---|---|
| Відеокарта NVIDIA | Роздільна здатність 1024x768, немає 3D | Повна роздільна здатність, 3D, ігри |
| Звукова карта Realtek | Базовий звук, без еквалайзера | Повний звук, еквалайзер, ефекти |
| Тачпад Synaptics | Базові рухи курсора | Жести, прокрутка, мультитач |
| Wi-Fi адаптер Intel | Базове з'єднання | Розширені налаштування, Bluetooth |

## 7. Де знайти драйвери

### 7.1. Windows Update (автоматичний пошук)

Найпростіший і найбезпечніший спосіб. Windows автоматично шукає та встановлює драйвери для виявлених пристроїв. Сучасні версії Windows 10/11 досить ефективно знаходять драйвери для більшості обладнання.

**Як перевірити:** Параметри → Оновлення та безпека → Windows Update → Перевірити наявність оновлень → Додаткові оновлення (Optional updates) — тут часто містяться оновлення драйверів.

### 7.2. Сайт виробника пристрою або ноутбука

**Найнадійніший спосіб** отримати правильний драйвер від виробника:

- **Для ноутбуків та ПК від виробника:** Перейти на сайт підтримки (Dell, HP, Lenovo, ASUS, Acer тощо), ввести модель пристрою або серійний номер, завантажити драйвери для вашої версії Windows.
- **Для окремих компонентів:** Перейти на сайт виробника компонента (nvidia.com, amd.com, intel.com, realtek.com тощо).

**Приклади сайтів:**
- Dell: dell.com/support
- HP: support.hp.com
- Lenovo: support.lenovo.com
- ASUS: asus.com/ua/support
- NVIDIA: nvidia.com/drivers
- AMD: amd.com/en/support
- Intel: intel.com/content/www/us/en/download-center

> **Важливо!** Завжди переконуйтесь, що ви на **офіційному сайті** виробника. Зловмисники часто створюють фальшиві сайти, що імітують сайти виробників.

### 7.3. Диспетчер пристроїв (Device Manager)

Дозволяє оновити драйвер конкретного пристрою прямо із системи (детальніше — у розділі 8).

### 7.4. Програми для автоматичного пошуку драйверів

Існують сторонні програми, що обіцяють автоматично знайти та встановити всі драйвери: **DriverPack Solution**, **Snappy Driver Installer**, **Driver Booster** тощо.

**Ризики використання таких програм:**

| Ризик | Опис |
|---|---|
| **Шкідливе ПЗ** | Деякі програми містять рекламне ПЗ (adware), небажані програми (PUP), а іноді й віруси |
| **Обманливий інтерфейс** | Інтерфейс спеціально заплутаний, щоб користувач випадково встановив зайве ПЗ |
| **Ненадійні джерела драйверів** | Драйвери беруться не від виробника — немає гарантії їх якості та безпеки |
| **Нестабільність системи** | Встановлення неправильного або несумісного драйвера може призвести до збоїв |
| **Антивіруси блокують** | Багато антивірусних програм (Malwarebytes, Kaspersky, ESET) визначають ці програми як потенційно небажані |

> **Рекомендація:** IT-фахівці не радять використовувати сторонні програми для автоматичного пошуку драйверів. Набагато безпечніше скористатися Windows Update або завантажити драйвери з офіційного сайту виробника.

## 8. Диспетчер пристроїв (Device Manager)

### Що таке Диспетчер пристроїв?

**Диспетчер пристроїв (Device Manager)** — це вбудований інструмент Windows для перегляду та керування всіма апаратними пристроями комп'ютера та їхніми драйверами.

### Як відкрити Диспетчер пристроїв

Є кілька способів:

1. **Через команду:** натиснути `Win + R`, ввести `devmgmt.msc`, натиснути Enter.
2. **Через контекстне меню «Пуск»:** натиснути `Win + X`, вибрати «Диспетчер пристроїв» (Device Manager).
3. **Через пошук:** ввести «Диспетчер пристроїв» у рядку пошуку Windows.
4. **Через Панель керування:** Панель керування → Обладнання та звук → Диспетчер пристроїв.

### Категорії пристроїв

Усі пристрої в Диспетчері згруповані за категоріями:

| Категорія | Що містить |
|---|---|
| Відеоадаптери (Display adapters) | Відеокарти (GPU) |
| Звукові, ігрові та відео пристрої (Sound, video and game controllers) | Звукові карти, мікрофони |
| Мережеві адаптери (Network adapters) | Wi-Fi, Ethernet, Bluetooth адаптери |
| Дискові накопичувачі (Disk drives) | Жорсткі диски, SSD |
| Контролери USB (Universal Serial Bus controllers) | USB-порти та підключені пристрої |
| Клавіатури (Keyboards) | Клавіатура |
| Миші та інші вказівні пристрої (Mice and other pointing devices) | Миша, тачпад |
| Монітори (Monitors) | Підключені монітори |
| Процесори (Processors) | Центральний процесор (CPU) |
| Системні пристрої (System devices) | Системні контролери, чіпсет |

### Значки стану пристроїв

| Значок | Значення |
|---|---|
| Звичайна піктограма (без позначок) | Пристрій працює нормально |
| **Жовтий трикутник з окличним знаком** (⚠) | Проблема з пристроєм — драйвер не встановлений, пошкоджений або конфліктує |
| **Знак питання** (?) | Невідомий пристрій — Windows не змогла визначити пристрій і не знайшла для нього драйвер |
| **Червоний хрестик** | Пристрій вимкнений (відключений вручну) |
| **Стрілка вниз** | Пристрій відключений |

### Дії з драйверами в Диспетчері пристроїв

Щоб виконати дії з драйвером, потрібно натиснути правою кнопкою миші на пристрій і вибрати **«Властивості»** (Properties), потім перейти на вкладку **«Драйвер»** (Driver).

**Доступні дії:**

1. **Оновити драйвер (Update Driver)**
   - «Автоматичний пошук драйверів» — Windows шукає драйвер в інтернеті та на комп'ютері.
   - «Виконати пошук драйверів на цьому комп'ютері» — вказати папку, куди ви попередньо завантажили драйвер.

2. **Відкотити драйвер (Roll Back Driver)**
   - Повертає попередню версію драйвера. Корисно, якщо нова версія драйвера спричинила проблеми. Кнопка доступна лише якщо раніше було встановлено іншу версію.

3. **Видалити пристрій (Uninstall Device)**
   - Видаляє драйвер пристрою. Після перезавантаження Windows спробує знайти та встановити драйвер заново.

4. **Вимкнути пристрій (Disable Device)**
   - Тимчасово вимикає пристрій без видалення драйвера.

### Перегляд властивостей пристрою

У вікні **Властивості** пристрою є кілька вкладок:

- **Загальне (General)** — стан пристрою, чи працює він справно.
- **Драйвер (Driver)** — інформація про драйвер: постачальник, дата, версія, цифровий підпис.
- **Відомості (Details)** — детальна технічна інформація, зокрема **Hardware ID** — унікальний ідентифікатор пристрою, що допомагає знайти потрібний драйвер.
- **Ресурси (Resources)** — апаратні ресурси, що використовує пристрій (переривання, адреси пам'яті).
- **Події (Events)** — журнал подій, пов'язаних із пристроєм.

## 9. Типові проблеми після встановлення ОС та їх вирішення

Після свіжого встановлення Windows користувачі часто стикаються з кількома характерними проблемами, майже всі з яких пов'язані з відсутніми драйверами.

### Проблема 1: Немає звуку

| | Опис |
|---|---|
| **Симптом** | Звук не відтворюється, значок динаміка в треї може мати хрестик |
| **Причина** | Не встановлений аудіо-драйвер (зазвичай Realtek High Definition Audio) |
| **Вирішення** | 1) Запустити Windows Update. 2) Відкрити Диспетчер пристроїв → Звукові пристрої → Оновити драйвер. 3) Завантажити драйвер з сайту виробника ноутбука/материнської плати |

### Проблема 2: Низька роздільна здатність екрана

| | Опис |
|---|---|
| **Симптом** | Зображення розмите, роздільна здатність лише 1024x768 або 800x600, елементи інтерфейсу великі |
| **Причина** | Працює стандартний драйвер «Microsoft Basic Display Adapter» замість драйвера відеокарти |
| **Вирішення** | 1) Визначити модель відеокарти (Intel, NVIDIA, AMD). 2) Завантажити драйвер з офіційного сайту: intel.com, nvidia.com або amd.com. 3) Встановити та перезавантажити комп'ютер |

### Проблема 3: Не працює Wi-Fi

| | Опис |
|---|---|
| **Симптом** | Немає списку доступних мереж Wi-Fi, значок мережі з хрестиком |
| **Причина** | Не встановлений драйвер бездротового мережевого адаптера (Wi-Fi) |
| **Вирішення** | 1) Підключити комп'ютер до інтернету через кабель Ethernet (якщо можливо). 2) Запустити Windows Update. 3) Або: завантажити драйвер Wi-Fi з іншого комп'ютера на флешку, перенести та встановити |
| **Порада** | Перед встановленням ОС корисно заздалегідь завантажити драйвер Wi-Fi та мережевої карти на USB-накопичувач |

### Проблема 4: Не працює тачпад (на ноутбуці)

| | Опис |
|---|---|
| **Симптом** | Тачпад не реагує на дотик або працює лише базове переміщення курсора (без жестів) |
| **Причина** | Не встановлений драйвер тачпада від виробника (Synaptics, ELAN, Alps тощо) |
| **Вирішення** | Завантажити драйвер тачпада з сайту виробника ноутбука для конкретної моделі |

### Проблема 5: Невідомі пристрої в Диспетчері пристроїв

| | Опис |
|---|---|
| **Симптом** | У Диспетчері пристроїв є пристрої з жовтим знаком оклику або знаком питання |
| **Вирішення** | 1) Натиснути правою кнопкою на невідомий пристрій → Властивості → Відомості → Hardware ID. 2) Скопіювати ідентифікатор. 3) Знайти драйвер за цим ідентифікатором на сайті виробника або через пошукову систему |

## 10. Точки відновлення (System Restore)

### Що таке точка відновлення?

**Точка відновлення (Restore Point)** — це «знімок» стану системи на певний момент часу. Вона зберігає:
- Системні файли Windows.
- Реєстр Windows (де зберігаються налаштування системи та програм).
- Встановлені драйвери.
- Налаштування системи.

**Що НЕ зберігається:** особисті файли користувача (документи, фото, відео, музика).

### Навіщо потрібні точки відновлення?

Якщо після встановлення програми, драйвера або оновлення система почала працювати нестабільно, можна «відкотити» стан системи до точки відновлення, створеної до цих змін. При цьому ваші особисті файли не будуть видалені, але програми та драйвери, встановлені після точки відновлення, буде видалено.

### Як увімкнути захист системи

1. Натиснути `Win + R`, ввести `sysdm.cpl`, натиснути Enter.
2. Перейти на вкладку **«Захист системи»** (System Protection).
3. Вибрати системний диск (зазвичай C:) і натиснути **«Налаштувати»** (Configure).
4. Вибрати **«Увімкнути захист системи»** (Turn on system protection).
5. Встановити максимальний обсяг дискового простору для точок відновлення (рекомендовано 5–10% від розміру диска).
6. Натиснути OK.

### Як створити точку відновлення вручну

1. Відкрити **Захист системи** (як описано вище).
2. Натиснути кнопку **«Створити»** (Create).
3. Ввести описову назву (наприклад, «Після встановлення всіх драйверів»).
4. Натиснути **«Створити»** і дочекатися завершення.

### Як відновити систему з точки відновлення

1. Відкрити **Захист системи** → натиснути **«Відновлення системи»** (System Restore).
2. Вибрати потрібну точку відновлення зі списку.
3. Натиснути **«Далі»** (Next), підтвердити і дочекатися завершення.
4. Комп'ютер перезавантажиться.

> **Зверніть увагу:** Точки відновлення також створюються автоматично перед встановленням оновлень Windows, програм та драйверів (якщо захист системи увімкнений).

### Відновлення, якщо Windows не завантажується

Якщо система не завантажується, можна скористатися точкою відновлення через **середовище відновлення Windows (Windows RE)**:
1. Завантажити комп'ютер з інсталяційного USB/DVD.
2. Вибрати **«Відновлення системи»** (Repair your computer).
3. Перейти до **Усунення неполадок → Додаткові параметри → Відновлення системи** (Troubleshoot → Advanced options → System Restore).

## 11. Активація Windows

### Що таке активація Windows?

**Активація** — це процес підтвердження того, що ваша копія Windows є ліцензійною (легальною). Активація прив'язує ліцензійний ключ до конкретного комп'ютера.

### Навіщо потрібна активація?

- Без активації Windows працює з обмеженнями: водяний знак на робочому столі, неможливість змінити фоновий малюнок, обмежені налаштування персоналізації.
- Ліцензійна копія гарантує отримання оновлень безпеки.
- Використання неактивованої (піратської) Windows є порушенням ліцензійної угоди.

### Як перевірити стан активації

**Спосіб 1: Через Параметри (Settings)**
1. Натиснути `Win + I` → **Система** (System) → **Активація** (Activation).
2. Якщо Windows активовано, буде напис «Windows активовано» з зеленою позначкою.

**Спосіб 2: Через командний рядок (команда slmgr)**
1. Відкрити командний рядок від імені адміністратора (натиснути `Win + X` → «Термінал Windows (Адміністратор)»).
2. Ввести команду:
   ```
   slmgr /xpr
   ```
3. З'явиться вікно з інформацією:
   - «Комп'ютер активовано на постійній основі» — Windows активовано назавжди.
   - Або буде показано дату закінчення активації.

**Додаткові команди:**
- `slmgr /dli` — базова інформація про ліцензію (тип ліцензії, стан активації).
- `slmgr /dlv` — детальна інформація про ліцензію (ідентифікатор активації, кількість переактивацій тощо).

### Типи ліцензій Windows

| Тип ліцензії | Опис |
|---|---|
| **OEM** | Ліцензія від виробника комп'ютера, прив'язана до конкретного пристрою. Зазвичай вже активована при покупці нового ПК |
| **Retail (коробкова)** | Купується окремо, може бути перенесена на інший комп'ютер |
| **Volume (корпоративна)** | Для організацій, що купують ліцензії масово |
| **Цифрова ліцензія** | Прив'язана до облікового запису Microsoft, активується автоматично при вході |

## 13. Глосарій

| Термін | Визначення |
|---|---|
| **Драйвер (Driver)** | Спеціальна програма, що забезпечує взаємодію операційної системи з конкретним апаратним пристроєм, «перекладаючи» команди ОС у інструкції, зрозумілі пристрою |
| **Диспетчер пристроїв (Device Manager)** | Вбудований інструмент Windows для перегляду, керування та діагностики всіх апаратних пристроїв і їхніх драйверів. Відкривається командою `devmgmt.msc` |
| **Windows Update** | Служба автоматичного оновлення Windows, що забезпечує завантаження та встановлення оновлень безпеки, виправлень помилок, оновлення драйверів та нових функцій |
| **Kernel-mode (режим ядра)** | Привілейований режим роботи процесора з повним доступом до апаратних ресурсів. Драйвери цього режиму працюють у спільному адресному просторі з ядром ОС |
| **User-mode (режим користувача)** | Непривілейований режим роботи процесора з обмеженим доступом до апаратури. Драйвери цього режиму працюють в ізольованому адресному просторі |
| **Driver Signing (підпис драйвера)** | Цифровий підпис, що підтверджує автентичність та цілісність драйвера. Microsoft перевіряє та підписує драйвери перед їх розповсюдженням |
| **DSE (Driver Signature Enforcement)** | Механізм Windows, що примусово перевіряє цифровий підпис драйверів перед їх завантаженням, блокуючи непідписані драйвери |
| **Generic драйвер** | Стандартний (універсальний) драйвер, що входить до складу Windows та забезпечує базову функціональність пристрою |
| **Vendor-specific драйвер** | Драйвер, розроблений виробником конкретного пристрою, що забезпечує повну функціональність та оптимальну продуктивність |
| **BSOD (Blue Screen of Death)** | «Синій екран смерті» — критична помилка Windows, що часто виникає через збій драйвера режиму ядра. Вимагає перезавантаження комп'ютера |
| **Точка відновлення (Restore Point)** | «Знімок» стану системи (системних файлів, реєстру, драйверів) на певний момент часу, що дозволяє відкотити систему до робочого стану |
| **System Restore (Відновлення системи)** | Вбудована функція Windows для повернення системи до попереднього стану за допомогою точок відновлення |
| **Активація Windows** | Процес підтвердження ліцензійності копії Windows шляхом прив'язки ліцензійного ключа до конкретного комп'ютера |
| **slmgr** | Software Licensing Management Tool — утиліта командного рядка Windows для керування ліцензуванням та активацією |
| **Hardware ID** | Унікальний ідентифікатор апаратного пристрою, що дозволяє точно визначити модель пристрою та знайти для нього потрібний драйвер |
| **OEM (Original Equipment Manufacturer)** | Виробник оригінального обладнання. OEM-ліцензія Windows прив'язана до комп'ютера, з яким була продана |
| **PUP (Potentially Unwanted Program)** | Потенційно небажана програма — ПЗ, яке може встановлюватися разом з корисними програмами без явної згоди користувача (рекламне ПЗ, панелі інструментів тощо) |
| **UAC (User Account Control)** | Контроль облікових записів — функція безпеки Windows, що запитує підтвердження перед виконанням дій, які потребують прав адміністратора |
| **Руткіт (Rootkit)** | Шкідлива програма, що маскує свою присутність у системі, часто працюючи на рівні ядра ОС для уникнення виявлення антивірусами |
