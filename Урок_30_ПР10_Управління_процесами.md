## 1. Мета роботи

- Закріпити навички роботи з Диспетчером завдань (Task Manager) на поглибленому рівні.
- Навчитися аналізувати процеси через вкладку «Деталі»: PID, стан, пріоритет, кількість потоків, CPU Time.
- Оволодіти практичними навичками зміни пріоритетів процесів та спостереження за впливом на продуктивність.
- Навчитися створювати штучне навантаження на CPU та аналізувати графіки продуктивності.
- Використовувати команди `tasklist`, `taskkill` та `wmic` у командному рядку (CMD).
- Ознайомитися з Монітором ресурсів (resmon) та командлетами PowerShell для управління процесами.

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Клавіатура та миша.
- Командний рядок (CMD).
- Windows PowerShell.
- Веббраузер (Microsoft Edge, Google Chrome або Mozilla Firefox).

---

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не завершуйте системні процеси Windows** (`csrss.exe`, `lsass.exe`, `winlogon.exe`, `smss.exe`, `services.exe`) — це може призвести до «синього екрану» (BSOD) або примусового перезавантаження.
4. **Не змінюйте пріоритет на «Реального часу» (Realtime)** без дозволу викладача — система може стати нереагуючою.
5. При роботі з `taskkill` завершуйте лише ті процеси, які ви самі запустили (Блокнот, Калькулятор тощо).
6. Не зупиняйте системні служби Windows.
7. Після завершення роботи закрийте всі програми, запущені під час практичної, та приведіть робоче місце до початкового стану.

## 4. Короткий теоретичний вступ

### 4.1. Процеси, потоки, пріоритети — нагадування

**Процес** — це програма у стані виконання. Кожен процес має власний адресний простір в оперативній пам'яті, унікальний ідентифікатор (PID) та один або більше потоків.

**Потік (Thread)** — це найменша одиниця виконання всередині процесу. Саме потокам операційна система виділяє процесорний час. Один процес може мати десятки та сотні потоків.

**Пріоритет** визначає, скільки процесорного часу ОС виділяє процесу порівняно з іншими. Шість рівнів пріоритету (від найвищого до найнижчого):

| Рівень | Англійська назва | Числове значення | Застосування |
|--------|------------------|------------------|--------------|
| Реального часу | Realtime | 24 | Лише для критичних системних задач. **Небезпечно!** |
| Високий | High | 13 | Задачі, що потребують максимуму ресурсів |
| Вище за нормальний | Above Normal | 10 | Трохи більше ресурсів, ніж звичайним |
| Нормальний | Normal | 8 | Стандартний для більшості програм |
| Нижче за нормальний | Below Normal | 6 | Фонові задачі, що не потребують швидкої реакції |
| Низький | Low (Idle) | 4 | Мінімум ресурсів — лише коли CPU вільний |

> **Нагадування з ПР №4:** Зміна пріоритету діє лише до перезавантаження комп'ютера. Після перезапуску всі пріоритети повертаються до значення «Нормальний».

---

### 4.2. Стани процесу

Кожен процес у будь-який момент часу перебуває в одному зі станів:

| Стан | Англійська назва | Опис |
|------|------------------|------|
| **Виконується** | Running | Процес активний, його потоки отримують процесорний час |
| **Призупинено** | Suspended | Процес тимчасово заморожений ОС (наприклад, UWP-додаток у фоні) |
| **Не відповідає** | Not Responding | Процес не обробляє повідомлення Windows понад 5 секунд |

У вкладці «Деталі» Диспетчера завдань стан кожного процесу відображається у стовпці **«Стан» (Status)**.

> **Що таке Suspended?** Windows 10/11 може автоматично призупиняти фонові UWP-додатки (наприклад, Калькулятор, Погода, Карти) для економії ресурсів. Такі процеси не споживають CPU, але залишаються в пам'яті.

---

### 4.3. Дерево процесів — батьківські та дочірні

Коли один процес створює інший, він стає **батьківським** (Parent), а новий процес — **дочірнім** (Child). Це створює **ієрархію** — дерево процесів.

**Приклад ієрархії:**
```
explorer.exe (PID 5120) — батьківський
├── notepad.exe (PID 7344) — дочірній
├── cmd.exe (PID 8012) — дочірній
│   └── tasklist.exe (PID 9100) — дочірній (онук explorer.exe)
└── chrome.exe (PID 6200) — дочірній
    ├── chrome.exe (PID 6204) — вкладка 1
    ├── chrome.exe (PID 6208) — вкладка 2
    └── chrome.exe (PID 6212) — GPU-процес
```

**Parent Process ID (PPID)** — це PID батьківського процесу. За допомогою PID та PPID можна відтворити дерево процесів і зрозуміти, хто кого створив.

Переглянути дерево процесів можна командою:
```
wmic process get processid,parentprocessid,name
```

### 4.4. CPU Time та реальне навантаження

**CPU Time** (час ЦП) — це сумарний час, протягом якого процесор виконував інструкції конкретного процесу з моменту його запуску.

**Різниця між CPU % та CPU Time:**

| Показник | Що означає | Приклад |
|----------|------------|---------|
| **CPU %** | Поточне навантаження на процесор у цю мить | Chrome: 15% (зараз активно обробляє сторінку) |
| **CPU Time** | Загальний час CPU за весь час роботи процесу | Chrome: 00:05:23 (за 2 години роботи використав 5 хвилин процесорного часу) |

CPU Time видно на вкладці «Деталі» Диспетчера завдань. Щоб побачити цей стовпець, може знадобитися додати його: правий клік на заголовку будь-якого стовпця --> «Вибрати стовпці» --> позначити «Час ЦП» (CPU Time).

> **Чому CPU Time важливий?** Процес може мати CPU% = 0% у конкретну мить (нічого не робить), але мати великий CPU Time, що означає — за весь час він активно навантажував процесор. Це корисно для виявлення програм, що «з'їдають» ресурси протягом тривалого часу.

## 5. Практичні завдання

### Завдання 1. Відкрити Диспетчер завдань і перемкнути в розширений режим

1. Відкрийте Диспетчер завдань комбінацією клавіш **Ctrl + Shift + Esc**.
2. Якщо Диспетчер відкрився у **спрощеному режимі** (видно лише список застосунків без деталей), натисніть кнопку **«Більше деталей»** (More details) внизу вікна.
   - У Windows 11 (22H2+) спрощеного режиму може не бути — одразу відкриється повний інтерфейс з бічною панеллю.
3. Переконайтеся, що ви бачите всі вкладки (або розділи бічної панелі): Процеси, Продуктивність, Журнал додатків, Автозавантаження, Користувачі, Деталі, Служби.
4. Перейдіть на вкладку **«Продуктивність» (Performance)** і запишіть у зошит:
   - Загальну кількість **процесів** у системі.
   - Загальну кількість **потоків** у системі.
   - Поточне завантаження **ЦП** у відсотках.
5. Запишіть у зошит три способи відкриття Диспетчера завдань, які ви знаєте.

---

### Завдання 2. Запустити 5+ програм і спостерігати створення процесів

1. Залиште Диспетчер завдань відкритим на вкладці **«Процеси»**.
2. По черзі запустіть такі програми:
   - **Блокнот** (Notepad) — через меню «Пуск» або Win + R --> `notepad`
   - **Калькулятор** (Calculator) — через меню «Пуск»
   - **Paint** — через меню «Пуск» або Win + R --> `mspaint`
   - **Командний рядок** (CMD) — Win + R --> `cmd`
   - **Провідник** — Win + E (нове вікно)
   - **Веббраузер** (Edge або Chrome) — через панель задач або меню «Пуск»
3. Після запуску кожної програми повертайтеся до Диспетчера завдань і знаходьте новий процес у списку.
4. Заповніть таблицю у зошиті:

| № | Програма | Ім'я процесу | ЦП % | Пам'ять (МБ) |
|---|----------|--------------|-------|--------------|
| 1 | Блокнот | | | |
| 2 | Калькулятор | | | |
| 3 | Paint | | | |
| 4 | CMD | | | |
| 5 | Провідник | | | |
| 6 | Веббраузер | | | |

5. Зверніть увагу: Провідник (explorer.exe) зазвичай вже запущений — нове вікно може не створити окремий процес. Запишіть, чи з'явився новий процес чи ні.
6. **Не закривайте** запущені програми — вони знадобляться для наступних завдань.

---

### Завдання 3. Знайти PID кожного процесу і записати

1. Перейдіть на вкладку **«Деталі»** (Details) Диспетчера завдань.
2. Знайдіть у списку кожен процес, який ви запустили у завданні 2.
3. Заповніть таблицю у зошиті:

| № | Ім'я процесу | PID | Стан (Status) | Ім'я користувача |
|---|--------------|-----|---------------|------------------|
| 1 | notepad.exe | | | |
| 2 | CalculatorApp.exe | | | |
| 3 | mspaint.exe | | | |
| 4 | cmd.exe | | | |
| 5 | explorer.exe | | | |
| 6 | (ім'я браузера) | | | |

4. Зверніть увагу на стовпець **«Стан» (Status)**:
   - Більшість процесів мають стан **«Виконується»** (Running).
   - Калькулятор може мати стан **«Призупинено»** (Suspended), якщо він не на передньому плані — це особливість UWP-додатків у Windows 10/11.
5. Запишіть у зошит відповідь на запитання: чому кожен процес має унікальний PID? Чи може два процеси мати однаковий PID одночасно?

### Завдання 4. Переглянути вкладку «Деталі» — додаткові стовпці

1. На вкладці **«Деталі»** натисніть правою кнопкою миші на заголовку будь-якого стовпця (наприклад, «Ім'я»).
2. Оберіть **«Вибрати стовпці»** (Select columns).
3. У вікні, що відкрилося, позначте галочками такі стовпці (якщо вони ще не увімкнені):
   - **Потоки** (Threads) — кількість потоків процесу
   - **Час ЦП** (CPU Time) — загальний час використання CPU
   - **Базовий пріоритет** (Base Priority) — числовий пріоритет процесу
4. Натисніть ОК.
5. Заповніть розширену таблицю для ваших процесів:

| Ім'я процесу | PID | Стан | Пріоритет | Потоки | CPU Time |
|--------------|-----|------|-----------|--------|----------|
| notepad.exe | | | | | |
| mspaint.exe | | | | | |
| cmd.exe | | | | | |
| (браузер).exe | | | | | |

6. Відсортуйте процеси за стовпцем **«Потоки»** (натисніть на заголовок). Запишіть: який процес має найбільшу кількість потоків? Скільки потоків у нього?
7. Відсортуйте за **«Час ЦП»** (CPU Time). Запишіть: який процес використав найбільше процесорного часу?

### Завдання 5. Змінити пріоритет процесу і спостерігати вплив

1. Переконайтеся, що **Блокнот** (notepad.exe) все ще запущений.
2. На вкладці «Деталі» знайдіть процес `notepad.exe`.
3. Натисніть правою кнопкою миші --> **«Задати пріоритет» (Set Priority)**.
4. Переконайтеся, що поточний пріоритет — **«Нормальний» (Normal)**.
5. Змініть пріоритет на **«Високий» (High)**. У діалоговому вікні підтвердіть зміну.
6. Перевірте у стовпці «Базовий пріоритет» — значення має змінитися.
7. Тепер знайдіть процес `mspaint.exe` (Paint). Змініть його пріоритет на **«Нижче за нормальний» (Below Normal)**.
8. Заповніть таблицю:

| Процес | Пріоритет ДО | Пріоритет ПІСЛЯ | Числове значення |
|--------|-------------|-----------------|------------------|
| notepad.exe | Normal (8) | High (13) | |
| mspaint.exe | Normal (8) | Below Normal (6) | |

9. Поверніть обидва процеси до пріоритету **«Нормальний» (Normal)**.
10. Запишіть у зошит: як зміна пріоритету впливає на виконання процесу? У якій ситуації підвищення пріоритету дійсно має сенс?

> **Пояснення:** При низькому загальному навантаженні на CPU різниця між пріоритетами майже непомітна — процесорного часу вистачає всім. Різниця стає відчутною, коли CPU завантажений на 90-100%: процеси з високим пріоритетом отримують час першими, а процеси з низьким — лише те, що залишилося.

### Завдання 6. Створити навантаження на CPU і спостерігати графік

1. Перейдіть на вкладку **«Продуктивність» (Performance)** --> розділ **«ЦП» (CPU)**.
2. Запишіть поточне завантаження CPU у відсотках (до навантаження): ______%
3. Відкрийте **веббраузер** (Chrome або Edge).
4. Відкрийте **10–15 нових вкладок** із різними вебсайтами (наприклад, YouTube, Wikipedia, новинні сайти). Бажано відкрити хоча б одну вкладку з відеороликом.
5. Поверніться до Диспетчера завдань і спостерігайте за графіком ЦП:
   - Як змінилося завантаження CPU?
   - Чи видно «сплеск» на графіку?
6. Запишіть максимальне навантаження CPU, яке ви спостерігали: ______%
7. Перейдіть на вкладку **«Процеси»** і відсортуйте за стовпцем **«ЦП» (CPU)**.
8. Запишіть: який процес (або група процесів) створює найбільше навантаження?
9. Закрийте всі вкладки браузера, крім однієї. Поверніться на вкладку «Продуктивність» і спостерігайте, як графік CPU повертається до початкового рівня.
10. Запишіть у зошит: завантаження CPU після закриття вкладок: ______%

### Завдання 7. Використати tasklist у CMD

1. Перейдіть до вікна **командного рядка** (CMD), яке ви відкрили раніше. Якщо ви його закрили — відкрийте знову: Win + R --> `cmd` --> Enter.
2. Введіть команду та запишіть результат:

**Крок 1 — Базовий список процесів:**
```
tasklist
```
Перегляньте вивід. Зверніть увагу на стовпці: Ім'я образу, PID, Ім'я сесії, Пам'ять.

**Крок 2 — Розширена інформація:**
```
tasklist /v
```
Знайдіть процес `notepad.exe` у виводі. Запишіть його **пріоритет** та **стан** (Status).

**Крок 3 — Фільтрація за іменем:**
```
tasklist /fi "imagename eq notepad.exe"
```
Запишіть результат — має відобразитися лише процес Блокнота.

**Крок 4 — Фільтрація за використанням пам'яті:**
```
tasklist /fi "memusage gt 51200"
```
Ця команда покаже процеси, що використовують понад 50 МБ пам'яті. Запишіть кількість таких процесів та імена трьох найважчих.

3. Заповніть таблицю у зошиті:

| Команда | Що робить | Кількість результатів |
|---------|-----------|-----------------------|
| `tasklist` | | |
| `tasklist /v` | | |
| `tasklist /fi "imagename eq notepad.exe"` | | |
| `tasklist /fi "memusage gt 51200"` | | |

### Завдання 8. Завершити процес через taskkill

1. Переконайтеся, що **Блокнот** (notepad.exe) запущений. Якщо ні — відкрийте його.
2. Дізнайтеся PID Блокнота:
   ```
   tasklist /fi "imagename eq notepad.exe"
   ```
   Запишіть PID: ______

3. **Завершіть процес за PID:**
   ```
   taskkill /PID <число>
   ```
   (замініть `<число>` на реальний PID, який ви записали)

4. Перевірте: вікно Блокнота має закритися. Виконайте:
   ```
   tasklist /fi "imagename eq notepad.exe"
   ```
   Має бути повідомлення: «Інформація: нічого не знайдено...» або порожній результат.

## 6. Монітор ресурсів (resmon) — аналіз потоків

Монітор ресурсів — це розширений інструмент моніторингу, що надає детальнішу інформацію, ніж Диспетчер завдань.

**Як відкрити:**
- Win + R --> `resmon` --> Enter
- Або через Диспетчер завдань --> вкладка «Продуктивність» --> посилання «Відкрити монітор ресурсів» внизу

**Вкладка «ЦП» (CPU) у Моніторі ресурсів:**

1. Відкрийте Монітор ресурсів (`resmon`).
2. Перейдіть на вкладку **«ЦП»**.
3. У верхній частині — список процесів із прапорцями.
4. Поставте прапорець біля процесу вашого **веббраузера** (наприклад, `chrome.exe`).
5. У нижній частині з'явиться панель **«Потоки»** (Threads), що показує всі потоки обраного процесу:

| Стовпець | Що показує |
|----------|------------|
| **Ідентифікатор потоку (Thread ID)** | Унікальний номер потоку |
| **Поточний ЦП %** | Навантаження цього потоку на процесор |
| **Середній ЦП %** | Середнє навантаження за час спостереження |
| **Модуль** | DLL або компонент, до якого належить потік |
| **Ідеальний процесор** | Ядро CPU, на якому потік переважно працює |

6. Запишіть у зошит: скільки потоків має процес вашого браузера? Який потік створює найбільше навантаження на CPU?

> **Порада:** Монітор ресурсів дозволяє побачити те, чого немає у Диспетчері завдань — окремі потоки процесу, їхнє навантаження на конкретні ядра CPU, а також залежності між процесами та файлами/реєстром.

## 7. PowerShell: Get-Process та Stop-Process

PowerShell — це сучасна командна оболонка Windows, яка надає більш потужні інструменти, ніж CMD.

**Як відкрити PowerShell:**
- Win + R --> `powershell` --> Enter
- Або правий клік на «Пуск» --> «Windows PowerShell»

### Основні команди для роботи з процесами:

**1. Переглянути всі процеси:**
```powershell
Get-Process
```
Виводить список усіх процесів зі стовпцями: Handles, NPM, PM, WS, CPU, Id, ProcessName.

**2. Переглянути конкретний процес:**
```powershell
Get-Process -Name notepad
```

**3. Переглянути процеси, відсортовані за використанням пам'яті (топ-10):**
```powershell
Get-Process | Sort-Object WorkingSet64 -Descending | Select-Object -First 10 Name, Id, @{N='Memory(MB)';E={[math]::Round($_.WorkingSet64/1MB,1)}}
```

**4. Завершити процес:**
```powershell
Stop-Process -Name notepad
```
або за PID:
```powershell
Stop-Process -Id 7344
```

**5. Переглянути кількість потоків процесу:**
```powershell
Get-Process -Name chrome | Select-Object Name, Id, Threads | Format-List
```

### Практичне завдання з PowerShell:

1. Відкрийте PowerShell.
2. Запустіть Блокнот: введіть `notepad` і натисніть Enter (або запустіть через меню «Пуск»).
3. Виконайте:
   ```powershell
   Get-Process -Name notepad
   ```
   Запишіть у зошит: Id (PID), CPU (час CPU у секундах), WS (Working Set — пам'ять).

4. Завершіть процес:
   ```powershell
   Stop-Process -Name notepad
   ```
5. Перевірте, що Блокнот закрився:
   ```powershell
   Get-Process -Name notepad
   ```
   Має з'явитися помилка — процес не знайдено.

6. Запишіть у зошит порівняння: які переваги має PowerShell (`Get-Process`, `Stop-Process`) перед CMD (`tasklist`, `taskkill`)?
