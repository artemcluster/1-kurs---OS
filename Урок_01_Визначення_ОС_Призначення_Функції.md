# 1. Що таке операційна система

## 1.1. Формальне визначення

**Операційна система (ОС)** -- це комплекс взаємопов'язаних програм, який забезпечує управління апаратними ресурсами комп'ютера, виконання прикладних програм та взаємодію між користувачем і комп'ютером.

Іншими словами:

> **Операційна система** -- це набір програм, які призначені для забезпечення взаємодії всіх пристроїв комп'ютера та надання користувачеві можливості виконувати різні дії на комп'ютері.

## 1.2. Простіші визначення для розуміння

- ОС -- це **головна програма комп'ютера**, яка запускається першою при увімкненні та працює весь час, поки комп'ютер увімкнений.
- ОС -- це **посередник** (проміжна ланка) між людиною та "залізом" комп'ютера.
- ОС -- це **менеджер ресурсів**, який розподіляє процесорний час, пам'ять, доступ до пристроїв між усіма програмами.

## 1.3. Аналогії для початківців

Щоб краще зрозуміти, що таке операційна система, уявімо кілька аналогій:

### Аналогія 1: ОС -- це диригент оркестру

Уявіть, що апаратні компоненти комп'ютера (процесор, оперативна пам'ять, жорсткий диск) -- це **музиканти оркестру**. Кожен з них дуже талановитий, але без диригента грали б хто що хоче, і результат був би просто шумом. **Операційна система -- це диригент**, який координує роботу кожного "музиканта", щоб разом вони створювали злагоджену "музику" -- тобто комп'ютер працював коректно.

### Аналогія 2: ОС -- це менеджер готелю

Уявіть великий готель. Є кімнати (пам'ять), персонал (пристрої), гості (програми). **Менеджер готелю (ОС)** вирішує, якого гостя в яку кімнату заселити, яких працівників кому призначити, стежить, щоб ніхто не конфліктував, і щоб усі були задоволені. Без менеджера настав би хаос.

### Аналогія 3: ОС -- це панель приладів автомобіля

Щоб увімкнути кондиціонер у машині, вам не потрібно лізти під капот -- ви просто натискаєте кнопку на панелі. Так само ОС надає вам **зручний інтерфейс** для управління складними механізмами комп'ютера: вам не потрібно знати машинну мову -- ви просто клікаєте мишкою по значках.

# 2. Коротка історія розвитку ОС

Операційні системи пройшли довгий шлях розвитку -- від повної відсутності до надскладних систем, які ми використовуємо сьогодні.

| Рік | Подія | Деталі |
|-----|-------|--------|
| **До 1950-х** | Без ОС | Програми вводили вручну через перемикачі та перфокарти. Кожна програма повинна була містити власний код для управління пристроями. |
| **1956** | Перша ОС -- **GM-NAA I/O** | Створена підрозділом General Motors для комп'ютера IBM 704. Це була система пакетної обробки. |
| **1961** | **MCP** (Burroughs) | Перша ОС, написана повністю мовою високого рівня (ESPOL, діалект ALGOL 60). |
| **1969** | Створення **Unix** | Кен Томпсон та Денніс Рітчі в лабораторіях AT&T Bell Labs створили Unix -- систему, що стала основою для багатьох сучасних ОС. |
| **1973** | Unix переписано мовою **C** | Денніс Рітчі створив мову C, і Unix був переписаний на ній, що зробило систему переносимою на інші комп'ютери. |
| **1974** | **CP/M** | Перша популярна ОС для мікрокомп'ютерів на базі процесорів Intel 8080. |
| **1981** | **MS-DOS** | Microsoft придбала QDOS ("Quick and Dirty Operating System") і перетворила її на MS-DOS -- головну ОС для перших IBM PC. Командний рядок без графіки. |
| **1983** | Проект **GNU** | Річард Столмен заснував проект GNU для створення вільної Unix-подібної ОС. |
| **1984** | **Mac OS** (Apple Macintosh) | Apple випустила Macintosh з графічним інтерфейсом та мишкою -- революція в зручності. |
| **1985** | **Windows 1.0** | Microsoft випустила першу версію Windows -- графічну оболонку поверх MS-DOS. |
| **1991** | **Linux** | Фінський студент **Лінус Торвальдс** (21 рік) створив ядро Linux. 25 серпня 1991 року він оголосив про свій проект, а 5 жовтня вийшла версія 0.02. |
| **1995** | **Windows 95** | Масова ОС з графічним інтерфейсом для широкого загалу. Кнопка "Пуск". |
| **2001** | **Windows XP** та **Mac OS X** | Windows XP стала однією з найпопулярніших ОС в історії. Apple перейшла на нове ядро (на базі Unix). |
| **2007** | **iOS** (iPhone) | Apple представила iPhone з мобільною ОС iOS. |
| **2008** | **Android** | Google випустила мобільну ОС Android на базі ядра Linux. |
| **2015** | **Windows 10** | Microsoft об'єднала настільну та мобільну версії. |
| **2021** | **Windows 11** | Остання велика версія Windows з оновленим дизайном. |

### Ключові постаті

- **Кен Томпсон** та **Денніс Рітчі** -- творці Unix (1969) та мови програмування C.
- **Білл Гейтс** -- співзасновник Microsoft, компанії, що створила MS-DOS та Windows.
- **Стів Джобс** -- співзасновник Apple, під його керівництвом створено Mac OS та iOS.
- **Лінус Торвальдс** -- фінський програміст, творець ядра Linux (1991) та системи контролю версій Git.
- **Річард Столмен** -- засновник проекту GNU та руху за вільне програмне забезпечення.

# 3. Навіщо потрібна операційна система

## 3.1. Що буде, якщо ОС немає?

Без операційної системи комп'ютер -- це просто набір електронних пристроїв, які нічого корисного зробити не можуть. Ось що конкретно станеться:

1. **Комп'ютер не завантажиться.** При увімкненні BIOS (базова система введення-виведення, вшита в материнську плату) шукає пристрій із встановленою ОС. Якщо не знайде -- покаже помилку: "No bootable device found" або "Operating system not found".

2. **Жодна програма не запуститься.** Усі програми (браузер, текстовий редактор, ігри) потребують ОС як середовище для роботи. Без неї вони не можуть взаємодіяти з процесором, пам'яттю чи дисплеєм.

3. **Мережа не працюватиме.** ОС надає драйвери та протоколи для мережевих адаптерів (Wi-Fi, Ethernet). Без неї -- жодного доступу до Інтернету.

4. **Не буде інтерфейсу.** Ні значків, ні вікон, ні меню. Без ОС ви не побачите на екрані нічого, з чим можна працювати.

5. **Потрібно буде писати в машинному коді.** Щоб виконати будь-яку найпростішу дію (наприклад, вивести букву на екран), вам доведеться самостійно писати програму машинною мовою, яка безпосередньо керує кожним електронним компонентом.

## 3.2. Чому ОС необхідна?

Операційна система необхідна, тому що вона:

- **Спрощує роботу з комп'ютером** -- замість машинного коду ви використовуєте зрозумілий графічний інтерфейс.
- **Економить час** -- ОС автоматично керує ресурсами, і вам не потрібно думати про розподіл пам'яті чи доступ до диска.
- **Дозволяє запускати багато програм одночасно** -- слухати музику, писати реферат і шукати інформацію в Інтернеті одночасно.
- **Захищає дані** -- обмежує доступ, контролює дозволи, захищає від вірусів.
- **Забезпечує сумісність** -- різні програми від різних виробників працюють разом завдяки стандартам, які надає ОС.

# 4. Призначення операційної системи

Операційна система має **два головних призначення**:

### 4.1. Зручність (розширена віртуальна машина)

ОС перетворює складний апаратний комплекс на зрозумілу "віртуальну машину", з якою зручно працювати. Замість того, щоб знати, на якому фізичному секторі диска зберігається файл, ви просто бачите папку "Документи" і файл "Реферат.docx".

### 4.2. Ефективність (менеджер ресурсів)

ОС раціонально розподіляє ресурси комп'ютера між усіма програмами та процесами:
- **Процесорний час** -- який процес виконувати зараз.
- **Оперативну пам'ять** -- скільки виділити кожній програмі.
- **Дисковий простір** -- де зберігати файли.
- **Пристрої введення/виведення** -- хто може використовувати принтер, мікрофон тощо.

# 5. Основні функції ОС

Операційна система виконує шість основних функцій:

---

## 5.1. Управління процесами

### Що таке процес?

**Процес** -- це програма, яка виконується в даний момент. Коли ви відкриваєте браузер -- це один процес, коли запускаєте текстовий редактор -- це інший процес.

### Стани процесу

Кожен процес за час свого існування проходить через кілька станів:

```
   Новий --> Готовий --> Виконується --> Завершений
                  ^          |
                  |          v
                  +--- Очікування
```

- **Новий (New)** -- процес щойно створений.
- **Готовий (Ready)** -- процес чекає своєї черги на використання процесора.
- **Виконується (Running)** -- процес безпосередньо працює на процесорі.
- **Очікування (Waiting/Blocked)** -- процес чекає якоїсь події (наприклад, завершення запису на диск).
- **Завершений (Terminated)** -- процес закінчив роботу.

### Що робить ОС з процесами?

- **Планування (scheduling)** -- визначає порядок доступу процесів до процесора. Використовує алгоритми, такі як Round Robin ("по колу"), пріоритетне планування тощо.
- **Багатозадачність (multitasking)** -- дозволяє виконувати багато процесів "одночасно" (насправді процесор дуже швидко перемикається між ними).
- **Синхронізація** -- запобігає конфліктам, коли два процеси намагаються одночасно використати один ресурс.
- **Міжпроцесна взаємодія (IPC)** -- дозволяє процесам обмінюватися даними між собою.

**Приклад з реального життя:** Ви одночасно слухаєте музику у Spotify, набираєте текст у Word і завантажуєте файл із Інтернету. ОС розподіляє процесорний час між цими трьома процесами так швидко, що вам здається, ніби все працює одночасно.

---

## 5.2. Управління пам'яттю

### Що таке оперативна пам'ять (RAM)?

**Оперативна пам'ять (RAM -- Random Access Memory)** -- це "робочий стіл" комп'ютера. Коли ви відкриваєте програму, вона завантажується з жорсткого диска в оперативну пам'ять, тому що доступ до RAM у сотні разів швидший, ніж до диска.

### Що робить ОС з пам'яттю?

- **Виділення пам'яті (allocation)** -- коли програма запускається, ОС виділяє їй необхідний обсяг оперативної пам'яті.
- **Звільнення пам'яті (deallocation)** -- коли програма завершується, ОС повертає цю пам'ять для інших програм.
- **Ізоляція процесів** -- ОС гарантує, що одна програма не може "зайти" в пам'ять іншої програми (це важливо для безпеки та стабільності).
- **Віртуальна пам'ять** -- якщо оперативної пам'яті не вистачає, ОС використовує частину жорсткого диска як додаткову пам'ять (так званий "файл підкачки" або swap).

**Аналогія:** Уявіть, що оперативна пам'ять -- це робочий стіл, а жорсткий диск -- шафа. Коли на столі місця не вистачає, ви тимчасово ховаєте деякі речі в шафу (swap), а коли вони знову потрібні -- дістаєте назад.

### Що буває без управління пам'яттю?

Якщо пам'яттю ніхто не керує, виникають **витоки пам'яті (memory leaks)** -- програми займають пам'ять, але не звільняють її. Комп'ютер поступово уповільнюється і може "зависнути".

## 5.3. Управління файловою системою

### Що таке файлова система?

**Файлова система** -- це спосіб організації та зберігання даних на носіях (жорсткому диску, SSD, флеш-накопичувачі). Вона перетворює хаотичні блоки даних на диску у зрозумілу ієрархію папок і файлів.

### Що робить ОС із файлами?

- **Створення, копіювання, переміщення, видалення** файлів і папок.
- **Організація в ієрархію** -- файли зберігаються в папках, які можуть вкладатися одна в одну (деревоподібна структура).
- **Управління атрибутами** -- кожен файл має атрибути: ім'я, розмір, тип, дата створення, дата зміни, права доступу.
- **Управління правами доступу** -- хто може читати файл, хто може редагувати, хто може видалити (read, write, execute).
- **Пошук файлів** за іменем, типом, датою тощо.
- **Резервне копіювання** -- створення копій для захисту від втрати даних.

### Приклади файлових систем

| Файлова система | Де використовується |
|-----------------|---------------------|
| **NTFS** | Windows (основна для жорстких дисків) |
| **FAT32** | Флеш-накопичувачі, старі системи |
| **exFAT** | Флеш-накопичувачі великого обсягу |
| **ext4** | Linux |
| **APFS** | macOS та iOS |

**Приклад:** Коли ви зберігаєте файл "Реферат.docx" у папку "Документи", ОС визначає, в які фізичні сектори жорсткого диска записати дані, оновлює таблицю файлової системи і запам'ятовує шлях до файлу. Ви ж просто бачите файл у папці.

## 5.4. Управління пристроями (введення/виведення)

### Що таке пристрої введення/виведення (I/O)?

- **Пристрої введення:** клавіатура, мишка, сканер, мікрофон, веб-камера, сенсорний екран.
- **Пристрої виведення:** монітор, принтер, колонки, навушники.
- **Пристрої введення/виведення:** жорсткий диск, SSD, мережевий адаптер, USB-флешка.

### Що робить ОС з пристроями?

- **Взаємодія через драйвери.** Драйвер -- це спеціальна програма-перекладач, яка "пояснює" операційній системі, як працювати з конкретним пристроєм. Наприклад, драйвер принтера Samsung "перекладає" команди ОС у формат, зрозумілий саме цьому принтеру.
- **Відстеження стану пристроїв** -- ОС знає, які пристрої підключені, які вільні, а які зайняті.
- **Розподіл доступу** -- якщо два процеси хочуть одночасно друкувати на принтері, ОС організовує чергу (спулінг).
- **Буферизація** -- тимчасове зберігання даних у буфері для вирівнювання різниці швидкостей між пристроями (процесор працює набагато швидше, ніж принтер).
- **Кешування** -- зберігання часто використовуваних даних у швидкій пам'яті для прискорення доступу.
- **Plug and Play** -- автоматичне розпізнавання нових підключених пристроїв без перезавантаження.

**Приклад:** Коли ви підключаєте USB-флешку, ОС автоматично виявляє її, завантажує відповідний драйвер, визначає файлову систему флешки і показує вам її вміст у провіднику.

## 5.5. Інтерфейс користувача

**Інтерфейс користувача** -- це набір засобів і правил, через які людина взаємодіє з комп'ютером. ОС надає два основних типи інтерфейсу:

### CLI -- командний рядок (Command Line Interface)

- Користувач вводить **текстові команди** з клавіатури.
- Потрібно знати синтаксис команд (конкретні слова та їх параметри).
- **Переваги:** швидкий, потужний, споживає мало ресурсів, підтримує сценарії (скрипти) для автоматизації.
- **Недоліки:** складний для початківців, потрібно запам'ятовувати команди.
- **Приклади:** Bash (Linux, macOS), PowerShell та CMD (Windows).

Приклад команди в Windows CMD:
```
dir C:\Users\Student\Documents
```
(показує список файлів у папці "Documents")

Приклад команди в Linux Bash:
```
ls /home/student/documents
```

### GUI -- графічний інтерфейс (Graphical User Interface)

- Користувач взаємодіє через **графічні елементи**: вікна, значки (іконки), меню, кнопки.
- Використовує мишку (або сенсорний екран) та клавіатуру.
- **Переваги:** інтуїтивно зрозумілий, легко навчитися, зручний для більшості задач.
- **Недоліки:** споживає більше ресурсів (пам'яті та процесора), може бути менш потужним для складних адміністративних задач.
- **Приклади:** Робочий стіл Windows, macOS Finder, GNOME та KDE у Linux.

### Порівняння CLI та GUI

| Характеристика | CLI (командний рядок) | GUI (графічний інтерфейс) |
|----------------|----------------------|---------------------------|
| Введення | Тільки клавіатура | Клавіатура + мишка |
| Складність | Висока для початківців | Низька, інтуїтивний |
| Швидкість роботи | Вища | Нижча |
| Використання ресурсів | Мінімальне | Значне |
| Гнучкість | Дуже висока | Обмежена |
| Автоматизація | Легко через скрипти | Складніше |

**Важливо:** Сучасні ОС підтримують обидва типи інтерфейсу. Наприклад, Windows має і графічний робочий стіл, і командний рядок (CMD, PowerShell). Справжній фахівець повинен уміти працювати і з тим, і з іншим.

## 5.6. Безпека та контроль доступу

### Що робить ОС для захисту?

- **Автентифікація користувачів** -- перевірка особистості через логін та пароль, PIN-код, біометрію (відбиток пальця, сканування обличчя).
- **Розмежування прав доступу** -- різні користувачі мають різні права. Адміністратор може змінювати системні налаштування, а звичайний користувач -- ні.
- **Захист файлів** -- ОС визначає, хто може читати, редагувати чи видаляти конкретний файл (права: read, write, execute).
- **Захист процесів** -- один процес не може втрутитися в роботу іншого. Якщо одна програма "зависне", інші продовжують працювати.
- **Брандмауер (фаєрвол)** -- контролює мережевий трафік, блокує підозрілі підключення.
- **Шифрування даних** -- захист конфіденційної інформації від несанкціонованого читання.
- **Оновлення безпеки** -- ОС регулярно отримує патчі, що виправляють знайдені вразливості.

**Приклад:** У Windows є система облікових записів: "Адміністратор" має повний доступ, а "Стандартний користувач" не може встановлювати програми без дозволу адміністратора. Це захищає комп'ютер від випадкового чи навмисного пошкодження.

## 5.7. Мережева підтримка

Сучасні ОС забезпечують підключення комп'ютера до мережі та Інтернету:

- **Підтримка мережевих протоколів** -- TCP/IP, HTTP, FTP, DNS тощо.
- **Управління мережевими адаптерами** -- Ethernet (провідне підключення), Wi-Fi (бездротове), Bluetooth.
- **Надання мережевих служб** -- спільний доступ до файлів і принтерів, віддалений робочий стіл, VPN.
- **Призначення IP-адрес** -- через DHCP або ручне налаштування.
- **Захист мережевого з'єднання** -- фаєрвол, шифрування трафіку.

# 6. ОС як посередник між апаратурою та користувачем

## 6.1. Трирівнева архітектура

Структуру взаємодії можна уявити як "сендвіч" з трьох рівнів:

```
+------------------------------------------+
|           КОРИСТУВАЧ / ПРОГРАМИ           |  <-- Верхній рівень
+------------------------------------------+
|          ОПЕРАЦІЙНА СИСТЕМА (ОС)          |  <-- Середній рівень
+------------------------------------------+
|      АПАРАТНЕ ЗАБЕЗПЕЧЕННЯ (HARDWARE)     |  <-- Нижній рівень
+------------------------------------------+
```

- **Апаратне забезпечення (Hardware):** Процесор (CPU), оперативна пам'ять (RAM), жорсткий диск (HDD/SSD), материнська плата, відеокарта, мережевий адаптер, пристрої введення/виведення.
- **Операційна система:** Посередник, який приймає запити від програм та користувача, перетворює їх у команди для апаратури та повертає результат назад.
- **Користувач / Прикладні програми:** Людина, яка працює з комп'ютером, та програми, якими вона користується (браузер, текстовий редактор, ігри тощо).

## 6.2. Як це працює на практиці?

Коли ви натискаєте кнопку "Зберегти" в текстовому редакторі:

1. **Програма** (Word, Notepad) надсилає запит операційній системі: "Запиши ці дані у файл з такою назвою".
2. **ОС** визначає, на який диск і в який сектор записати дані, взаємодіє з драйвером жорсткого диска.
3. **Драйвер** передає команду контролеру диска (апаратура).
4. **Апаратура** фізично записує дані на диск.
5. **ОС** підтверджує програмі, що збереження пройшло успішно.
6. **Програма** показує вам повідомлення "Файл збережено".

Увесь цей ланцюг відбувається за частки секунди, і все завдяки ОС як посереднику.

## 6.3. Ядро ОС (Kernel)

В центрі кожної операційної системи знаходиться **ядро (kernel)** -- це основна частина ОС, яка безпосередньо взаємодіє з апаратурою.

- **Ядро** працює в привілейованому режимі і має повний доступ до всього обладнання.
- **Оболонка (shell)** -- це зовнішній рівень ОС, через який з нею взаємодіє користувач (CLI або GUI).

```
+------------------------+
|     Оболонка (Shell)   |  <-- Те, що бачить користувач
+------------------------+
|     Ядро (Kernel)      |  <-- Серце системи
+------------------------+
|      Апаратура          |
+------------------------+
```

# 7. Системне та прикладне програмне забезпечення

## 7.1. Класифікація програмного забезпечення

Усе програмне забезпечення (ПЗ) комп'ютера поділяється на **три категорії**:

### 1. Системне програмне забезпечення

**Системне ПЗ** -- це сукупність програм, призначених для управління апаратною частиною комп'ютера та забезпечення роботи інших програм.

Системне ПЗ **не вирішує конкретних практичних задач користувача**, а лише забезпечує роботу інших програм.

**Складові системного ПЗ:**
- **Операційна система** (Windows, Linux, macOS) -- головний компонент.
- **Драйвери пристроїв** -- програми для управління конкретними пристроями.
- **Утиліти** -- допоміжні програми для обслуговування системи (антивіруси, архіватори, програми для дефрагментації, очищення диска).
- **Завантажувач (bootloader)** -- програма, що завантажує ОС при включенні комп'ютера.

### 2. Прикладне програмне забезпечення

**Прикладне ПЗ** -- це програми, призначені для розв'язання конкретних практичних задач користувача.

Прикладне ПЗ **працює поверх операційної системи** і використовує ресурси, які надає ОС.

**Приклади прикладного ПЗ:**
- **Текстові редактори:** Microsoft Word, LibreOffice Writer, Google Docs.
- **Електронні таблиці:** Microsoft Excel, LibreOffice Calc.
- **Браузери:** Google Chrome, Mozilla Firefox, Microsoft Edge.
- **Графічні редактори:** Adobe Photoshop, GIMP, Canva.
- **Медіапрогравачі:** VLC, Windows Media Player.
- **Комунікаційні програми:** Telegram, Viber, Zoom.
- **Ігри:** будь-які комп'ютерні ігри.

### 3. Інструментальне програмне забезпечення

**Інструментальне ПЗ** -- це комплекс програмних засобів для створення нового або модифікації існуючого програмного забезпечення.

**Приклади:**
- **Середовища розробки (IDE):** Visual Studio, IntelliJ IDEA, PyCharm.
- **Компілятори та інтерпретатори:** GCC, Python interpreter.
- **Системи контролю версій:** Git.

## 7.2. Де знаходиться ОС у цій класифікації?

```
Програмне забезпечення (ПЗ)
|
+--- Системне ПЗ
|    |
|    +--- >>> ОПЕРАЦІЙНА СИСТЕМА <<< (головний компонент!)
|    +--- Драйвери
|    +--- Утиліти
|    +--- Завантажувач
|
+--- Прикладне ПЗ
|    |
|    +--- Текстові редактори
|    +--- Браузери
|    +--- Ігри
|    +--- ...
|
+--- Інструментальне ПЗ
     |
     +--- Середовища розробки
     +--- Компілятори
     +--- ...
```

**Ключова різниця:**
- Системне ПЗ (включаючи ОС) написане мовами низького рівня (C, Assembler), близькими до "мови" комп'ютера.
- Прикладне ПЗ написане мовами високого рівня (Python, Java, C#), орієнтованими на зручність для програміста.
- Прикладне ПЗ **не може працювати без** системного ПЗ (без ОС).
- Системне ПЗ **може працювати без** прикладного.

# 8. Типи та приклади сучасних ОС

## 8.1. Настільні (десктопні) операційні системи

### Microsoft Windows

- **Розробник:** Microsoft Corporation (засновник -- Білл Гейтс).
- **Рік першого випуску:** 1985 (Windows 1.0).
- **Поточна версія:** Windows 11 (випущена 5 жовтня 2021 року).
- **Частка ринку (настільні ПК, 2025):** приблизно **71%**.
- **Ліцензія:** Пропрієтарна (платна).
- **Основні характеристики:**
  - Найпоширеніша настільна ОС у світі.
  - Графічний інтерфейс з "Робочим столом", "Панеллю задач", меню "Пуск".
  - Широка сумісність з програмами та іграми.
  - Підтримка сенсорних екранів.
  - Вбудовані засоби безпеки (Windows Defender, фаєрвол).
  - Найкраща сумісність з іграми.
- **Версії Windows:** Windows 95 -> 98 -> XP -> Vista -> 7 -> 8 -> 10 -> 11.
- **Версії Windows 11 vs 10 (2025):** Windows 11 -- 54,3% серед користувачів Windows, Windows 10 -- 45,7%.

### macOS

- **Розробник:** Apple Inc. (засновник -- Стів Джобс).
- **Рік першого випуску:** 1984 (як Mac OS, сучасна Mac OS X -- з 2001 року).
- **Частка ринку (настільні ПК, 2025):** приблизно **16%**.
- **Ліцензія:** Пропрієтарна, працює тільки на комп'ютерах Apple (iMac, MacBook).
- **Основні характеристики:**
  - Базується на Unix (ядро Darwin).
  - Відоме гарним дизайном та стабільністю.
  - Популярна серед дизайнерів, відеоредакторів, музикантів.
  - Тісна інтеграція з іншими пристроями Apple (iPhone, iPad).
  - Висока безпека -- менше вірусів порівняно з Windows.

### Linux

- **Розробник:** Відкрита спільнота (ядро створив Лінус Торвальдс у 1991 р.).
- **Частка ринку (настільні ПК, 2025):** приблизно **4%**.
- **Ліцензія:** Вільне ПЗ (безкоштовна, з відкритим вихідним кодом, ліцензія GNU GPL).
- **Основні характеристики:**
  - Відкритий вихідний код -- будь-хто може переглядати, змінювати та розповсюджувати.
  - Існує у вигляді різних **дистрибутивів** (Ubuntu, Fedora, Debian, Linux Mint, Arch Linux та ін.).
  - Домінує у серверному сегменті (~63% серверів у світі).
  - **100%** з 500 найпотужніших суперкомп'ютерів світу працюють на Linux (з 2017 року).
  - Висока безпека та стабільність.
  - Безкоштовна -- ідеальна для навчання.
  - Популярна серед програмістів та системних адміністраторів.

### ChromeOS

- **Розробник:** Google.
- **Частка ринку (настільні ПК, 2025):** приблизно **1,8%**.
- **Основна ідея:** Легка ОС на базі Linux, орієнтована на роботу через браузер Chrome та хмарні сервіси.
- **Використовується** переважно в бюджетних ноутбуках (Chromebook) для навчальних закладів.

## 8.2. Мобільні операційні системи

### Android

- **Розробник:** Google (на базі ядра Linux).
- **Рік першого випуску:** 2008.
- **Поточна версія:** Android 15.
- **Частка ринку (мобільні пристрої, 2025):** приблизно **72%**.
- **Ліцензія:** Частково відкритий вихідний код.
- **Основні характеристики:**
  - Найпоширеніша мобільна ОС у світі.
  - Використовується на смартфонах та планшетах Samsung, Xiaomi, OnePlus, Google Pixel та багатьох інших.
  - Висока гнучкість та можливість налаштування.
  - Великий магазин додатків -- Google Play Store.
  - Домінує в країнах, що розвиваються (86% в Африці, 91% в Латинській Америці).

### iOS

- **Розробник:** Apple Inc.
- **Рік першого випуску:** 2007 (разом з першим iPhone).
- **Поточна версія:** iOS 18.
- **Частка ринку (мобільні пристрої, 2025):** приблизно **28%**.
- **Ліцензія:** Пропрієтарна (тільки на iPhone).
- **Основні характеристики:**
  - Відома високою безпекою та конфіденційністю.
  - Плавна та стабільна робота.
  - Тісна інтеграція з екосистемою Apple (Mac, iPad, Apple Watch).
  - Магазин додатків -- App Store.

## 8.3. Загальна частка ринку (всі пристрої, 2025)

| Операційна система | Частка (усі пристрої) | Частка (настільні ПК) | Частка (мобільні) |
|--------------------|----------------------|----------------------|-------------------|
| **Android** | ~44% | -- | ~72% |
| **Windows** | ~28% | ~71% | -- |
| **iOS** | ~16% | -- | ~28% |
| **macOS** | ~4% | ~16% | -- |
| **Linux** | ~1% | ~4% | -- |

**Цікаві факти:**
- Якщо рахувати **всі пристрої** (смартфони, планшети, комп'ютери, сервери), то Android -- найпопулярніша ОС у світі.
- Якщо рахувати тільки **настільні ПК** -- лідирує Windows.
- На **серверах** домінує Linux (~63%).
- У **суперкомп'ютерах** -- 100% Linux.
- **59,6%** світового інтернет-трафіку генерується мобільними пристроями, 38,1% -- настільними.

# Словник ключових термінів

| Термін | Визначення |
|--------|------------|
| **Операційна система (ОС)** | Комплекс програм для управління ресурсами комп'ютера та забезпечення взаємодії з користувачем |
| **Ядро (Kernel)** | Центральна частина ОС, що безпосередньо взаємодіє з апаратурою |
| **Оболонка (Shell)** | Зовнішній рівень ОС для взаємодії з користувачем |
| **Процес** | Програма, що виконується в даний момент |
| **Оперативна пам'ять (RAM)** | Швидка пам'ять для тимчасового зберігання даних під час роботи програм |
| **Файлова система** | Спосіб організації та зберігання файлів на носіях даних |
| **Драйвер** | Програма для управління конкретним пристроєм |
| **CLI** | Command Line Interface -- командний рядок |
| **GUI** | Graphical User Interface -- графічний інтерфейс |
| **Системне ПЗ** | Програми для управління апаратурою та забезпечення роботи інших програм |
| **Прикладне ПЗ** | Програми для розв'язання конкретних задач користувача |
