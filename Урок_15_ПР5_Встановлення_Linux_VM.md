## 1. Мета роботи

- Навчитися створювати віртуальну машину для Linux у програмі VirtualBox.
- Виконати повний цикл встановлення операційної системи Ubuntu Linux на віртуальну машину.
- Ознайомитися з робочим середовищем GNOME та основними інструментами Ubuntu.
- Порівняти процес встановлення Linux із встановленням Windows.

## 2. Теоретичний вступ

### 2.1. Що таке Linux

**Linux** — це сімейство вільних та відкритих операційних систем, побудованих на базі ядра Linux, яке у 1991 році створив фінський програміст **Лінус Торвальдс**. На відміну від Windows або macOS, Linux поширюється безкоштовно, і будь-хто може переглядати, змінювати та розповсюджувати його вихідний код.

Основні особливості Linux:

| Характеристика | Опис |
|---|---|
| **Безкоштовність** | Більшість дистрибутивів Linux можна завантажити та використовувати абсолютно безкоштовно |
| **Відкритий код** | Вихідний код доступний для перегляду, зміни та розповсюдження (ліцензія GPL) |
| **Безпека** | Linux значно менше схильний до вірусів та шкідливого ПЗ, ніж Windows |
| **Стабільність** | Сервери на Linux можуть працювати роками без перезавантаження |
| **Різноманіття** | Існують сотні дистрибутивів для різних цілей |

> **Цікавий факт:** Linux використовується на більшості серверів в Інтернеті, на суперкомп'ютерах, у смартфонах Android, у маршрутизаторах, "розумних" телевізорах та навіть на Міжнародній космічній станції.

### 2.2. Дистрибутиви Linux

**Дистрибутив** — це готовий набір програмного забезпечення на основі ядра Linux, який включає: ядро, графічне середовище, системні утиліти, набір прикладних програм та інсталятор. Різні дистрибутиви створюються для різних потреб.

| Дистрибутив | Опис | Для кого підходить |
|---|---|---|
| **Ubuntu** | Найпопулярніший дистрибутив для початківців. Розробляється компанією Canonical. Має зручний інсталятор, велику спільноту та багато навчальних матеріалів | Початківці, студенти, розробники |
| **Linux Mint** | Заснований на Ubuntu, має інтерфейс, схожий на Windows. Дуже дружній до новачків | Ті, хто переходить з Windows |
| **Fedora** | Розробляється за підтримки Red Hat. Завжди містить найновіші технології та програмне забезпечення | Досвідчені користувачі, розробники |
| **Debian** | Один з найстаріших дистрибутивів (з 1993 р.). Дуже стабільний та надійний. Ubuntu побудований на основі Debian | Сервери, досвідчені користувачі |
| **Arch Linux** | Мінімалістичний дистрибутив, який користувач збирає "з нуля" під себе | Просунуті користувачі |

> **Для нашої практичної роботи ми обираємо Ubuntu** — це найзручніший дистрибутив для навчання з великою спільнотою та детальною документацією. Поточна версія з довготривалою підтримкою (LTS) — **Ubuntu 24.04 LTS "Noble Numbat"** (підтримка до квітня 2029 року).

### 2.3. Робочі середовища Linux

На відміну від Windows, де графічний інтерфейс є невід'ємною частиною системи, у Linux можна вибрати та встановити різні **робочі середовища** (Desktop Environments, DE). Робоче середовище визначає зовнішній вигляд системи: як виглядають вікна, панелі, меню, анімації тощо.

| Робоче середовище | Опис | Вимоги до ресурсів |
|---|---|---|
| **GNOME** | Стандартне середовище Ubuntu. Сучасний, мінімалістичний дизайн. Використовує верхню панель та бічний док | Середні — високі (від 2 ГБ RAM) |
| **KDE Plasma** | Потужне та гнучке середовище з величезними можливостями налаштування. Інтерфейс нагадує Windows | Середні (від 2 ГБ RAM) |
| **XFCE** | Легке та швидке середовище. Ідеальне для старих або слабких комп'ютерів | Низькі (від 512 МБ RAM) |
| **Cinnamon** | Стандартне середовище Linux Mint. Класичний інтерфейс, схожий на Windows 7 | Середні (від 2 ГБ RAM) |
| **MATE** | Продовження класичного GNOME 2. Просте та стабільне | Низькі (від 1 ГБ RAM) |

> **У нашій роботі** Ubuntu 24.04 LTS використовує середовище **GNOME 46** з протоколом відображення Wayland.

## 3. Підготовка до роботи

Перед початком практичної роботи потрібно переконатися, що все готово:

**1. VirtualBox вже встановлений** (ми встановлювали його на попередньому уроці). Переконайтеся, що програма запускається без помилок.

**2. Завантажити ISO-образ Ubuntu:**

1. Відкрийте браузер та перейдіть на офіційний сайт: **https://ubuntu.com/download/desktop**
2. Натисніть кнопку **"Download Ubuntu 24.04.4 LTS"** (або іншу доступну LTS-версію).
3. Дочекайтеся завершення завантаження. Файл матиме назву на кшталт `ubuntu-24.04.4-desktop-amd64.iso` (розмір приблизно 5.8 ГБ).
4. Запам'ятайте або запишіть місце збереження файлу (наприклад, папка "Завантаження").

> **Що таке ISO-образ?** ISO-файл — це точна копія оптичного диска (CD/DVD), збережена у вигляді одного файлу. Він містить усі файли операційної системи, необхідні для встановлення. У VirtualBox ми підключаємо ISO як віртуальний DVD-привід.

> **Що таке LTS?** LTS (Long Term Support) — версія з довготривалою підтримкою. Ubuntu випускає нову LTS-версію кожні 2 роки (у квітні парних років). LTS-версії отримують оновлення безпеки протягом 5 років, тому вони рекомендовані для навчання та роботи.

## 4. Практична частина — Створення віртуальної машини

### 4.1. Крок 1. Створення нової VM у VirtualBox

1. Запустіть програму **Oracle VM VirtualBox**.
2. Натисніть кнопку **"Створити" (New)** на панелі інструментів зверху.
3. У вікні, що з'явилося, заповніть поля:
   - **Ім'я (Name):** `Ubuntu` (коли ви введете це ім'я, VirtualBox автоматично встановить правильний тип та версію).
   - **Тека машини (Machine Folder):** залиште за замовчуванням або оберіть зручне розташування.
   - **ISO-образ (ISO Image):** натисніть на випадаючий список, оберіть **"Інше..." (Other...)** та знайдіть завантажений файл `ubuntu-24.04.4-desktop-amd64.iso`.
   - **Тип (Type):** Linux (має встановитися автоматично).
   - **Версія (Version):** Ubuntu (64-bit) (має встановитися автоматично).
4. Поставте позначку **"Пропустити автоматичне встановлення" (Skip Unattended Installation)** — це дозволить нам пройти весь процес встановлення вручну та побачити кожен крок.
5. Натисніть **"Далі" (Next)**.

### 4.2. Крок 2. Налаштування оперативної пам'яті та процесора

1. У розділі **"Апаратне забезпечення" (Hardware)** встановіть:
   - **Оперативна пам'ять (Base Memory):** рекомендовано **4096 МБ (4 ГБ)**. Мінімум — 2048 МБ (2 ГБ). Повзунок повинен залишатися в **зеленій зоні**, щоб не забрати занадто багато пам'яті у основної системи.
   - **Процесори (Processors):** встановіть **2 ядра (CPUs)**. Повзунок також має бути в зеленій зоні.
2. За бажанням увімкніть прапорець **"Увімкнути EFI" (Enable EFI)** — це сучасний режим завантаження (але для VM це не обов'язково).
3. Натисніть **"Далі" (Next)**.

> **Порада:** Якщо ваш комп'ютер має 8 ГБ RAM або більше, виділіть VM 4 ГБ. Якщо лише 4 ГБ RAM — виділіть VM 2 ГБ, щоб основна система продовжувала працювати стабільно.

### 4.3. Крок 3. Створення віртуального диска

1. У розділі **"Віртуальний жорсткий диск" (Virtual Hard Disk)** оберіть **"Створити віртуальний жорсткий диск зараз" (Create a Virtual Hard Disk Now)**.
2. Встановіть розмір диска: рекомендовано **25-30 ГБ**. Для навчальних цілей цього достатньо.
3. Залиште прапорець **"Попередньо виділити повний розмір" (Pre-allocate Full Size)** — вимкненим. Це означає, що файл віртуального диска буде динамічним: він займатиме на вашому реальному диску лише стільки місця, скільки фактично використовується в VM.
4. Натисніть **"Далі" (Next)**.
5. Перегляньте підсумок налаштувань та натисніть **"Готово" (Finish)**.

> **Динамічний vs Фіксований диск:** Динамічний диск спочатку займає мало місця і збільшується в міру потреби (максимум до встановленого розміру). Фіксований диск одразу займає весь вказаний обсяг на вашому реальному диску, але працює трохи швидше. Для навчання краще обрати динамічний.

### 4.4. Крок 4. Додаткові налаштування VM

Після створення VM, виділіть її у списку та натисніть **"Налаштування" (Settings)**:

1. **Система (System) → Материнська плата (Motherboard):**
   - Переконайтеся, що порядок завантаження правильний: Оптичний диск (Optical) → Жорсткий диск (Hard Disk).

2. **Дисплей (Display) → Екран (Screen):**
   - Встановіть **Відеопам'ять (Video Memory): 128 МБ** (максимальне значення).
   - **Графічний контролер (Graphics Controller):** залиште **VMSVGA** або оберіть **VBoxSVGA**.

3. **Загальне (General) → Додатково (Advanced):**
   - **Спільний буфер обміну (Shared Clipboard):** встановіть **Двонаправлений (Bidirectional)** — це дозволить копіювати текст між основною системою та VM.
   - **Drag'n'Drop:** встановіть **Двонаправлений (Bidirectional)** — для перетягування файлів між системами.

4. Натисніть **"ОК"** для збереження налаштувань.

### 4.5. Крок 5. Підключення ISO-образу Ubuntu

Якщо ви не підключили ISO на кроці 1, зробіть це зараз:

1. Оберіть створену VM у списку та натисніть **"Налаштування" (Settings)**.
2. Перейдіть до розділу **"Носії" (Storage)**.
3. У розділі "Контролер: IDE" (або "Контролер: SATA") натисніть на значок **порожнього диска** (з написом "Порожньо" / "Empty").
4. Праворуч, біля напису "Оптичний привід", натисніть на **значок диска** та оберіть **"Вибрати файл диска..." (Choose a disk file...)**.
5. Знайдіть та оберіть завантажений файл `ubuntu-24.04.4-desktop-amd64.iso`.
6. Натисніть **"ОК"**.

## 5. Практична частина — Встановлення Ubuntu

### 5.1. Крок 6. Запуск віртуальної машини

1. Виділіть VM "Ubuntu" у списку.
2. Натисніть кнопку **"Запустити" (Start)** на панелі інструментів.
3. Відкриється нове вікно — екран віртуальної машини.
4. З'явиться меню завантаження GRUB. Оберіть **"Try or Install Ubuntu"** та натисніть **Enter**.
5. Зачекайте, поки завантажиться середовище встановлення (це може зайняти 1-3 хвилини).

> **Захоплення миші:** Коли ви натиснете всередині вікна VM, VirtualBox "захопить" мишу. Щоб повернути мишу до основної системи, натисніть **праву клавішу Ctrl** (це "хост-клавіша" VirtualBox).

### 5.2. Крок 7. Вибір мови

1. Після завантаження з'явиться вікно інсталятора Ubuntu.
2. У списку мов оберіть **"Українська" (Ukrainian)**.
3. Натисніть **"Далі" (Next)**.

### 5.3. Крок 8. Спеціальні можливості

1. З'явиться екран **"Доступність" (Accessibility)** — це нова функція Ubuntu 24.04.
2. Тут можна налаштувати: екранну клавіатуру, збільшення екрана, високу контрастність тощо.
3. Для звичайного встановлення **пропустіть цей крок** — натисніть **"Далі" (Next)**.

### 5.4. Крок 9. Розкладка клавіатури

1. Оберіть розкладку клавіатури: **Українська (Ukrainian)**.
2. Ви можете перевірити розкладку у тестовому полі внизу екрана — введіть кілька символів, щоб переконатися, що все працює правильно.
3. Натисніть **"Далі" (Next)**.

> **Примітка:** Можна додати другу розкладку (наприклад, англійську) вже після встановлення системи.

### 5.5. Крок 10. Підключення до мережі

1. Якщо VM підключена до Інтернету через NAT (за замовчуванням у VirtualBox), інсталятор запропонує **"Використовувати дротове з'єднання" (Use Wired Connection)**.
2. Оберіть цей варіант та натисніть **"Далі" (Next)**.

> **Навіщо потрібен Інтернет?** Підключення до мережі дозволяє завантажити оновлення безпеки та додаткові драйвери під час встановлення.

### 5.6. Крок 11. Тип встановлення

1. Оберіть **"Встановити Ubuntu" (Install Ubuntu)**.
2. На наступному екрані оберіть тип встановлення:
   - **Інтерактивне встановлення (Interactive Installation)** — ми обираємо саме цей варіант, щоб пройти всі кроки вручну.
3. Натисніть **"Далі" (Next)**.

### 5.7. Крок 12. Вибір додатків

1. Оберіть набір додатків:
   - **Стандартний набір (Default Selection)** — лише базові утиліти та веббраузер.
   - **Розширений набір (Extended Selection)** — додатково офісний пакет, медіаплеєр, утиліти.
2. Рекомендуємо обрати **"Розширений набір"** для повноцінного знайомства з Ubuntu.
3. Натисніть **"Далі" (Next)**.

### 5.8. Крок 13. Стороннє програмне забезпечення

1. Якщо VM підключена до Інтернету, з'явиться запит про встановлення стороннього ПЗ:
   - **"Встановити рекомендоване пропрієтарне програмне забезпечення"** — поставте позначку. Це встановить додаткові кодеки для аудіо та відео, а також драйвери.
   - **"Завантажити та встановити оновлення під час встановлення"** — поставте позначку.
2. Натисніть **"Далі" (Next)**.

### 5.9. Крок 14. Розмітка диска

1. Оберіть варіант розмітки диска:
   - **"Використати весь диск" (Erase disk and install Ubuntu)** — найпростіший варіант. Інсталятор автоматично створить усі необхідні розділи.
2. Натисніть **"Далі" (Next)**.

> **Це безпечно!** Оскільки ми встановлюємо Ubuntu на **віртуальну машину**, "стерти диск" означає стерти лише **віртуальний** диск VM. Ваш реальний жорсткий диск та основна операційна система залишаться без змін.

### 5.10. Крок 15. Часовий пояс

1. Оберіть часовий пояс на карті або введіть місто: **Київ (Kyiv)**.
2. Часовий пояс: **Europe/Kyiv (UTC+2 / UTC+3)**.
3. Натисніть **"Далі" (Next)**.

> **Важливо:** Запам'ятайте або запишіть пароль! Він потрібен для входу в систему та для виконання адміністративних дій (команда `sudo`).

### 5.12. Крок 17. Процес встановлення та перезавантаження

1. Перегляньте підсумок налаштувань та натисніть **"Встановити" (Install)**.
2. Розпочнеться процес встановлення. Це може зайняти **10-20 хвилин** залежно від швидкості комп'ютера та Інтернет-з'єднання.
3. Під час встановлення на екрані з'являтимуться слайди з інформацією про Ubuntu — можете ознайомитися з ними.
4. Після завершення натисніть **"Перезавантажити зараз" (Restart Now)**.
5. Коли з'явиться повідомлення "Please remove the installation medium, then press ENTER", просто натисніть **Enter** (VirtualBox автоматично від'єднає ISO-образ).
6. Після перезавантаження з'явиться екран входу — введіть свій пароль.

## 6. Встановлення Guest Additions

**Guest Additions** — це набір додаткових драйверів та утиліт від VirtualBox, які значно покращують роботу гостьової ОС. Вони забезпечують:

- Автоматичну зміну роздільної здатності екрана при зміні розміру вікна VM.
- Спільний буфер обміну (копіювання/вставка між основною ОС та VM).
- Перетягування файлів (drag-and-drop) між системами.
- Спільні папки.
- Покращену продуктивність графіки.

### Покрокова інструкція:

1. Увійдіть у систему Ubuntu у VM.
2. Відкрийте **Термінал** (натисніть клавішу `Super`/`Windows`, введіть "Terminal" та натисніть Enter).
3. Встановіть необхідні пакети, ввівши команди:

```bash
sudo apt update
sudo apt install build-essential linux-headers-$(uname -r) -y
```

4. Введіть ваш пароль, коли система запитає (символи пароля не відображаються — це нормально).
5. У меню вікна VirtualBox (зверху) натисніть: **Пристрої (Devices) → Вставити образ диска Гостьових доповнень... (Insert Guest Additions CD image...)**.
6. Якщо з'явиться запит на автозапуск — натисніть **"Запустити" (Run)**. Якщо ні, виконайте у терміналі:

```bash
cd /media/$USER/VBox_GAs_*/
sudo ./VBoxLinuxAdditions.run
```

7. Дочекайтеся завершення встановлення (з'явиться повідомлення "VirtualBox Guest Additions: Running kernel modules will not be replaced...").
8. Перезавантажте VM командою:

```bash
sudo reboot
```

9. Після перезавантаження спробуйте змінити розмір вікна VM — роздільна здатність екрана Ubuntu має змінюватися автоматично.

## 7. Перше знайомство з Ubuntu

### 7.1. Робочий стіл GNOME

Після входу в систему ви побачите робочий стіл із середовищем **GNOME**. Основні елементи:

| Елемент | Розташування | Призначення |
|---|---|---|
| **Верхня панель** | Зверху екрана | Показує час, дату, індикатори звуку, мережі, батареї. Зліва — кнопка "Дії" (Activities) |
| **Док-панель (Dash)** | Зліва екрана | Панель швидкого запуску з іконками улюблених програм. Аналог панелі задач у Windows |
| **Кнопка "Дії" (Activities)** | Лівий верхній кут | Відкриває огляд усіх відкритих вікон та пошук програм. Також активується натисканням клавіші **Super** (Windows) |
| **Робочий стіл** | Центральна частина екрана | Основна робоча область. На відміну від Windows, у GNOME на робочому столі зазвичай немає іконок |
| **Системне меню** | Правий верхній кут | Налаштування звуку, мережі, живлення, вимкнення/перезавантаження |

> **Клавіша Super (Windows):** Натискання цієї клавіші відкриває огляд "Дії" (Activities), де можна побачити всі відкриті вікна та скористатися пошуком для запуску будь-якої програми.

### 7.2. Файловий менеджер Nautilus

**Nautilus** (офіційна назва — "Файли" / "Files") — це вбудований файловий менеджер Ubuntu, аналог Провідника Windows.

Основні елементи Nautilus:
- **Бічна панель зліва** — швидкий доступ до основних папок: Домівка, Документи, Завантаження, Музика, Зображення тощо.
- **Адресний рядок зверху** — показує поточний шлях.
- **Область вмісту** — файли та папки поточного каталогу.

Структура домашньої папки Linux:

| Папка | Призначення | Аналог у Windows |
|---|---|---|
| `/home/ваш_логін/` | Домашня папка користувача | `C:\Users\ваше_ім'я\` |
| `Документи` | Для документів | `Документи` |
| `Завантаження` | Для завантажених файлів | `Завантаження` |
| `Зображення` | Для фото та зображень | `Зображення` |
| `Робочий стіл` | Файли на робочому столі | `Робочий стіл` |

### 7.3. Термінал та базові команди

**Термінал** — це програма для введення текстових команд, аналог командного рядка (cmd) у Windows, але значно потужніший. У Linux термінал є основним інструментом для адміністрування системи.

Як відкрити термінал:
- Натисніть **Ctrl + Alt + T** (клавіатурне скорочення).
- Або натисніть клавішу **Super**, введіть "Terminal" та натисніть Enter.

Основні команди для початку:

| Команда | Що робить | Приклад |
|---|---|---|
| `pwd` | Показує поточну директорію (де ви зараз знаходитесь) | `pwd` → `/home/ivan` |
| `ls` | Показує вміст поточної директорії (список файлів та папок) | `ls` → `Документи  Завантаження  Зображення` |
| `ls -la` | Показує детальний список файлів, включно з прихованими | `ls -la` |
| `cd` | Переходить до іншої директорії | `cd Документи` |
| `cd ..` | Переходить на рівень вгору | `cd ..` |
| `mkdir` | Створює нову директорію (папку) | `mkdir МояПапка` |
| `cat` | Показує вміст текстового файлу | `cat файл.txt` |
| `sudo apt update` | Оновлює список доступних пакетів (програм) | `sudo apt update` |
| `sudo apt upgrade` | Встановлює доступні оновлення системи | `sudo apt upgrade` |
| `clear` | Очищує екран терміналу | `clear` |

Спробуйте виконати у терміналі наступні команди по черзі:

```bash
pwd
ls
cd Документи
pwd
mkdir TestFolder
ls
cd ..
sudo apt update
```

> **Що таке `sudo`?** Команда `sudo` (Super User DO) дозволяє виконувати команди з правами адміністратора. При першому використанні у сесії система запитає ваш пароль. Це механізм безпеки Linux — звичайний користувач не може змінювати системні файли без підтвердження.

### 7.4. Менеджер додатків

**Ubuntu Software** (Центр додатків) — це графічний магазин програм, аналог Microsoft Store у Windows. Тут можна:

- Знаходити та встановлювати нові програми (браузери, офісні пакети, ігри, утиліти).
- Оновлювати вже встановлені програми.
- Видаляти непотрібні програми.

Щоб відкрити: знайдіть іконку **"Ubuntu Software"** (або "App Center") на док-панелі або через пошук (клавіша Super).

Популярні безкоштовні програми, доступні в Ubuntu:

| Категорія | Програма | Опис |
|---|---|---|
| Офіс | LibreOffice | Повноцінний офісний пакет (аналог Microsoft Office) |
| Браузер | Firefox | Встановлений за замовчуванням |
| Графіка | GIMP | Редактор зображень (аналог Photoshop) |
