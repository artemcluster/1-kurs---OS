## 1. IPv4-адреса: формат та двійкове представлення

### Що таке IP-адреса?

**IP-адреса** (Internet Protocol address) — це унікальна числова адреса, яку отримує кожен пристрій у комп'ютерній мережі. Вона необхідна для того, щоб пристрої могли знаходити один одного та обмінюватися даними. Можна провести аналогію з поштовою адресою: щоб лист дійшов до адресата, потрібно вказати точну адресу.

### Формат IPv4-адреси

IPv4-адреса складається з **32 біт** (4 байти), які записуються як **4 октети** (групи по 8 біт), розділені крапками. Кожен октет записується у **десятковій нотації** — числом від 0 до 255.

**Приклад:**

```
192.168.1.10
```

Тут:
- 1-й октет: **192**
- 2-й октет: **168**
- 3-й октет: **1**
- 4-й октет: **10**

### Двійкове представлення

Комп'ютер працює з числами у двійковій системі (0 та 1). Кожен октет IP-адреси — це 8 біт, де кожен біт може бути 0 або 1.

**Переведемо 192.168.1.10 у двійкову форму:**

| Десяткове | Двійкове |
|-----------|----------|
| 192 | 11000000 |
| 168 | 10101000 |
| 1 | 00000001 |
| 10 | 00001010 |

Повний запис: `11000000.10101000.00000001.00001010`

**Як переводити десяткове число в двійкове:**

Ваги бітів у октеті (зліва направо): 128, 64, 32, 16, 8, 4, 2, 1.

Приклад для числа 192:
- 192 >= 128? Так -> 1, залишок: 192 - 128 = 64
- 64 >= 64? Так -> 1, залишок: 64 - 64 = 0
- 0 >= 32? Ні -> 0
- 0 >= 16? Ні -> 0
- 0 >= 8? Ні -> 0
- 0 >= 4? Ні -> 0
- 0 >= 2? Ні -> 0
- 0 >= 1? Ні -> 0

Результат: **11000000**

### Діапазон IPv4-адрес

Оскільки кожен октет може мати значення від 0 до 255, повний діапазон IPv4-адрес:

```
0.0.0.0 — 255.255.255.255
```

Усього можливих адрес: 2^32 = **4 294 967 296** (приблизно 4,3 мільярда).

## 2. Класи IP-адрес

Історично IPv4-адреси поділялися на **5 класів** (A, B, C, D, E). Хоча сучасна мережа переважно використовує безкласову маршрутизацію (CIDR), розуміння класів залишається важливим.

### Таблиця класів IP-адрес

| Клас | Діапазон | Маска за замовчуванням | CIDR | Кількість мереж | Кількість хостів |
|------|----------|----------------------|------|-----------------|-----------------|
| **A** | 1.0.0.0 -- 126.255.255.255 | 255.0.0.0 | /8 | 126 | 16 777 214 |
| **B** | 128.0.0.0 -- 191.255.255.255 | 255.255.0.0 | /16 | 16 384 | 65 534 |
| **C** | 192.0.0.0 -- 223.255.255.255 | 255.255.255.0 | /24 | 2 097 152 | 254 |
| **D** | 224.0.0.0 -- 239.255.255.255 | -- | -- | Multicast | -- |
| **E** | 240.0.0.0 -- 255.255.255.255 | -- | -- | Зарезервований | -- |

### Характеристика кожного класу

**Клас A** — для дуже великих мереж. Перший октет визначає мережу, решта три — хости. Перший біт завжди `0`.

**Клас B** — для середніх та великих мереж. Перші два октети визначають мережу, останні два — хости. Перші два біти завжди `10`.

**Клас C** — для невеликих мереж. Перші три октети визначають мережу, останній — хости. Перші три біти завжди `110`.

**Клас D** — зарезервований для **мультикаст** (групова розсилка). Використовується для передачі даних одразу групі пристроїв (наприклад, потокове відео). Не має поняття мережевої частини та частини хоста.

**Клас E** — **зарезервований** для експериментальних та дослідницьких цілей. Не використовується у звичайних мережах.

### Як визначити клас за першим октетом?

| Перший октет | Клас |
|-------------|------|
| 1--126 | A |
| 128--191 | B |
| 192--223 | C |
| 224--239 | D |
| 240--255 | E |

## 3. Приватні та публічні IP-адреси. NAT

### Публічні IP-адреси

**Публічна (public) IP-адреса** — це унікальна адреса, яка видима з будь-якого місця в Інтернеті. Її призначає Інтернет-провайдер (ISP). Два пристрої не можуть мати однакову публічну IP-адресу одночасно.

### Приватні IP-адреси

**Приватна (private) IP-адреса** — це адреса, яка використовується тільки всередині локальної мережі (дім, офіс, школа). Ці адреси **не маршрутизуються** в Інтернеті, тобто їх не видно зовні. Визначені стандартом **RFC 1918**.

### Діапазони приватних адрес

| Діапазон | CIDR | Клас | Кількість адрес |
|----------|------|------|-----------------|
| 10.0.0.0 -- 10.255.255.255 | 10.0.0.0/8 | A | 16 777 216 |
| 172.16.0.0 -- 172.31.255.255 | 172.16.0.0/12 | B | 1 048 576 |
| 192.168.0.0 -- 192.168.255.255 | 192.168.0.0/16 | C | 65 536 |

**Приклади з повсякденного життя:**
- Ваш домашній Wi-Fi роутер зазвичай призначає пристроям адреси виду `192.168.0.x` або `192.168.1.x`.
- У комп'ютерному класі школи комп'ютери можуть мати адреси `10.0.0.x`.

### NAT (Network Address Translation)

**NAT** — це технологія, що дозволяє кільком пристроям з приватними IP-адресами виходити в Інтернет через одну публічну IP-адресу.

**Як це працює:**
1. Комп'ютер з адресою `192.168.1.10` надсилає запит на веб-сайт.
2. Роутер отримує пакет і замінює приватну адресу (`192.168.1.10`) на свою публічну адресу (наприклад, `203.0.113.5`).
3. Веб-сервер відповідає на публічну адресу роутера.
4. Роутер отримує відповідь і пересилає її на правильний комп'ютер у локальній мережі.

```
[ПК: 192.168.1.10] ---> [Роутер: NAT] ---> [Інтернет: 203.0.113.5]
[ПК: 192.168.1.11] -/       |
[ПК: 192.168.1.12] -/       |
                     Усі використовують одну публічну адресу
```

**Навіщо NAT:**
- Економія публічних IPv4-адрес (їх обмежена кількість).
- Додатковий рівень безпеки — пристрої з приватними адресами не видимі напряму з Інтернету.

## 4. Спеціальні адреси

Деякі IP-адреси мають особливе призначення і не можуть бути призначені звичайним пристроям.

| Адреса | Назва | Призначення |
|--------|-------|-------------|
| **127.0.0.1** | Localhost (loopback) | Вказує на сам пристрій. Використовується для тестування мережевих програм без реального мережевого з'єднання. Весь діапазон 127.0.0.0/8 зарезервований для цього. |
| **0.0.0.0** | Невизначена адреса | Означає "будь-яка адреса" або "адреса не призначена". Використовується при запуску DHCP-клієнта (коли пристрій ще не отримав IP). |
| **255.255.255.255** | Широкомовна (broadcast) | Пакет, надісланий на цю адресу, отримають **усі** пристрої в локальній мережі. Використовується, наприклад, у процесі DHCP Discover. |
| **169.254.x.x** | APIPA (link-local) | Автоматично призначається, якщо DHCP-сервер недоступний. Забезпечує базову локальну комунікацію. |

**Приклад використання localhost:** Якщо ви розробляєте веб-сайт на своєму комп'ютері, ви можете відкрити його у браузері за адресою `http://127.0.0.1` — сайт буде доступний тільки вам, без виходу в мережу.

## 5. Маска підмережі

### Що таке маска підмережі?

**Маска підмережі** (subnet mask) — це 32-бітне число, яке визначає, яка частина IP-адреси відноситься до **мережі**, а яка — до **хоста** (конкретного пристрою).

- Біти зі значенням **1** — мережева частина.
- Біти зі значенням **0** — частина хоста.

### Приклад

Маска `255.255.255.0` у двійковому вигляді:

```
11111111.11111111.11111111.00000000
|--- мережа (24 біти) ---||хост(8)|
```

Це означає: перші 3 октети — адреса мережі, останній октет — адреса хоста в цій мережі.

### Логічна операція AND

Щоб визначити адресу мережі, до якої належить пристрій, потрібно виконати побітову операцію **AND** між IP-адресою та маскою.

**Правила AND:**
- 1 AND 1 = 1
- 1 AND 0 = 0
- 0 AND 1 = 0
- 0 AND 0 = 0

**Приклад:**

```
IP-адреса:     192.168.1.10   = 11000000.10101000.00000001.00001010
Маска:         255.255.255.0  = 11111111.11111111.11111111.00000000
               ─────────────────────────────────────────────────────
Адреса мережі: 192.168.1.0    = 11000000.10101000.00000001.00000000
```

Результат: пристрій з IP `192.168.1.10` і маскою `255.255.255.0` належить до мережі `192.168.1.0`.

### CIDR-нотація (безкласова маршрутизація)

**CIDR** (Classless Inter-Domain Routing) — це сучасний спосіб запису маски підмережі. Замість повного запису маски використовується **число після символу "/"**, яке вказує кількість одиничних бітів у масці.

| Маска | CIDR | Мережеві біти | Хостові біти | Кількість хостів |
|-------|------|--------------|-------------|-----------------|
| 255.0.0.0 | /8 | 8 | 24 | 16 777 214 |
| 255.255.0.0 | /16 | 16 | 16 | 65 534 |
| 255.255.255.0 | /24 | 24 | 8 | 254 |
| 255.255.255.128 | /25 | 25 | 7 | 126 |
| 255.255.255.192 | /26 | 26 | 6 | 62 |
| 255.255.255.224 | /27 | 27 | 5 | 30 |
| 255.255.255.240 | /28 | 28 | 4 | 14 |
| 255.255.255.248 | /29 | 29 | 3 | 6 |
| 255.255.255.252 | /30 | 30 | 2 | 2 |

**Формула кількості хостів:** 2^n - 2, де n -- кількість хостових бітів. Віднімаємо 2, бо одна адреса -- це адреса мережі, а інша -- broadcast-адреса.

**Приклад запису:** `192.168.1.0/24` означає мережу `192.168.1.0` з маскою `255.255.255.0`.

## 6. Приклад розрахунку підмережі

### Задача

Дано: IP-адреса **192.168.10.130/26**. Визначити:
- адресу мережі
- broadcast-адресу
- діапазон хостів
- кількість доступних хостів

### Крок 1: Визначити маску

CIDR /26 означає, що маска має 26 одиниць та 6 нулів:

```
Маска: 11111111.11111111.11111111.11000000 = 255.255.255.192
```

### Крок 2: Знайти адресу мережі (AND)

```
IP:    192.168.10.130  = 11000000.10101000.00001010.10000010
Маска: 255.255.255.192 = 11111111.11111111.11111111.11000000
       ─────────────────────────────────────────────────────
AND:   192.168.10.128  = 11000000.10101000.00001010.10000000
```

**Адреса мережі: 192.168.10.128**

### Крок 3: Знайти broadcast-адресу

Для broadcast-адреси потрібно всі хостові біти (останні 6 біт) встановити в 1:

```
Мережа: 11000000.10101000.00001010.10000000
                                    ^^^^^^ -- хостові біти
Broadcast: 11000000.10101000.00001010.10111111 = 192.168.10.191
```

**Broadcast-адреса: 192.168.10.191**

### Крок 4: Діапазон хостів

- **Перший хост:** адреса мережі + 1 = **192.168.10.129**
- **Останній хост:** broadcast - 1 = **192.168.10.190**

### Крок 5: Кількість хостів

2^6 - 2 = 64 - 2 = **62 хости**

### Підсумок

| Параметр | Значення |
|----------|----------|
| IP-адреса | 192.168.10.130/26 |
| Маска підмережі | 255.255.255.192 |
| Адреса мережі | 192.168.10.128 |
| Broadcast-адреса | 192.168.10.191 |
| Перший хост | 192.168.10.129 |
| Останній хост | 192.168.10.190 |
| Кількість хостів | 62 |

## 7. Шлюз за замовчуванням (Default Gateway)

### Що це таке?

**Шлюз за замовчуванням** (default gateway) — це мережевий пристрій (зазвичай **роутер**), через який пристрій надсилає дані, коли адресат знаходиться **за межами локальної мережі**.

### Як це працює?

1. Комп'ютер хоче надіслати дані на IP-адресу (наприклад, на сервер Google).
2. За допомогою маски підмережі комп'ютер визначає: **"Чи належить адресат до моєї мережі?"**
3. **Якщо так** — дані надсилаються напряму адресату в локальній мережі.
4. **Якщо ні** — дані надсилаються на **шлюз за замовчуванням** (роутер), який вже знає, куди їх передати далі.

### Приклад

```
Комп'ютер:       IP = 192.168.1.10,  маска = 255.255.255.0
Шлюз:            IP = 192.168.1.1    (роутер)
Сервер Google:   IP = 142.250.185.14 (інша мережа)
```

Комп'ютер бачить, що `142.250.185.14` не належить до мережі `192.168.1.0/24`, тому надсилає пакет роутеру `192.168.1.1`, а роутер пересилає його далі.

### Аналогія

Шлюз за замовчуванням -- це як **вхідні двері вашого будинку**. Щоб дістатися куди-небудь за межі будинку (локальної мережі), ви повинні спочатку вийти через двері (шлюз).

## 8. DNS -- система доменних імен

### Що таке DNS?

**DNS** (Domain Name System) — це система, яка перетворює **доменні імена** (наприклад, `google.com`) на **IP-адреси** (наприклад, `142.250.185.14`). DNS можна порівняти з **телефонною книгою Інтернету**: замість того, щоб запам'ятовувати числові адреси, ми використовуємо зрозумілі імена.

### Як працює DNS (спрощено)

```
1. Ви вводите "google.com" у браузері
              |
2. Браузер перевіряє локальний кеш (чи запам'ятав адресу раніше)
              |  (якщо ні)
3. Запит надсилається DNS-серверу (зазвичай сервер провайдера)
              |
4. DNS-сервер шукає відповідність:
   google.com --> 142.250.185.14
              |
5. IP-адреса повертається браузеру
              |
6. Браузер з'єднується з сервером за IP-адресою
              |
7. Сторінка завантажується
```

### Ієрархія DNS

DNS має ієрархічну структуру:

- **Кореневі сервери** (Root) — верхній рівень, знають, де шукати домени верхнього рівня.
- **TLD-сервери** (Top-Level Domain) — обслуговують домени `.com`, `.org`, `.ua`, `.net` тощо.
- **Авторитативні сервери** — містять точні записи конкретних доменів.

### Типи DNS-записів

| Тип | Призначення |
|-----|-------------|
| **A** | Зв'язує домен з IPv4-адресою |
| **AAAA** | Зв'язує домен з IPv6-адресою |
| **MX** | Вказує поштовий сервер для домену |
| **CNAME** | Псевдонім (один домен посилається на інший) |

### Популярні DNS-сервери

| Постачальник | Первинний DNS | Вторинний DNS |
|-------------|--------------|--------------|
| Google | 8.8.8.8 | 8.8.4.4 |
| Cloudflare | 1.1.1.1 | 1.0.0.1 |

## 9. DHCP -- автоматичне призначення IP

### Що таке DHCP?

**DHCP** (Dynamic Host Configuration Protocol) — це протокол, який **автоматично** призначає IP-адреси та інші мережеві параметри пристроям у мережі. Завдяки DHCP не потрібно вручну вводити IP-адресу, маску, шлюз та DNS на кожному комп'ютері.

### Що видає DHCP-сервер?

- IP-адресу
- Маску підмережі
- Шлюз за замовчуванням
- Адреси DNS-серверів
- Час оренди (lease time)

### Процес DORA

DHCP працює за схемою **DORA** — чотири кроки:

```
КЛІЄНТ                           СЕРВЕР
   |                                |
   |--- 1. DISCOVER (broadcast) --->|  "Хто тут DHCP-сервер?"
   |                                |
   |<-- 2. OFFER -------------------|  "Я! Пропоную адресу 192.168.1.10"
   |                                |
   |--- 3. REQUEST (broadcast) ---->|  "Дякую, я хочу саме цю адресу"
   |                                |
   |<-- 4. ACKNOWLEDGE -------------|  "Готово! Адреса твоя на 8 годин"
   |                                |
```

**Крок 1 -- Discover (Виявлення):**
Новий пристрій підключається до мережі, але ще не має IP-адреси. Він надсилає **широкомовний** (broadcast) запит DHCPDISCOVER на адресу `255.255.255.255`, використовуючи `0.0.0.0` як адресу відправника. Цей запит отримують усі пристрої в мережі.

**Крок 2 -- Offer (Пропозиція):**
DHCP-сервер отримує запит і відповідає повідомленням DHCPOFFER, в якому пропонує вільну IP-адресу, маску, шлюз, DNS та тривалість оренди.

**Крок 3 -- Request (Запит):**
Клієнт отримує пропозицію (або кілька пропозицій від різних серверів) і надсилає DHCPREQUEST, підтверджуючи вибір конкретного сервера та адреси. Це повідомлення також broadcast, щоб інші сервери дізналися, що їхні пропозиції відхилені.

**Крок 4 -- Acknowledge (Підтвердження):**
Сервер надсилає DHCPACK, підтверджуючи, що адреса успішно призначена клієнту. Тепер пристрій може працювати в мережі.

### DHCP-порти

- DHCP-сервер працює на порту **UDP 67**
- DHCP-клієнт працює на порту **UDP 68**

### Оренда адреси (Lease)

DHCP-сервер видає IP-адресу **не назавжди**, а на певний час — це називається **оренда** (lease). Типовий строк оренди — **8 днів** (може налаштовуватися адміністратором).

- Коли половина строку оренди мине, клієнт автоматично запитує **продовження** (renewal).
- Якщо сервер не відповідає, клієнт спробує ще раз.
- Після завершення оренди адреса повертається в пул і може бути видана іншому пристрою.

### Статична vs динамічна IP-адреса

| Параметр | Статична IP | Динамічна IP (DHCP) |
|----------|------------|-------------------|
| Налаштування | Вручну адміністратором | Автоматично через DHCP |
| Зміна | Не змінюється | Може змінюватися |
| Використання | Сервери, принтери, роутери | Комп'ютери, телефони, планшети |
| Переваги | Стабільність, передбачуваність | Зручність, немає конфліктів |
| Недоліки | Трудомістке налаштування | Адреса може змінитися |

**Коли потрібна статична IP:** серверу (веб-сервер, DNS-сервер), мережевому принтеру, роутеру -- тобто пристроям, до яких інші мають звертатися за фіксованою адресою.

**Коли підходить динамічна IP:** звичайним комп'ютерам, ноутбукам, смартфонам -- їм не потрібна постійна адреса.

## 10. IPv6 -- коротко про протокол нового покоління

### Навіщо потрібен IPv6?

IPv4-адрес лише ~4,3 мільярда, і вони **вже вичерпані** (пул IANA був розподілений ще у 2011 році). З кожним днем до Інтернету підключається все більше пристроїв: смартфони, планшети, розумні годинники, датчики IoT тощо. **IPv6** був створений для вирішення цієї проблеми.

### Формат IPv6-адреси

- **128 біт** (замість 32 біт у IPv4)
- Записується як **8 груп по 4 шістнадцяткові символи**, розділені двокрапками
- Використовуються цифри 0--9 та літери a--f

**Приклад:**

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

### Скорочення запису

**Правило 1 -- прибрати ведучі нулі:**

`2001:0db8:0000:0042` --> `2001:db8:0:42`

**Правило 2 -- замінити послідовні групи нулів на `::`** (можна лише один раз):

`2001:0db8:0000:0000:0000:0000:0000:0001` --> `2001:db8::1`

### Порівняння IPv4 та IPv6

| Характеристика | IPv4 | IPv6 |
|----------------|------|------|
| Розмір адреси | 32 біти | 128 біт |
| Кількість адрес | ~4,3 x 10^9 | ~3,4 x 10^38 |
| Нотація | Десяткова з крапками | Шістнадцяткова з двокрапками |
| Приклад | 192.168.1.1 | 2001:db8::1 |
| Конфігурація | Ручна або DHCP | Автоконфігурація (SLAAC) або DHCPv6 |
| Broadcast | Є | Немає (замінений multicast) |

### Чому перехід на IPv6 повільний?

- Більшість існуючого обладнання та програм розраховані на IPv4.
- NAT та приватні адреси дозволили "розтягнути" ресурс IPv4.
- Потрібна підтримка IPv6 на всіх пристроях маршруту.
- Багато мереж працюють у **dual-stack** режимі (підтримують і IPv4, і IPv6 одночасно).

## Глосарій

| Термін | Визначення |
|--------|------------|
| **IP-адреса** | Унікальна числова адреса пристрою в мережі, що використовується для ідентифікації та обміну даними |
| **IPv4** | Четверта версія інтернет-протоколу; адреса складається з 32 біт (4 октети) |
| **IPv6** | Шоста версія інтернет-протоколу; адреса складається з 128 біт (8 груп по 4 hex-символи) |
| **Октет** | Група з 8 біт; один із чотирьох компонентів IPv4-адреси (значення від 0 до 255) |
| **Маска підмережі** | 32-бітне число, що визначає межу між мережевою частиною та частиною хоста в IP-адресі |
| **CIDR** | Classless Inter-Domain Routing -- безкласова міждоменна маршрутизація; сучасний спосіб запису масок (наприклад, /24) |
| **Підмережа (subnet)** | Логічний поділ IP-мережі на менші сегменти для ефективного управління та безпеки |
| **Broadcast** | Широкомовна адреса; пакет, надісланий на неї, отримають усі пристрої мережі |
| **Localhost (127.0.0.1)** | Спеціальна адреса зворотного зв'язку; вказує на сам пристрій |
| **Приватна IP-адреса** | Адреса, що використовується лише в локальній мережі та не маршрутизується в Інтернеті (RFC 1918) |
| **Публічна IP-адреса** | Глобально унікальна адреса, видима з будь-якої точки Інтернету |
| **NAT** | Network Address Translation -- технологія підміни приватних адрес на публічну для виходу в Інтернет |
| **Шлюз за замовчуванням** | Мережевий пристрій (роутер), через який дані надсилаються за межі локальної мережі |
| **DNS** | Domain Name System -- система, що перетворює доменні імена (google.com) на IP-адреси |
| **DHCP** | Dynamic Host Configuration Protocol -- протокол автоматичного призначення IP-адрес у мережі |
| **DORA** | Discover, Offer, Request, Acknowledge -- чотири кроки процесу отримання IP через DHCP |
| **Оренда (Lease)** | Обмежений час, на який DHCP-сервер видає IP-адресу клієнту |
| **Статична IP** | IP-адреса, що призначається вручну і не змінюється |
| **Динамічна IP** | IP-адреса, що автоматично призначається через DHCP і може змінюватися |
| **Multicast** | Групова передача даних одночасно кільком одержувачам |
| **APIPA** | Automatic Private IP Addressing -- автоматичне призначення адреси 169.254.x.x при недоступності DHCP |
| **RFC** | Request for Comments -- документи, що визначають стандарти Інтернету |
| **Dual-stack** | Режим одночасної роботи IPv4 та IPv6 на одному пристрої |
