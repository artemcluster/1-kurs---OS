## 1. Мета роботи

- Зрозуміти принцип роботи віртуальної пам'яті та файлу підкачки у Windows 10/11.
- Навчитися переглядати параметри оперативної пам'яті через Диспетчер завдань.
- Оволодіти навичками перегляду та зміни налаштувань файлу підкачки.
- Спостерігати за використанням пам'яті під навантаженням.
- Ознайомитися з інструментами моніторингу: Монітор ресурсів (resmon), systeminfo, PowerShell.
- Навчитися повертати налаштування віртуальної пам'яті до стану за замовчуванням.

---

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Клавіатура та миша.
- Права адміністратора (для зміни налаштувань файлу підкачки).
- Командний рядок (CMD) та PowerShell.
- Браузер (для створення навантаження на пам'ять).

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не вимикайте файл підкачки повністю** — це може призвести до нестабільної роботи системи, вильотів програм та BSOD (синього екрану смерті).
4. **Після завершення роботи обов'язково поверніть налаштування файлу підкачки до автоматичного режиму** (завдання 10).
5. При зміні розміру файлу підкачки не встановлюйте значення менше 800 МБ — це може спричинити проблеми з роботою системи.
6. Не перезавантажуйте комп'ютер без дозволу викладача (деякі зміни набувають чинності після перезавантаження).
7. Після завершення роботи приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Що таке віртуальна пам'ять

Кожна програма, яку ви запускаєте, потребує **оперативну пам'ять (RAM)** — швидку пам'ять, де зберігаються дані під час роботи. Але оперативна пам'ять обмежена: на навчальних комп'ютерах зазвичай 4–16 ГБ. Що станеться, якщо відкрити одночасно браузер із 30 вкладками, текстовий редактор, гру та кілька інших програм? RAM закінчиться.

Саме тут допомагає **віртуальна пам'ять** — механізм, при якому операційна система використовує частину жорсткого диска (або SSD) як додаткову «оперативну пам'ять». Дані, до яких програми давно не зверталися, переміщуються з RAM на диск, звільняючи місце для активних програм.

**Як це працює:**

```
┌─────────────────────────────────────┐
│          ВІРТУАЛЬНА ПАМ'ЯТЬ        │
│    (те, що «бачить» кожна програма) │
│                                     │
│   ┌───────────┐   ┌───────────────┐ │
│   │    RAM    │   │ Файл підкачки │ │
│   │  (швидка) │   │   (повільна)  │ │
│   │  4–16 ГБ  │   │  1,5–24 ГБ   │ │
│   └───────────┘   └───────────────┘ │
└─────────────────────────────────────┘
```

Операційна система автоматично переміщує дані між RAM та файлом підкачки. Цей процес називається **підкачка** (paging або swapping). Для програм це прозоро — вони «думають», що у них набагато більше пам'яті, ніж є фізично.

---

### 4.2. Файл підкачки (pagefile.sys)

**Файл підкачки** (page file) — це прихований системний файл `pagefile.sys`, що зберігається у корені системного диска (зазвичай `C:\`). Він є фізичним представленням віртуальної пам'яті на диску.

**Основні характеристики pagefile.sys:**

| Параметр | Опис |
|----------|------|
| **Розташування** | Зазвичай `C:\pagefile.sys` (можна перенести на інший диск) |
| **Видимість** | Прихований системний файл (не видно у Провіднику за замовчуванням) |
| **Розмір** | За замовчуванням керується автоматично Windows (зазвичай 1–4 ГБ) |
| **Режими** | Автоматичне керування / Вказаний вручну розмір / Відсутній (не рекомендується) |

**Ключові терміни, які ви побачите в Диспетчері завдань:**

| Термін | Значення |
|--------|----------|
| **Загальний обсяг (Total)** | Скільки RAM фізично встановлено на комп'ютері |
| **Використовується (In use)** | Скільки RAM зараз використовують програми та система |
| **Доступно (Available)** | Скільки RAM можна негайно виділити новим програмам |
| **Committed** | Загальний обсяг пам'яті (RAM + файл підкачки), який система пообіцяла програмам. Формат: «використано / ліміт» |
| **Cached** | Дані, які система тримає в RAM для швидшого доступу (кеш файлів, SuperFetch) |
| **Paged Pool** | Область пам'яті ядра, яку МОЖНА вивантажити у файл підкачки |
| **Non-paged Pool** | Область пам'яті ядра, яка ЗАВЖДИ залишається в RAM (критичні системні дані) |

> **Аналогія:** Уявіть RAM як робочий стіл, а файл підкачки — як шухляду під столом. Поки документів мало, все лежить на столі (в RAM). Коли місця не вистачає, ви ховаєте рідко потрібні документи у шухляду (файл підкачки). Коли вони знадобляться — дістаєте назад, але це займає більше часу.

### 4.3. Навіщо налаштовувати файл підкачки

За замовчуванням Windows автоматично керує розміром файлу підкачки. Для більшості користувачів це оптимальний варіант. Однак є ситуації, коли ручне налаштування має сенс:

| Ситуація | Рекомендація |
|----------|-------------|
| Мало RAM (4 ГБ або менше) | Збільшити файл підкачки вручну для стабільності |
| Маленький системний SSD | Перенести файл підкачки на інший диск з більшим обсягом |
| Помилки BSOD (PAGE_FAULT_IN_NONPAGED_AREA) | Збільшити або зафіксувати розмір файлу підкачки |
| Декілька фізичних дисків | Розподілити файли підкачки по дисках для підвищення швидкості |
| Багато RAM (32+ ГБ) | Зазвичай автоматичне керування є достатнім |

**Як потрапити до налаштувань файлу підкачки у Windows 10/11:**

1. Натисніть **Win + R**, введіть `sysdm.cpl`, натисніть Enter.
2. Перейдіть на вкладку **«Додатково»** (Advanced).
3. У розділі **«Швидкодія»** (Performance) натисніть кнопку **«Параметри...»** (Settings...).
4. У вікні, що відкрилося, перейдіть на вкладку **«Додатково»** (Advanced).
5. У розділі **«Віртуальна пам'ять»** (Virtual Memory) натисніть кнопку **«Змінити...»** (Change...).

**Альтернативний шлях:** Пошук у меню «Пуск» --> «Настроювання представлення та продуктивності системи» (Adjust the appearance and performance of Windows).

---

### 4.4. Рекомендації щодо оптимального розміру

Загальна формула для ручного налаштування:

| Обсяг RAM | Початковий розмір (Initial) | Максимальний розмір (Maximum) |
|-----------|---------------------------|------------------------------|
| 4 ГБ | 6 000 МБ (1,5x RAM) | 12 000 МБ (3x RAM) |
| 8 ГБ | 12 000 МБ (1,5x RAM) | 24 000 МБ (3x RAM) |
| 16 ГБ | 8 000 МБ (0,5x RAM) | 16 000 МБ (1x RAM) |
| 32 ГБ і більше | 4 000–8 000 МБ | 16 000 МБ |

> **Зверніть увагу:** Чим більше RAM на комп'ютері, тим менш критичний файл підкачки. Для системи з 4 ГБ RAM файл підкачки — це необхідність. Для системи з 32 ГБ — це страховка на крайній випадок. Але навіть із великим обсягом RAM файл підкачки не слід відключати повністю.

**Різниця між початковим і максимальним розміром:**
- **Початковий розмір** (Initial size) — мінімальний розмір файлу, що створюється при завантаженні системи.
- **Максимальний розмір** (Maximum size) — до якого розміру файл може зрости під навантаженням.
- **Порада:** Якщо встановити однакові значення (початковий = максимальний), файл підкачки не буде фрагментуватися, що дещо підвищить продуктивність.

### 4.5. Hard Faults — що це і як їх спостерігати

Коли програма звертається до даних, які були переміщені з RAM у файл підкачки, операційна система повинна зчитати їх з диска назад у RAM. Цей процес називається **Hard Fault** (жорстка помилка сторінки, або сторінкова помилка з диска).

**Важливо:** Незважаючи на назву, Hard Fault — це **не помилка** у звичному розумінні. Це нормальний механізм роботи віртуальної пам'яті. Однак велика кількість Hard Faults свідчить про те, що системі не вистачає RAM і вона надто часто звертається до файлу підкачки, що уповільнює роботу.

**Де спостерігати Hard Faults:**
- **Монітор ресурсів** (resmon.exe) --> вкладка **«Пам'ять»** (Memory) --> графік **«Hard Faults/sec»** (Жорсткі помилки сторінок/с) у правій частині вікна.
- У таблиці на вкладці «Пам'ять» можна побачити стовпець **«Hard Faults»** біля кожного процесу.

| Кількість Hard Faults/sec | Інтерпретація |
|---------------------------|---------------|
| 0–10 | Нормально, система працює в межах наявної RAM |
| 10–50 | Помірне навантаження, може бути коротким піком під час запуску програм |
| 50–100+ | Постійно — ознака нестачі RAM, система активно використовує файл підкачки |
| Сотні і більше | Серйозна нестача RAM, значне уповільнення роботи |

---

### 4.6. Чого НЕ варто робити з віртуальною пам'яттю

1. **Не вимикайте файл підкачки повністю.** Навіть якщо у вас 16 або 32 ГБ RAM, деякі системні функції потребують файлу підкачки:
   - Створення дампів пам'яті при збоях (для діагностики BSOD)
   - Робота деяких програм, які очікують наявність файлу підкачки
   - Windows використовує файл підкачки для оптимізації, а не лише як «overflow»

2. **Не встановлюйте занадто маленький розмір.** Якщо файл підкачки менший за 800 МБ, система може стати нестабільною.

3. **Не забувайте повернути налаштування.** Якщо ви змінили розмір файлу підкачки для експерименту — обов'язково поверніть автоматичне керування або задайте рекомендований розмір.

4. **Не переносьте файл підкачки на повільний зовнішній диск або USB-флешку.** Це різко погіршить продуктивність.

## 5. Практичні завдання

### Завдання 1. Переглянути поточний обсяг RAM через Диспетчер завдань

1. Натисніть **Ctrl + Shift + Esc** для відкриття Диспетчера завдань (Task Manager).
2. Якщо відкрився спрощений вигляд — натисніть **«Докладніше»** (More details) внизу вікна.
3. Перейдіть на вкладку **«Продуктивність»** (Performance).
4. У лівій панелі натисніть на **«Пам'ять»** (Memory).
5. Заповніть таблицю у зошиті:

| Параметр | Значення |
|----------|----------|
| Загальний обсяг RAM (Total) | |
| Використовується (In use) | |
| Доступно (Available) | |
| Committed (використано / ліміт) | |
| Cached (кешовано) | |
| Paged Pool (вивантажуваний пул) | |
| Non-paged Pool (невивантажуваний пул) | |
| Швидкість RAM (Speed), МГц | |
| Кількість слотів (Slots used) | |
| Форм-фактор (Form factor) | |

6. Зверніть увагу на графік використання пам'яті у верхній частині. Який відсоток RAM використовується зараз? Запишіть.

> **Пояснення значення Committed:** Наприклад, «4,2 ГБ / 11,9 ГБ» означає: з 11,9 ГБ загального доступного простору (RAM + файл підкачки) система вже пообіцяла програмам 4,2 ГБ. Якщо перше число наближається до другого — системі може не вистачити пам'яті.

---

### Завдання 2. Переглянути поточні налаштування файлу підкачки

1. Натисніть **Win + R**, введіть `sysdm.cpl`, натисніть Enter.
2. Перейдіть на вкладку **«Додатково»** (Advanced).
3. У розділі «Швидкодія» натисніть **«Параметри...»** (Settings...).
4. У вікні «Параметри швидкодії» перейдіть на вкладку **«Додатково»** (Advanced).
5. Знайдіть розділ **«Віртуальна пам'ять»** (Virtual Memory). Запишіть, який обсяг вказано як «Загальний обсяг файлу підкачки на всіх дисках» (Total paging file size for all drives).
6. Натисніть кнопку **«Змінити...»** (Change...).
7. Заповніть таблицю:

| Параметр | Значення |
|----------|----------|
| Чи ввімкнено «Автоматично керувати розміром файлу підкачки для всіх дисків»? | |
| На якому диску розміщено файл підкачки? | |
| Тип налаштування (Автоматичне / Вказаний розмір / Без файлу) | |
| Поточний розмір файлу підкачки (МБ) | |
| Мінімально допустимий (Minimum allowed), МБ | |
| Рекомендований (Recommended), МБ | |
| Поточний виділений (Currently allocated), МБ | |

8. **Не змінюйте нічого на цьому етапі!** Натисніть «Скасувати» (Cancel) для виходу.

---

### Завдання 3. Змінити розмір файлу підкачки вручну

> **УВАГА!** Виконуйте це завдання уважно та запам'ятайте поточні значення, щоб потім повернути їх у завданні 10.

1. Повторіть кроки 1–6 із завдання 2, щоб відкрити вікно налаштування віртуальної пам'яті.
2. Зніміть галочку **«Автоматично керувати розміром файлу підкачки для всіх дисків»** (Automatically manage paging file size for all drives).
3. Виберіть диск **C:** у списку.
4. Оберіть пункт **«Указати розмір»** (Custom size).
5. Введіть значення:
   - **Початковий розмір (Initial size), МБ:** `2048` (2 ГБ)
   - **Максимальний розмір (Maximum size), МБ:** `4096` (4 ГБ)
6. Натисніть кнопку **«Задати»** (Set).
7. Натисніть **ОК** у всіх вікнах.
8. Якщо система запропонує перезавантажити комп'ютер — **НЕ перезавантажуйте** зараз, натисніть «Ні» або «Пізніше».
9. Запишіть у зошит, які значення ви встановили, та чи з'явилося повідомлення про необхідність перезавантаження.

> **Примітка:** Деякі зміни файлу підкачки набувають чинності лише після перезавантаження. Однак Windows дозволяє змінювати максимальний розмір без перезавантаження, якщо він збільшується.

---

### Завдання 4. Створити навантаження на RAM та спостерігати за використанням пам'яті

1. Відкрийте **Диспетчер завдань** (Ctrl + Shift + Esc) --> вкладка **«Продуктивність»** --> **«Пам'ять»**.
2. Розмістіть вікно Диспетчера завдань у правій частині екрана, щоб бачити графік.
3. Запишіть **поточні значення**:

| Параметр | До навантаження | Після навантаження |
|----------|----------------|-------------------|
| Використовується (In use) | | |
| Доступно (Available) | | |
| Committed | | |

4. Тепер створіть навантаження на пам'ять:
   - Відкрийте **браузер** (Chrome, Edge або Firefox).
   - Відкрийте **15–20 нових вкладок** із різними веб-сайтами (наприклад, wikipedia.org, youtube.com, різні новинні сайти).
   - Відкрийте **Блокнот** (Notepad), **Калькулятор**, **Paint**, **Провідник** (2–3 вікна).
   - Якщо є можливість — відкрийте будь-яку «важку» програму (Word, Excel тощо).
5. Зачекайте 30 секунд, поки система стабілізується.
6. Запишіть **нові значення** у стовпець «Після навантаження».
7. Порівняйте значення та запишіть відповіді:
   - На скільки збільшилося використання RAM?
   - Чи змінився обсяг Committed?
   - Який відсоток RAM використовується тепер?

### Завдання 5. Моніторинг Hard Faults через Монітор ресурсів

1. Натисніть **Win + R**, введіть `resmon`, натисніть Enter.
2. Перейдіть на вкладку **«Пам'ять»** (Memory).
3. У правій частині вікна знайдіть три графіки:
   - **Used Physical Memory** (Використана фізична пам'ять)
   - **Commit Charge** (Виділена пам'ять)
   - **Hard Faults/sec** (Жорсткі помилки сторінок/с)
4. Спостерігайте за графіком **Hard Faults/sec** протягом 1 хвилини. Запишіть:

| Параметр | Значення |
|----------|----------|
| Середнє значення Hard Faults/sec (приблизно) | |
| Чи були піки? Якщо так — якої величини? | |
| Який процес має найбільше Hard Faults? (подивіться у таблиці ліворуч, стовпець Hard Faults) | |

5. **Не закриваючи Монітор ресурсів**, відкрийте ще 5–10 вкладок у браузері або запустіть нову програму. Спостерігайте за зміною графіка Hard Faults/sec.
6. Запишіть: чи збільшилася кількість Hard Faults після додаткового навантаження?

> **Інтерпретація:** Якщо Hard Faults/sec постійно перевищує 50–100, це означає, що система часто звертається до файлу підкачки — можлива нестача RAM. На комп'ютерах із достатнім обсягом RAM (8+ ГБ) значення зазвичай близьке до нуля, з короткими піками при запуску нових програм.

### Завдання 6. Перегляд інформації через CMD: systeminfo

1. Натисніть **Win + R**, введіть `cmd`, натисніть Enter.
2. Введіть команду:
   ```
   systeminfo | find "Memory"
   ```
   (у Windows з українською локалізацією може знадобитися: `systeminfo | find "пам"` або `systeminfo | find "мор"`)
3. Дочекайтеся виконання (може тривати 15–30 секунд, оскільки systeminfo збирає повну інформацію про систему).
4. Запишіть результати:

| Параметр | Значення |
|----------|----------|
| Total Physical Memory (Загальний обсяг фізичної пам'яті) | |
| Available Physical Memory (Доступна фізична пам'ять) | |
| Virtual Memory: Max Size (Віртуальна пам'ять: макс. розмір) | |
| Virtual Memory: Available (Віртуальна пам'ять: доступна) | |
| Virtual Memory: In Use (Віртуальна пам'ять: використовується) | |

5. Обчисліть і запишіть:
   - Скільки віртуальної пам'яті використовується у відсотках? (In Use / Max Size * 100%)
   - Чому значення «Virtual Memory: Max Size» більше за «Total Physical Memory»?

> **Підказка:** Якщо команда `find "Memory"` не знаходить рядків — спробуйте виконати просто `systeminfo` і знайти потрібні рядки вручну. Запишіть результати.

### Завдання 7. Перегляд інформації через PowerShell: Win32_PageFileUsage

1. Натисніть **Win + R**, введіть `powershell`, натисніть Enter.
2. Введіть команду:
   ```powershell
   Get-CimInstance -ClassName Win32_PageFileUsage | Select-Object Name, AllocatedBaseSize, CurrentUsage, PeakUsage
   ```
3. Запишіть результат:

| Параметр | Значення |
|----------|----------|
| Name (шлях до файлу підкачки) | |
| AllocatedBaseSize (виділений розмір, МБ) | |
| CurrentUsage (поточне використання, МБ) | |
| PeakUsage (пікове використання, МБ) | |

4. Тепер введіть команду для перевірки режиму керування:
   ```powershell
   (Get-CimInstance Win32_ComputerSystem).AutomaticManagedPagefile
   ```
5. Запишіть результат: **True** (автоматичне керування) або **False** (ручне налаштування).

6. Додатково введіть команду для перегляду налаштувань:
   ```powershell
   Get-CimInstance -ClassName Win32_PageFileSetting | Select-Object Name, InitialSize, MaximumSize
   ```
7. Запишіть: чи збігаються значення InitialSize та MaximumSize з тими, які ви встановили у завданні 3?

> **Примітка:** Клас `Win32_PageFileUsage` показує **поточний стан** файлу підкачки (під час роботи системи), а клас `Win32_PageFileSetting` — **налаштування**, які визначають параметри при завантаженні.

### Завдання 10. Повернути налаштування файлу підкачки до автоматичного режиму

> **Це завдання є ОБОВ'ЯЗКОВИМ! Не пропускайте його!**

1. Натисніть **Win + R**, введіть `sysdm.cpl`, натисніть Enter.
2. Вкладка **«Додатково»** --> **«Параметри...»** (у розділі «Швидкодія»).
3. Вкладка **«Додатково»** --> **«Змінити...»** (у розділі «Віртуальна пам'ять»).
4. Поставте галочку **«Автоматично керувати розміром файлу підкачки для всіх дисків»** (Automatically manage paging file size for all drives).
5. Натисніть **ОК** у всіх вікнах.
6. Підтвердіть у зошиті: «Налаштування файлу підкачки повернуто до автоматичного режиму».
7. Перевірте через PowerShell:
   ```powershell
   (Get-CimInstance Win32_ComputerSystem).AutomaticManagedPagefile
   ```
   Результат має бути **True**.

> **Чому це важливо:** Якщо залишити вручну налаштований файл підкачки з малим розміром, наступний користувач або адміністратор може зіткнутися з проблемами продуктивності. Завжди залишайте систему в робочому стані!

6. Уявіть ситуацію: на комп'ютері з 4 ГБ RAM файл підкачки вимкнено повністю. Користувач відкрив браузер із 30 вкладками, Word, Excel та Photoshop. Що станеться? Опишіть можливі наслідки.
