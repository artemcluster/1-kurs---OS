## 1. Загрози безпеці операційних систем

Сучасні операційні системи постійно перебувають під загрозою з боку різноманітних шкідливих програм та методів атак. Щодня у світі виявляється приблизно **560 000 нових зразків шкідливого ПЗ**. Розуміння основних загроз -- перший крок до ефективного захисту.

### 1.1. Основні типи загроз

**Віруси** -- програми, що вбудовуються в інші файли (програми, документи) і активуються при їх запуску. Вірус не може розповсюджуватись самостійно -- йому потрібна дія користувача (запуск файлу, відкриття документа). Різновиди вірусів:
- *Файлові інфектори* -- прикріплюються до виконуваних файлів (.exe, .com)
- *Макровіруси* -- заражають макроси в документах Microsoft Office
- *Перезаписувальні віруси* -- знищують вміст файлів, замінюючи їх шкідливим кодом
- *Поліморфні віруси* -- змінюють свій код при кожному копіюванні, щоб уникнути виявлення

**Трояни (троянські програми)** -- маскуються під корисне програмне забезпечення, але виконують шкідливі дії у фоновому режимі. На відміну від вірусів, трояни не розмножуються самостійно -- користувач сам завантажує та встановлює їх, вважаючи легітимними програмами. Трояни часто слугують механізмом доставки для інших типів шкідливого ПЗ.

**Черв'яки (Worms)** -- самостійно розповсюджуються мережею, експлуатуючи вразливості системи. На відміну від вірусів, не потребують дій користувача для поширення. Їхня здатність автономно розмножуватись робить їх особливо небезпечними -- один черв'як може заразити тисячі комп'ютерів за лічені години.

**Ransomware (програми-вимагачі)** -- шифрують файли або блокують доступ до системи й вимагають викуп за розшифрування. Сучасні атаки використовують тактику "подвійного вимагання": зловмисники не лише шифрують дані, а й викрадають їх, погрожуючи публічним оприлюдненням. Середня вартість відновлення після ransomware-атаки у 2025 році складає приблизно 1,53 мільйона доларів. Існує модель Ransomware-as-a-Service (RaaS), за якої кіберзлочинці продають доступ до ransomware за підписку.

**Spyware (шпигунське ПЗ)** -- збирає конфіденційну інформацію з заражених систем без відома користувача. Може відстежувати відвідані веб-сайти, перехоплювати натискання клавіш (кейлогер), записувати облікові дані для входу.

**Adware (рекламне ПЗ)** -- відображає небажану рекламу та спливаючі вікна. Зазвичай потрапляє разом із завантаженням безкоштовного програмного забезпечення. Генерує дохід для розповсюджувача, примусово показуючи рекламу.

**Rootkits** -- надають зловмиснику віддалений контроль над комп'ютером жертви з повними правами адміністратора. Можуть бути впроваджені в ядро ОС, прошивку або гіпервізор. Rootkits надзвичайно складно виявити стандартними засобами, оскільки вони маскують свою присутність на найглибшому рівні системи.

### 1.2. Методи атак

**Фішинг (Phishing)** -- метод шахрайства, при якому зловмисники створюють підроблені веб-сайти або надсилають електронні листи, що імітують легітимні організації (банки, поштові сервіси, соціальні мережі), з метою виманити логіни, паролі, дані банківських карток. Ознаки фішингу:
- "Терміновий" тон повідомлення, що створює штучний тиск
- Граматичні помилки у нібито офіційних листах
- Посилання, що ведуть не на офіційний домен (наприклад, g00gle.com замість google.com)

**Соціальна інженерія** -- маніпулювання людьми з метою отримання конфіденційної інформації або доступу до систем. Зловмисники експлуатують довіру, страх, цікавість або бажання допомогти. Приклади: дзвінки нібито від "технічної підтримки", підкидання заражених USB-флешок, прохання "підтримати" посилання від знайомого в соцмережах (чий акаунт було зламано).

## 2. Класифікація шкідливого програмного забезпечення

| Тип шкідливого ПЗ | Спосіб розповсюдження | Основна дія | Потребує дій користувача? | Приклади |
|---|---|---|---|---|
| **Вірус** | Через заражені файли | Руйнування файлів, порушення роботи програм | Так (запуск файлу) | ILOVEYOU, Melissa, CIH |
| **Троян** | Маскування під легітимне ПЗ | Крадіжка даних, створення бекдорів | Так (встановлення програми) | Zeus, Emotet, SnakeKeylogger |
| **Черв'як** | Самостійно через мережу | Перевантаження мережі, доставка іншого ПЗ | Ні | WannaCry, Mirai, Blaster |
| **Ransomware** | Фішинг, вразливості | Шифрування файлів, вимагання викупу | Часто так (відкриття вкладення) | WannaCry, Petya, LockBit |
| **Spyware** | Разом з іншим ПЗ, вразливості | Крадіжка персональних даних | Не завжди | Pegasus, FinFisher, CoolWebSearch |
| **Adware** | Разом з безкоштовним ПЗ | Показ небажаної реклами | Частково (встановлення) | Fireball, DollarRevenue |
| **Rootkit** | Через вразливості, фішинг | Приховане керування системою | Не завжди | Necurs, ZeroAccess, Azazel |
| **Кейлогер** | Трояни, фізичний доступ | Перехоплення натискань клавіш | Не завжди | Ardamax, Revealer Keylogger |
| **Бекдор** | Трояни, вразливості | Створення прихованого віддаленого доступу | Ні | Back Orifice, Poison Ivy |
| **Криптомайнер** | Веб-скрипти, трояни | Видобуток криптовалюти за рахунок ресурсів жертви | Не завжди | Coinhive, XMRig |

### Сучасні тенденції (2025)

- **Інфостілери** (infostealer) -- програми, що викрадають облікові дані, куки браузера, токени автентифікації. Складають **37% усіх інцидентів** зі шкідливим ПЗ у 2025 році.
- **Безфайлове шкідливе ПЗ (Fileless malware)** -- існує лише в оперативній пам'яті, використовує легітимні системні інструменти (PowerShell) для виконання. Надзвичайно складне для виявлення.
- **Malware-as-a-Service (MaaS)** -- комерційні пропозиції шкідливого ПЗ за підписку складають понад **60%** усіх спостережених кампаній.
- **AI-malware** -- новий клас шкідливого ПЗ, що використовує штучний інтелект для адаптації, маскування та більш ефективних атак.

## 3. Брандмауер (Firewall)

### 3.1. Що таке брандмауер і як він працює

**Брандмауер (firewall, мережевий екран)** -- це захисний бар'єр між комп'ютером та мережею, який контролює вхідний і вихідний мережевий трафік на основі заданих правил безпеки.

**Принцип роботи -- фільтрація пакетів за правилами:**
1. Кожен мережевий пакет, що надходить до комп'ютера або виходить з нього, проходить через брандмауер.
2. Брандмауер аналізує параметри пакета: IP-адресу джерела та призначення, протокол (TCP, UDP, ICMP), порт джерела та призначення.
3. Пакет порівнюється з набором правил (правила вхідного та вихідного трафіку).
4. На основі результату порівняння пакет **дозволяється** або **блокується**.

```
Інтернет  ──►  [ Брандмауер ]  ──►  Комп'ютер
                  │ Правила:    │
                  │ ✓ Дозволити │
                  │ ✗ Заблокувати│
```

### 3.2. Брандмауер Windows (Windows Defender Firewall)

Брандмауер Windows -- вбудований хост-брандмауер, який увімкнений за замовчуванням у всіх версіях Windows. Він є частиною підсистеми безпеки операційної системи.

#### Три мережеві профілі

Брандмауер Windows використовує три мережеві профілі, кожен з яких має власний набір правил:

| Профіль | Коли активний | Рівень обмежень | Приклад |
|---|---|---|---|
| **Доменний (Domain)** | Комп'ютер підключено до мережі з контролером домену Active Directory | Середній (керується адміністратором) | Корпоративна мережа офісу |
| **Приватний (Private)** | Підключення до довіреної домашньої або робочої мережі | Помірний | Домашня Wi-Fi мережа |
| **Публічний (Public)** | Підключення до загальнодоступної мережі | Найвищий (найсуворіші правила) | Wi-Fi у кав'ярні, аеропорту, готелі |

Windows автоматично визначає тип мережі за допомогою служби **Network Location Awareness (NLA)** і застосовує відповідний профіль.

#### Правила вхідного та вихідного трафіку

- **Правила вхідного трафіку (Inbound Rules)** -- контролюють трафік, що надходить до системи. Захищають від несанкціонованого віддаленого доступу та мережевого сканування.
- **Правила вихідного трафіку (Outbound Rules)** -- контролюють трафік, що виходить із системи. Запобігають витоку даних та блокують зв'язок шкідливого ПЗ із зовнішніми серверами.

Кожне правило визначає:
- Програму або службу
- Протокол (TCP/UDP)
- Порт (або діапазон портів)
- IP-адресу (або діапазон адрес)
- Дію: дозволити, блокувати або дозволити за умови безпечного з'єднання (IPsec)

#### Як відкрити брандмауер Windows

| Команда | Що відкриває | Як запустити |
|---|---|---|
| `firewall.cpl` | Базові налаштування брандмауера | Win+R, ввести `firewall.cpl`, Enter |
| `wf.msc` | Розширені налаштування (Windows Firewall with Advanced Security) | Win+R, ввести `wf.msc`, Enter |

Також можна відкрити через: **Параметри** --> **Конфіденційність і безпека** --> **Безпека Windows** --> **Брандмауер і захист мережі**.

#### Дозвіл або блокування програм через брандмауер

Щоб дозволити або заблокувати програму:
1. Відкрити `firewall.cpl`
2. Натиснути **"Дозволити програму або функцію через брандмауер Windows"**
3. Натиснути **"Змінити налаштування"** (потрібні права адміністратора)
4. Знайти програму у списку або натиснути **"Дозволити іншу програму..."**
5. Встановити або зняти позначки для **Приватної** та **Публічної** мережі
6. Натиснути **OK**

Через розширені налаштування (`wf.msc`) можна створити детальніші правила: вказати конкретні порти, протоколи, IP-адреси та умови застосування.

> **Важливо:** Microsoft рекомендує ніколи не вимикати брандмауер повністю, оскільки це позбавляє систему захисту IPsec, протидії мережевим атакам, служби захисту Windows та фільтрів завантаження.

## 4. Антивірусний захист

### 4.1. Методи виявлення шкідливого ПЗ

| Метод | Як працює | Переваги | Недоліки |
|---|---|---|---|
| **Сигнатурний** | Порівнює файли з базою відомих "відбитків" (сигнатур) шкідливого ПЗ | Високоточний для відомих загроз, швидкий | Не виявляє нові (невідомі) загрози |
| **Евристичний** | Аналізує код програми на наявність підозрілих конструкцій та шаблонів | Виявляє нові модифікації відомих загроз | Вищий рівень хибних спрацювань |
| **Поведінковий (Behavioral)** | Відстежує поведінку програм у реальному часі: дії з файлами, реєстром, мережею | Виявляє невідомі загрози за їхніми діями | Загроза спрацьовує до виявлення |
| **Хмарний (Cloud-based)** | Надсилає підозрілі файли до хмарних серверів для аналізу потужнішими засобами | Швидке оновлення, колективний інтелект мільйонів пристроїв | Потребує інтернет-з'єднання |

Сучасні антивіруси зазвичай поєднують усі чотири методи для максимального захисту.

### 4.2. Microsoft Defender (Windows Defender Antivirus)

Microsoft Defender -- вбудований антивірус Windows, який забезпечує комплексний захист без додаткової оплати. У 2025 році отримує відмінні оцінки від незалежних лабораторій AV-TEST та AV-Comparatives.

**Основні можливості:**

**Захист у реальному часі (Real-time protection):**
- Безперервне сканування файлів, програм та запущених процесів
- Блокування підозрілої активності до нанесення шкоди
- Використання евристичного та поведінкового аналізу

**Типи сканування:**
- *Швидке сканування (Quick Scan)* -- перевіряє найбільш вразливі місця системи (~5 хв)
- *Повне сканування (Full Scan)* -- перевіряє всі файли та програми на диску (30-60+ хв)
- *Настроюване сканування (Custom Scan)* -- перевірка обраних папок або дисків
- *Автономне сканування (Offline Scan)* -- перезавантажує систему та сканує до завантаження ОС, ефективне проти rootkits

**Оновлення:**
- Автоматичне оновлення сигнатурних баз через Windows Update
- Хмарний захист (MAPS -- Microsoft Advanced Protection Service) для аналізу нових загроз
- Функція "Блокування при першому виявленні" (Block at First Sight) -- машинне навчання прогнозує шкідливість невідомого файлу

**Журнал захисту (Protection History):**
- Відображає всі виявлені загрози та вжиті дії
- Показує видалені потенційно небажані програми
- Фіксує стан ключових служб безпеки

**Захист від ransomware:**
- Контрольований доступ до папок (Controlled Folder Access) -- запобігає несанкціонованому доступу до захищених папок
- Інтеграція з OneDrive для відновлення даних після атаки

### 4.3. Інші популярні антивіруси

| Антивірус | Особливості | Платний/безкоштовний |
|---|---|---|
| **Kaspersky** | Потужний евристичний аналіз, захист онлайн-платежів, батьківський контроль. Розроблений у Росії, що викликає суперечки щодо довіри у деяких країнах | Є безкоштовна та платна версії |
| **Norton** | Комплексний захист, вбудований VPN, менеджер паролів, моніторинг Dark Web | Платний (є пробний період) |
| **Bitdefender** | Один із найкращих показників виявлення, низьке навантаження на систему, антифішинг | Є безкоштовна та платна версії |
| **Avast** | Широкі можливості у безкоштовній версії, Wi-Fi інспектор, менеджер паролів | Є безкоштовна та платна версії |

> **Зверніть увагу:** Якщо встановити сторонній антивірус, Microsoft Defender автоматично вимкнеться. При видаленні стороннього антивіруса Defender автоматично увімкнеться знову.

## 5. Безпека Windows (Windows Security)

**Windows Security (Безпека Windows)** -- це центральна панель управління безпекою, вбудована в Windows 10/11. Вона об'єднує всі засоби захисту в одному місці.

**Як відкрити:** Пуск --> Параметри --> Конфіденційність і безпека --> Безпека Windows, або знайти "Безпека Windows" через пошук.

### Розділи Windows Security

| Розділ | Призначення |
|---|---|
| **Захист від вірусів і загроз** (Virus & threat protection) | Управління антивірусом, запуск сканувань, налаштування захисту в реальному часі, оновлення баз, захист від ransomware |
| **Захист облікових записів** (Account protection) | Налаштування Windows Hello (вхід за допомогою обличчя, відбитка пальця, PIN), динамічне блокування |
| **Брандмауер і захист мережі** (Firewall & network protection) | Управління брандмауером для кожного профілю (приватний, публічний, доменний) |
| **Керування програмами та браузером** (App & browser control) | Microsoft Defender SmartScreen -- захист від небезпечних програм, файлів, сайтів та завантажень |
| **Безпека пристрою** (Device security) | Ізоляція ядра (Core Isolation), модуль TPM, безпечне завантаження (Secure Boot), шифрування даних |
| **Продуктивність і справність пристрою** (Device performance & health) | Звіти про стан системи, ємність сховища, стан драйверів |
| **Сімейні параметри** (Family options) | Батьківський контроль, звіти про активність дітей, обмеження часу використання |
| **Журнал захисту** (Protection history) | Історія всіх дій захисту, виявлені загрози та їх обробка |

### Індикатори стану

- **Зелений** -- пристрій захищено, рекомендованих дій немає
- **Жовтий** -- є рекомендація з безпеки, яку варто розглянути
- **Червоний** -- попередження, що потребує негайної уваги

## 6. BitLocker -- шифрування дисків

**BitLocker** -- вбудована технологія шифрування дисків у Windows (доступна в редакціях Pro, Enterprise, Education).

### Що робить BitLocker

BitLocker шифрує весь вміст диска, роблячи дані нечитабельними без ключа розшифрування. Навіть якщо зловмисник фізично отримає доступ до жорсткого диска (вкраде ноутбук), він не зможе прочитати дані.

### Як працює

- Використовує алгоритм шифрування AES (128 або 256 біт)
- Працює разом з модулем **TPM (Trusted Platform Module)** -- апаратним чіпом безпеки на материнській платі
- Ключ шифрування може зберігатись у TPM, на USB-носії або захищатись паролем

### Апаратне прискорення BitLocker (2025-2026)

Microsoft впроваджує **апаратно-прискорений BitLocker** починаючи з Windows 11 25H2:
- Криптографічні операції перекладаються з процесора на спеціалізований апаратний модуль у сучасних процесорах
- Швидкість читання/запису зростає майже вдвічі порівняно з програмним шифруванням
- Навантаження на процесор зменшується приблизно на 70%
- Покращується час автономної роботи ноутбуків

### Як увімкнути BitLocker

1. Відкрити **Панель керування** --> **Система та безпека** --> **Шифрування диска BitLocker**
2. Або через командний рядок: `manage-bde -on C:`
3. Обрати спосіб розблокування (TPM, пароль, USB-ключ)
4. Зберегти ключ відновлення (обов'язково!) -- в обліковий запис Microsoft, у файл або роздрукувати
5. Обрати режим шифрування та запустити процес

> **Перевірка стану:** команда `manage-bde -status` покаже стан шифрування усіх дисків.

## 7. Windows Update -- критичність оновлень для безпеки

### Чому оновлення критично важливі

Близько **30% кібератак на організації починаються з експлуатації відомих вразливостей**, для яких вже існують виправлення. Оновлення системи -- один із найпростіших і найефективніших способів захисту.

### Типи оновлень Windows

| Тип оновлення | Опис | Частота |
|---|---|---|
| **Оновлення безпеки (Security Updates)** | Виправляють критичні вразливості | Щомісяця ("Patch Tuesday" -- другий вівторок місяця) та позапланово для критичних загроз |
| **Накопичувальні оновлення (Cumulative Updates)** | Містять усі попередні виправлення + нові | Щомісяця |
| **Оновлення функцій (Feature Updates)** | Нова функціональність ОС | 1-2 рази на рік |
| **Оновлення драйверів** | Виправлення та покращення для обладнання | За потребою |
| **Оновлення визначень (Definition Updates)** | Нові сигнатури для Microsoft Defender | Кілька разів на день |

### Як перевірити оновлення

**Параметри** --> **Windows Update** --> **Перевірити наявність оновлень**

Або через командний рядок:
```
UsoClient StartScan
```

> **Порада:** Увімкніть автоматичне оновлення та не відкладайте встановлення оновлень безпеки. Кожен день затримки -- це день, протягом якого ваша система вразлива до відомих загроз.

## 8. Базові правила кібергігієни

Дослідження Microsoft показує, що п'ять базових практик кібергігієни здатні запобігти **понад 99% кібератак**. Ось десять основних правил:

### 10 правил кібергігієни

**1. Використовуйте надійні та унікальні паролі**
- Довжина паролю важливіша за складність -- використовуйте парольні фрази від 16 символів
- Не використовуйте один пароль для різних акаунтів
- Використовуйте менеджер паролів (Bitwarden, KeePass, 1Password)

**2. Увімкніть багатофакторну автентифікацію (MFA)**
- Навіть якщо зловмисник дізнається ваш пароль, без другого фактора він не отримає доступ
- Використовуйте додатки-автентифікатори (Google Authenticator, Microsoft Authenticator) замість SMS

**3. Регулярно оновлюйте систему та програми**
- Встановлюйте оновлення безпеки одразу після їх появи
- Увімкніть автоматичне оновлення для ОС та основних програм

**4. Будьте пильні щодо фішингу**
- Не натискайте на підозрілі посилання в листах та повідомленнях
- Перевіряйте адресу відправника та URL-адресу сайту перед введенням даних
- Наведіть курсор на посилання, щоб побачити реальну адресу

**5. Обережно поводьтесь з публічними Wi-Fi мережами**
- Не вводьте конфіденційні дані (паролі, банківські дані) у публічних мережах
- Використовуйте VPN для шифрування трафіку в публічних мережах
- Вимикайте Wi-Fi та Bluetooth, коли не використовуєте

**6. Створюйте резервні копії даних**
- Дотримуйтесь правила 3-2-1: 3 копії даних, на 2 різних носіях, 1 копія за межами основного місця
- Регулярно перевіряйте можливість відновлення з резервних копій

**7. Встановлюйте програми лише з надійних джерел**
- Завантажуйте програми з офіційних сайтів або магазинів додатків
- Не встановлюйте "зламане" (піратське) програмне забезпечення -- воно часто містить шкідливий код

**8. Захищайте мобільні пристрої**
- Встановіть PIN-код або біометричний захист
- Увімкніть функцію віддаленого стирання даних у разі втрати
- Не встановлюйте додатки з невідомих джерел

**9. Контролюйте дозволи програм**
- Перевіряйте, які дозволи запитують програми (камера, мікрофон, місцезнаходження)
- Відкликайте непотрібні дозволи
- Видаляйте програми, якими не користуєтесь

**10. Навчайтесь та будьте в курсі**
- Слідкуйте за новинами кібербезпеки
- Навчайте рідних та друзів базовим правилам безпеки
- Пам'ятайте: безпека -- це постійний процес, а не одноразова дія

## 9. Засоби безпеки в Linux

Linux має власний набір інструментів безпеки, відмінний від Windows.

### 9.1. Брандмауери в Linux

**iptables** -- класичний інструмент фільтрації пакетів у Linux, що працює через підсистему Netfilter ядра Linux. Довгий час був стандартним брандмауером, але поступово замінюється на nftables.

**nftables** -- сучасний наступник iptables (включений в ядро Linux з версії 3.13). Переваги:
- Спрощений синтаксис правил
- Вища продуктивність
- Єдиний інструмент замість кількох (iptables, ip6tables, arptables, ebtables)

Приклад правила nftables для блокування вхідного трафіку на порт 22 (SSH):
```
nft add rule inet filter input tcp dport 22 drop
```

**UFW (Uncomplicated Firewall)** -- спрощений інтерфейс для управління брандмауером в Ubuntu/Debian. UFW є "фронтендом" для iptables/nftables, що робить налаштування брандмауера доступним для звичайних користувачів.

Основні команди UFW:
```bash
sudo ufw enable              # Увімкнути брандмауер
sudo ufw disable             # Вимкнути брандмауер
sudo ufw status              # Переглянути стан та правила
sudo ufw allow 80/tcp        # Дозволити HTTP-трафік
sudo ufw deny 23/tcp         # Заблокувати Telnet
sudo ufw allow OpenSSH       # Дозволити SSH за назвою профілю
```

### 9.2. Антивірус для Linux: ClamAV

**ClamAV** -- безкоштовний антивірус з відкритим кодом для Linux. Хоча Linux значно менш вразливий до вірусів, ніж Windows, ClamAV корисний для:
- Сканування файлів, що передаються на Windows-машини
- Перевірки поштових серверів
- Захисту файлових серверів у змішаних мережах

Встановлення та використання:
```bash
sudo apt install clamav clamav-daemon    # Встановлення
sudo freshclam                           # Оновлення бази сигнатур
clamscan -r /home                        # Рекурсивне сканування папки
```

| Інструмент | Аналог у Windows | Призначення |
|---|---|---|
| iptables / nftables | Windows Defender Firewall (ядро) | Фільтрація мережевих пакетів |
| UFW | Інтерфейс firewall.cpl | Зручне управління правилами |
| ClamAV | Microsoft Defender Antivirus | Антивірусний захист |

## 11. Глосарій

| Термін | Визначення |
|---|---|
| **Брандмауер (Firewall)** | Програмний або апаратний засіб, що фільтрує мережевий трафік за заданими правилами, дозволяючи або блокуючи з'єднання |
| **Антивірус** | Програма для виявлення, блокування та видалення шкідливого програмного забезпечення |
| **Вірус** | Шкідлива програма, що вбудовується в інші файли та активується при їх запуску |
| **Троян** | Шкідлива програма, що маскується під легітимне ПЗ; не розмножується самостійно |
| **Черв'як (Worm)** | Шкідлива програма, що самостійно розповсюджується мережею без участі користувача |
| **Ransomware** | Програма-вимагач, що шифрує файли жертви та вимагає викуп за розшифрування |
| **Spyware** | Шпигунське ПЗ, що таємно збирає інформацію про дії користувача |
| **Adware** | Рекламне ПЗ, що примусово показує рекламу |
| **Rootkit** | Програма, що забезпечує прихований адміністративний доступ до системи |
| **Фішинг (Phishing)** | Шахрайство з використанням підроблених сайтів або листів для виманювання конфіденційних даних |
| **Соціальна інженерія** | Маніпулювання людьми для отримання доступу до інформації або систем |
| **Сигнатура** | Унікальний "відбиток" (послідовність байтів) відомого шкідливого ПЗ, за яким антивірус розпізнає загрозу |
| **Евристичний аналіз** | Метод виявлення загроз шляхом аналізу структури та поведінки коду на наявність підозрілих шаблонів |
| **Поведінковий аналіз** | Метод виявлення загроз через моніторинг дій програми у реальному часі |
| **Хмарний захист** | Аналіз підозрілих файлів на потужних хмарних серверах з використанням колективного інтелекту |
| **BitLocker** | Вбудована технологія повнодискового шифрування у Windows |
| **TPM (Trusted Platform Module)** | Апаратний модуль безпеки на материнській платі для зберігання криптографічних ключів |
| **MFA (Multi-Factor Authentication)** | Багатофакторна автентифікація -- підтвердження особи двома або більше способами |
| **IPsec** | Набір протоколів для захисту даних, що передаються через IP-мережі, шляхом автентифікації та шифрування |
| **iptables / nftables** | Інструменти фільтрації мережевих пакетів у ядрі Linux |
| **UFW** | Uncomplicated Firewall -- спрощений інтерфейс управління брандмауером в Ubuntu/Debian |
| **ClamAV** | Безкоштовний антивірус з відкритим кодом для Linux |
| **Кібергігієна** | Набір регулярних дій та звичок для підтримання безпеки в цифровому середовищі |
| **VPN (Virtual Private Network)** | Віртуальна приватна мережа, що шифрує інтернет-з'єднання для захисту даних |
| **Patch Tuesday** | Другий вівторок кожного місяця, коли Microsoft випускає планові оновлення безпеки |
