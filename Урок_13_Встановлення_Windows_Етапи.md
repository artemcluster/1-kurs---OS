## 1. Вступ. Навіщо знати процес встановлення Windows

Встановлення операційної системи — одна з базових навичок будь-якого IT-спеціаліста. Навіть якщо у повсякденній роботі ви користуєтесь комп'ютером із уже встановленою ОС, рано чи пізно виникають ситуації, коли потрібно:

- встановити ОС на новий комп'ютер без операційної системи;
- перевстановити Windows після критичного збою або зараження вірусами;
- оновити стару версію Windows до нової;
- налаштувати комп'ютер для іншої людини або організації.

Розуміння етапів інсталяції дозволяє уникнути типових помилок, зберегти важливі дані та правильно налаштувати систему з самого початку.

## 2. Способи встановлення Windows

Існує три основних способи встановлення (інсталяції) Windows. Кожен із них має своє призначення та особливості.

### 2.1. Чиста інсталяція (Clean Install)

**Чиста інсталяція** — це встановлення Windows «з нуля» на повністю відформатований диск або на новий комп'ютер. Під час чистої інсталяції:

- видаляються всі попередні дані з системного розділу;
- встановлюється свіжа копія операційної системи;
- усі програми потрібно встановлювати заново;
- драйвери пристроїв також встановлюються заново.

**Коли використовують:**
- на новому комп'ютері без ОС;
- при серйозних збоях у роботі системи;
- при зараженні вірусами, яке неможливо усунути;
- коли потрібна «ідеально чиста» система без сміття від попередніх програм.

**Переваги:** максимальна швидкодія та стабільність нової системи, відсутність «залишків» старих програм.

**Недоліки:** потребує резервного копіювання даних, усі програми треба встановлювати заново.

### 2.2. Оновлення (Upgrade)

**Оновлення** — це встановлення нової версії Windows поверх існуючої з максимальним збереженням даних. Наприклад, оновлення з Windows 10 до Windows 11.

Під час оновлення:

- зберігаються особисті файли користувача;
- зберігаються (за вибором) встановлені програми та налаштування;
- системні файли замінюються на нові;
- стара версія Windows зберігається у папці `Windows.old` протягом 10 днів (для можливості повернення).

**Коли використовують:**
- коли потрібно перейти на нову версію Windows;
- коли не хочуть втрачати встановлені програми та налаштування.

**Переваги:** зручність — не потрібно перевстановлювати все заново.

**Недоліки:** можуть залишитися проблеми від старої системи, система може працювати повільніше, ніж після чистої інсталяції.

### 2.3. Відновлення (Recovery / Reset)

**Відновлення** — це повернення системи до робочого стану без використання зовнішнього інсталяційного носія. Windows 10 та Windows 11 мають вбудовану функцію «Скинути цей ПК» (Reset this PC), яка пропонує два варіанти:

- **Зберегти мої файли** — видаляються програми та налаштування, але особисті файли залишаються;
- **Видалити все** — повне очищення системи (аналог чистої інсталяції без зовнішнього носія).

**Коли використовують:**
- при незначних збоях у роботі системи;
- перед продажем або передачею комп'ютера іншій людині;
- коли немає під рукою інсталяційного USB-носія.

**Шлях:** Параметри → Система → Відновлення → Скинути цей ПК.

## 3. Системні вимоги Windows 10 та Windows 11

Перед встановленням слід переконатися, що комп'ютер відповідає мінімальним системним вимогам.

| Параметр | Windows 10 | Windows 11 |
|----------|-----------|------------|
| **Процесор** | 1 ГГц або швидший | 1 ГГц, 2+ ядра, 64-біт |
| **Оперативна пам'ять (RAM)** | 1 ГБ (32-біт) / 2 ГБ (64-біт) | 4 ГБ |
| **Вільне місце на диску** | 16 ГБ (32-біт) / 20 ГБ (64-біт) | 64 ГБ |
| **Відеокарта** | DirectX 9 з драйвером WDDM 1.0 | DirectX 12 з драйвером WDDM 2.0 |
| **Дисплей** | 800 x 600 | 720p (1280 x 720), 9" або більше |
| **TPM** | Не обов'язково | **TPM 2.0 — обов'язково** |
| **Secure Boot** | Не обов'язково | **Обов'язково** |
| **UEFI** | Підтримується, але не обов'язково | **Обов'язково (замість Legacy BIOS)** |
| **Архітектура** | 32-біт або 64-біт | **Тільки 64-біт** |

> **Зверніть увагу:** Windows 11 має значно суворіші вимоги до апаратного забезпечення порівняно з Windows 10. Головна нововведення — обов'язкова наявність модуля TPM 2.0 та підтримка Secure Boot.

## 4. Підготовка до встановлення: створення завантажувального носія

Перед чистою інсталяцією потрібно створити **завантажувальний USB-носій** (флешку) з дистрибутивом Windows. Для цього знадобиться:

- USB-флешка обсягом **8 ГБ або більше**;
- комп'ютер з доступом до Інтернету;
- офіційний інструмент **Media Creation Tool** від Microsoft (завантажується безкоштовно з сайту microsoft.com).

### Покрокове створення завантажувальної флешки:

1. Завантажте **Media Creation Tool** з офіційного сайту Microsoft.
2. Запустіть програму та прийміть ліцензійну угоду.
3. Оберіть «Створити інсталяційний носій для іншого ПК».
4. Оберіть мову, випуск (edition) та архітектуру (64-біт).
5. Оберіть «USB-носій» та вкажіть вашу флешку.
6. Дочекайтесь завершення завантаження та запису.

> **Увага!** Під час запису всі дані на флешці будуть видалені. Заздалегідь скопіюйте важливі файли.

**Альтернативний спосіб:** Завантажити ISO-образ Windows з сайту Microsoft і записати його на флешку за допомогою безкоштовної програми **Rufus**. Rufus також дозволяє обійти перевірку вимог TPM 2.0 та Secure Boot при створенні завантажувальної флешки Windows 11 (для навчальних або тестових цілей).

## 5. Етапи встановлення Windows 10/11 — покрокова інструкція

### Етап 1. Налаштування BIOS/UEFI для завантаження з USB

Щоб комп'ютер завантажився з USB-флешки, а не з жорсткого диска, потрібно змінити порядок завантаження в BIOS/UEFI:

1. **Увімкніть або перезавантажте** комп'ютер.
2. **Натисніть клавішу** для входу в BIOS/UEFI відразу після ввімкнення. Зазвичай це:
   - **Del** або **F2** — для більшості настільних ПК та ноутбуків;
   - **F10** — HP;
   - **F12** — Lenovo, Dell (для швидкого меню завантаження — Boot Menu);
   - **Esc** — деякі моделі ASUS, Acer.
3. Знайдіть розділ **Boot** (Завантаження).
4. Встановіть **USB-пристрій** першим у порядку завантаження (Boot Priority / Boot Order).
5. Переконайтесь, що увімкнено режим **UEFI** (а не Legacy/CSM) — це особливо важливо для Windows 11.
6. Збережіть зміни та перезавантажте (зазвичай клавіша **F10** → «Save & Exit»).

> **Швидке меню завантаження (Boot Menu):** Замість входу в BIOS можна натиснути F12 (або F8, Esc — залежно від виробника) під час увімкнення, щоб одноразово обрати пристрій для завантаження.

### Етап 2. Завантаження з інсталяційного носія

1. Після перезавантаження з'явиться повідомлення: **«Press any key to boot from CD or DVD...»** або подібне. Натисніть будь-яку клавішу.
2. З'явиться логотип Windows і почнеться завантаження програми встановлення.
3. Зачекайте — це може тривати від 30 секунд до кількох хвилин.

### Етап 3. Вибір мови, формату часу та клавіатури

На першому екрані програми встановлення потрібно обрати:

- **Мова встановлення** — наприклад, «Українська» або «English»;
- **Формат часу та валюти** — наприклад, «Українська (Україна)»;
- **Метод введення (клавіатура)** — наприклад, «Українська».

Після вибору натисніть **«Далі»** (Next).

### Етап 4. «Інсталювати зараз»

На наступному екрані натисніть велику кнопку **«Інсталювати зараз»** (Install now).

> Також тут є посилання «Відновлення системи» (Repair your computer) у лівому нижньому куті — воно використовується для відновлення існуючої системи, а не для нової інсталяції.

### Етап 5. Введення ключа продукту

Програма запитає **ключ продукту** (Product Key) — 25-символьний код формату XXXXX-XXXXX-XXXXX-XXXXX-XXXXX.

- Якщо у вас є ключ — введіть його.
- Якщо ви раніше вже встановлювали Windows на цьому комп'ютері — натисніть **«У мене немає ключа продукту»** (I don't have a product key). Windows активується автоматично після встановлення через цифрову ліцензію, прив'язану до обладнання.
- Якщо ви встановлюєте Windows для ознайомлення — також можна пропустити цей крок.

### Етап 6. Вибір версії (редакції) Windows

Якщо ви пропустили введення ключа, програма запропонує обрати **редакцію** Windows:

| Редакція | Призначення |
|----------|-------------|
| **Windows Home** | Для домашнього використання. Базовий набір функцій |
| **Windows Pro** | Для професіоналів та малого бізнесу. Додатково: BitLocker, Remote Desktop, групові політики, приєднання до домену |
| **Windows Education** | Для навчальних закладів. Функціональність на рівні Pro/Enterprise |
| **Windows Pro for Workstations** | Для потужних робочих станцій із серверним обладнанням |

> **Порада:** Оберіть ту версію, яка відповідає вашому ключу продукту. Якщо ви навчаєтесь — зазвичай обирають **Home** або **Pro**.

### Етап 7. Ліцензійна угода

Прочитайте (або принаймні перегляньте) **Ліцензійну угоду Microsoft** (Microsoft Software License Terms). Поставте позначку **«Я приймаю умови ліцензії»** та натисніть **«Далі»**.

> Без прийняття ліцензійної угоди продовжити встановлення неможливо.

### Етап 8. Тип інсталяції

Програма пропонує два варіанти:

- **Оновлення (Upgrade)** — встановлення Windows зі збереженням файлів, програм і параметрів. Цей варіант доступний лише якщо на комп'ютері вже є попередня версія Windows.
- **Вибіркова: тільки встановлення Windows (Custom)** — чиста інсталяція. Дозволяє обрати розділ диска, відформатувати його або створити нові розділи. **Саме цей варіант обирають для чистої інсталяції.**

### Етап 9. Розмітка диска (розділи)

Це один із найважливіших етапів. Ви побачите список усіх дисків та розділів комп'ютера.

**Основні дії:**

- **Видалити** (Delete) — видаляє існуючий розділ. **Увага: усі дані на ньому будуть втрачені!**
- **Створити** (New) — створює новий розділ на нерозміченому просторі.
- **Форматувати** (Format) — очищує розділ (файлова система NTFS).
- **Далі** (Next) — починає встановлення на обраний розділ.

**Типовий сценарій для чистої інсталяції на новому диску:**

1. Оберіть «Нерозмічений простір» (Unallocated Space).
2. Натисніть «Далі» — Windows автоматично створить потрібні розділи:
   - **Розділ відновлення** (Recovery) — ~500 МБ;
   - **Системний розділ EFI** — ~100 МБ (для UEFI-завантаження);
   - **Зарезервований розділ Microsoft** (MSR) — ~16 МБ;
   - **Основний розділ** — решта простору (тут буде встановлена Windows).

**Типовий сценарій для перевстановлення:**

1. Видаліть старий розділ з Windows (зазвичай «Диск 0 Розділ 2» або подібний).
2. Оберіть нерозмічений простір, натисніть «Далі».
3. Windows створить потрібні розділи автоматично.

> **Важливо:** Якщо у вас є окремий розділ із даними (наприклад, диск D:), **не видаляйте** його — тоді ваші файли збережуться.

### Етап 10. Копіювання файлів та встановлення

Після натискання «Далі» починається автоматичний процес. Комп'ютер виконує:

1. **Копіювання файлів Windows** — файли копіюються з USB-носія на жорсткий диск.
2. **Підготовка файлів для встановлення** — розпакування та підготовка.
3. **Встановлення компонентів** — інсталяція драйверів, служб, системних компонентів.
4. **Встановлення оновлень** — якщо є доступ до Інтернету.
5. **Завершення** — фіналізація встановлення.

Цей процес займає **від 10 до 30 хвилин** залежно від швидкості диска (SSD значно швидше за HDD).

> Під час цього етапу комп'ютер **перезавантажиться 1–2 рази** автоматично. Це нормально — не натискайте жодних клавіш під час перезавантаження.

### Етап 11. Первинне налаштування (OOBE — Out-of-Box Experience)

Після останнього перезавантаження починається **первинне налаштування** — так звана процедура OOBE (Out-of-Box Experience). Це серія екранів, де ви налаштовуєте систему «під себе».

#### 11.1. Вибір регіону (країни)

Оберіть вашу країну — наприклад, **«Україна»**. Це вплине на формат дати, часу, валюти та регіональні параметри. Натисніть **«Так»** (Yes).

#### 11.2. Розкладка клавіатури

Оберіть основну розкладку клавіатури — наприклад, **«Українська»**. Можна додати другу розкладку (наприклад, «Англійська (США)»). Натисніть «Так» та «Пропустити» (або «Додати розкладку»).

#### 11.3. Підключення до мережі

Windows запропонує **підключитися до мережі Wi-Fi** або виявить кабельне підключення (Ethernet) автоматично.

- У Windows 11 підключення до Інтернету є **обов'язковим** для продовження налаштування.
- У Windows 10 цей крок можна пропустити, натиснувши «У мене немає Інтернету» → «Продовжити з обмеженим налаштуванням».

#### 11.4. Обліковий запис

**Windows 11 (за замовчуванням)** вимагає входу з **обліковим записом Microsoft** (електронна пошта @outlook.com, @hotmail.com, @live.com або будь-яка інша, прив'язана до акаунту Microsoft). Це дає можливості:
- синхронізації налаштувань між пристроями;
- доступу до OneDrive (хмарне сховище);
- завантаження застосунків із Microsoft Store;
- відновлення пароля через Інтернет.

**Локальний обліковий запис** — альтернатива без підключення до хмарних сервісів Microsoft. У Windows 11 створити локальний обліковий запис під час первинного налаштування складніше — Microsoft поступово обмежує цю можливість. У версії Windows 11 Pro можна обрати «Налаштування для роботи» → «Приєднання до домену». У Windows 10 локальний обліковий запис створюється легко через опцію «Автономний обліковий запис».

#### 11.5. Створення PIN-коду

Windows запропонує створити **PIN-код** — короткий цифровий код для швидкого входу в систему замість пароля. PIN-код зберігається локально на пристрої (не передається в Інтернет), що робить його безпечнішим за пароль у деяких сценаріях.

- Мінімум 4 цифри;
- Можна ввімкнути використання літер та символів.

#### 11.6. Параметри конфіденційності

Windows запропонує налаштувати **параметри конфіденційності**:

- **Розташування** (Location) — чи дозволяти програмам визначати ваше місцезнаходження;
- **Діагностичні дані** — які дані про роботу системи надсилати Microsoft;
- **Рукописний ввід та введення тексту** — чи надсилати дані для покращення розпізнавання;
- **Персоналізований досвід** — чи використовувати ваші дані для підбору реклами та рекомендацій;
- **Рекламний ідентифікатор** — чи дозволяти програмам використовувати унікальний рекламний ID.

> **Порада:** Для максимальної приватності рекомендується вимкнути всі або більшість цих параметрів. Їх завжди можна змінити пізніше в «Параметрах» Windows.

### Етап 12. Робочий стіл — встановлення завершено

Після завершення всіх кроків OOBE ви побачите **робочий стіл Windows** — це означає, що встановлення успішно завершено.

**Подальші дії після встановлення:**

1. **Перевірити Windows Update** — завантажити та встановити всі оновлення.
2. **Встановити драйвери** — особливо для відеокарти (NVIDIA, AMD, Intel), мережевої карти, звуку. Windows Update встановить більшість драйверів автоматично, але для відеокарти краще завантажити драйвер з офіційного сайту виробника.
3. **Встановити необхідне програмне забезпечення** — браузер, офісний пакет, антивірус тощо.
4. **Активувати Windows** — якщо система не активувалася автоматично.
5. **Створити точку відновлення** — на випадок майбутніх проблем.

## 6. Особливості встановлення Windows 11 порівняно з Windows 10

| Характеристика | Windows 10 | Windows 11 |
|----------------|-----------|------------|
| **TPM 2.0** | Не вимагається | Обов'язковий |
| **Secure Boot** | Не вимагається | Обов'язковий |
| **UEFI** | Підтримує і Legacy BIOS | Тільки UEFI |
| **Архітектура процесора** | 32-біт та 64-біт | Тільки 64-біт |
| **Мінімум RAM** | 2 ГБ (64-біт) | 4 ГБ |
| **Мінімум місця на диску** | 20 ГБ | 64 ГБ |
| **Інтернет під час OOBE** | Необов'язковий | Обов'язковий |
| **Обліковий запис Microsoft** | Можна легко створити локальний | Фактично обов'язковий (локальний — через обхідні методи) |
| **Підтримка процесорів** | Широка (починаючи з дуже старих) | Обмежена: Intel 8-го покоління+, AMD Ryzen 2000+ |
| **Підтримка** | Закінчилась 14 жовтня 2025 р. | Активна підтримка |

### Що таке TPM 2.0?

**TPM (Trusted Platform Module)** — це спеціальний апаратний модуль безпеки, вбудований у материнську плату (або реалізований програмно через процесор — fTPM у AMD, PTT у Intel). TPM зберігає криптографічні ключі та забезпечує:

- шифрування диска (BitLocker);
- безпечне завантаження (Secure Boot);
- апаратну автентифікацію (Windows Hello);
- захист від фізичного втручання.

**Як перевірити наявність TPM:**
- Натисніть **Win + R**, введіть `tpm.msc`, натисніть Enter.
- Якщо з'явиться вікно з інформацією про TPM та «Версія специфікації: 2.0» — модуль є та підтримується.

### Що таке Secure Boot?

**Secure Boot** (безпечне завантаження) — технологія UEFI, яка перевіряє цифрові підписи всіх компонентів, що завантажуються під час старту комп'ютера. Це запобігає завантаженню шкідливого програмного забезпечення (руткітів, буткітів) до запуску операційної системи.

## 7. Типові проблеми при встановленні та їх вирішення

### Проблема 1: Комп'ютер не завантажується з USB-флешки

**Причини та рішення:**
- У BIOS/UEFI не змінено порядок завантаження — увійдіть у BIOS та поставте USB першим.
- Флешка створена неправильно — перезапишіть її за допомогою Media Creation Tool або Rufus.
- Увімкнено режим Legacy замість UEFI (або навпаки) — переключіть режим у BIOS.
- USB-порт не працює — спробуйте інший порт (краще USB 2.0 на задній панелі).

### Проблема 2: Диск не відображається під час встановлення

**Причини та рішення:**
- Відсутній драйвер контролера SATA/NVMe — завантажте драйвер **Intel RST (Rapid Storage Technology)** або драйвер контролера вашої материнської плати на іншу флешку. На етапі вибору диска натисніть **«Завантажити драйвер»** (Load Driver) та вкажіть шлях до драйвера.
- У BIOS увімкнено режим RAID замість AHCI — змініть на **AHCI** у налаштуваннях SATA.
- Кабель SATA пошкоджений або погано підключений — перевірте з'єднання.

### Проблема 3: Помилка «Windows не може бути встановлена на цей диск» (GPT/MBR)

**Причини та рішення:**
- Диск має стиль розмітки MBR, а комп'ютер завантажується в режимі UEFI (потрібен GPT), або навпаки.
- **Рішення:** На етапі вибору диска натисніть **Shift + F10** для виклику командного рядка. Введіть:
  ```
  diskpart
  list disk
  select disk 0
  clean
  convert gpt
  exit
  ```
  Після цього оновіть список дисків і продовжте встановлення.

> **Увага:** Команда `clean` видаляє ВСІ дані з диска!

### Проблема 4: Помилка «Цей комп'ютер не відповідає вимогам Windows 11»

**Причини та рішення:**
- Вимкнено TPM 2.0 — увімкніть у BIOS (розділ Security → TPM/fTPM/PTT).
- Вимкнено Secure Boot — увімкніть у BIOS.
- Непідтримуваний процесор — для навчальних цілей можна скористатися Rufus із увімкненою опцією обходу перевірок.

### Проблема 5: Встановлення «зависає» на певному етапі

**Причини та рішення:**
- Пошкоджений інсталяційний носій — створіть флешку заново.
- Проблеми з оперативною пам'яттю — протестуйте RAM (Windows Memory Diagnostic або memtest86).
- Перегрів — перевірте систему охолодження.
- Від'єднайте всі зайві USB-пристрої та периферію.

### Проблема 6: Немає драйвера мережі після встановлення

**Рішення:** Заздалегідь завантажте драйвери мережевої карти (Wi-Fi або Ethernet) з сайту виробника ноутбука/материнської плати на флешку. Встановіть їх після інсталяції Windows.

## 8. Активація Windows — що це і навіщо

### Що таке активація?

**Активація Windows** — це процес підтвердження того, що ваша копія Windows є ліцензійною (справжньою) і використовується на дозволеній кількості пристроїв відповідно до умов ліцензійної угоди Microsoft.

Активація пов'язує ліцензію з конкретним апаратним забезпеченням (конфігурацією) комп'ютера.

### Навіщо потрібна активація?

- Підтверджує легальність використання Windows;
- Знімає обмеження (без активації: водяний знак на робочому столі, неможливість змінити шпалери, обмежений доступ до персоналізації);
- Дозволяє отримувати оновлення безпеки;
- Забезпечує технічну підтримку від Microsoft.

### Два способи активації

#### 1. Ключ продукту (Product Key)

**Ключ продукту** — це 25-символьний буквено-цифровий код формату:
```
XXXXX-XXXXX-XXXXX-XXXXX-XXXXX
```

Ключ може знаходитися:
- на наклейці на корпусі комп'ютера (OEM-ліцензія);
- у коробці з диском (Retail-ліцензія);
- в електронному листі підтвердження покупки;
- у особистому кабінеті Microsoft Store.

**Типи ліцензій за ключем продукту:**

| Тип ліцензії | Опис | Переносність |
|-------------|------|-------------|
| **OEM** | Встановлена виробником ПК | Прив'язана до конкретного пристрою, не переноситься |
| **Retail** | Придбана окремо у магазині | Можна переносити на інший комп'ютер |
| **Volume** | Для організацій (один ключ на багато ПК) | Використовується в корпоративному середовищі |

#### 2. Цифрова ліцензія (Digital License)

**Цифрова ліцензія** (цифрове право) — це метод активації без введення ключа продукту. Інформація про ліцензію зберігається на серверах Microsoft та прив'язується до апаратної конфігурації комп'ютера.

Цифрова ліцензія з'являється, коли:
- ви оновили Windows 7/8.1 до Windows 10 безкоштовно;
- ви придбали Windows у Microsoft Store;
- ви раніше активували Windows на цьому комп'ютері.

**При перевстановленні** Windows на тому самому комп'ютері — система активується автоматично (ключ вводити не потрібно, достатньо пропустити крок введення ключа).

### Як перевірити стан активації?

**Шлях:** Параметри → Система → Активація.

Можливі статуси:
- **«Windows активовано»** — все гаразд;
- **«Windows активовано за допомогою цифрової ліцензії»** — активація через серверну прив'язку;
- **«Windows не активовано»** — потрібно ввести ключ або усунути проблему.

## 10. Глосарій

| Термін | Визначення |
|--------|-----------|
| **BIOS (Basic Input/Output System)** | Базова система введення-виведення — мікропрограма, що зберігається на мікросхемі материнської плати та відповідає за ініціалізацію обладнання при ввімкненні комп'ютера. Застаріла технологія, замінена UEFI |
| **UEFI (Unified Extensible Firmware Interface)** | Сучасний інтерфейс мікропрограми, що замінив BIOS. Підтримує графічний інтерфейс, GPT-розмітку, Secure Boot, завантаження з великих дисків |
| **TPM (Trusted Platform Module)** | Апаратний модуль безпеки на материнській платі, який зберігає криптографічні ключі та забезпечує захист даних. Версія 2.0 обов'язкова для Windows 11 |
| **Secure Boot** | Технологія безпечного завантаження UEFI, що перевіряє цифрові підписи програм під час старту системи, запобігаючи завантаженню шкідливого ПЗ |
| **GPT (GUID Partition Table)** | Сучасний стандарт розмітки дисків, що підтримує диски понад 2 ТБ та до 128 розділів. Необхідний для завантаження в режимі UEFI |
| **MBR (Master Boot Record)** | Застарілий стандарт розмітки дисків. Підтримує диски до 2 ТБ та до 4 основних розділів. Використовується з Legacy BIOS |
| **OOBE (Out-of-Box Experience)** | Процедура первинного налаштування Windows після встановлення: вибір регіону, клавіатури, мережі, облікового запису, параметрів конфіденційності |
| **Чиста інсталяція (Clean Install)** | Встановлення ОС «з нуля» з форматуванням системного розділу та видаленням усіх попередніх даних |
| **Оновлення (Upgrade)** | Встановлення нової версії ОС поверх існуючої зі збереженням файлів, програм та налаштувань |
| **Відновлення (Recovery/Reset)** | Повернення системи до заводського стану за допомогою вбудованих засобів Windows без зовнішнього носія |
| **Активація Windows** | Процес підтвердження ліцензійності копії Windows шляхом прив'язки ключа продукту або цифрової ліцензії до апаратного забезпечення комп'ютера |
| **Ключ продукту (Product Key)** | 25-символьний буквено-цифровий код, що використовується для активації Windows |
| **Цифрова ліцензія (Digital License)** | Метод активації Windows без ключа; ліцензія зберігається на серверах Microsoft та прив'язана до апаратної конфігурації ПК |
| **Media Creation Tool** | Офіційна утиліта Microsoft для створення завантажувального USB-носія або ISO-образу Windows |
| **Rufus** | Безкоштовна програма для створення завантажувальних USB-носіїв з ISO-образів. Підтримує обхід перевірки вимог Windows 11 |
| **NTFS (New Technology File System)** | Файлова система, що використовується Windows для системного розділу. Підтримує права доступу, шифрування, стиснення, великі файли |
| **Розділ (Partition)** | Логічно відокремлена область на фізичному диску, яка сприймається системою як окремий диск |
| **Форматування** | Процес підготовки розділу диска до запису даних шляхом створення файлової системи. Видаляє всі наявні дані |
| **Boot Menu** | Меню швидкого вибору пристрою завантаження, що викликається під час старту комп'ютера спеціальною клавішею (F12, F8, Esc) |
| **ISO-образ** | Файл-образ оптичного диска, що містить точну копію вмісту диска. Використовується для створення завантажувальних носіїв |
