## 1. Мережеві можливості сучасних ОС — огляд

Сучасні операційні системи (Windows, Linux, macOS) мають вбудовану підтримку мережі. Це означає, що мережеві функції глибоко інтегровані в ядро та системні компоненти ОС, а не є окремими додатками.

### Основні мережеві можливості ОС

| Можливість | Опис |
|---|---|
| **Спільний доступ до файлів** | Користувачі можуть відкривати доступ до папок і файлів на своєму комп'ютері для інших користувачів мережі. У Windows це реалізується через протокол SMB (Server Message Block), у Linux — через Samba або NFS (Network File System). |
| **Спільний доступ до принтерів** | ОС дозволяє "розшарити" принтер, підключений до одного комп'ютера, щоб інші комп'ютери мережі могли на ньому друкувати. |
| **Доступ до Інтернету** | ОС забезпечує підключення до мережі Інтернет через вбудовані мережеві протоколи TCP/IP. Стек протоколів TCP/IP є базовим для всіх сучасних ОС. |
| **VPN (Virtual Private Network)** | Віртуальна приватна мережа дозволяє створити захищений "тунель" через публічну мережу (Інтернет). ОС має вбудовані засоби для створення та підключення до VPN — це використовується для безпечного віддаленого доступу до корпоративної мережі. |
| **Віддалений робочий стіл** | Дозволяє керувати іншим комп'ютером через мережу так, ніби ви сидите безпосередньо за ним. У Windows це RDP (Remote Desktop Protocol), у Linux — SSH, VNC, XRDP. |

### Чому мережеві засоби вбудовані в ОС?

Раніше мережеві функції встановлювалися окремо як додаткове програмне забезпечення. Сучасні ОС (починаючи з Windows NT, Linux) мають мережеві засоби, вбудовані безпосередньо в ядро. Це забезпечує:

- **Високу продуктивність** — мережевий код працює на рівні ядра, без зайвих "прошарків".
- **Надійність** — мережеві компоненти тестуються разом з ОС.
- **Простоту налаштування** — користувач одразу після встановлення ОС має доступ до мережі.

## 2. Типи мереж: LAN, WAN, MAN, PAN, WLAN

Комп'ютерні мережі класифікують за **географічним охопленням** — тобто за відстанню, на якій вони працюють.

### Порівняльна таблиця типів мереж

| Тип | Повна назва | Українською | Охоплення | Приклади |
|-----|-------------|-------------|-----------|----------|
| **PAN** | Personal Area Network | Персональна мережа | до 10 м | Bluetooth-навушники, бездротова миша, з'єднання телефону з ноутбуком |
| **LAN** | Local Area Network | Локальна мережа | до 1–5 км | Мережа в класі, офісі, будинку; комп'ютерний клас у коледжі |
| **WLAN** | Wireless Local Area Network | Бездротова локальна мережа | до 100–300 м (Wi-Fi) | Wi-Fi мережа вдома, у кафе, у навчальному закладі |
| **MAN** | Metropolitan Area Network | Міська (магістральна) мережа | до 50–60 км | Мережа кабельного телебачення міста, мережа університетського містечка |
| **WAN** | Wide Area Network | Глобальна мережа | сотні — тисячі км | Інтернет, корпоративна мережа між філіями в різних містах/країнах |

### Детальний опис кожного типу

**PAN (Personal Area Network)** — найменша мережа, що охоплює простір навколо однієї людини. Типові технології: Bluetooth, USB, інфрачервоний порт. Приклад: ви підключаєте бездротові навушники до смартфона — це PAN.

**LAN (Local Area Network)** — локальна мережа в межах одного приміщення, будівлі або групи будівель. Саме LAN ми бачимо у комп'ютерних класах, офісах, вдома. Для з'єднання використовується кабель Ethernet (UTP-кабель з роз'ємом RJ-45) або Wi-Fi. LAN має високу швидкість передачі даних (100 Мбіт/с — 10 Гбіт/с) і низьку затримку.

**WLAN (Wireless LAN)** — це та сама локальна мережа, але без проводів. Працює за стандартом IEEE 802.11 (Wi-Fi). Дозволяє підключати ноутбуки, смартфони, планшети без кабелів. Зручна для мобільних користувачів, але має менший радіус дії та нижчу швидкість порівняно з дротовою LAN.

**MAN (Metropolitan Area Network)** — мережа масштабу міста. Об'єднує кілька LAN в межах міста або великого кампусу. Приклади: мережа провайдера кабельного інтернету у місті, мережа між корпусами великого університету.

**WAN (Wide Area Network)** — глобальна мережа, що охоплює величезні території (країни, континенти). Найвідоміший приклад WAN — Інтернет. WAN об'єднує безліч LAN та MAN. Для з'єднання використовуються оптоволоконні кабелі, супутниковий зв'язок, підводні кабелі.

## 3. Мережеві топології

**Топологія мережі** — це спосіб фізичного або логічного з'єднання комп'ютерів у мережу. Топологія визначає, як саме пристрої підключені один до одного.

### 3.1. Топологія "Зірка" (Star)

```
        [ПК1]
          |
[ПК4]---[HUB/SWITCH]---[ПК2]
          |
        [ПК3]
```

**Принцип:** усі комп'ютери підключені до одного центрального пристрою — комутатора (switch) або концентратора (hub). Обмін даними між будь-якими двома комп'ютерами відбувається через центральний вузол.

| Переваги | Недоліки |
|-----------|----------|
| Вихід з ладу одного комп'ютера не впливає на роботу мережі | Якщо зламається центральний пристрій — вся мережа не працює |
| Легко додавати нові комп'ютери | Потрібно більше кабелю, ніж у шині |
| Простий пошук несправностей | Кількість підключень обмежена портами комутатора |
| Висока швидкість роботи | Вища вартість обладнання |

> **Зірка — найпоширеніша топологія** у сучасних локальних мережах. Саме її ви бачите у комп'ютерних класах.

### 3.2. Топологія "Шина" (Bus)

```
[ПК1]---[ПК2]---[ПК3]---[ПК4]---[ПК5]
|                                       |
[Термінатор]                   [Термінатор]
```

**Принцип:** усі комп'ютери підключені до одного спільного кабелю (магістралі). Дані, відправлені одним комп'ютером, проходять по кабелю і доступні всім іншим. На кінцях кабелю встановлюють термінатори — спеціальні заглушки, що поглинають сигнал.

| Переваги | Недоліки |
|-----------|----------|
| Проста в реалізації | Розрив кабелю зупиняє всю мережу |
| Мінімальні витрати кабелю | Складний пошук несправностей |
| Не потрібен центральний пристрій | Одночасно передає дані лише один комп'ютер |
| Низька вартість | Зі зростанням кількості ПК продуктивність падає |

> Шина використовувалася у старих мережах (Ethernet на коаксіальному кабелі). Сьогодні практично не застосовується.

### 3.3. Топологія "Кільце" (Ring)

```
    [ПК1]
   /      \
[ПК4]    [ПК2]
   \      /
    [ПК3]
```

**Принцип:** кожний комп'ютер з'єднаний з двома сусідніми, утворюючи замкнене кільце. Дані передаються по колу від одного комп'ютера до іншого в одному напрямку (або в обох при подвійному кільці).

| Переваги | Недоліки |
|-----------|----------|
| Не потрібен центральний пристрій | Вихід з ладу одного ПК зупиняє всю мережу |
| Кожен ПК підсилює сигнал перед передачею далі | Для додавання нового ПК треба зупиняти мережу |
| Може підтримувати велику кількість вузлів | Кожен новий вузол уповільнює мережу |
| Подвійне кільце підвищує надійність | Складний пошук несправностей |

> Кільце використовується у технологіях Token Ring та FDDI, але у сучасних LAN зустрічається рідко.

### 3.4. Змішана (гібридна) топологія

На практиці великі мережі використовують **комбінацію** різних топологій. Наприклад, "зірка-зірка" — кілька комутаторів-зірок об'єднані між собою. Або "зірка-шина" — комутатори (зірки) з'єднані спільною магістраллю. Такий підхід дозволяє поєднувати переваги різних топологій.

## 4. Мережеве обладнання

Для побудови мережі потрібне спеціальне обладнання. Розглянемо основні пристрої.

### 4.1. Мережева карта (NIC — Network Interface Card)

**Що це:** плата (або вбудований чіп), яка дозволяє комп'ютеру підключатися до мережі.

**Як працює:** мережева карта перетворює дані комп'ютера у сигнали, придатні для передачі по мережевому кабелю або бездротовим каналом, і навпаки. Кожна мережева карта має унікальну **MAC-адресу** (Media Access Control) — 48-бітний ідентифікатор, що присвоюється виробником. Приклад MAC-адреси: `00:1A:2B:3C:4D:5E`.

**Типи:**
- **Дротова** (Ethernet) — підключається кабелем RJ-45.
- **Бездротова** (Wi-Fi) — підключається через радіосигнал.

У сучасних комп'ютерах і ноутбуках мережеві карти зазвичай вбудовані в материнську плату.

### 4.2. Комутатор (Switch)

**Що це:** пристрій, що об'єднує кілька комп'ютерів у локальну мережу.

**Як працює:** комутатор зберігає у пам'яті **таблицю MAC-адрес** — відповідність "MAC-адреса пристрою — порт комутатора". Коли надходить кадр (пакет даних канального рівня), комутатор читає MAC-адресу отримувача і відправляє кадр лише на потрібний порт. Це значно зменшує навантаження на мережу порівняно з концентратором (hub), який надсилає дані на всі порти.

**Рівень моделі OSI:** канальний (L2).

> Комутатор — це "розумний" пристрій, на відміну від концентратора (hub), який просто дублює сигнал на всі порти.

### 4.3. Маршрутизатор (Router)

**Що це:** пристрій, що з'єднує різні мережі між собою та визначає оптимальний маршрут передачі даних.

**Як працює:** маршрутизатор працює з **IP-адресами** (мережевий рівень). Він аналізує IP-адресу призначення у кожному пакеті та, використовуючи **таблицю маршрутизації**, вирішує, через який інтерфейс і на який наступний вузол відправити пакет. Домашній роутер також виконує функцію NAT (Network Address Translation) — перетворює приватні IP-адреси домашньої мережі в одну публічну IP-адресу для виходу в Інтернет.

**Рівень моделі OSI:** мережевий (L3).

> Простіше кажучи: комутатор з'єднує пристрої **всередині** однієї мережі, а маршрутизатор з'єднує **різні мережі** між собою.

### 4.4. Точка доступу (Access Point, AP)

**Що це:** пристрій, що створює бездротову мережу Wi-Fi, до якої підключаються пристрої без кабелю.

**Як працює:** точка доступу з'єднана кабелем з комутатором або маршрутизатором і транслює радіосигнал Wi-Fi. Бездротові пристрої (ноутбуки, смартфони) підключаються до неї по радіоканалу. AP розширює зону покриття бездротової мережі.

> У домашніх умовах роутер і точка доступу часто об'єднані в одному пристрої — "Wi-Fi роутер".

### 4.5. Модем (Modem)

**Що це:** пристрій, що перетворює цифровий сигнал комп'ютера в аналоговий сигнал для передачі по телефонній лінії (або іншому каналу) — і навпаки.

**Як працює:** назва "модем" — скорочення від **мо**дулятор-**дем**одулятор. Модулятор перетворює цифровий сигнал в аналоговий для передачі, демодулятор виконує зворотне перетворення при прийомі.

**Типи модемів:**
- **ADSL-модем** — працює через телефонну лінію, забезпечує асиметричну швидкість (швидкість завантаження вища за швидкість відправки).
- **Кабельний модем** — використовує коаксіальний кабель кабельного телебачення.
- **Стільниковий (3G/4G/5G) модем** — працює через мережу мобільного оператора.
- **Оптичний модем (ONT/ONU)** — перетворює оптичний сигнал з оптоволоконного кабелю.

### Зведена таблиця мережевого обладнання

| Пристрій | Функція | Рівень OSI |
|----------|---------|------------|
| Мережева карта (NIC) | Підключення комп'ютера до мережі | L1 (фізичний) / L2 (канальний) |
| Комутатор (Switch) | Об'єднання пристроїв у LAN за MAC-адресами | L2 (канальний) |
| Маршрутизатор (Router) | Маршрутизація пакетів між мережами за IP-адресами | L3 (мережевий) |
| Точка доступу (AP) | Створення бездротової мережі Wi-Fi | L1 / L2 |
| Модем | Перетворення цифрового сигналу в аналоговий і навпаки | L1 (фізичний) |

## 5. Архітектура мережевих засобів ОС

Операційна система має складну внутрішню структуру для роботи з мережею. Ця структура називається **архітектурою мережевих засобів**.

### 5.1. Мережевий стек (Network Stack)

**Мережевий стек** — це програмна реалізація набору мережевих протоколів в операційній системі. Він організований у вигляді рівнів (шарів), де кожен рівень виконує свою функцію та взаємодіє з сусідніми рівнями.

```
┌─────────────────────────────────────┐
│  Прикладний рівень (додатки)        │  ← Браузер, пошта, месенджер
│  HTTP, FTP, DNS, SMTP               │
├─────────────────────────────────────┤
│  Транспортний рівень                │  ← Надійна доставка даних
│  TCP, UDP                           │
├─────────────────────────────────────┤
│  Мережевий рівень                   │  ← Маршрутизація пакетів
│  IP, ICMP                           │
├─────────────────────────────────────┤
│  Канальний рівень                   │  ← Передача кадрів
│  Ethernet, Wi-Fi (802.11), ARP      │
├─────────────────────────────────────┤
│  Фізичний рівень                    │  ← Передача бітів
│  Мережева карта, кабель, радіосигнал│
└─────────────────────────────────────┘
```

**Принцип роботи:** коли ви відкриваєте веб-сторінку, дані проходять **зверху вниз** по стеку при відправці (кожен рівень додає свій заголовок — це називається **інкапсуляція**), і **знизу вверх** при прийомі (кожен рівень знімає свій заголовок — **деінкапсуляція**).

### 5.2. Мережеві протоколи

**Протокол** — це набір правил, за якими пристрої обмінюються даними у мережі.

| Протокол | Рівень | Призначення |
|----------|--------|-------------|
| **TCP** (Transmission Control Protocol) | Транспортний | Надійна доставка даних з підтвердженням отримання |
| **UDP** (User Datagram Protocol) | Транспортний | Швидка доставка без підтвердження (для відео, ігор) |
| **IP** (Internet Protocol) | Мережевий | Адресація та маршрутизація пакетів |
| **HTTP/HTTPS** | Прикладний | Передача веб-сторінок |
| **DNS** (Domain Name System) | Прикладний | Перетворення доменних імен в IP-адреси |
| **DHCP** (Dynamic Host Configuration Protocol) | Прикладний | Автоматичне призначення IP-адрес |
| **ARP** (Address Resolution Protocol) | Канальний | Перетворення IP-адреси в MAC-адресу |
| **FTP** (File Transfer Protocol) | Прикладний | Передача файлів |

### 5.3. Мережеві драйвери

**Мережевий драйвер** — це програма, що забезпечує взаємодію ОС з конкретною мережевою картою. Драйвер "знає", як саме спілкуватися з апаратним забезпеченням конкретного виробника.

Мережеві драйвери працюють на **найнижчому рівні** мережевого стеку. Вони:

- Ініціалізують мережеву карту при завантаженні ОС.
- Передають кадри даних від ОС до мережевої карти для відправки.
- Приймають вхідні кадри від мережевої карти та передають їх вищим рівням стеку.
- Використовують **кільцевий буфер (ring buffer)** та механізм **DMA (Direct Memory Access)** для ефективної передачі великих обсягів даних без навантаження процесора.

> Без правильного драйвера мережева карта не працюватиме, навіть якщо вона фізично справна.

### 5.4. Мережеві служби (сервіси)

**Мережеві служби** — це фонові програми (демони в Linux, служби/сервіси в Windows), що виконують мережеві функції автоматично, без участі користувача.

| Служба | Функція |
|--------|---------|
| **DHCP-клієнт** | Автоматично отримує IP-адресу, маску підмережі, шлюз та DNS-сервер від DHCP-сервера при підключенні до мережі |
| **DNS-клієнт** | Перетворює доменні імена (наприклад, google.com) в IP-адреси (наприклад, 142.250.74.46). Також кешує (зберігає) попередні DNS-запити для прискорення |
| **NetBIOS** | Служба імен для локальних мереж Windows. Дозволяє знаходити комп'ютери за їх іменами у мережевому оточенні |
| **Служба спільного доступу (SMB/CIFS)** | Забезпечує спільний доступ до файлів та принтерів у мережі Windows |
| **Брандмауер (Firewall)** | Фільтрує вхідний та вихідний мережевий трафік за заданими правилами для захисту комп'ютера |

### 5.5. Мережеві API (інтерфейси програмування)

**API (Application Programming Interface)** — це набір функцій, які програмісти використовують для написання мережевих програм.

**Winsock (Windows Sockets)** — мережевий API для Windows. Будь-яка програма у Windows, що працює з мережею (браузер, месенджер, онлайн-гра), використовує Winsock для відправки та отримання даних. Winsock базується на концепції сокетів.

**Berkeley Sockets (BSD Sockets)** — мережевий API для Unix/Linux систем. Це стандарт, що з'явився у 1983 році в ОС BSD Unix і став основою для мережевого програмування у всьому світі. Winsock фактично є адаптацією Berkeley Sockets для Windows.

**Що таке сокет?** Сокет — це кінцева точка мережевого з'єднання. Його можна уявити як "розетку", через яку програма відправляє та отримує дані. Сокет визначається парою: **IP-адреса + номер порту**. Наприклад, веб-сервер слухає на сокеті `192.168.1.1:80` (IP-адреса : порт).

### Загальна схема архітектури мережевих засобів ОС

```
┌───────────────────────────────────────────────────┐
│               ДОДАТКИ КОРИСТУВАЧА                 │
│         (браузер, месенджер, пошта)               │
├───────────────────────────────────────────────────┤
│              МЕРЕЖЕВІ API                         │
│         (Winsock / Berkeley Sockets)              │
├───────────────────────────────────────────────────┤
│             МЕРЕЖЕВІ СЛУЖБИ                       │
│      (DHCP-клієнт, DNS-клієнт, NetBIOS)          │
├───────────────────────────────────────────────────┤
│           МЕРЕЖЕВІ ПРОТОКОЛИ                      │
│         (TCP/IP, UDP, ICMP, ARP)                  │
├───────────────────────────────────────────────────┤
│            МЕРЕЖЕВІ ДРАЙВЕРИ                      │
│     (драйвер Ethernet, драйвер Wi-Fi)            │
├───────────────────────────────────────────────────┤
│           МЕРЕЖЕВЕ ОБЛАДНАННЯ                     │
│      (мережева карта, Wi-Fi адаптер)              │
└───────────────────────────────────────────────────┘
```

## 6. Клієнт-серверна та однорангова моделі

Існують два основних способи організації взаємодії комп'ютерів у мережі.

### 6.1. Клієнт-серверна модель (Client-Server)

```
    [Клієнт 1] ──┐
    [Клієнт 2] ──┼── [СЕРВЕР]
    [Клієнт 3] ──┘
```

**Принцип:** є виділений сервер (потужний комп'ютер), який надає послуги (зберігає файли, керує принтерами, надає доступ до Інтернету). Клієнти (робочі станції) — звертаються до сервера з запитами.

**Приклади з життя:**
- Ви відкриваєте сайт — ваш браузер (клієнт) надсилає запит веб-серверу, який відповідає веб-сторінкою.
- Шкільний комп'ютерний клас: всі ПК (клієнти) працюють з файлами, що зберігаються на сервері.

| Переваги | Недоліки |
|-----------|----------|
| Централізоване управління та безпека | Висока вартість сервера та обслуговування |
| Легке резервне копіювання (все на сервері) | Якщо сервер зламається — мережа не працює |
| Надійність та стабільність | Потрібен кваліфікований адміністратор |
| Масштабованість (можна додавати клієнтів) | Сервер може стати "вузьким місцем" при перевантаженні |

### 6.2. Однорангова модель (Peer-to-Peer, P2P)

```
    [ПК1] ── [ПК2]
      |    ×    |
    [ПК3] ── [ПК4]
```

**Принцип:** немає виділеного сервера. Кожен комп'ютер є рівноправним учасником (peer) і може одночасно бути і клієнтом, і сервером. Кожен ПК може надавати свої ресурси іншим і використовувати ресурси інших.

**Приклади з життя:**
- Домашня мережа, де два комп'ютери обмінюються файлами.
- Торрент-мережі (BitTorrent): кожен учасник одночасно завантажує і роздає файли.

| Переваги | Недоліки |
|-----------|----------|
| Просто та дешево налаштувати | Складне управління при великій кількості ПК |
| Не потрібен виділений сервер | Низький рівень безпеки |
| Немає єдиної точки відмови | Резервне копіювання — відповідальність кожного |
| Ідеально для малих мереж (до 10 ПК) | Продуктивність залежить від кожного учасника |

### Порівняльна таблиця

| Характеристика | Клієнт-сервер | Однорангова (P2P) |
|----------------|---------------|-------------------|
| Виділений сервер | Так | Ні |
| Безпека | Висока (централізована) | Низька |
| Вартість | Висока | Низька |
| Масштабованість | Добра | Обмежена |
| Адміністрування | Потрібен спеціаліст | Кожен сам |
| Типовий розмір | Від 10 до тисяч ПК | До 10 ПК |

## 7. Мережеві засоби Windows

Windows має зручні графічні та командні інструменти для налаштування мережі.

### 7.1. Центр мережевих підключень і спільного доступу

Це головна панель управління мережею у Windows. Тут можна:

- Переглянути активні мережеві підключення.
- Змінити параметри адаптера.
- Налаштувати нове підключення (VPN, PPPoE тощо).
- Увімкнути/вимкнути спільний доступ до файлів та принтерів.

**Як відкрити:** Панель керування → Мережа та Інтернет → Центр мережевих підключень і спільного доступу.

### 7.2. ncpa.cpl — Мережеві підключення

**ncpa.cpl** (Network Control Panel Applet) — це системний аплет, що відкриває вікно **"Мережеві підключення"** зі списком усіх мережевих адаптерів (Ethernet, Wi-Fi, VPN тощо).

**Як відкрити:** натиснути `Win + R`, ввести `ncpa.cpl`, натиснути Enter.

**Що можна зробити:**
- Увімкнути або вимкнути мережевий адаптер.
- Налаштувати IP-адресу, маску підмережі, шлюз та DNS (правий клік → Властивості → TCP/IPv4).
- Перейменувати підключення.
- Діагностувати проблеми з підключенням.

### 7.3. Корисні мережеві команди Windows (командний рядок)

| Команда | Призначення |
|---------|-------------|
| `ipconfig` | Показує поточну мережеву конфігурацію (IP, маска, шлюз) |
| `ipconfig /all` | Детальна інформація про всі мережеві адаптери |
| `ipconfig /release` | Звільняє поточну IP-адресу (від DHCP) |
| `ipconfig /renew` | Запитує нову IP-адресу від DHCP-сервера |
| `ping <адреса>` | Перевіряє зв'язок з іншим комп'ютером |
| `tracert <адреса>` | Показує маршрут пакетів до вказаної адреси |
| `nslookup <ім'я>` | Запитує DNS-сервер для визначення IP-адреси домену |
| `netstat` | Показує активні мережеві з'єднання |

## 8. Мережеві засоби Linux

Linux має потужні інструменти для управління мережею, як графічні, так і командні.

### 8.1. NetworkManager

**NetworkManager** — це служба (демон), яка автоматично управляє мережевими підключеннями у Linux. Вона:

- Автоматично виявляє і підключається до дротових та бездротових мереж.
- Управляє VPN-з'єднаннями.
- Дозволяє зберігати кілька профілів підключень (наприклад, "Домашня мережа", "Робота") та перемикатися між ними.
- Має графічний інтерфейс (nm-applet — значок мережі у панелі задач) та текстовий інтерфейс (nmtui).

NetworkManager є стандартним менеджером мережі у більшості дистрибутивів Linux (Ubuntu, Fedora, CentOS та інших).

### 8.2. ifconfig та ip — командні утиліти

**ifconfig** (interface configuration) — класична утиліта для перегляду та налаштування мережевих інтерфейсів. Показує IP-адреси, MAC-адреси, стан інтерфейсів.

> **Важливо:** ifconfig вважається застарілою (deprecated). У сучасних дистрибутивах Linux її замінює утиліта **ip** з пакету iproute2.

**ip** — сучасна утиліта для управління мережевими інтерфейсами, маршрутами, тунелями тощо.

| Завдання | ifconfig (старий) | ip (сучасний) |
|----------|-------------------|---------------|
| Показати всі інтерфейси | `ifconfig` | `ip addr show` |
| Показати конкретний інтерфейс | `ifconfig eth0` | `ip addr show eth0` |
| Призначити IP-адресу | `ifconfig eth0 192.168.1.10 netmask 255.255.255.0` | `ip addr add 192.168.1.10/24 dev eth0` |
| Увімкнути інтерфейс | `ifconfig eth0 up` | `ip link set eth0 up` |
| Вимкнути інтерфейс | `ifconfig eth0 down` | `ip link set eth0 down` |
| Показати таблицю маршрутизації | `route -n` | `ip route show` |

### 8.3. nmcli — командний інтерфейс NetworkManager

**nmcli** (NetworkManager Command Line Interface) — потужний командний інструмент для управління NetworkManager без графічного інтерфейсу. Ідеально підходить для серверів та автоматизації через скрипти.

**Основні команди nmcli:**

| Команда | Призначення |
|---------|-------------|
| `nmcli general` | Загальний статус NetworkManager |
| `nmcli device status` | Стан усіх мережевих пристроїв |
| `nmcli connection show` | Список усіх збережених підключень |
| `nmcli connection up "Назва"` | Активувати підключення |
| `nmcli connection down "Назва"` | Деактивувати підключення |
| `nmcli device wifi list` | Показати доступні Wi-Fi мережі |
| `nmcli device wifi connect "SSID" password "пароль"` | Підключитися до Wi-Fi |

### 8.4. Інші корисні мережеві команди Linux

| Команда | Призначення |
|---------|-------------|
| `ping <адреса>` | Перевірка зв'язку з хостом |
| `traceroute <адреса>` | Маршрут пакетів до хоста |
| `ss -tulnp` | Показати відкриті порти та з'єднання (замінює netstat) |
| `dig <домен>` | DNS-запит (аналог nslookup) |
| `hostname -I` | Показати IP-адреси комп'ютера |

## Глосарій

| Термін | Визначення |
|--------|------------|
| **LAN (Local Area Network)** | Локальна мережа — мережа в межах будівлі або групи будівель (до 1–5 км). |
| **WAN (Wide Area Network)** | Глобальна мережа — мережа, що охоплює великі географічні відстані (Інтернет). |
| **MAN (Metropolitan Area Network)** | Міська мережа — мережа масштабу міста або великого кампусу (до 50–60 км). |
| **PAN (Personal Area Network)** | Персональна мережа — мережа навколо однієї людини (до 10 м), наприклад Bluetooth. |
| **WLAN (Wireless LAN)** | Бездротова локальна мережа, що працює за технологією Wi-Fi (IEEE 802.11). |
| **Топологія мережі** | Спосіб фізичного та логічного з'єднання комп'ютерів у мережі (зірка, шина, кільце тощо). |
| **NIC (Network Interface Card)** | Мережева карта — пристрій для підключення комп'ютера до мережі. |
| **MAC-адреса** | Унікальний 48-бітний апаратний ідентифікатор мережевої карти, що присвоюється виробником. |
| **Switch (Комутатор)** | Мережевий пристрій, що з'єднує комп'ютери в LAN та передає кадри за MAC-адресами. |
| **Router (Маршрутизатор)** | Пристрій, що з'єднує різні мережі та маршрутизує пакети за IP-адресами. |
| **Access Point (Точка доступу)** | Пристрій, що створює бездротову Wi-Fi мережу для підключення пристроїв. |
| **Modem (Модем)** | Пристрій для перетворення цифрового сигналу в аналоговий і навпаки (модулятор-демодулятор). |
| **Мережевий стек (Network Stack)** | Багаторівнева програмна реалізація мережевих протоколів в ОС. |
| **TCP/IP** | Основний набір протоколів Інтернету. TCP забезпечує надійну доставку, IP — адресацію та маршрутизацію. |
| **DHCP** | Протокол автоматичного призначення IP-адрес пристроям у мережі. |
| **DNS** | Система доменних імен — перетворює доменні імена (google.com) в IP-адреси. |
| **Мережевий драйвер** | Програма, що забезпечує взаємодію ОС з конкретною моделлю мережевої карти. |
| **Сокет (Socket)** | Кінцева точка мережевого з'єднання, що визначається парою IP-адреса:порт. |
| **Winsock** | Мережевий API для Windows, що реалізує інтерфейс сокетів. |
| **Berkeley Sockets** | Стандартний мережевий API для Unix/Linux систем, основа мережевого програмування. |
| **VPN (Virtual Private Network)** | Віртуальна приватна мережа — захищений "тунель" через публічну мережу. |
| **RDP (Remote Desktop Protocol)** | Протокол віддаленого робочого столу у Windows. |
| **NAT (Network Address Translation)** | Технологія перетворення приватних IP-адрес у публічну для доступу до Інтернету. |
| **ncpa.cpl** | Системний аплет Windows для відкриття вікна "Мережеві підключення". |
| **NetworkManager** | Служба автоматичного управління мережею у Linux. |
| **nmcli** | Командний інтерфейс NetworkManager у Linux. |
| **P2P (Peer-to-Peer)** | Однорангова мережа — кожен комп'ютер є одночасно клієнтом і сервером. |
| **Клієнт-серверна модель** | Архітектура мережі, де виділений сервер надає послуги, а клієнти звертаються до нього із запитами. |
| **Інкапсуляція** | Процес додавання заголовків кожного рівня мережевого стеку при передачі даних. |
| **SMB (Server Message Block)** | Протокол спільного доступу до файлів та принтерів у мережах Windows. |
| **OSI (Open Systems Interconnection)** | Еталонна 7-рівнева модель мережевої взаємодії відкритих систем. |
