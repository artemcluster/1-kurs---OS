## 1. Мета роботи

- Ознайомитися з основними системними утилітами Windows 10/11.
- Навчитися запускати утиліти через діалогове вікно "Виконати" (Win+R).
- Отримати практичні навички збору інформації про систему, діагностики та обслуговування комп'ютера.
- Опанувати базові команди командного рядка (CMD) та PowerShell.

---

## 2. Короткий теоретичний вступ

**Системні утиліти** — це спеціальні програми, вбудовані в операційну систему Windows, які призначені для діагностики, налаштування, обслуговування та моніторингу роботи комп'ютера.

### Навіщо потрібні системні утиліти?

- **Діагностика** — визначення характеристик комп'ютера, пошук проблем з обладнанням та програмами.
- **Обслуговування** — очищення диска від тимчасових файлів, оптимізація файлової системи.
- **Моніторинг** — спостереження за використанням процесора, оперативної пам'яті, дисків та мережі в реальному часі.
- **Налаштування** — зміна параметрів завантаження системи, управління службами та автозавантаженням.
- **Автоматизація** — виконання команд через командний рядок, планування задач.

### Як запустити утиліту?

Найшвидший спосіб запуску більшості утиліт:

1. Натисніть комбінацію клавіш **Win + R** — відкриється діалогове вікно "Виконати".
2. Введіть назву утиліти (наприклад, `msconfig`).
3. Натисніть **Enter** або кнопку **OK**.

> **Важливо!** Деякі утиліти потребують прав адміністратора. У такому випадку система може запитати підтвердження через вікно контролю облікових записів (UAC).

## 4. Практичне завдання 1 — Конфігурація системи (msconfig)

### Що це таке?

**MSConfig** (System Configuration / Конфігурація системи) — утиліта для налаштування способу завантаження Windows, управління службами та програмами автозапуску. Використовується для діагностики проблем із завантаженням та продуктивністю системи.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `msconfig`, натисніть **Enter**.

**Крок 2.** Ознайомтеся з вкладкою **"Загальне" (General)**:
- **Звичайне завантаження** — запуск Windows з усіма драйверами та службами.
- **Діагностичне завантаження** — запуск лише базових служб Windows (схоже на безпечний режим).
- **Вибіркове завантаження** — дозволяє обирати, які служби та елементи автозапуску завантажувати.

> Запишіть у зошит, який тип завантаження обрано на вашому комп'ютері.

**Крок 3.** Перейдіть на вкладку **"Завантаження" (Boot)**:
- Тут відображається список встановлених операційних систем.
- Зверніть увагу на параметри: "Безпечне завантаження" (Safe boot), "Без GUI", "Журнал завантаження".
- Кнопка **"Додаткові параметри"** дозволяє обмежити кількість процесорів та обсяг пам'яті для завантаження.

> Запишіть назву ОС та час очікування (тайм-аут), які вказані на цій вкладці.

**Крок 4.** Перейдіть на вкладку **"Служби" (Services)**:
- Тут відображається список усіх служб Windows.
- Поставте позначку **"Не відображати служби Microsoft"** — залишаться лише сторонні служби.

> Запишіть, скільки всього служб відображається та скільки з них — сторонні.

**Крок 5.** Перейдіть на вкладку **"Автозавантаження" (Startup)**:
- У Windows 10/11 ця вкладка містить посилання **"Відкрити диспетчер завдань"**.
- Натисніть це посилання — відкриється Диспетчер завдань на вкладці "Автозавантаження".

> Запишіть 3-5 програм з автозавантаження та їхній вплив на завантаження (низький / середній / високий).

**Крок 6.** Перейдіть на вкладку **"Сервіс" (Tools)**:
- Це список корисних системних утиліт із можливістю їх швидкого запуску.
- Виберіть будь-яку утиліту та натисніть **"Запуск"**.

> **Увага!** Не змінюйте жодних налаштувань! Натисніть "Скасувати" для закриття вікна без збереження змін.

## 5. Практичне завдання 2 — Відомості про систему (msinfo32)

### Що це таке?

**MSInfo32** (System Information / Відомості про систему) — утиліта, яка відображає вичерпну інформацію про апаратне та програмне забезпечення комп'ютера. Це один з найкращих інструментів для збору повних характеристик системи.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `msinfo32`, натисніть **Enter**.

**Крок 2.** Ознайомтеся з розділом **"Зведення про систему" (System Summary)**:
- Тут відображаються: назва ОС, версія, виробник системи, модель, тип, процесор, версія BIOS/UEFI, обсяг фізичної пам'яті (RAM).

> Запишіть у зошит:
> - Назва ОС та її версія
> - Назва процесора
> - Обсяг оперативної пам'яті (RAM)
> - Версія BIOS/UEFI

**Крок 3.** Розгорніть розділ **"Апаратні ресурси" (Hardware Resources)**:
- **Конфлікти/Спільне використання** — пристрої, що спільно використовують ресурси.
- **DMA** — канали прямого доступу до пам'яті.
- **IRQ** — запити на переривання (кожному пристрою призначено свій номер).
- **Вводи-виводи (I/O)** — порти введення/виведення.

> Перегляньте список IRQ. Запишіть, який пристрій використовує IRQ 0 та IRQ 1.

**Крок 4.** Розгорніть розділ **"Компоненти" (Components)**:
- **Дисплей** — інформація про відеокарту та драйвер.
- **Звук** — звукові пристрої.
- **Запам'ятовуючі пристрої > Диски** — інформація про жорсткі диски.
- **Мережа > Адаптер** — мережеві адаптери.

> Запишіть назву відеокарти та обсяг відеопам'яті з розділу "Дисплей".

**Крок 5.** Розгорніть розділ **"Програмне середовище" (Software Environment)**:
- **Системні драйвери** — список драйверів та їхній стан.
- **Змінні середовища** — системні змінні (PATH, TEMP тощо).
- **Мережеві підключення** — активні підключення.
- **Служби** — список системних служб та їхній стан (запущена/зупинена).

> Перейдіть до "Служби". Запишіть 3 служби зі станом "Запущена".

## 6. Практичне завдання 3 — Засіб діагностики DirectX (dxdiag)

### Що це таке?

**DxDiag** (DirectX Diagnostic Tool) — утиліта для перегляду інформації про компоненти та драйвери DirectX, встановлені в системі. DirectX — це набір технологій Microsoft для мультимедіа, ігор та графіки.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `dxdiag`, натисніть **Enter**.

> При першому запуску система може запитати дозвіл на перевірку цифрових підписів драйверів — натисніть "Так".

**Крок 2.** Ознайомтеся з вкладкою **"Система" (System)**:
- Назва комп'ютера, операційна система, мова, виробник системи, модель.
- Процесор, обсяг оперативної пам'яті.
- Версія DirectX (наприклад, DirectX 12).

> Запишіть версію DirectX та обсяг оперативної пам'яті.

**Крок 3.** Перейдіть на вкладку **"Дисплей" (Display)**:
- Назва відеоадаптера (відеокарти), виробник, тип мікросхеми.
- Обсяг відеопам'яті (VRAM).
- Стан функцій DirectDraw, Direct3D, прискорення AGP.
- У розділі "Примітки" внизу вказано, чи є проблеми.

> Запишіть назву відеокарти, обсяг відеопам'яті та чи є повідомлення про проблеми в "Примітках".

**Крок 4.** Перейдіть на вкладку **"Звук" (Sound)**:
- Назва звукового пристрою, тип обладнання.
- Інформація про драйвер.
- У розділі "Примітки" — стан звукової підсистеми.

> Запишіть назву звукового пристрою та вміст "Приміток".

**Крок 5.** Перейдіть на вкладку **"Введення" (Input)**:
- Список пристроїв введення: клавіатура, миша, ігрові контролери.

**Крок 6.** Натисніть кнопку **"Зберегти всі відомості..."** — збережіть файл звіту на Робочий стіл під назвою `dxdiag_звіт.txt`.

## 7. Практичне завдання 4 — Очищення диска (cleanmgr)

### Що це таке?

**Cleanmgr** (Disk Cleanup / Очищення диска) — утиліта для видалення непотрібних тимчасових файлів, кешу, звітів про помилки та інших даних, що займають місце на диску.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `cleanmgr`, натисніть **Enter**.

**Крок 2.** Виберіть диск для очищення (зазвичай **C:**) та натисніть **OK**.

**Крок 3.** Зачекайте, поки утиліта проаналізує диск.

**Крок 4.** У вікні результатів ви побачите список категорій файлів для видалення:
- Тимчасові файли Інтернету
- Тимчасові файли Windows
- Кеш ескізів
- Файли звітів про помилки Windows
- Завантажені файли програм
- Кошик

> Зверніть увагу на стовпець "Обсяг дискового простору, що звільняється" для кожної категорії.

**Крок 5.** Запишіть у зошит:
- Загальний обсяг простору, який можна звільнити.
- Три найбільші категорії за обсягом.

**Крок 6.** Натисніть кнопку **"Очистити системні файли"** — з'являться додаткові категорії (журнали оновлень Windows, попередні встановлення Windows тощо).

> **Увага!** Під час практичної роботи НЕ натискайте "OK" для видалення файлів, якщо не отримали дозвіл від викладача. Закрийте вікно кнопкою "Скасувати".

## 8. Практичне завдання 5 — Дефрагментація та оптимізація дисків (dfrgui)

### Що це таке?

**Dfrgui** (Optimize Drives / Оптимізація дисків) — утиліта для аналізу та оптимізації (дефрагментації) дисків. Дефрагментація збирає розрізнені фрагменти файлів в одне ціле, що прискорює читання даних з жорсткого диска (HDD).

> **Примітка:** Для SSD-дисків замість дефрагментації виконується операція TRIM, яка також підвищує продуктивність, але працює інакше.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `dfrgui`, натисніть **Enter**.

**Крок 2.** У вікні "Оптимізація дисків" ви побачите:
- Список усіх дисків (C:, D: тощо).
- Тип носія (жорсткий диск / твердотільний диск).
- Поточний стан (відсоток фрагментації або "OK").
- Дата останньої оптимізації.

> Запишіть у зошит:
> - Кількість дисків на вашому комп'ютері.
> - Тип кожного диска (HDD чи SSD).
> - Поточний стан оптимізації.

**Крок 3.** Виберіть диск та натисніть **"Аналізувати"** — утиліта перевірить рівень фрагментації.

**Крок 4.** Натисніть **"Змінити параметри"** — перегляньте розклад автоматичної оптимізації (зазвичай раз на тиждень).

> **Увага!** Не запускайте оптимізацію без дозволу викладача — процес може тривати довго.

## 9. Практичне завдання 6 — Монітор ресурсів (resmon)

### Що це таке?

**Resmon** (Resource Monitor / Монітор ресурсів) — потужний інструмент для моніторингу використання апаратних ресурсів у реальному часі. Він надає значно детальнішу інформацію, ніж Диспетчер завдань.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `resmon`, натисніть **Enter**.

**Крок 2.** Ознайомтеся з вкладкою **"Огляд" (Overview)**:
- Тут відображаються зведені дані по CPU, Диску, Мережі та Пам'яті.
- Праворуч — графіки використання ресурсів за останні 60 секунд.

**Крок 3.** Перейдіть на вкладку **"ЦП" (CPU)**:
- Список усіх процесів з відсотком використання процесора.
- Графіки завантаження кожного ядра процесора.

> Запишіть 3 процеси, які найбільше навантажують процесор у даний момент.

**Крок 4.** Перейдіть на вкладку **"Пам'ять" (Memory)**:
- Список процесів із зазначенням обсягу використаної пам'яті.
- Графічна шкала внизу показує розподіл фізичної пам'яті: використовувана, змінена, очікувана, вільна.

> Запишіть:
> - Загальний обсяг фізичної пам'яті.
> - Скільки пам'яті використовується зараз.
> - Який процес займає найбільше пам'яті.

**Крок 5.** Перейдіть на вкладку **"Диск" (Disk)**:
- Процеси, які виконують операції читання/запису на диск.
- Швидкість читання та запису для кожного процесу.

**Крок 6.** Перейдіть на вкладку **"Мережа" (Network)**:
- Процеси, які використовують мережеве підключення.
- IP-адреси, до яких підключається кожен процес.
- Швидкість надсилання та отримання даних.

> Запишіть 2-3 процеси, які використовують мережу, та IP-адреси їхніх підключень.

## 10. Практичне завдання 7 — Командний рядок (cmd)

### Що це таке?

**CMD** (Command Prompt / Командний рядок) — текстовий інтерфейс для виконання команд операційної системи. Виник ще в епоху MS-DOS (1987) і залишається важливим інструментом для системного адміністрування.

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `cmd`, натисніть **Enter**.

**Крок 2.** Виконайте по черзі наступні команди (після кожної команди натискайте **Enter**):

#### Команда `ver`
```
ver
```
Показує версію операційної системи Windows.

> Запишіть результат.

#### Команда `hostname`
```
hostname
```
Показує мережеве ім'я комп'ютера.

> Запишіть ім'я комп'ютера.

#### Команда `systeminfo`
```
systeminfo
```
Показує детальну інформацію про систему: назву ОС, версію, дату встановлення, час завантаження, виробника, модель, тип системи, процесор, обсяг пам'яті, мережеві адаптери.

> Запишіть:
> - Назву ОС та версію
> - Дату першого встановлення системи
> - Час останнього завантаження

#### Команда `ipconfig`
```
ipconfig
```
Показує конфігурацію мережевих адаптерів: IP-адресу, маску підмережі, шлюз за замовчуванням.

Для детальнішої інформації використайте:
```
ipconfig /all
```

> Запишіть вашу IPv4-адресу, маску підмережі та основний шлюз.

#### Команда `tasklist`
```
tasklist
```
Показує список усіх запущених процесів з їхніми ідентифікаторами (PID) та обсягом використаної пам'яті.

> Запишіть 5 процесів з найбільшим споживанням пам'яті (стовпець "Память").

#### Команда `dir`
```
dir
```
Показує вміст поточної директорії (папки) — список файлів та підпапок.

Спробуйте також:
```
dir /a
```
— показує всі файли, включно з прихованими та системними.

#### Команда `cls`
```
cls
```
Очищує екран командного рядка від усього попереднього тексту.

**Крок 3.** Спробуйте виконати ще декілька корисних команд:

```
echo Hello, World!
```
— виводить текст на екран.

```
date /t
```
— показує поточну дату.

```
time /t
```
— показує поточний час.

## 11. Практичне завдання 8 — Windows PowerShell

### Що це таке?

**Windows PowerShell** — це сучасна оболонка командного рядка та мова сценаріїв від Microsoft, побудована на платформі .NET. PowerShell значно потужніший за CMD і є основним інструментом для адміністрування сучасних систем Windows.

### Головні відмінності PowerShell від CMD

| Ознака | CMD | PowerShell |
|--------|-----|------------|
| Рік створення | 1987 | 2006 |
| Тип виводу | Звичайний текст | Об'єкти .NET |
| Синтаксис команд | Прості команди DOS | Командлети: Дієслово-Іменник (наприклад, `Get-Process`) |
| Файли сценаріїв | `.bat` (пакетні файли) | `.ps1` |
| Крос-платформність | Лише Windows | Windows, macOS, Linux (з версії 6) |

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `powershell`, натисніть **Enter**.

> Зверніть увагу: вікно PowerShell відрізняється від CMD — зазвичай має синій фон замість чорного.

**Крок 2.** Виконайте наступні команди:

#### Перегляд версії PowerShell
```powershell
$PSVersionTable
```
> Запишіть версію PowerShell.

#### Список процесів (аналог tasklist)
```powershell
Get-Process
```

#### Список запущених служб
```powershell
Get-Service | Where-Object {$_.Status -eq "Running"}
```

#### Інформація про комп'ютер
```powershell
Get-ComputerInfo | Select-Object OsName, OsVersion, CsProcessors, CsTotalPhysicalMemory
```

#### Перевірка IP-адреси (аналог ipconfig)
```powershell
Get-NetIPAddress
```

**Крок 3.** Переконайтеся, що в PowerShell працюють і звичайні CMD-команди:
```powershell
dir
hostname
ipconfig
cls
```

> Це працює завдяки **псевдонімам (aliases)** — PowerShell розпізнає команди CMD та перенаправляє їх до відповідних командлетів.

## 12. Практичне завдання 9 — Редактор реєстру (regedit)

### Що це таке?

**Реєстр Windows** — це ієрархічна база даних, яка зберігає всі налаштування операційної системи, обладнання, програм та користувачів. **Regedit** — графічний інструмент для перегляду та редагування реєстру.

> **УВАГА! Реєстр — це критично важлива частина системи. Неправильне редагування може призвести до збоїв або непрацездатності Windows. У цій практичній роботі ми ТІЛЬКИ ПЕРЕГЛЯДАЄМО реєстр, НІЧОГО НЕ ЗМІНЮЄМО!**

### Структура реєстру — 5 кореневих розділів

| Розділ | Скорочення | Призначення |
|--------|-----------|-------------|
| HKEY_CLASSES_ROOT | HKCR | Асоціації файлів та об'єкти COM |
| HKEY_CURRENT_USER | HKCU | Налаштування поточного користувача |
| HKEY_LOCAL_MACHINE | HKLM | Загальносистемні параметри обладнання та ПЗ |
| HKEY_USERS | HKU | Профілі всіх користувачів системи |
| HKEY_CURRENT_CONFIG | HKCC | Поточна конфігурація обладнання |

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `regedit`, натисніть **Enter**. Підтвердіть запит UAC.

**Крок 2.** Вікно Редактора реєстру складається з двох частин:
- **Ліва панель** — дерево розділів (ключів).
- **Права панель** — параметри (значення) обраного розділу.

**Крок 3.** Перейдіть до розділу з інформацією про ОС:
```
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion
```
- Послідовно розгортайте: HKEY_LOCAL_MACHINE → SOFTWARE → Microsoft → Windows NT → CurrentVersion.

> Запишіть значення параметрів:
> - **ProductName** — назва продукту (наприклад, Windows 10 Pro)
> - **CurrentBuild** — номер збірки
> - **RegisteredOwner** — зареєстрований власник

**Крок 4.** Перейдіть до розділу автозапуску:
```
HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
```

> Запишіть, які програми прописані в автозапуску поточного користувача.

**Крок 5.** Перейдіть до розділу шпалер робочого столу:
```
HKEY_CURRENT_USER\Control Panel\Desktop
```
Знайдіть параметр **WallPaper** — він містить шлях до файлу поточних шпалер.

> **Нагадування: НЕ змінюйте жодних значень!** Закрийте редактор реєстру.

## 13. Практичне завдання 10 — Планувальник завдань (taskschd.msc)

### Що це таке?

**Планувальник завдань** (Task Scheduler) — утиліта Windows, яка дозволяє автоматично запускати програми або сценарії за розкладом (у визначений час, при завантаженні системи, при вході користувача тощо).

### Покрокові дії

**Крок 1.** Натисніть **Win + R**, введіть `taskschd.msc`, натисніть **Enter**.

**Крок 2.** Вікно Планувальника завдань складається з трьох частин:
- **Ліва панель** — дерево папок із завданнями.
- **Центральна панель** — список завдань у вибраній папці та їхні властивості.
- **Права панель** — панель дій.

**Крок 3.** У лівій панелі натисніть на **"Бібліотека планувальника завдань"** (Task Scheduler Library).
- У центральній частині з'явиться список усіх запланованих завдань.

> Запишіть 5 будь-яких завдань з їхніми назвами та станом (Готово / Працює / Вимкнено).

**Крок 4.** Двічі клацніть на будь-якому завданні, щоб відкрити його властивості. Перегляньте вкладки:
- **Загальне** — назва, опис, параметри безпеки.
- **Тригери** — умови запуску (за розкладом, при вході, при запуску тощо).
- **Дії** — що саме виконується (запуск програми, відправлення повідомлення тощо).
- **Журнал** (History) — історія виконання завдання.

> Запишіть для одного завдання: його назву, тригер (коли запускається) та дію (що виконує).

**Крок 5.** Розгорніть папки в лівій панелі (наприклад, Microsoft → Windows) — тут зберігаються системні заплановані завдання.

> **Увага!** Не створюйте, не вмикайте та не вимикайте жодних завдань без дозволу викладача.

## 14. Підсумок

У цій практичній роботі ви ознайомилися з основними системними утилітами Windows:

| Утиліта | Головне призначення |
|---------|-------------------|
| msconfig | Діагностика завантаження, управління службами |
| msinfo32 | Повна інформація про апаратуру та ПЗ |
| dxdiag | Діагностика графіки, звуку та DirectX |
| cleanmgr | Очищення диска від тимчасових файлів |
| dfrgui | Дефрагментація та оптимізація дисків |
| resmon | Моніторинг ресурсів у реальному часі |
| cmd | Текстовий інтерфейс для системних команд |
| PowerShell | Потужна оболонка з мовою сценаріїв |
| regedit | Перегляд системного реєстру |
| taskschd.msc | Перегляд запланованих системних завдань |

Вміння працювати з цими утилітами допоможе вам діагностувати проблеми з комп'ютером, збирати інформацію про систему та виконувати базове обслуговування.
