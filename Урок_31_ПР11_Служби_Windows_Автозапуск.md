## 1. Мета роботи

- Засвоїти поняття служби (service) як фонової програми без графічного інтерфейсу.
- Навчитися переглядати, запускати, зупиняти служби та змінювати їх тип запуску через графічну консоль `services.msc`.
- Оволодіти командами управління службами в CMD (`sc`, `net start`, `net stop`) та PowerShell (`Get-Service`, `Start-Service`, `Stop-Service`).
- Ознайомитися зі способами управління автозапуском програм: Диспетчер завдань, `msconfig`, папка Startup, Планувальник завдань, реєстр Windows.
- Зрозуміти, які служби є критичними і чому їх не можна вимикати.

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Клавіатура та миша.
- Командний рядок (CMD), запущений від імені адміністратора.
- Windows PowerShell, запущений від імені адміністратора.

---

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не вимикайте та не зупиняйте критичні системні служби** (Remote Procedure Call, Windows Event Log, Plug and Play, Cryptographic Services тощо) — це може призвести до збою або «синього екрану смерті» (BSOD).
4. **Не змінюйте тип запуску на «Вимкнено»** для служб, у яких ви не впевнені. Використовуйте тип «Ручний» замість «Вимкнено» — це безпечніше.
5. Після завершення кожного завдання **повертайте всі налаштування до початкового стану**.
6. Не видаляйте та не додавайте програми до автозапуску без дозволу викладача.
7. Після завершення роботи приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Що таке служба (Service)

**Служба** (англ. *service*) — це спеціальний тип програми в Windows, яка:

- працює **у фоновому режимі**, без видимого вікна або графічного інтерфейсу;
- запускається автоматично **при завантаженні системи** (або за запитом іншої програми);
- може працювати **незалежно від того, чи увійшов користувач у систему**;
- керується спеціальним системним компонентом — **Диспетчером керування службами** (SCM — Service Control Manager).

Служби забезпечують роботу ключових функцій операційної системи: мережу, друк, оновлення, безпеку, звук, підключення пристроїв тощо.

> **Аналогія:** Уявіть службу як невидимого працівника в офісі. Він не сидить за столом із монітором, не спілкується з вами напряму, але виконує важливу роботу «за кулісами»: забезпечує електрику, водопостачання, охорону. Без нього офіс не зможе нормально функціонувати, хоча ви його навіть не бачите.

---

### 4.2. Відмінність служби від звичайного процесу

| Характеристика | Звичайний процес (програма) | Служба (service) |
|----------------|----------------------------|-------------------|
| **Інтерфейс** | Має вікно, кнопки, меню (GUI) | Не має графічного інтерфейсу |
| **Запуск** | Запускається користувачем (подвійний клік) | Запускається автоматично системою або за запитом |
| **Залежність від користувача** | Працює лише коли користувач увійшов у систему | Може працювати без входу користувача в систему |
| **Управління** | Закривається через кнопку «Х» або через Диспетчер завдань | Управляється через `services.msc`, CMD або PowerShell |
| **Обліковий запис** | Працює від імені поточного користувача | Працює від імені системного облікового запису (Local System, Local Service, Network Service) |
| **Приклади** | Блокнот, Chrome, Word | Windows Update, Print Spooler, DHCP Client |

Кожна служба у Windows має два імені:
- **Відображуване ім'я** (Display Name) — зрозуміла назва для людини, наприклад: «Диспетчер друку» або «Print Spooler».
- **Системне ім'я** (Service Name) — коротке внутрішнє ім'я для команд, наприклад: `Spooler`, `wuauserv`, `Dhcp`.

---

### 4.3. Типи запуску служб

Windows підтримує чотири типи запуску для служб. Тип запуску визначає, **коли і як** операційна система запускає службу.

| Тип запуску | Опис | Значення в реєстрі |
|-------------|------|---------------------|
| **Автоматичний** (Automatic) | Служба запускається автоматично разом із завантаженням Windows. Використовується для критично важливих служб, які мають бути готові одразу після старту системи. | `Start = 2`, `DelayedAutostart = 0` |
| **Автоматичний (відкладений запуск)** (Automatic Delayed Start) | Служба запускається автоматично, але через кілька секунд після завантаження Windows. Це прискорює початковий запуск системи, бо не всі служби стартують одночасно. | `Start = 2`, `DelayedAutostart = 1` |
| **Ручний** (Manual) | Служба не запускається автоматично. Вона стартує лише тоді, коли її запитає інша програма або користувач запустить її вручну. | `Start = 3` |
| **Вимкнено** (Disabled) | Служба повністю заблокована і не може бути запущена ні автоматично, ні вручну (поки тип запуску не буде змінено). | `Start = 4` |

> **Порада:** Якщо ви не впевнені, чи потрібна якась служба, краще встановити її тип запуску на **«Ручний»**, а не **«Вимкнено»**. У ручному режимі Windows зможе запустити службу, якщо вона буде потрібна, а у вимкненому — ні, що може призвести до помилок.

### 4.4. Важливі служби Windows

Нижче наведено перелік основних служб, з якими ви працюватимете:

| Відображуване ім'я | Системне ім'я | Призначення | Тип запуску за замовчуванням |
|---------------------|---------------|-------------|----------------------------|
| **Windows Update** | `wuauserv` | Пошук, завантаження та встановлення оновлень Windows. Забезпечує безпеку та стабільність системи. | Ручний (активується автоматично за потреби) |
| **DHCP Client** | `Dhcp` | Автоматично отримує IP-адресу та параметри мережі від DHCP-сервера. Без цієї служби комп'ютер не зможе автоматично підключитися до мережі. | Автоматичний |
| **DNS Client** | `Dnscache` | Кешує (зберігає) результати DNS-запитів, прискорюючи доступ до вебсайтів. Також перетворює доменні імена (google.com) в IP-адреси. | Автоматичний |
| **Print Spooler** (Диспетчер друку) | `Spooler` | Управляє чергою друку — приймає завдання на друк від програм і відправляє їх на принтер у правильному порядку. | Автоматичний |
| **Windows Defender Firewall** | `mpssvc` | Брандмауер Windows — захищає комп'ютер від несанкціонованого мережевого доступу. Блокує підозрілі вхідні та вихідні з'єднання. | Автоматичний |
| **Windows Audio** | `Audiosrv` | Забезпечує роботу звукової підсистеми. Без цієї служби комп'ютер не відтворюватиме жодних звуків. | Автоматичний |
| **Windows Audio Endpoint Builder** | `AudioEndpointBuilder` | Управляє звуковими пристроями (динаміки, навушники, мікрофон). Працює спільно зі службою Windows Audio. | Автоматичний |
| **Plug and Play** | `PlugPlay` | Автоматично розпізнає підключене обладнання (USB-пристрої, клавіатури, миші) та встановлює драйвери. | Автоматичний |
| **Remote Procedure Call (RPC)** | `RpcSs` | Базова служба, від якої залежить робота багатьох інших служб і компонентів Windows. Неможливо зупинити. | Автоматичний |
| **Windows Event Log** | `EventLog` | Записує всі системні події, помилки та попередження в журнали Windows. Необхідний для діагностики проблем. | Автоматичний |

### 4.5. Консоль служб services.msc

**Консоль служб** — це графічний інструмент для управління службами Windows. Відкрити її можна кількома способами:

| Спосіб | Дія |
|--------|-----|
| **Win + R → services.msc** | Натиснути Win + R, ввести `services.msc`, натиснути Enter |
| **Пошук у меню "Пуск"** | Ввести «Служби» або «Services» у рядку пошуку |
| **Панель управління** | Панель управління → Адміністрування → Служби |
| **Комп'ютер → Управління** | Правий клік на «Цей ПК» → Управління → Служби та застосунки → Служби |

**Інтерфейс консолі служб:**

Консоль відображає список усіх служб у вигляді таблиці з такими стовпцями:

| Стовпець | Опис |
|----------|------|
| **Ім'я (Name)** | Відображуване ім'я служби |
| **Опис (Description)** | Короткий опис призначення служби |
| **Стан (Status)** | Поточний стан: «Працює» (Running) або порожнє поле (зупинена) |
| **Тип запуску (Startup Type)** | Автоматичний, Ручний, Вимкнено тощо |
| **Вхід від імені (Log On As)** | Обліковий запис, від якого працює служба |

**Дії зі службою:**

Для управління конкретною службою потрібно двічі клікнути на ній. Відкриється вікно властивостей служби з кількома вкладками:

- **Загальне (General)** — тут можна побачити системне ім'я, відображуване ім'я, опис, шлях до виконуваного файлу, тип запуску. Тут же знаходяться кнопки: «Запустити» (Start), «Зупинити» (Stop), «Призупинити» (Pause), «Відновити» (Resume).
- **Вхід (Log On)** — вказує, від імені якого облікового запису працює служба.
- **Відновлення (Recovery)** — визначає, що робити при збої служби (перезапустити, запустити програму, перезавантажити комп'ютер).
- **Залежності (Dependencies)** — показує, від яких служб залежить обрана служба, та які служби залежать від неї.

### 4.6. Управління службами з командного рядка (CMD)

Для управління службами з CMD потрібно запустити командний рядок **від імені адміністратора** (правий клік → «Запуск від імені адміністратора»).

#### Команди `sc` (Service Control)

Команда `sc` — це потужний інструмент для управління службами. Він працює **асинхронно** — тобто відправляє команду і не чекає її завершення.

| Команда | Призначення | Приклад |
|---------|-------------|---------|
| `sc query` | Показати стан усіх запущених служб | `sc query` |
| `sc query <ім'я>` | Показати стан конкретної служби | `sc query Spooler` |
| `sc queryex state= all` | Показати всі служби (і запущені, і зупинені) | `sc queryex state= all` |
| `sc start <ім'я>` | Запустити службу | `sc start Spooler` |
| `sc stop <ім'я>` | Зупинити службу | `sc stop Spooler` |
| `sc config <ім'я> start= auto` | Змінити тип запуску на «Автоматичний» | `sc config Spooler start= auto` |
| `sc config <ім'я> start= demand` | Змінити тип запуску на «Ручний» | `sc config Spooler start= demand` |
| `sc config <ім'я> start= disabled` | Змінити тип запуску на «Вимкнено» | `sc config Spooler start= disabled` |
| `sc qc <ім'я>` | Показати конфігурацію служби (тип запуску, шлях) | `sc qc Spooler` |

> **Важливо:** У команді `sc config` після знаку `=` обов'язково має бути **пробіл**. Наприклад: `start= auto` (правильно), `start=auto` (неправильно — не спрацює!).

#### Команди `net start` / `net stop`

Команди `net` — це старіший, але простіший спосіб управління службами. Вони працюють **синхронно** — тобто чекають, поки служба повністю запуститься або зупиниться.

| Команда | Призначення | Приклад |
|---------|-------------|---------|
| `net start` | Показати список усіх запущених служб | `net start` |
| `net start <ім'я>` | Запустити службу | `net start Spooler` |
| `net stop <ім'я>` | Зупинити службу | `net stop Spooler` |

**Відмінності `sc` від `net`:**

| Функція | `sc` | `net` |
|---------|------|-------|
| Запуск/зупинка | Так | Так |
| Запит стану | Так | Ні |
| Зміна конфігурації | Так | Ні |
| Створення/видалення служб | Так | Ні |
| Синхронний режим (чекає завершення) | Ні | Так |
| Приймає відображуване ім'я | Ні (лише системне ім'я) | Так |

### 4.7. Управління службами з PowerShell

PowerShell надає зручний набір командлетів (cmdlets) для управління службами. Для виконання потрібно запустити PowerShell **від імені адміністратора**.

| Командлет | Призначення | Приклад |
|-----------|-------------|---------|
| `Get-Service` | Отримати список усіх служб | `Get-Service` |
| `Get-Service -Name <ім'я>` | Отримати інформацію про конкретну службу | `Get-Service -Name Spooler` |
| `Get-Service -DisplayName "*Audio*"` | Знайти служби за частиною відображуваного імені | `Get-Service -DisplayName "*Audio*"` |
| `Start-Service -Name <ім'я>` | Запустити службу | `Start-Service -Name Spooler` |
| `Stop-Service -Name <ім'я>` | Зупинити службу | `Stop-Service -Name Spooler` |
| `Restart-Service -Name <ім'я>` | Перезапустити службу (зупинити + запустити) | `Restart-Service -Name Spooler` |
| `Set-Service -Name <ім'я> -StartupType <тип>` | Змінити тип запуску | `Set-Service -Name Spooler -StartupType Manual` |

**Корисні комбінації PowerShell:**

Вивести тільки запущені служби:
```powershell
Get-Service | Where-Object {$_.Status -eq "Running"}
```

Вивести тільки зупинені служби:
```powershell
Get-Service | Where-Object {$_.Status -eq "Stopped"}
```

Отримати інформацію у вигляді зручної таблиці:
```powershell
Get-Service | Select-Object Name, DisplayName, Status, StartType | Format-Table -AutoSize
```

### 4.8. Автозапуск програм — загальні відомості

**Автозапуск (автозавантаження)** — це механізм, який дозволяє програмам автоматично запускатися при вході користувача в систему або при завантаженні Windows.

**Навіщо потрібен автозапуск:**
- Автоматичний запуск антивірусу для захисту системи.
- Запуск хмарних сервісів (OneDrive, Dropbox, Google Drive) для синхронізації файлів.
- Запуск месенджерів (Telegram, Discord, Skype) для отримання повідомлень.

**Чому забагато програм в автозапуску — це погано:**
- Збільшується час завантаження Windows (кожна програма потребує ресурсів при старті).
- Витрачається оперативна пам'ять (RAM) на програми, які, можливо, не потрібні одразу.
- Збільшується навантаження на процесор (CPU) у перші хвилини після завантаження.

---

### 4.9. Способи управління автозапуском

В Windows існує кілька місць, де програми можуть «реєструватися» для автозапуску. Важливо знати всі ці місця для повного контролю.

#### 1. Диспетчер завдань — вкладка «Автозавантаження» (Startup Apps)

Найпростіший і найзручніший спосіб для звичайних користувачів.

- Відкрити: **Ctrl + Shift + Esc** → вкладка **«Автозавантаження»** (Startup Apps).
- Показує список програм, які запускаються разом із Windows.
- Для кожної програми вказано: ім'я, видавець, стан (Увімкнено/Вимкнено), **вплив на запуск** (Високий/Середній/Низький/Не оцінено).
- Можна **увімкнути** або **вимкнути** автозапуск програми (правий клік → Увімкнути/Вимкнути).
- **Обмеження:** не можна додати нову програму до автозапуску через цей інтерфейс.

#### 2. Утиліта msconfig (Конфігурація системи)

- Відкрити: **Win + R → msconfig → Enter**.
- Вкладка **«Служби» (Services)** — показує всі служби Windows.
- Прапорець **«Не відображати служби Microsoft»** (Hide all Microsoft services) — дуже корисна функція, яка дозволяє побачити лише **сторонні служби** (встановлені програмами). Це допомагає виявити небажане ПЗ.
- Вкладка «Автозавантаження» (Startup) у Windows 10/11 перенаправляє до Диспетчера завдань.

#### 3. Папка «Автозавантаження» (Startup)

Класичний і прозорий метод — розміщення ярлика програми у спеціальній папці.

| Папка | Команда для відкриття | Сфера дії |
|-------|----------------------|-----------|
| Автозавантаження поточного користувача | `shell:startup` | Програми запускаються тільки для поточного користувача |
| Автозавантаження всіх користувачів | `shell:common startup` | Програми запускаються для всіх користувачів (потрібні права адміністратора) |

Повний шлях до папки поточного користувача:
```
C:\Users\<Ім'я_користувача>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup
```

Повний шлях до загальної папки:
```
C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup
```

Щоб додати програму до автозапуску — створіть ярлик потрібної програми у цій папці. Щоб прибрати — видаліть ярлик.

#### 4. Планувальник завдань (Task Scheduler — taskschd.msc)

Потужний інструмент для створення автоматичних задач із гнучкими умовами.

- Відкрити: **Win + R → taskschd.msc → Enter**.
- Дозволяє налаштувати запуск програми при різних подіях: при вході в систему, при завантаженні комп'ютера, за розкладом (час, день тижня).
- Можна задати умови: запускати лише при підключенні до мережі, лише при живленні від мережі тощо.
- Деякі шкідливі програми використовують Планувальник для автозапуску, тому важливо перевіряти його вміст.

#### 5. Реєстр Windows — ключі Run

Програми можуть реєструватися в автозапуску через **реєстр Windows** — центральну базу даних налаштувань системи.

Основні ключі реєстру для автозапуску:

| Ключ реєстру | Сфера дії |
|--------------|-----------|
| `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Run` | Автозапуск для поточного користувача |
| `HKEY_LOCAL_MACHINE\Software\Microsoft\Windows\CurrentVersion\Run` | Автозапуск для всіх користувачів |
| `HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\RunOnce` | Одноразовий запуск (запис видаляється після виконання) |

Для перегляду: **Win + R → regedit → Enter**, потім перейти за потрібним шляхом.

> **Увага:** Реєстр — це критично важливий компонент Windows. Неправильне редагування реєстру може призвести до серйозних проблем із системою. Ніколи не змінюйте і не видаляйте записи, якщо не впевнені в їх призначенні.

### 4.10. Безпека: які служби НЕ можна вимикати

Деякі служби є **критично важливими** для роботи Windows. Їх вимкнення може призвести до збоїв, втрати мережевого з'єднання, проблем із безпекою або навіть до неможливості завантажити систему.

**Служби, які категорично НЕ можна вимикати:**

| Служба | Системне ім'я | Чому не можна вимикати |
|--------|---------------|----------------------|
| **Remote Procedure Call (RPC)** | `RpcSs` | Базова служба Windows. Від неї залежить робота десятків інших служб. Система не дозволить її зупинити. |
| **Windows Event Log** | `EventLog` | Записує всі системні події. Без неї неможливо діагностувати проблеми. |
| **Plug and Play** | `PlugPlay` | Розпізнає підключене обладнання. Без неї USB-пристрої не працюватимуть. |
| **Cryptographic Services** | `CryptSvc` | Перевіряє цифрові підписи, сертифікати та оновлення. Без неї неможливо встановлювати оновлення та безпечне ПЗ. |
| **Windows Update** | `wuauserv` | Забезпечує оновлення безпеки. Вимкнення залишає систему вразливою до атак. |
| **Windows Defender Firewall** | `mpssvc` | Захищає від мережевих атак. Без неї комп'ютер стає відкритим для зловмисників. |
| **DHCP Client** | `Dhcp` | Отримує IP-адресу. Без неї комп'ютер не зможе підключитися до мережі. |
| **DNS Client** | `Dnscache` | Перетворює доменні імена в IP-адреси. Без неї не працюватиме доступ до вебсайтів за іменами. |
| **Windows Audio** | `Audiosrv` | Забезпечує звук. Без неї жоден звук не відтворюватиметься. |

**Служби, які можна безпечно вимкнути (якщо не використовуються):**

| Служба | Коли можна вимкнути |
|--------|---------------------|
| **Print Spooler** | Якщо комп'ютер не підключений до принтера і ви не друкуєте |
| **Fax** | Якщо ви не використовуєте факсимільний зв'язок |
| **Bluetooth Support Service** | Якщо комп'ютер не має Bluetooth або ви його не використовуєте |
| **Remote Desktop Services** | Якщо ви не використовуєте віддалений робочий стіл |
| **Xbox-служби** | Якщо ви не використовуєте Xbox-функції Windows |
| **Downloaded Maps Manager** | Якщо ви не використовуєте додаток «Карти» Windows |

> **Золоте правило:** Перед вимкненням будь-якої служби створіть **точку відновлення системи** (System Restore Point). Якщо щось піде не так, ви зможете повернути систему до робочого стану.

**Запишіть у зошит:** загальну кількість служб, кількість працюючих, кількість автоматичних, кількість вимкнених.

---

### Завдання 3. Зупинка та запуск служби Print Spooler

1. Знайдіть службу **Print Spooler** (Диспетчер друку) у консолі `services.msc`.
2. Запишіть її поточний стан та тип запуску.
3. Натисніть правою кнопкою миші на службі → **«Зупинити»** (Stop).
4. Переконайтеся, що стан змінився (зник напис «Працює»).
5. Спробуйте відкрити будь-який текстовий документ і надрукувати його (**Ctrl + P**). Запишіть, що сталося.
6. Поверніться до консолі служб, натисніть правою кнопкою миші на Print Spooler → **«Запустити»** (Start).
7. Переконайтеся, що служба знову працює.

**Запишіть у зошит:** що відбувається при спробі друку з вимкненою службою Print Spooler.

---

### Завдання 4. Зміна типу запуску служби

1. У консолі `services.msc` знайдіть службу **Print Spooler**.
2. Двічі клікніть на ній для відкриття вікна властивостей.
3. Перейдіть на вкладку **«Загальне» (General)**.
4. Запишіть поточний тип запуску.
5. Змініть тип запуску на **«Ручний» (Manual)** → натисніть **«Застосувати»** (Apply) → **«ОК»**.
6. Перевірте, що тип запуску змінився у списку.
7. **Поверніть тип запуску на початкове значення** (Автоматичний).

**Запишіть у зошит:** перегляньте вкладку **«Залежності» (Dependencies)** для служби Print Spooler. Запишіть, від яких служб вона залежить та які служби залежать від неї.

---

### Завдання 5. Управління службами через CMD — команди sc

1. Відкрийте командний рядок **від імені адміністратора** (правий клік на Пуск → «Командний рядок (Адміністратор)» або «Термінал (Адміністратор)»).
2. Виконайте команду `sc query Spooler` та запишіть результат (стан, тип).
3. Виконайте команду `sc query wuauserv` та запишіть стан служби Windows Update.
4. Виконайте команду `sc stop Spooler` — зупиніть службу Print Spooler.
5. Виконайте знову `sc query Spooler` — переконайтеся, що стан змінився на STOPPED.
6. Виконайте команду `sc start Spooler` — запустіть службу назад.
7. Перевірте стан: `sc query Spooler`.
8. Виконайте команду `sc qc Spooler` — перегляньте конфігурацію служби. Запишіть шлях до виконуваного файлу (BINARY_PATH_NAME).

**Запишіть у зошит:** результати виконання кожної команди (стан, тип запуску, шлях до файлу).

---

### Завдання 6. Управління службами через CMD — команди net

1. У тому ж вікні CMD виконайте команду `net start` — перегляньте список усіх запущених служб.
2. Запишіть 5 будь-яких служб зі списку.
3. Виконайте команду `net stop Spooler` — зупиніть службу Print Spooler. Зверніть увагу: команда чекає завершення операції.
4. Виконайте команду `net start Spooler` — запустіть службу назад.
5. Порівняйте поведінку `sc stop` та `net stop`: яка з команд завершилася швидше? Яка з них чекала повної зупинки?

**Запишіть у зошит:** різницю між `sc` та `net` (синхронний vs асинхронний режим).

---

### Завдання 7. Управління службами через PowerShell

1. Відкрийте **Windows PowerShell від імені адміністратора**.
2. Виконайте команду:
   ```powershell
   Get-Service
   ```
   Перегляньте список служб. Зверніть увагу на стовпці: Status, Name, DisplayName.
3. Виконайте команду:
   ```powershell
   Get-Service -Name Spooler
   ```
   Запишіть стан (Status) служби Print Spooler.
4. Знайдіть усі служби, пов'язані зі звуком:
   ```powershell
   Get-Service -DisplayName "*Audio*"
   ```
   Запишіть їх імена та стан.
5. Зупиніть службу Print Spooler:
   ```powershell
   Stop-Service -Name Spooler
   ```
6. Перевірте стан:
   ```powershell
   Get-Service -Name Spooler
   ```
7. Запустіть службу назад:
   ```powershell
   Start-Service -Name Spooler
   ```
8. Виконайте команду для відображення зручної таблиці:
   ```powershell
   Get-Service | Select-Object Name, DisplayName, Status, StartType | Format-Table -AutoSize
   ```

**Запишіть у зошит:** порівняння трьох способів управління службами (services.msc, CMD, PowerShell) — які переваги має кожен.

**Завдання 6.** Створіть ярлик програми «Калькулятор» (`calc.exe`) у папці автозавантаження (`shell:startup`). Перезавантажте комп'ютер (або вийдіть та увійдіть знову). Перевірте, чи запустився Калькулятор автоматично. Після перевірки — видаліть ярлик з папки автозавантаження.

**Завдання 7.** Порівняйте вміст двох ключів реєстру: `HKEY_CURRENT_USER\...\Run` та `HKEY_LOCAL_MACHINE\...\Run`. Чим вони відрізняються за призначенням? Які програми зареєстровані в кожному?

**Завдання 8.** Дослідіть вкладку «Відновлення» (Recovery) у властивостях служби Windows Update. Запишіть, які дії налаштовані при першій, другій та подальших помилках. Поясніть, навіщо потрібна така функціональність.

**Завдання 9.** Використовуючи Планувальник завдань (`taskschd.msc`), створіть просте завдання, яке відкриватиме Блокнот при вході в систему. Після перевірки — видаліть це завдання.

**Завдання 10.** Напишіть пакетний файл (`.bat`), який за допомогою команди `sc query` перевіряє стан трьох служб: Spooler, wuauserv та Audiosrv, і виводить результати на екран. Підказка: у файлі розмістіть три рядки з командою `sc query` для кожної служби.

**Завдання 11.** Уявіть ситуацію: після встановлення нової програми комп'ютер став завантажуватися дуже повільно (замість 30 секунд — 3 хвилини). Опишіть покроковий алгоритм діагностики: які інструменти ви використаєте, що перевірите, як виправите проблему.
