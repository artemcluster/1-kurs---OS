# КОНСПЕКТ УРОКУ №45

## Тема: Практична робота №16 — Налаштування мережевого підключення Windows

---

## Мета роботи

Навчитися переглядати та налаштовувати мережеві підключення у Windows 10/11, працювати з вікном «Мережеві підключення» (ncpa.cpl), аналізувати стан мережевих адаптерів, змінювати параметри TCP/IP, використовувати команди `ipconfig` для діагностики та управління мережевим підключенням.

---

## Теоретичний вступ

### 1. Мережевий адаптер

**Мережевий адаптер** (Network Adapter) — це апаратний або програмний компонент, який забезпечує підключення комп'ютера до мережі. Кожен комп'ютер може мати один або кілька мережевих адаптерів:

- **Ethernet (дротовий)** — підключення через кабель RJ-45 до комутатора або маршрутизатора. Типові виробники: Realtek, Intel.
- **Wi-Fi (бездротовий)** — підключення до бездротової мережі. Типові виробники: Intel, Qualcomm, Realtek.
- **Віртуальні адаптери** — створюються програмним забезпеченням (VPN-клієнти, VirtualBox, VMware, Hyper-V).

Кожен мережевий адаптер має **MAC-адресу** (Media Access Control) — унікальний фізичний ідентифікатор, що складається з 6 байтів у шістнадцятковому форматі (наприклад, `00-1A-2B-3C-4D-5E`). MAC-адреса призначається виробником і зазвичай не змінюється.

---

### 2. IP-адреса та маска підмережі (нагадування)

**IP-адреса** — це логічна адреса пристрою в мережі. У IPv4 вона складається з 4 октетів, розділених крапками (наприклад, `192.168.1.100`).

**Маска підмережі** — визначає, яка частина IP-адреси є адресою мережі, а яка — адресою вузла (хоста). Типова маска для домашньої мережі: `255.255.255.0` (/24).

**Шлюз за замовчуванням** (Default Gateway) — це IP-адреса маршрутизатора, через який комп'ютер виходить в Інтернет. Зазвичай: `192.168.1.1` або `192.168.0.1`.

---

### 3. DHCP — автоматичне отримання IP-адреси

**DHCP** (Dynamic Host Configuration Protocol) — протокол, що дозволяє комп'ютерам автоматично отримувати мережеві параметри від DHCP-сервера (зазвичай — маршрутизатора):

- IP-адресу;
- маску підмережі;
- шлюз за замовчуванням;
- адреси DNS-серверів.

DHCP-сервер видає IP-адресу на певний термін — це називається **оренда** (lease). Після закінчення терміну оренди комп'ютер повинен оновити (продовжити) свою адресу.

| Параметр | Приклад значення |
|----------|-----------------|
| IP-адреса | 192.168.1.105 |
| Маска підмережі | 255.255.255.0 |
| Шлюз за замовчуванням | 192.168.1.1 |
| DNS-сервер | 192.168.1.1 (або 8.8.8.8) |
| DHCP-сервер | 192.168.1.1 |
| Термін оренди | 24 години (типово) |

---

### 4. DNS — система доменних імен (нагадування)

**DNS** (Domain Name System) — перетворює доменні імена (наприклад, `google.com`) на IP-адреси (наприклад, `142.250.74.206`). Без DNS довелося б запам'ятовувати числові адреси кожного сайту.

**Популярні публічні DNS-сервери:**

| Провайдер | Первинний DNS | Вторинний DNS |
|-----------|---------------|---------------|
| Google DNS | `8.8.8.8` | `8.8.4.4` |
| Cloudflare | `1.1.1.1` | `1.0.0.1` |
| OpenDNS | `208.67.222.222` | `208.67.220.220` |

Переваги використання публічних DNS: стабільніша робота, вища швидкість розв'язання імен, додатковий захист від фішингу та шкідливих сайтів.

---

### 5. Профілі мережі у Windows

Windows визначає тип підключеної мережі за допомогою **мережевого профілю**, який впливає на параметри безпеки:

| Профіль | Опис | Мережеве виявлення | Спільний доступ |
|---------|------|-------------------|-----------------|
| **Приватна** (Private) | Довірена мережа (дім, офіс) | Увімкнено | Увімкнено |
| **Загальнодоступна** (Public) | Недовірена мережа (кафе, аеропорт) | Вимкнено | Вимкнено |
| **Доменна** (Domain) | Корпоративна мережа з доменом Active Directory | Визначається адміністратором | Визначається адміністратором |

> **Важливо:** У публічній мережі комп'ютер прихований від інших пристроїв — це підвищує безпеку у відкритих Wi-Fi мережах. У приватній мережі комп'ютер видимий, що дозволяє обмінюватися файлами та використовувати спільні принтери.

---

### 6. Команди ipconfig — огляд

| Команда | Призначення |
|---------|-------------|
| `ipconfig` | Базова інформація: IP-адреса, маска, шлюз для всіх адаптерів |
| `ipconfig /all` | Детальна інформація: MAC-адреса, DHCP-сервер, DNS-сервери, термін оренди |
| `ipconfig /release` | Звільнити поточну IP-адресу (відмовитися від DHCP-оренди) |
| `ipconfig /renew` | Отримати нову IP-адресу від DHCP-сервера |
| `ipconfig /flushdns` | Очистити кеш DNS-розв'язувача |
| `ipconfig /displaydns` | Показати вміст кешу DNS |

## Практичні завдання

### Завдання 1. Відкрити «Мережеві підключення» (ncpa.cpl) — перегляд адаптерів

1. Натисніть `Win + R`, введіть `ncpa.cpl`, натисніть **Enter**.
2. Відкриється вікно **«Мережеві підключення»** — тут відображаються всі мережеві адаптери комп'ютера.
3. Перегляньте список адаптерів. Зверніть увагу на:
   - назви адаптерів (наприклад, «Ethernet», «Wi-Fi», «Підключення через Bluetooth»);
   - стан кожного адаптера: **підключено** (активне з'єднання), **вимкнено** (сірий значок), **мережевий кабель не підключено** (червоний хрест);
   - тип з'єднання — дротове чи бездротове.
4. Запишіть у зошит:
   - скільки адаптерів відображається;
   - назву та стан кожного адаптера.
5. Зробіть знімок екрана вікна «Мережеві підключення».

> **Альтернативні способи відкриття:** Панель керування → Мережа та Інтернет → Центр мережевих підключень і спільного доступу → «Зміна параметрів адаптера». Або: ПКМ на значку мережі в треї → «Відкрити параметри мережі та Інтернету» → «Додаткові параметри мережі» → «Додаткові параметри мережевого адаптера».

---

### Завдання 2. Переглянути стан мережевого підключення

1. У вікні «Мережеві підключення» (ncpa.cpl) двічі клацніть на **активному** адаптері (Ethernet або Wi-Fi).
2. Відкриється вікно **«Стан»** підключення. Перегляньте та запишіть:

| Параметр | Ваше значення |
|----------|---------------|
| Тип підключення (IPv4/IPv6) | |
| Швидкість (Speed) | |
| Тривалість підключення (Duration) | |
| Відправлено байтів (Bytes Sent) | |
| Прийнято байтів (Bytes Received) | |

3. Зверніть увагу: швидкість для Ethernet зазвичай 100 Мбіт/с або 1 Гбіт/с, для Wi-Fi — залежить від стандарту (наприклад, Wi-Fi 5 — до 867 Мбіт/с, Wi-Fi 6 — до 2400 Мбіт/с).
4. Зробіть знімок екрана вікна «Стан».

---

### Завдання 3. Переглянути властивості адаптера — TCP/IPv4, TCP/IPv6

1. У вікні «Стан» підключення натисніть кнопку **«Властивості»** (Properties).
2. У вікні «Властивості» відображається список мережевих компонентів. Знайдіть:
   - **Протокол Інтернету версії 4 (TCP/IPv4)** — основний протокол для IPv4-адресації.
   - **Протокол Інтернету версії 6 (TCP/IPv6)** — протокол для IPv6-адресації.
   - Інші компоненти: «Клієнт для мереж Microsoft», «Спільний доступ до файлів і принтерів», «Планувальник пакетів QoS» тощо.
3. Запишіть назви всіх компонентів, що відображаються у списку.
4. Виділіть **TCP/IPv4** та натисніть кнопку **«Властивості»**.
5. У вікні властивостей TCP/IPv4 зверніть увагу на два варіанти:
   - **Отримати IP-адресу автоматично** — DHCP (автоматичне налаштування).
   - **Використовувати таку IP-адресу** — статичне (ручне) налаштування.
6. Аналогічно перегляньте налаштування DNS:
   - **Отримати адресу DNS-сервера автоматично** — DNS від DHCP.
   - **Використовувати такі адреси DNS-серверів** — ручне налаштування DNS.
7. Зробіть знімок екрана. **Не змінюйте нічого на цьому етапі — лише переглядайте!**
8. Натисніть **«Скасувати»**, щоб закрити вікно без змін.

---

### Завдання 4. Записати поточні мережеві налаштування

1. Повторно відкрийте вікно «Стан» підключення (двічі клацнути на адаптері в ncpa.cpl).
2. Натисніть кнопку **«Відомості»** (Details).
3. Відкриється вікно **«Відомості про мережеве підключення»**. Заповніть таблицю:

| Параметр | Ваше значення |
|----------|---------------|
| Опис (назва адаптера) | |
| Фізична адреса (MAC) | |
| DHCP увімкнено (Так/Ні) | |
| IPv4-адреса | |
| Маска підмережі IPv4 | |
| Шлюз за замовчуванням | |
| DNS-сервер IPv4 | |
| DHCP-сервер IPv4 | |
| Оренда отримана (Lease Obtained) | |
| Оренда закінчується (Lease Expires) | |

4. Зверніть увагу: якщо у полі «DHCP увімкнено» стоїть **«Так»** — IP-адреса отримана автоматично. Якщо **«Ні»** — налаштована статична адреса.
5. Зробіть знімок екрана вікна «Відомості».

> **Важливо:** Запишіть ці дані — вони знадобляться для порівняння після виконання наступних завдань.

---

### Завдання 5. CMD: ipconfig — базова інформація

1. Відкрийте **Командний рядок** (CMD): натисніть `Win + R`, введіть `cmd`, натисніть **Enter**.
2. Введіть команду:
   ```
   ipconfig
   ```
3. Ця команда виводить для кожного мережевого адаптера:
   - **IPv4-адреса** — поточна IP-адреса.
   - **Маска підмережі** — визначає межі мережі.
   - **Шлюз за замовчуванням** — адреса маршрутизатора.
4. Знайдіть свій активний адаптер (Ethernet або Wi-Fi) і запишіть:

| Параметр | Значення з ipconfig |
|----------|-------------------|
| Назва адаптера | |
| IPv4-адреса | |
| Маска підмережі | |
| Шлюз за замовчуванням | |

5. Порівняйте ці дані з даними із завдання 4. Вони повинні збігатися.
6. Зробіть знімок екрана результату команди `ipconfig`.

---

### Завдання 6. CMD: ipconfig /all — детальна інформація

1. У Командному рядку введіть команду:
   ```
   ipconfig /all
   ```
2. Ця команда виводить повну інформацію про всі мережеві адаптери. Знайдіть свій активний адаптер і заповніть таблицю:

| Параметр | Значення з ipconfig /all |
|----------|------------------------|
| Опис (Description) | |
| Фізична адреса (Physical Address / MAC) | |
| DHCP увімкнено (DHCP Enabled) | |
| IPv4-адреса | |
| Маска підмережі | |
| Шлюз за замовчуванням | |
| DHCP-сервер (DHCP Server) | |
| DNS-сервери (DNS Servers) | |
| Оренда отримана (Lease Obtained) | |
| Оренда закінчується (Lease Expires) | |

3. Зверніть увагу на додаткові поля, яких не було у звичайному `ipconfig`:
   - **Фізична адреса (MAC)** — апаратна адреса мережевого адаптера.
   - **DHCP-сервер** — IP-адреса сервера, що видав IP-адресу.
   - **Термін оренди** — дата та час початку і закінчення DHCP-оренди.
4. Порівняйте дані з вікном «Відомості» (завдання 4). Запишіть, чи збігаються всі параметри.
5. Зробіть знімок екрана.

---

### Завдання 7. Змінити DNS-сервер вручну (Google DNS)

> **Увага:** Це завдання передбачає зміну налаштувань мережі. Після виконання ми повернемо все до автоматичних налаштувань (завдання 10).

**Спосіб 1 — через ncpa.cpl (класичний):**

1. Відкрийте вікно «Мережеві підключення» (`Win + R` → `ncpa.cpl`).
2. ПКМ на активному адаптері → **«Властивості»**.
3. Виділіть **«Протокол Інтернету версії 4 (TCP/IPv4)»** → натисніть **«Властивості»**.
4. Оберіть **«Використовувати такі адреси DNS-серверів»**.
5. Введіть:
   - **Основний DNS-сервер:** `8.8.8.8`
   - **Додатковий DNS-сервер:** `8.8.4.4`
6. Натисніть **OK** → **OK** (закрийте обидва вікна).

**Спосіб 2 — через Параметри Windows 11:**

1. Відкрийте **Параметри** (`Win + I`) → **Мережа та Інтернет**.
2. Оберіть **Wi-Fi** або **Ethernet** (залежно від підключення).
3. Натисніть **«Властивості обладнання»** (Hardware properties).
4. У рядку **«Призначення DNS-сервера»** натисніть **«Редагувати»**.
5. Змініть з «Автоматично» на **«Вручну»**.
6. Увімкніть **IPv4** і введіть:
   - Основний DNS: `8.8.8.8`
   - Додатковий DNS: `8.8.4.4`
7. Натисніть **«Зберегти»**.

**Перевірка зміни:**

8. Відкрийте CMD та введіть:
   ```
   ipconfig /all
   ```
9. Знайдіть рядок **DNS Servers** — він повинен показувати `8.8.8.8` та `8.8.4.4`.
10. Для додаткової перевірки введіть:
    ```
    nslookup google.com
    ```
11. У рядку **Server** має відображатися `dns.google` або `8.8.8.8`.
12. Зробіть знімок екрана результатів обох команд.

---

### Завдання 8. CMD: ipconfig /release та ipconfig /renew — оновлення DHCP-оренди

> **Увага:** Після виконання `ipconfig /release` комп'ютер тимчасово втратить мережеве підключення. Відразу після цього виконайте `ipconfig /renew`.

1. Відкрийте **Командний рядок від імені адміністратора**: натисніть `Win`, введіть `cmd`, ПКМ → **«Запуск від імені адміністратора»**.
2. Перед початком запишіть поточну IP-адресу:
   ```
   ipconfig
   ```
3. Виконайте команду для звільнення IP-адреси:
   ```
   ipconfig /release
   ```
4. Зверніть увагу: поле IPv4-адреса тепер показує `0.0.0.0` — комп'ютер відмовився від IP-адреси.
5. Спробуйте виконати:
   ```
   ipconfig
   ```
   Переконайтесь, що адреса дійсно звільнена.
6. Тепер отримайте нову IP-адресу:
   ```
   ipconfig /renew
   ```
7. Дочекайтеся завершення (може зайняти 5–15 секунд).
8. Перевірте нову адресу:
   ```
   ipconfig
   ```
9. Заповніть таблицю:

| Параметр | До release | Після renew |
|----------|-----------|-------------|
| IPv4-адреса | | |
| Маска підмережі | | |
| Шлюз за замовчуванням | | |

10. Запишіть: чи змінилася IP-адреса після оновлення оренди? (У більшості випадків DHCP-сервер видає ту саму адресу.)
11. Зробіть знімок екрана.

---

### Завдання 9. CMD: ipconfig /flushdns — очищення кешу DNS

1. У Командному рядку (від імені адміністратора) перегляньте поточний кеш DNS:
   ```
   ipconfig /displaydns
   ```
2. Зверніть увагу: у кеші зберігаються записи про раніше відвідані сайти (доменні імена та їх IP-адреси). Запишіть 2–3 записи з кешу.
3. Очистіть кеш DNS:
   ```
   ipconfig /flushdns
   ```
4. Ви побачите повідомлення: **«Кеш зіставлювача DNS успішно очищено»** (Successfully flushed the DNS Resolver Cache).
5. Перевірте, що кеш порожній:
   ```
   ipconfig /displaydns
   ```
6. Тепер відкрийте браузер та перейдіть на будь-який сайт (наприклад, `google.com`).
7. Поверніться до CMD та знову виконайте:
   ```
   ipconfig /displaydns
   ```
8. Зверніть увагу: у кеші з'явилися нові записи для щойно відвіданого сайту.
9. Запишіть свої спостереження. Зробіть знімок екрана.

> **Коли це корисно?** Очищення кешу DNS допомагає, коли сайт змінив IP-адресу, але комп'ютер все ще використовує стару (закешовану) адресу. Також корисно після зміни DNS-серверів (як у завданні 7).

### Завдання 10. Параметри → Мережа та Інтернет — перегляд стану та повернення до автоматичних налаштувань

**Частина A — перегляд стану мережі:**

1. Відкрийте **Параметри** (`Win + I`) → **Мережа та Інтернет**.
2. На головній сторінці перегляньте:
   - Стан підключення (Підключено / Не підключено).
   - Тип мережі (Ethernet або Wi-Fi).
   - Профіль мережі (Приватна або Загальнодоступна).
3. Натисніть **«Додаткові параметри мережі»** (Advanced network settings).
4. У розділі **«Мережеві адаптери»** перегляньте список усіх адаптерів та їх стан.
5. Натисніть на активний адаптер і перегляньте:
   - Стан медіа (Media state).
   - Швидкість підключення (Link speed).
   - Тривалість (Duration).
   - Байтів відправлено/прийнято.
6. Натисніть **«Переглянути додаткові властивості»** (View additional properties) — тут відображаються: IP-адреса, DNS, MAC-адреса, виробник адаптера, версія драйвера.
7. Запишіть основну інформацію. Зробіть знімок екрана.

**Частина B — перегляд профілю мережі (Центр мережевих підключень):**

8. У **Параметри → Мережа та Інтернет** натисніть **«Властивості»** біля назви активної мережі.
9. У верхній частині вікна ви побачите тип мережевого профілю:
   - **Приватна мережа** — комп'ютер видимий у мережі, увімкнено спільний доступ.
   - **Загальнодоступна мережа** — комп'ютер прихований, спільний доступ вимкнено.
10. Запишіть, який профіль встановлено на вашому комп'ютері.
11. Також можна переглянути профіль через класичну Панель керування: `Win + R` → `control /name Microsoft.NetworkAndSharingCenter` → у Центрі мережевих підключень біля назви мережі вказано її тип.

**Частина C — повернення DNS до автоматичних налаштувань:**

12. Відкрийте `ncpa.cpl` (`Win + R` → `ncpa.cpl`).
13. ПКМ на активному адаптері → **«Властивості»**.
14. Виділіть **TCP/IPv4** → **«Властивості»**.
15. Оберіть **«Отримати адресу DNS-сервера автоматично»**.
16. Натисніть **OK** → **OK**.
17. Перевірте у CMD:
    ```
    ipconfig /all
    ```
18. Переконайтеся, що DNS-сервер знову отримується автоматично (від DHCP).
19. Зробіть знімок екрана для підтвердження.

> **Важливо:** Завжди повертайте налаштування до початкового стану після експериментів! У навчальному кабінеті залишені ручні налаштування можуть спричинити проблеми для наступних користувачів.

## Контрольні запитання для самоперевірки

1. Що таке мережевий адаптер? Які типи мережевих адаптерів може мати комп'ютер (назвіть три)?
2. Якою командою відкривається вікно «Мережеві підключення»? Назвіть щонайменше два способи його відкриття.
3. Що таке MAC-адреса? Чим вона відрізняється від IP-адреси? Наведіть приклад формату MAC-адреси.
4. Яка різниця між автоматичним (DHCP) та статичним налаштуванням IP-адреси? Коли використовується кожен варіант?
5. Що таке DHCP-оренда (lease)? Що відбувається, коли термін оренди закінчується?
6. Що показує команда `ipconfig`? Чим вона відрізняється від `ipconfig /all`?
7. Що робить команда `ipconfig /release`? Що станеться з мережевим підключенням після її виконання?
8. Що робить команда `ipconfig /renew`? Чому її зазвичай виконують після `ipconfig /release`?
9. Що таке DNS і для чого він потрібен? Що станеться, якщо DNS-сервер не працює?
10. Навіщо очищувати кеш DNS командою `ipconfig /flushdns`? Наведіть приклад ситуації, коли це необхідно.
11. Які переваги має використання Google DNS (8.8.8.8, 8.8.4.4) замість DNS провайдера?
12. Чим відрізняється приватний мережевий профіль від загальнодоступного? Який профіль безпечніше використовувати у публічних Wi-Fi мережах?
13. Як перевірити, чи використовує комп'ютер DHCP або статичну IP-адресу, за допомогою CMD? Яку команду потрібно ввести та на яке поле звернути увагу?
14. Чому важливо повертати мережеві налаштування до автоматичних (DHCP) після експериментів зі статичними параметрами?
15. Яку інформацію можна побачити у вікні «Відомості про мережеве підключення» (Network Connection Details)? Перелічіть щонайменше шість параметрів.
