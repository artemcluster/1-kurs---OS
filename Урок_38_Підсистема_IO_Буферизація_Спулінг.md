## 1. Підсистема введення-виведення в ОС

### 1.1. Що таке введення-виведення?

**Введення-виведення (Input/Output, I/O)** --- це обмін даними між центральним процесором (CPU), оперативною пам'яттю та зовнішніми (периферійними) пристроями.

- **Введення (Input)** --- читання даних із зовнішнього пристрою в оперативну пам'ять (наприклад, натискання клавіші на клавіатурі).
- **Виведення (Output)** --- запис даних з оперативної пам'яті на зовнішній пристрій (наприклад, виведення тексту на екран або друк на принтері).

Одне з головних завдань операційної системи --- забезпечити зручний та ефективний обмін даними між програмами і периферійними пристроями комп'ютера. Цю роботу виконує **підсистема введення-виведення**.

### 1.2. Загальна структура підсистеми I/O

Підсистема введення-виведення побудована як ієрархія рівнів --- від прикладної програми до фізичного пристрою. Кожен рівень виконує свою функцію і "спілкується" лише з сусідніми рівнями.

```
 ┌─────────────────────────────────────────────┐
 │        ПРИКЛАДНА ПРОГРАМА                   │
 │   (виклик функцій: read, write, print...)   │
 └──────────────────┬──────────────────────────┘
                    │  системний виклик (API)
                    ▼
 ┌─────────────────────────────────────────────┐
 │      ПІДСИСТЕМА I/O ЯДРА ОС                │
 │  (менеджер I/O, буферизація, кешування,     │
 │   планування запитів, спулінг)              │
 └──────────────────┬──────────────────────────┘
                    │
                    ▼
 ┌─────────────────────────────────────────────┐
 │         ФАЙЛОВА СИСТЕМА                     │
 │  (FAT, NTFS, ext4 --- логічна організація)  │
 └──────────────────┬──────────────────────────┘
                    │
                    ▼
 ┌─────────────────────────────────────────────┐
 │         ДРАЙВЕР ПРИСТРОЮ                   │
 │  (програмний модуль, що "розуміє"           │
 │   конкретний пристрій)                      │
 └──────────────────┬──────────────────────────┘
                    │  команди контролеру
                    ▼
 ┌─────────────────────────────────────────────┐
 │         КОНТРОЛЕР ПРИСТРОЮ                  │
 │  (апаратна мікросхема на пристрої           │
 │   або материнській платі)                   │
 └──────────────────┬──────────────────────────┘
                    │  електричні сигнали
                    ▼
 ┌─────────────────────────────────────────────┐
 │         ФІЗИЧНИЙ ПРИСТРІЙ                   │
 │  (жорсткий диск, принтер, клавіатура,       │
 │   монітор, мережева карта тощо)             │
 └─────────────────────────────────────────────┘
```

### 1.3. Опис кожного рівня

| Рівень | Що робить | Приклад |
|---|---|---|
| **Прикладна програма** | Викликає функції API для роботи з пристроями | Програма Word викликає функцію "друк" |
| **Підсистема I/O ядра** | Приймає запити від програм, керує буферами, кешем, чергами | Ядро Windows приймає запит на друк |
| **Файлова система** | Перетворює логічні операції (відкрити файл) у фізичні адреси на диску | NTFS знаходить, де на диску лежить файл |
| **Драйвер пристрою** | Перетворює загальні команди на конкретні команди для пристрою | Драйвер принтера HP формує дані у форматі PCL |
| **Контролер** | Апаратна частина --- виконує команди драйвера, керує пристроєм | SATA-контролер керує обертанням диска |
| **Фізичний пристрій** | Виконує реальну роботу --- друкує, записує, зчитує | Принтер наносить фарбу на папір |

> **Головна ідея:** програмісту не потрібно знати деталі роботи кожного пристрою. ОС надає єдиний, уніфікований інтерфейс для всіх пристроїв.

## 2. Способи організації введення-виведення

Існують три основні способи, якими CPU може обмінюватися даними з периферійними пристроями.

### 2.1. Програмне введення-виведення (Polling / Опитування)

**Принцип:** процесор сам постійно перевіряє (опитує) стан пристрою --- "Чи готові дані?"

```
 ┌───────┐         ┌────────────┐
 │  CPU  │──────>  │  Пристрій  │
 │       │  "Готовий?"           │
 │       │<──────  │  "Ні"      │
 │       │──────>  │  "Готовий?" │
 │       │<──────  │  "Ні"      │
 │       │──────>  │  "Готовий?" │
 │       │<──────  │  "Так!"    │
 │       │<══════  │  [Дані]    │
 └───────┘         └────────────┘
```

**Аналогія:** ви стоїте біля мікрохвильовки і кожні 5 секунд відкриваєте дверцята --- "Чи готова їжа?"

**Переваги:**
- Проста реалізація
- Не потребує додаткового апаратного забезпечення
- Передбачуваний час відповіді

**Недоліки:**
- CPU витрачає час на безкорисне очікування (busy waiting)
- Процесор не може виконувати інші завдання під час опитування
- Дуже неефективно для повільних пристроїв

### 2.2. Введення-виведення за перериваннями (Interrupt-Driven I/O)

**Принцип:** CPU надсилає команду пристрою і далі займається іншими справами. Коли пристрій готовий --- він сам повідомляє CPU через **переривання (interrupt)**.

```
 ┌───────┐         ┌────────────┐
 │  CPU  │──────>  │  Пристрій  │
 │       │  "Читай дані"        │
 │       │                      │
 │  ...виконує інші задачі...   │
 │       │                      │
 │       │<══════  │ ПЕРЕРИВАННЯ│
 │       │         │ + [Дані]   │
 └───────┘         └────────────┘
```

**Аналогія:** ви ставите їжу в мікрохвильовку і йдете дивитися серіал. Коли їжа готова --- мікрохвильовка пищить (переривання), і ви йдете забирати їжу.

**Переваги:**
- CPU вільний для інших завдань, поки пристрій працює
- Більш ефективне використання процесорного часу

**Недоліки:**
- Обробка переривання потребує часу (збереження стану, виклик обробника)
- При великих обсягах даних CPU все одно повинен передавати кожен байт/слово
- Складніша реалізація, ніж polling

### 2.3. Прямий доступ до пам'яті (DMA --- Direct Memory Access)

**Принцип:** спеціальний апаратний контролер --- **DMA-контролер** --- передає дані напряму між пристроєм та оперативною пам'яттю, **без участі CPU**. Процесор лише ініціює передачу і отримує переривання по завершенні.

```
 ┌───────┐    1. Команда    ┌──────────────┐
 │  CPU  │───────────────>  │ DMA-контролер│
 │       │                  │              │
 │  ...вільний...           │  2. Передача │
 │       │                  │     даних    │
 │       │                  │   ┌──────┐   │
 │       │                  │<──│Пристр│   │
 │       │                  │──>│ОЗП   │   │
 │       │                  │   └──────┘   │
 │       │<── 3. Переривання│  "Готово!"   │
 └───────┘                  └──────────────┘
```

**Аналогія:** ви замовляєте доставку їжі (DMA). Кур'єр (DMA-контролер) сам забирає замовлення з ресторану і привозить додому. Ви тільки зробили замовлення і отримали повідомлення "Доставлено".

**Переваги:**
- CPU майже не задіяний під час передачі даних
- Висока швидкість передачі великих обсягів даних
- Ідеально для дисків, мережевих карт, звукових карт

**Недоліки:**
- Потребує додаткового апаратного контролера DMA
- Складніша архітектура
- DMA-контролер і CPU можуть конкурувати за доступ до шини пам'яті

### 2.4. Порівняльна таблиця

| Характеристика | Програмне (Polling) | За перериваннями | DMA |
|---|---|---|---|
| **Участь CPU** | Постійна (100%) | Часткова | Мінімальна |
| **Ефективність CPU** | Низька | Середня | Висока |
| **Швидкість передачі** | Низька | Середня | Висока |
| **Складність реалізації** | Проста | Середня | Складна |
| **Додаткове обладнання** | Не потрібне | Контролер переривань | DMA-контролер |
| **Де використовують** | Прості пристрої (кнопки, датчики) | Клавіатура, миша | Диски, мережа, відео, звук |
| **Аналогія** | Стояти біля мікрохвильовки | Чекати на звуковий сигнал | Замовити доставку |

## 3. Буферизація (Buffering)

### 3.1. Що таке буфер?

**Буфер (buffer)** --- це тимчасова область пам'яті, яка використовується для зберігання даних під час їх передачі між двома пристроями або між пристроєм і програмою.

> **Аналогія з рестораном:** уявіть ресторан. Кухня готує страви швидко, а клієнт їсть повільно. **Офіціант** --- це буфер. Він забирає готову страву з кухні (швидкий виробник) і несе клієнту (повільний споживач). Поки клієнт їсть, кухня вже готує наступну страву. Без офіціанта кухар мусив би стояти і чекати, поки клієнт з'їсть попередню страву.

### 3.2. Навіщо потрібна буферизація?

Три основні причини:

1. **Вирівнювання швидкостей.** Пристрої працюють із різною швидкістю. Наприклад, процесор обробляє дані у мільярди разів швидше, ніж принтер друкує. Буфер дозволяє швидкому пристрою не чекати на повільний.

2. **Узгодження розмірів блоків даних.** Різні пристрої оперують блоками різного розміру. Мережева карта може отримувати дані пакетами по 1500 байт, а програма хоче зчитувати по 4096 байт. Буфер накопичує дані до потрібного розміру.

3. **Підтримка семантики копіювання.** Коли програма каже "записати ці дані на диск", ОС копіює дані в буфер ядра. Навіть якщо програма потім змінить свої дані, на диск запишеться оригінальна версія.

### 3.3. Типи буферизації

#### Одинарна буферизація (Single Buffering)

Використовується **один** буфер. Поки виробник (пристрій) заповнює буфер --- споживач (CPU) чекає. Поки споживач обробляє буфер --- виробник чекає.

```
  Виробник ──> [ БУФЕР ] ──> Споживач

  Крок 1: Виробник ПИШЕ  → Споживач чекає
  Крок 2: Виробник чекає → Споживач ЧИТАЄ
  Крок 3: Виробник ПИШЕ  → Споживач чекає
  ...
```

**Недолік:** виробник і споживач не можуть працювати одночасно --- один завжди чекає.

#### Подвійна буферизація (Double Buffering)

Використовуються **два** буфери. Поки споживач читає з одного буфера --- виробник вже заповнює другий. Потім вони міняються.

```
  Виробник ──> [ БУФЕР A ] ──> Споживач
               [ БУФЕР B ]

  Крок 1: Виробник ПИШЕ в A  → Споживач чекає
  Крок 2: Виробник ПИШЕ в B  → Споживач ЧИТАЄ з A
  Крок 3: Виробник ПИШЕ в A  → Споживач ЧИТАЄ з B
  ...
```

**Перевага:** виробник і споживач працюють паралельно --- система швидша.

**Приклад з життя:** подвійна буферизація широко використовується у відеоіграх. Один буфер відображається на екрані, а другий заповнюється наступним кадром. Коли кадр готовий --- буфери міняються місцями. Це запобігає "мерехтінню" зображення.

#### Циклічна буферизація (Circular / Ring Buffer)

Використовується **кілька буферів** (зазвичай більше двох), організованих у кільце. Виробник записує дані за допомогою вказівника запису, споживач зчитує за допомогою вказівника читання. Обидва вказівники рухаються по колу.

```
         Вказівник запису
              ↓
  ┌───┬───┬───┬───┬───┬───┐
  │ 5 │ 6 │   │   │ 3 │ 4 │  ← дані в буферах
  └───┴───┴───┴───┴───┴───┘
                      ↑
              Вказівник читання

  Буфери з'єднані в кільце:
  [0] → [1] → [2] → [3] → [4] → [5] → [0] → ...
```

**Переваги:**
- Обробка швидких серій (burst) введення-виведення
- Мінімальний час простою між виробником і споживачем
- Операції читання та запису --- лише переміщення вказівників (дуже швидко)
- Використовується у реальному часі: аудіо, відео, мережеві пакети

### 3.4. Приклади буферизації у повсякденній роботі

| Де | Що відбувається |
|---|---|
| **Буфер клавіатури** | Коли ви швидко набираєте текст, символи зберігаються в буфері клавіатури, навіть якщо програма ще не встигла їх обробити |
| **Буфер друку** | Дані для принтера спочатку зберігаються в буфері, щоб програма могла продовжити роботу, не чекаючи завершення друку |
| **Буфер відео** | Відеоплеєр завантажує наступні кадри в буфер заздалегідь, щоб відео не "замерзало" при короткочасних затримках |
| **Буфер мережі** | Мережева карта зберігає вхідні пакети в буфері, поки ОС не обробить їх |
| **Буфер звуку** | Звукова карта використовує буфер для безперервного відтворення аудіо |

## 4. Кешування (Caching)

### 4.1. Що таке кеш?

**Кеш (cache)** --- це швидка пам'ять, у якій зберігаються **копії** часто використовуваних даних для прискорення повторного доступу.

### 4.2. Головна відмінність від буферизації

| Властивість | Буфер | Кеш |
|---|---|---|
| **Що зберігає** | Оригінальні дані при передачі | **Копію** даних для повторного використання |
| **Мета** | Узгодити швидкості пристроїв | Прискорити повторний доступ |
| **Час зберігання** | До завершення передачі | Поки дані не будуть витіснені |
| **Принцип** | FIFO (першим прийшов --- першим пішов) | Локальність (часова та просторова) |

> **Проста аналогія:**
> - **Буфер** --- це стіл офіціанта, на якому стоять страви, що несуть клієнту (транзитне зберігання).
> - **Кеш** --- це холодильник удома, де ви зберігаєте копію улюбленого соусу, щоб щоразу не бігти в магазин (зберігання для швидкого повторного доступу).

### 4.3. Рівні кешу в комп'ютері

```
  Швидкість ▲    Розмір ▼
  ──────────────────────────
  │ Регістри CPU   │  ~1 КБ    │  < 1 нс     │
  │ Кеш L1         │  ~64 КБ   │  ~1 нс      │  для кожного ядра
  │ Кеш L2         │  ~256 КБ  │  ~3-10 нс   │  для кожного ядра
  │ Кеш L3         │  8-64 МБ  │  ~10-30 нс  │  спільний для всіх ядер
  │ Оперативна RAM │  8-64 ГБ  │  ~50-100 нс │
  │ SSD             │  256 ГБ+  │  ~100 мкс   │
  │ HDD             │  1 ТБ+    │  ~10 мс     │
  ──────────────────────────
```

| Рівень кешу | Розташування | Типовий розмір | Швидкість |
|---|---|---|---|
| **L1** | Всередині кожного ядра CPU | ~64 КБ | До 100 разів швидше за RAM |
| **L2** | Всередині кожного ядра CPU | ~256 КБ --- 1 МБ | ~25 разів швидше за RAM |
| **L3** | Спільний для всіх ядер CPU | 8 --- 64 МБ | ~2 рази швидше за RAM |
| **Дисковий кеш** | В оперативній пам'яті (RAM) | Змінний | Набагато швидше за фізичний диск |
| **Кеш файлової системи** | В RAM, керується ядром ОС | Значна частина вільної RAM | Запобігає повторним зверненням до диска |

**Як працює кешування (спрощено):**

1. Програма запитує дані (наприклад, прочитати файл).
2. ОС спочатку шукає ці дані в кеші (швидко!).
3. Якщо знайдено (**cache hit** --- влучання в кеш) --- дані повертаються негайно.
4. Якщо не знайдено (**cache miss** --- промах кешу) --- дані зчитуються з повільного пристрою, повертаються програмі та **зберігаються в кеші** для наступного разу.

## 5. Спулінг (Spooling)

### 5.1. Що таке спулінг?

**Спулінг (Spooling)** --- від англійського **Simultaneous Peripheral Operations On-Line** --- це техніка, при якій **диск використовується як великий буфер** для обміну даними між програмами та повільними периферійними пристроями.

> **Головна ідея:** програма не чекає, поки повільний пристрій (наприклад, принтер) закінчить роботу. Замість цього дані записуються у файл на диску (спул-файл), а спеціальна програма --- **спулер** --- потім передає ці дані пристрою у зручному для нього темпі.

### 5.2. Як працює спулінг (на прикладі друку)

```
  ┌──────────┐                    ┌──────────────┐
  │ Програма │    1. "Друкувати"  │              │
  │  Word    │──────────────────> │  Спул-файл   │
  └──────────┘     (швидко!)      │  на диску    │
                                  │  (.spl)      │
  ┌──────────┐    2. "Друкувати"  │              │
  │ Програма │──────────────────> │  Черга       │
  │  Excel   │     (швидко!)      │  завдань     │
  └──────────┘                    └──────┬───────┘
                                         │
                                    3. Спулер
                                    відправляє
                                    по одному
                                         │
                                         ▼
                                  ┌──────────────┐
                                  │   ПРИНТЕР    │
                                  │  (повільно)  │
                                  └──────────────┘
```

**Покроково:**

1. Користувач натискає "Друк" у Word. Програма **швидко** записує дані для друку у спул-файл на жорсткому диску.
2. Word повідомляє: "Друк завершено!" --- і користувач може продовжувати роботу.
3. Тим часом Excel теж надсилає документ на друк --- він стає другим у черзі.
4. **Спулер** (фонова служба ОС) бере завдання з черги по одному і відправляє дані на принтер.
5. Принтер друкує документи послідовно.

### 5.3. Переваги спулінгу

- **Кілька програм можуть "друкувати" одночасно** --- кожна записує дані в свій спул-файл, не чекаючи на принтер.
- **Програми звільняються миттєво** --- не треба чекати, поки принтер роздрукує всі сторінки.
- **Впорядкованість** --- завдання обробляються в порядку черги (FIFO).
- **Контроль** --- адміністратор може переглядати чергу, змінювати пріоритети, видаляти завдання.

### 5.4. Спулінг vs. Буферизація

| Властивість | Буферизація | Спулінг |
|---|---|---|
| **Де зберігаються дані** | В оперативній пам'яті (RAM) | На жорсткому диску |
| **Розмір** | Обмежений (кілька КБ --- МБ) | Великий (гігабайти) |
| **Кількість завдань** | Зазвичай одне | Черга з багатьох завдань |
| **Приклад** | Буфер клавіатури | Черга друку |
| **Швидкість доступу** | Дуже швидка | Повільніша, але об'ємніша |

## 6. Черга друку в Windows

### 6.1. Служба Print Spooler

У Windows за спулінг друку відповідає системна служба **Print Spooler** (Диспетчер друку). Вона запускається автоматично при завантаженні Windows і працює у фоновому режимі.

**Що робить служба Print Spooler:**
- Приймає завдання друку від усіх програм
- Зберігає їх у тимчасових файлах (спул-файлах) у папці `C:\Windows\System32\spool\PRINTERS`
- Перетворює дані у формат, зрозумілий принтеру (EMF, RAW, PCL, PostScript)
- Відправляє завдання на принтер у порядку черги

### 6.2. Як переглянути чергу друку

**Спосіб 1 --- через Параметри:**
1. Відкрийте **Пуск** (Start) > **Параметри** (Settings)
2. Перейдіть до **Bluetooth і пристрої** > **Принтери та сканери**
3. Оберіть потрібний принтер
4. Натисніть **Відкрити чергу друку** (Open print queue)

**Спосіб 2 --- через Панель керування:**
1. Відкрийте **Панель керування** > **Пристрої та принтери**
2. Двічі клацніть на принтері --- відкриється вікно черги друку

**У вікні черги друку відображається:**

| Стовпець | Опис |
|---|---|
| Ім'я документа | Назва файлу, що друкується |
| Стан | Друкується, Очікує, Помилка |
| Власник | Ім'я користувача, який відправив на друк |
| Сторінки | Кількість сторінок |
| Розмір | Обсяг даних |
| Час подачі | Коли завдання було надіслано |

### 6.3. Керування чергою друку

**Основні дії:**
- **Призупинити завдання** --- правий клік на завданні > Призупинити
- **Відновити завдання** --- правий клік > Відновити
- **Скасувати завдання** --- правий клік > Скасувати
- **Скасувати всі завдання** --- меню Принтер > Скасувати всі документи

### 6.4. Перезапуск служби Print Spooler

Якщо друк "зависає" і завдання не видаляються, потрібно перезапустити службу.

**Через командний рядок (від імені адміністратора):**
```
net stop spooler
net start spooler
```

**Через консоль "Служби":**
1. Натисніть **Win + R**, введіть `services.msc`, натисніть Enter
2. Знайдіть службу **Print Spooler** (Диспетчер друку)
3. Правий клік > **Перезапустити**

**Очищення "застряглих" завдань вручну:**
1. Зупиніть службу: `net stop spooler`
2. Відкрийте папку `C:\Windows\System32\spool\PRINTERS`
3. Видаліть усі файли у цій папці
4. Запустіть службу: `net start spooler`

## Глосарій

| Термін | Визначення |
|---|---|
| **Введення-виведення (I/O)** | Обмін даними між CPU / RAM та периферійними пристроями |
| **Підсистема I/O** | Компонент ядра ОС, що керує обміном даних із пристроями |
| **Драйвер пристрою** | Програмний модуль, що перетворює команди ОС на команди конкретного пристрою |
| **Контролер пристрою** | Апаратна мікросхема, що безпосередньо керує фізичним пристроєм |
| **Polling (опитування)** | Спосіб I/O, при якому CPU постійно перевіряє стан пристрою |
| **Переривання (Interrupt)** | Сигнал від пристрою до CPU про те, що операція завершена або потребує уваги |
| **DMA (Direct Memory Access)** | Прямий доступ до пам'яті --- передача даних між пристроєм і RAM без участі CPU |
| **DMA-контролер** | Спеціальний апаратний модуль, що виконує DMA-передачу |
| **Буфер (Buffer)** | Тимчасова область пам'яті для проміжного зберігання даних при передачі |
| **Буферизація (Buffering)** | Техніка використання буферів для узгодження швидкостей пристроїв |
| **Одинарна буферизація** | Використання одного буфера --- виробник і споживач працюють почергово |
| **Подвійна буферизація** | Використання двох буферів --- виробник і споживач можуть працювати паралельно |
| **Циклічний буфер (Ring Buffer)** | Кілька буферів, організованих у кільце --- для високошвидкісного I/O |
| **Кеш (Cache)** | Швидка пам'ять, що зберігає копії часто використовуваних даних |
| **Cache hit (влучання)** | Запитані дані знайдено в кеші |
| **Cache miss (промах)** | Запитаних даних у кеші немає --- потрібно зчитувати з повільнішої пам'яті |
| **L1, L2, L3** | Рівні кеш-пам'яті процесора (від найшвидшого до найбільшого) |
| **Спулінг (Spooling)** | Simultaneous Peripheral Operations On-Line --- використання диска як великого буфера для черги завдань |
| **Спулер (Spooler)** | Програма (служба ОС), що керує чергою завдань для периферійного пристрою |
| **Спул-файл** | Тимчасовий файл на диску, що містить дані завдання (наприклад, документ для друку) |
| **Print Spooler** | Служба Windows, що керує чергою друку |
| **Черга друку (Print Queue)** | Упорядкований список завдань друку, що очікують обробки |
| **FIFO** | First In --- First Out (перший прийшов --- перший пішов) --- принцип роботи черги |
