# КОНСПЕКТ УРОКУ №48

## Тема: Практична робота №18 — Спільний доступ до ресурсів у мережі

---

## Мета роботи

Навчитися налаштовувати спільний доступ до файлів і принтерів у локальній мережі Windows 10/11, керувати дозволами, підключати мережеві диски та спільні принтери, а також використовувати команди командного рядка для роботи з мережевими ресурсами.

---

## Теоретичний вступ

### 1. Спільний доступ до ресурсів (File/Printer Sharing)

**Спільний доступ** — це механізм операційної системи, який дозволяє надавати іншим комп'ютерам у мережі доступ до файлів, папок та принтерів, підключених до вашого ПК. Завдяки спільному доступу кілька користувачів можуть одночасно працювати з одними й тими самими документами, друкувати на одному принтері без необхідності фізичного підключення до нього.

Основні можливості спільного доступу у Windows:

- надання доступу до окремих папок або цілих дисків іншим користувачам мережі;
- спільне використання принтерів, підключених до одного комп'ютера;
- налаштування рівнів дозволів: лише читання, читання та запис, повний контроль;
- обмеження доступу за допомогою паролів та облікових записів.

> **Важливо:** У Windows існує обмеження — не більше **20 одночасних підключень** до спільних ресурсів на клієнтських версіях ОС (Windows 10/11). Для більшої кількості підключень потрібен Windows Server.

---

### 2. Протокол SMB (Server Message Block)

**SMB (Server Message Block)** — це мережевий протокол прикладного рівня, який забезпечує спільний доступ до файлів, принтерів і послідовних портів між вузлами мережі. Саме SMB використовується Windows для обміну файлами по локальній мережі.

Еволюція версій SMB:

| Версія | Підтримка ОС | Особливості |
|--------|-------------|-------------|
| **SMB 1.0** (CIFS) | Windows 2000, XP | Застаріла, **небезпечна** — вразлива до атак (WannaCry). За замовчуванням вимкнена у Windows 10/11 |
| **SMB 2.0** | Windows Vista / Server 2008 | Зменшення кількості команд, підтримка символічних посилань |
| **SMB 2.1** | Windows 7 / Server 2008 R2 | Покращена продуктивність, підтримка великих MTU |
| **SMB 3.0** | Windows 8 / Server 2012 | Наскрізне шифрування, багатоканальний доступ |
| **SMB 3.1.1** | Windows 10/11 / Server 2016+ | **Поточна версія.** Обов'язкове шифрування, перевірка цілісності, захист від атак |

**Порт SMB:** TCP **445** — основний порт, через який працює SMB у сучасних мережах.

> **Безпека:** SMB 1.0 вважається небезпечним і не рекомендується до використання. У Windows 10/11 він за замовчуванням вимкнений. Ніколи не вмикайте SMB 1.0 без крайньої необхідності.

---

### 3. UNC-шлях (Universal Naming Convention)

**UNC-шлях** — це стандартний формат запису шляху до мережевого ресурсу у Windows:

```
\\ім'я_комп'ютера\ім'я_спільного_ресурсу
```

Приклади:

| UNC-шлях | Що відкриває |
|----------|-------------|
| `\\PC-TEACHER\Документи` | Спільна папка «Документи» на комп'ютері PC-TEACHER |
| `\\192.168.1.10\Проєкти` | Спільна папка «Проєкти» за IP-адресою |
| `\\SERVER\HP_LaserJet` | Мережевий принтер HP LaserJet на комп'ютері SERVER |
| `\\PC01\C$` | Адміністративний спільний ресурс — диск C: на комп'ютері PC01 |

**Як використовувати UNC-шлях:**

1. Натисніть `Win + R`, введіть UNC-шлях (наприклад, `\\PC-TEACHER\Документи`) та натисніть Enter.
2. Або відкрийте **Провідник** і введіть UNC-шлях в адресний рядок.

---

### 4. Мережеве виявлення (Network Discovery)

**Мережеве виявлення** — це функція Windows, яка визначає, чи може ваш комп'ютер бачити інші комп'ютери в мережі та чи можуть інші комп'ютери бачити ваш.

Windows розрізняє два типи мережевих профілів:

| Профіль | Призначення | Мережеве виявлення |
|---------|-------------|-------------------|
| **Приватна мережа** | Домашня або робоча мережа, якій ви довіряєте | **Увімкнено** — комп'ютер видимий, спільний доступ дозволений |
| **Публічна мережа** | Кафе, аеропорт, бібліотека | **Вимкнено** — комп'ютер прихований, спільний доступ заблокований |

> **Правило:** Перш ніж налаштовувати спільний доступ, переконайтеся, що ваша мережа позначена як **Приватна**. Для цього відкрийте **Параметри → Мережа та Інтернет** → натисніть на ваше підключення (Ethernet або Wi-Fi) → встановіть профіль **Приватна мережа**.

Системні служби, необхідні для мережевого виявлення:

- **Function Discovery Provider Host** (fdPHost)
- **Function Discovery Resource Publication** (FDResPub)
- **SSDP Discovery** (SSDPSRV)
- **UPnP Device Host** (upnphost)

Якщо мережеве виявлення не працює, перевірте, чи запущені ці служби: `Win + R` → `services.msc` → знайдіть кожну службу → ПКМ → **Властивості** → тип запуску: **Автоматичний** → натисніть **Запустити**.

---

### 5. Дозволи спільного доступу

При наданні спільного доступу до папки можна встановити різні рівні дозволів:

| Рівень дозволу | Що дозволяє | Що забороняє |
|----------------|-------------|-------------|
| **Читання (Read)** | Переглядати файли, відкривати, копіювати | Змінювати, видаляти, створювати нові файли |
| **Читання/Запис (Read/Write)** | Переглядати, відкривати, копіювати, створювати, змінювати, видаляти файли | Зміна дозволів |
| **Повний контроль (Full Control)** | Усе вищезазначене + зміна дозволів | Нічого не забороняє |

Дозволи можна призначати:

- **Конкретному користувачу** — за ім'ям облікового запису Windows.
- **Групі Everyone** — усім користувачам мережі без обмежень.
- **Групі Authenticated Users** — лише автентифікованим (тим, хто ввів логін і пароль).

---

### 6. Адміністративні спільні ресурси

Windows автоматично створює **приховані спільні ресурси** (їх назви закінчуються на `$`), які не відображаються у списку мережевих ресурсів при звичайному перегляді:

| Ресурс | Шлях | Призначення |
|--------|------|-------------|
| **C$** (D$, E$ тощо) | Кореневий каталог відповідного диска | Віддалений доступ до всього вмісту диска. Створюється для кожного диска автоматично |
| **ADMIN$** | `C:\Windows` | Доступ до системної папки Windows. Використовується для віддаленого адміністрування |
| **IPC$** | — (немає шляху на диску) | Міжпроцесна комунікація (Inter-Process Communication). Забезпечує зв'язок між службами на різних комп'ютерах, автентифікацію, доступ до іменованих каналів (named pipes) |
| **PRINT$** | `C:\Windows\System32\spool\drivers` | Спільний доступ до драйверів принтерів |

> **Безпека:** Адміністративні ресурси доступні **лише** користувачам з правами локального адміністратора. Символ `$` у назві робить ресурс прихованим, але не захищеним — захист забезпечується лише через дозволи облікових записів.

## Практичні завдання

### Завдання 1. Увімкнення мережевого виявлення та спільного доступу до файлів

1. Переконайтеся, що ваша мережа позначена як **Приватна**:
   - Відкрийте **Параметри** (`Win + I`) → **Мережа та Інтернет**.
   - Натисніть на ваше активне підключення (Ethernet або Wi-Fi).
   - Встановіть тип мережевого профілю: **Приватна мережа**.
2. Увімкніть мережеве виявлення та спільний доступ:
   - **Параметри** → **Мережа та Інтернет** → **Додаткові параметри мережі** → **Додаткові параметри спільного доступу**.
   - У розділі **Приватні мережі** увімкніть:
     - **Мережеве виявлення** (Network Discovery) — перемикач у положення **Увімк.**
     - **Автоматичне налаштування підключених до мережі пристроїв** — увімкнути.
     - **Спільний доступ до файлів і принтерів** (File and Printer Sharing) — перемикач у положення **Увімк.**
   - У розділі **Усі мережі** перегляньте налаштування:
     - **Спільний доступ із захистом паролем** — залиште **увімкненим** (для безпеки) або вимкніть (для спрощення доступу у навчальній мережі — за вказівкою вчителя).
3. Зробіть знімок екрана вікна «Додаткові параметри спільного доступу» з увімкненими параметрами.

---

### Завдання 2. Створення папки та надання спільного доступу

1. Створіть на робочому столі папку з назвою **Спільна_<ваше прізвище>** (наприклад, `Спільна_Петренко`).
2. Покладіть у папку 2–3 текстових файли (створіть їх або скопіюйте будь-які).
3. Надайте спільний доступ до папки:
   - ПКМ на папці → **Властивості** → вкладка **Доступ** → натисніть **Спільний доступ...** (Share...).
   - У вікні «Мережевий доступ» оберіть користувача **Everyone** з випадаючого списку → натисніть **Додати**.
   - Встановіть рівень дозволу: **Читання** (Read).
   - Натисніть **Спільний доступ** (Share) → **Готово** (Done).
4. Зверніть увагу на **мережевий шлях** до вашої папки, що відображається у вікні підтвердження (наприклад, `\\PC01\Спільна_Петренко`). Запишіть його.
5. Зробіть знімок екрана вкладки **Доступ** у властивостях папки.

---

### Завдання 3. Налаштування дозволів для різних користувачів

1. Відкрийте властивості вашої спільної папки → вкладка **Доступ** → натисніть **Розширена настройка спільного доступу** (Advanced Sharing).
2. Переконайтеся, що прапорець **Відкрити спільний доступ до цієї папки** встановлено.
3. Натисніть кнопку **Дозволи** (Permissions). Перегляньте поточні дозволи:
   - Яка група/користувач має доступ?
   - Які дозволи встановлено (Повний контроль, Змінення, Читання)?
4. Додайте групу **Everyone** (якщо відсутня) та встановіть:
   - **Читання** — дозволити.
   - **Змінення** — дозволити (це дозволить створювати та редагувати файли).
   - **Повний контроль** — заборонити (залишити без відмітки).
5. Натисніть **Застосувати** → **OK**.
6. Заповніть таблицю:

| Група/Користувач | Повний контроль | Змінення | Читання |
|------------------|----------------|----------|---------|
| Everyone | | | |

7. Зробіть знімок екрана вікна дозволів.

---

### Завдання 4. Підключення до спільної папки з іншого комп'ютера

1. На **іншому комп'ютері** у класі (або попросіть сусіда підключитися до вашого ПК) відкрийте **Провідник**.
2. В адресний рядок введіть UNC-шлях вашого сусіда:
   ```
   \\ім'я_комп'ютера_сусіда
   ```
   (наприклад, `\\PC02` або `\\192.168.1.15`).
3. Натисніть **Enter**. Якщо з'явиться запит на введення імені користувача та пароля — введіть облікові дані, що вам повідомить сусід (або вчитель).
4. Знайдіть спільну папку сусіда та відкрийте її.
5. Спробуйте виконати дії відповідно до наданих дозволів:
   - **Прочитати** файл — відкрити текстовий документ.
   - **Створити** новий файл у спільній папці (якщо дозволено Змінення).
   - **Видалити** файл (якщо дозволено).
6. Запишіть результати: які дії вдалось виконати, а які — ні. Поясніть чому.
7. Зробіть знімок екрана Провідника з відкритою спільною папкою сусіда.

---

### Завдання 5. CMD: net share — перегляд спільних ресурсів

1. Відкрийте **Командний рядок** (CMD): `Win + R` → введіть `cmd` → **Enter**.
2. Введіть команду:
   ```
   net share
   ```
3. Перегляньте список усіх спільних ресурсів на вашому комп'ютері. Запишіть у таблицю:

| Ім'я ресурсу | Шлях (Resource) | Примітка (Remark) | Тип (звичайний / адміністративний) |
|-------------|----------------|-------------------|-----------------------------------|
| | | | |
| | | | |
| | | | |

4. Зверніть увагу на ресурси, що закінчуються на `$` — це приховані адміністративні ресурси.
5. Створіть новий спільний ресурс через CMD:
   ```
   net share TestShare=C:\Users\%USERNAME%\Desktop\Спільна_Петренко /grant:Everyone,READ
   ```
6. Повторіть команду `net share` та переконайтеся, що новий ресурс з'явився у списку.
7. Видаліть створений ресурс:
   ```
   net share TestShare /delete
   ```
8. Зробіть знімок екрана з результатами команд.

---

### Завдання 6. CMD: net use — підключення мережевого диска

1. У **Командному рядку** перегляньте поточні мережеві підключення:
   ```
   net use
   ```
2. Підключіть спільну папку сусіда як мережевий диск **Z:**
   ```
   net use Z: \\ім'я_ПК_сусіда\ім'я_спільної_папки
   ```
   Наприклад:
   ```
   net use Z: \\PC02\Спільна_Петренко
   ```
3. Якщо з'явиться запит пароля, введіть облікові дані або додайте параметр `/user`:
   ```
   net use Z: \\PC02\Спільна_Петренко /user:Ім'яКористувача Пароль
   ```
4. Відкрийте **Провідник** → **Цей ПК** — переконайтеся, що диск Z: з'явився у списку.
5. Відкрийте диск Z: та перевірте, що ви бачите файли зі спільної папки.
6. Від'єднайте мережевий диск:
   ```
   net use Z: /delete
   ```
7. Переконайтеся, що диск Z: зник із Провідника.
8. Запишіть усі виконані команди та їх результати.

---

### Завдання 7. Спільний доступ до принтера

> **Примітка:** Якщо у класі немає фізичного принтера — встановіть віртуальний принтер «Microsoft Print to PDF» або використайте будь-який уже встановлений принтер для демонстрації налаштувань.

1. Відкрийте **Параметри** (`Win + I`) → **Bluetooth і пристрої** → **Принтери та сканери**.
2. Оберіть принтер зі списку (наприклад, Microsoft Print to PDF).
3. Натисніть **Властивості принтера** (Printer Properties).
4. Перейдіть на вкладку **Доступ** (Sharing):
   - Встановіть прапорець **Спільний доступ до цього принтера** (Share this printer).
   - У полі **Ім'я спільного ресурсу** (Share name) введіть коротку назву без пробілів (наприклад, `PDF_Принтер`).
   - Натисніть **Застосувати** → **OK**.
5. Запишіть мережевий шлях до принтера: `\\ім'я_вашого_ПК\PDF_Принтер`.
6. Зробіть знімок екрана вкладки «Доступ» із налаштуваннями.

---

### Завдання 8. Підключення до мережевого принтера з іншого ПК

1. На **іншому комп'ютері** відкрийте **Параметри** → **Bluetooth і пристрої** → **Принтери та сканери**.
2. Натисніть **Додати пристрій** (Add device) → зачекайте кілька секунд.
3. Якщо принтер не знайдено автоматично, натисніть **Додати вручну** (Add manually).
4. Оберіть **Вибрати спільний принтер за іменем** (Select a shared printer by name).
5. Введіть мережевий шлях до принтера сусіда:
   ```
   \\ім'я_ПК_сусіда\ім'я_принтера
   ```
   Наприклад: `\\PC01\PDF_Принтер`.
6. Натисніть **Далі** (Next) → дочекайтеся встановлення.
7. Натисніть **Готово** (Finish).
8. Перевірте, що принтер з'явився у списку принтерів.

**Альтернативний спосіб через Провідник:**

1. Відкрийте Провідник → введіть в адресний рядок `\\ім'я_ПК_сусіда`.
2. У списку спільних ресурсів знайдіть принтер → ПКМ → **Підключити** (Connect).

9. Зробіть знімок екрана списку принтерів з доданим мережевим принтером.

---

### Завдання 9. Адміністративні спільні ресурси: C$, D$, ADMIN$, IPC$

1. У **Командному рядку** виконайте команду:
   ```
   net share
   ```
2. Знайдіть у списку ресурси, що закінчуються на `$`: **C$**, **ADMIN$**, **IPC$** та інші.
3. Заповніть таблицю:

| Ресурс | Шлях | Призначення |
|--------|------|-------------|
| C$ | | |
| ADMIN$ | | |
| IPC$ | | |

4. Спробуйте відкрити адміністративний ресурс свого комп'ютера через Провідник:
   - Натисніть `Win + R`, введіть: `\\localhost\C$` → натисніть **Enter**.
   - Що ви бачите? Запишіть результат.
5. Спробуйте відкрити адміністративний ресурс **іншого** комп'ютера:
   - `Win + R` → `\\ім'я_ПК_сусіда\C$` → **Enter**.

### Завдання 10. CMD: net view — перегляд комп'ютерів у мережі

1. У **Командному рядку** введіть:
   ```
   net view
   ```
   Ця команда відображає список комп'ютерів у вашій робочій групі (workgroup).
2. Запишіть список комп'ютерів, що відобразилися.
3. Перегляньте спільні ресурси конкретного комп'ютера:
   ```
   net view \\ім'я_ПК_сусіда
   ```
   Наприклад:
   ```
   net view \\PC02
   ```
4. Запишіть, які спільні ресурси бачить ваш комп'ютер на ПК сусіда.
5. Спробуйте переглянути ресурси за IP-адресою:
   ```
   net view \\192.168.1.15
   ```
6. Порівняйте результати: чи однакові списки ресурсів при зверненні за іменем та за IP?

> **Примітка:** Якщо команда `net view` повертає помилку «Список серверів для цієї робочої групи наразі недоступний» — це може означати, що мережеве виявлення вимкнене або служба NetBIOS не працює. Перевірте налаштування з Завдання 1.

7. Зробіть знімок екрана з результатами команд.

## Безпека спільного доступу

### Рекомендації щодо безпечного налаштування

1. **Тип мережі:** Завжди використовуйте профіль **Приватна мережа** лише у довіреному середовищі (дім, офіс, навчальний клас). У громадських місцях — лише **Публічна мережа**.

2. **Захист паролем:** Увімкніть **спільний доступ із захистом паролем** (Password Protected Sharing) — це вимагатиме від користувачів вводити логін і пароль для підключення.

3. **Принцип найменших привілеїв:** Надавайте мінімально необхідні дозволи:
   - Якщо достатньо перегляду — дозвіл **Читання**.
   - Не використовуйте **Everyone + Повний контроль** без крайньої потреби.
   - Надавайте доступ конкретним користувачам, а не групі Everyone.

4. **SMB версія:** Не вмикайте SMB 1.0 — він має критичні вразливості. Windows 10/11 використовує SMB 3.1.1 за замовчуванням.

5. **Брандмауер:** Переконайтеся, що правила брандмауера дозволяють спільний доступ до файлів і принтерів лише для приватного профілю мережі. Перевірка: **Панель керування** → **Брандмауер Windows** → **Дозвіл програми** → знайдіть «Спільний доступ до файлів і принтерів» — прапорець лише для **Приватна**.

6. **Приховані ресурси ($):** Адміністративні ресурси (C$, ADMIN$, IPC$) не можна переглянути звичайним способом, але можна підключитися, знаючи шлях. У робочій групі за замовчуванням доступ до них заборонений для віддалених адміністраторів (крім вбудованого Administrator).

### Порівняння: публічна vs приватна мережа

| Параметр | Приватна мережа | Публічна мережа |
|----------|----------------|----------------|
| Мережеве виявлення | Увімкнено | Вимкнено |
| Спільний доступ до файлів | Дозволено | Заблоковано |
| Спільний доступ до принтерів | Дозволено | Заблоковано |
| Брандмауер | Менш суворі правила | Максимально суворі правила |
| Рекомендоване місце | Дім, офіс, клас | Кафе, аеропорт, бібліотека |

## Зведена таблиця команд CMD

| Команда | Призначення | Приклад |
|---------|------------|---------|
| `net share` | Переглянути всі спільні ресурси на локальному ПК | `net share` |
| `net share Ім'я=Шлях` | Створити спільний ресурс | `net share Docs=C:\Docs /grant:Everyone,READ` |
| `net share Ім'я /delete` | Видалити спільний ресурс | `net share Docs /delete` |
| `net use` | Переглянути підключені мережеві диски | `net use` |
| `net use X: \\ПК\Папка` | Підключити мережевий диск | `net use Z: \\PC02\Share` |
| `net use X: /delete` | Від'єднати мережевий диск | `net use Z: /delete` |
| `net view` | Переглянути комп'ютери в мережі | `net view` |
| `net view \\ПК` | Переглянути ресурси конкретного ПК | `net view \\PC02` |
