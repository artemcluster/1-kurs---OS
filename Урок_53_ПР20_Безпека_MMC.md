# КОНСПЕКТ УРОКУ №53

## Тема: Практична робота №20 — Управління параметрами безпеки. Консолі MMC

---

## Мета роботи

Навчитися працювати з Microsoft Management Console (MMC) та її оснастками, налаштовувати параметри безпеки Windows через локальну політику безпеки, редактор групових політик, брандмауер та журнал подій, а також створювати власні консолі MMC.

---

## Теоретичний вступ

### 1. Microsoft Management Console (MMC) — фреймворк для адміністрування

**Microsoft Management Console (MMC)** — це компонент Windows, який надає єдиний інтерфейс для системного адміністрування. MMC сама по собі не виконує адміністративних функцій — вона є **фреймворком** (каркасом), до якого підключаються спеціалізовані інструменти — **оснастки (snap-ins)**.

MMC працює за принципом «порожньої консолі»: адміністратор додає до неї потрібні оснастки, налаштовує вигляд та зберігає консоль як файл `.msc`. Це дозволяє створювати персоналізовані набори інструментів для конкретних завдань.

**Запуск MMC:**

| Спосіб | Дія |
|--------|-----|
| Команда «Виконати» | `Win + R` → ввести `mmc` → Enter |
| Командний рядок | Ввести `mmc.exe` → Enter |
| PowerShell | Ввести `mmc` → Enter |
| Пошук | Натиснути `Win`, ввести «mmc» |

> **Важливо:** Для повноцінної роботи з більшістю оснасток потрібні права адміністратора. Деякі оснастки (secpol.msc, gpedit.msc, lusrmgr.msc) доступні лише у Windows Pro, Enterprise та Education — у Windows Home вони відсутні.

---

### 2. Оснастки (Snap-ins): що це та як додати

**Оснастка (snap-in)** — це модуль, що «вбудовується» у MMC і надає конкретну адміністративну функціональність. Оснастки реалізовані як COM-об'єкти у вигляді DLL-бібліотек.

**Типи оснасток:**

- **Автономні (stand-alone)** — працюють самостійно, виконуючи окрему адміністративну задачу (наприклад, Диспетчер пристроїв, Управління дисками).
- **Розширення (extension)** — додають функціональність до автономних оснасток.
- **Подвійного режиму (dual-mode)** — можуть працювати і як автономні, і як розширення.

**Як додати оснастку до MMC:**

1. Відкрити MMC (`Win + R` → `mmc`).
2. Меню **Файл → Додати або видалити оснастку** (або натиснути `Ctrl + M`).
3. У лівій частині вікна — список доступних оснасток. Обрати потрібну, натиснути **Додати**.
4. Якщо з'являється вибір — обрати **Локальний комп'ютер** (або інший комп'ютер для віддаленого адміністрування).
5. Натиснути **OK**.
6. Для збереження: **Файл → Зберегти як** → вказати ім'я файлу (формат `.msc`).

### 3. Важливі оснастки MMC (довідкова таблиця)

| Команда | Назва оснастки | Призначення |
|---------|---------------|-------------|
| `devmgmt.msc` | Диспетчер пристроїв | Перегляд та керування апаратними пристроями, драйверами |
| `diskmgmt.msc` | Керування дисками | Створення, форматування, зміна розмірів розділів дисків |
| `services.msc` | Служби | Перегляд та керування службами Windows (запуск, зупинка, тип запуску) |
| `eventvwr.msc` | Переглядач подій | Перегляд журналів подій Windows (Система, Безпека, Програма) |
| `secpol.msc` | Локальна політика безпеки | Налаштування політик паролів, блокування облікових записів, аудиту |
| `gpedit.msc` | Редактор локальної групової політики | Конфігурація комп'ютера та користувача через групові політики |
| `lusrmgr.msc` | Локальні користувачі та групи | Створення, видалення та керування обліковими записами та групами |
| `wf.msc` | Брандмауер Windows у режимі підвищеної безпеки | Налаштування правил вхідного та вихідного трафіку, IPsec |
| `compmgmt.msc` | Керування комп'ютером | Комплексна оснастка: об'єднує Диспетчер пристроїв, Управління дисками, Служби, Переглядач подій тощо |
| `certmgr.msc` | Сертифікати | Перегляд та керування цифровими сертифікатами поточного користувача |

### 4. Ключові поняття безпеки Windows

**Політика безпеки** — набір правил, що визначають, як система захищає ресурси: вимоги до паролів, обмеження входу, права користувачів, параметри аудиту.

**Аудит безпеки** — процес відстеження та реєстрації подій безпеки (входу в систему, доступу до файлів, змін політик). Результати записуються у журнал безпеки (Security Log).

**Ключові події безпеки:**

| Event ID | Подія | Опис |
|----------|-------|------|
| **4624** | Успішний вхід | Фіксує кожну успішну спробу входу в систему |
| **4625** | Невдалий вхід | Фіксує кожну невдалу спробу входу (неправильний пароль, заблокований обліковий запис) |

**Типи входу (Logon Type):**

| Тип | Опис |
|-----|------|
| Тип 2 | Інтерактивний — вхід через клавіатуру/мишу безпосередньо за комп'ютером |
| Тип 3 | Мережевий — доступ до спільних ресурсів через мережу |
| Тип 10 | Віддалений інтерактивний — вхід через Remote Desktop (RDP) |

## Практичні завдання

### Завдання 1. Відкрити MMC та додати оснастку «Локальна політика безпеки»

1. Натисніть `Win + R`, введіть `mmc`, натисніть **Enter**. Підтвердіть запит контролю облікових записів (UAC).
2. У порожній консолі MMC перейдіть до меню **Файл → Додати або видалити оснастку** (або `Ctrl + M`).
3. У списку доступних оснасток знайдіть **«Редактор об'єктів групової політики»** або іншу оснастку безпеки. Натисніть **Додати**.
4. Оберіть **«Локальний комп'ютер»**, натисніть **Готово**, потім **OK**.
5. Також відкрийте оснастку напряму: `Win + R` → `secpol.msc` → **Enter**.
6. Порівняйте: оснастка, додана вручну до MMC, та secpol.msc — це один і той самий інструмент, лише відкритий різними способами.
7. Зробіть знімок екрана консолі MMC з доданою оснасткою.

---

### Завдання 2. secpol.msc — Політики облікових записів: пароль та блокування

1. Натисніть `Win + R`, введіть `secpol.msc`, натисніть **Enter**.
2. У лівій панелі перейдіть: **Політики облікових записів → Політика паролів**.
3. Перегляньте та запишіть поточні значення кожного параметра:

| Параметр | Поточне значення | Рекомендоване значення |
|----------|-----------------|----------------------|
| Мінімальна довжина пароля | | 8 символів |
| Пароль має відповідати вимогам складності | | Увімкнено |
| Максимальний термін дії пароля | | 90 днів |
| Мінімальний термін дії пароля | | 1 день |
| Вести журнал паролів | | 5 паролів |

4. Двічі клацніть на **«Мінімальна довжина пароля»** — встановіть значення **8 символів**. Натисніть **OK**.
5. Увімкніть параметр **«Пароль має відповідати вимогам складності»** (якщо вимкнено).
6. Перейдіть до **Політики облікових записів → Політика блокування облікового запису**.
7. Перегляньте та запишіть:

| Параметр | Поточне значення |
|----------|-----------------|
| Порогове значення блокування облікового запису | |
| Тривалість блокування облікового запису | |
| Скидання лічильника блокування через | |

8. Встановіть **порогове значення блокування** = **5 невдалих спроб**. Windows автоматично запропонує значення для тривалості блокування та скидання лічильника (30 хвилин) — прийміть їх.
9. Зробіть знімок екрана з налаштованими параметрами.

> **Увага:** Не тестуйте блокування на єдиному обліковому записі адміністратора — це може призвести до втрати доступу до системи!

### Завдання 3. secpol.msc — Локальні політики: права користувачів та параметри безпеки

1. У вікні `secpol.msc` перейдіть: **Локальні політики → Призначення прав користувачів**.
2. Знайдіть та запишіть, яким користувачам/групам призначені такі права:

| Право | Призначено (користувачі/групи) |
|-------|-------------------------------|
| Дозволити локальний вхід | |
| Доступ до цього комп'ютера з мережі | |
| Завершення роботи системи | |
| Зміна системного часу | |

3. Перейдіть до **Локальні політики → Параметри безпеки**.
4. Знайдіть та запишіть значення параметрів:

| Параметр | Значення |
|----------|----------|
| Інтерактивний вхід: не відображати останнє ім'я користувача | |
| Облікові записи: стан облікового запису адміністратора | |
| Облікові записи: стан облікового запису гостя | |
| Інтерактивний вхід: текст повідомлення для користувачів під час входу | |

5. Зробіть знімок екрана розділу «Призначення прав користувачів».

### Завдання 4. gpedit.msc — Редактор групових політик

1. Натисніть `Win + R`, введіть `gpedit.msc`, натисніть **Enter**.
2. У лівій панелі перейдіть: **Конфігурація комп'ютера → Адміністративні шаблони**.
3. Дослідіть структуру. Перейдіть до **Конфігурація комп'ютера → Адміністративні шаблони → Система**.
4. Знайдіть та відкрийте політику **«Не відображати екран блокування»** (шлях: Конфігурація комп'ютера → Адміністративні шаблони → Панель керування → Персоналізація).
5. Зверніть увагу на три стани політики: **Не налаштовано**, **Увімкнено**, **Вимкнено**. Залиште значення **Не налаштовано**, натисніть **OK**.
6. Перейдіть до: **Конфігурація комп'ютера → Адміністративні шаблони → Компоненти Windows → Windows Update**.
7. Запишіть назви 5 будь-яких політик, що стосуються оновлення Windows.
8. Перейдіть до: **Конфігурація користувача → Адміністративні шаблони → Система**.
9. Знайдіть політику **«Заборонити доступ до засобів редагування реєстру»** — запишіть її поточний стан.
10. Зробіть знімок екрана дерева політик з розгорнутою гілкою «Адміністративні шаблони».

> **Увага:** Зміни в gpedit.msc мають високий пріоритет і перевизначають налаштування, зроблені через звичайний інтерфейс Windows. Будьте обережні з увімкненням обмежувальних політик!

### Завдання 5. Брандмауер Windows (wf.msc): перегляд та створення правила

1. Натисніть `Win + R`, введіть `wf.msc`, натисніть **Enter**.
2. На головній сторінці ознайомтеся зі станом брандмауера для кожного профілю:

| Профіль | Стан брандмауера | Вхідні підключення | Вихідні підключення |
|---------|-----------------|-------------------|-------------------|
| Профіль домену | | | |
| Приватний профіль | | | |
| Загальнодоступний профіль | | | |

3. У лівій панелі натисніть **«Правила для вхідних підключень»**.
4. Перегляньте список правил. Знайдіть та запишіть інформацію про 3 будь-які правила:

| Назва правила | Увімкнено | Дія (Дозволити/Блокувати) | Протокол | Порт |
|---------------|-----------|--------------------------|----------|------|
| | | | | |
| | | | | |
| | | | | |

5. **Створення правила для блокування порту:**
   - У правій панелі натисніть **«Створити правило»** (New Rule).
   - Тип правила: оберіть **«Порт»** → **Далі**.
   - Протокол: оберіть **TCP**. Вкажіть **конкретний локальний порт**: `23` (Telnet) → **Далі**.
   - Дія: оберіть **«Блокувати підключення»** → **Далі**.
   - Профілі: залиште всі три (Домен, Приватний, Загальнодоступний) → **Далі**.
   - Назва: введіть **«Блокування Telnet (порт 23) — ПР20»**. Опис: «Навчальне правило — блокування вхідного порту 23» → **Готово**.
6. Переконайтеся, що нове правило з'явилося у списку вхідних правил.
7. Зробіть знімок екрана з новоствореним правилом.

> **Після завершення роботи:** Видаліть створене правило (ПКМ → Видалити), щоб не залишати зайвих правил у системі.

### Завдання 6. Безпека Windows (Windows Defender): сканування та журнал

1. Відкрийте **Параметри** (`Win + I`) → **Оновлення та безпека** → **Безпека Windows** (або натисніть на значок щита у системному треї).
2. Перейдіть до розділу **«Захист від вірусів та загроз»**.
3. Запишіть:
   - Стан захисту в реальному часі (увімкнено/вимкнено).
   - Дату та час останнього сканування.
   - Кількість знайдених загроз (якщо є).
4. Натисніть **«Швидке сканування»** та дочекайтеся його завершення.
5. Після завершення запишіть результат: кількість перевірених файлів, знайдені загрози, час сканування.
6. Натисніть **«Журнал захисту»** (Protection history) — перегляньте останні записи.
7. Перейдіть до розділу **«Брандмауер та захист мережі»** — запишіть стан для кожного типу мережі (Мережа домену, Приватна мережа, Загальнодоступна мережа).
8. Зробіть знімок екрана результатів сканування.

### Завдання 7. eventvwr.msc — Журнал подій: безпека та події входу

1. Натисніть `Win + R`, введіть `eventvwr.msc`, натисніть **Enter**.
2. У лівій панелі перейдіть: **Журнали Windows → Безпека**.
3. Перегляньте список подій. Зверніть увагу на колонки: **Дата і час**, **Джерело**, **Код події (Event ID)**, **Категорія завдання**.
4. **Пошук подій входу в систему:**
   - У правій панелі натисніть **«Фільтр поточного журналу»**.
   - У полі **«Ідентифікатори подій»** (Event IDs) введіть: `4624, 4625`.
   - Натисніть **OK**.
5. Знайдіть подію з **Event ID 4624** (успішний вхід). Двічі клацніть на ній та запишіть:
   - Дата та час входу.
   - Ім'я облікового запису (Account Name).
   - Тип входу (Logon Type): 2 = інтерактивний, 3 = мережевий, 10 = RDP.
6. Знайдіть подію з **Event ID 4625** (невдалий вхід), якщо є. Запишіть:
   - Дата та час спроби.
   - Причина невдачі (Failure Reason / SubStatus).
7. Зніміть фільтр. Перейдіть до **Журнали Windows → Система** — знайдіть будь-яку подію рівня «Помилка» або «Попередження». Запишіть Код події та Опис.
8. Зробіть знімок екрана відфільтрованих подій безпеки.

> **Примітка:** Якщо подій з Event ID 4624/4625 немає — можливо, аудит входу не увімкнено. Це буде перевірено у Завданні 10 за допомогою `auditpol`.

### Завдання 8. compmgmt.msc — Керування комп'ютером: огляд основних розділів

1. Натисніть `Win + R`, введіть `compmgmt.msc`, натисніть **Enter**.
2. Ознайомтеся зі структурою — це комплексна консоль, що об'єднує кілька оснасток:

| Розділ | Вміст |
|--------|-------|
| **Службові програми** | Планувальник завдань, Переглядач подій, Спільні папки, Локальні користувачі та групи, Продуктивність, Диспетчер пристроїв |
| **Запам'ятовуючі пристрої** | Керування дисками, зйомні носії |
| **Служби та програми** | Служби Windows |

3. Розгорніть **Службові програми → Локальні користувачі та групи → Користувачі**. Запишіть список усіх облікових записів.
4. Перейдіть до **Локальні користувачі та групи → Групи**. Знайдіть групу **«Адміністратори»** — двічі клацніть та запишіть, які облікові записи є її членами.
5. Розгорніть **Запам'ятовуючі пристрої → Керування дисками** — запишіть кількість дисків та розділів.
6. Розгорніть **Служби та програми → Служби** — знайдіть службу **«Брандмауер Windows»** (Windows Defender Firewall), запишіть її стан та тип запуску.
7. Зробіть знімок екрана вікна «Керування комп'ютером» із розгорнутими розділами.

### Завдання 9. Створення власної консолі MMC

1. Натисніть `Win + R`, введіть `mmc`, натисніть **Enter**.
2. Перейдіть: **Файл → Додати або видалити оснастку** (`Ctrl + M`).
3. Додайте такі оснастки:
   - **Переглядач подій** (Event Viewer)
   - **Служби** (Services)
   - **Сертифікати** (Certificates) — обрати «Мій обліковий запис користувача»
   - **Монітор продуктивності** (Performance Monitor)
4. Для кожної оснастки: виберіть → натисніть **Додати** → оберіть «Локальний комп'ютер» (якщо пропонується) → **Готово**.
5. Натисніть **OK**. Переконайтеся, що у лівій панелі з'явилися всі 4 оснастки.
6. Збережіть консоль: **Файл → Зберегти як** → на робочому столі з ім'ям `МояКонсоль_ПР20.msc`.
7. Закрийте MMC. Відкрийте збережений файл `МояКонсоль_ПР20.msc` подвійним кліком — переконайтеся, що всі оснастки на місці.
8. Зробіть знімок екрана збереженої консолі.

### Завдання 10. CMD: auditpol — перегляд політики аудиту

1. Відкрийте **Командний рядок (CMD)** від імені адміністратора (ПКМ → Запуск від імені адміністратора).
2. Введіть команду:
   ```
   auditpol /get /category:*
   ```
3. Дочекайтеся виводу результатів. Команда покаже всі категорії та підкатегорії аудиту з їх поточним станом: **Успіх**, **Невдача**, **Успіх і невдача** або **Немає аудиту**.
4. Знайдіть та запишіть стан аудиту для таких підкатегорій:

| Категорія / Підкатегорія | Поточний стан |
|--------------------------|---------------|
| Вхід/Вихід → Вхід у систему (Logon) | |
| Вхід/Вихід → Вихід із системи (Logoff) | |
| Управління обліковими записами → Управління обліковим записом користувача | |
| Зміна політики → Зміна політики аудиту | |
| Система → Зміна стану безпеки | |

5. Для перегляду лише списку категорій аудиту виконайте:
   ```
   auditpol /list /category
   ```
6. Запишіть загальну кількість категорій аудиту.
7. Збережіть повний результат у файл:
   ```
   auditpol /get /category:* > C:\Users\%USERNAME%\Desktop\auditpol_result.txt
   ```
8. Зробіть знімок екрана результатів команди `auditpol`.

> **Примітка:** Якщо для підкатегорії «Вхід у систему» стоїть значення «Немає аудиту» — це пояснює, чому у Завданні 7 могли бути відсутні події з Event ID 4624/4625.
