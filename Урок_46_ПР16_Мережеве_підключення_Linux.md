# КОНСПЕКТ УРОКУ №46

## Тема: Практична робота №16 — Налаштування мережевого підключення Linux

---

## Мета роботи

Навчитися переглядати та аналізувати мережеві налаштування в Ubuntu Linux, використовувати команди для діагностики мережевого підключення, працювати з NetworkManager через командний рядок та графічний інтерфейс, налаштовувати статичну IP-адресу та повертати автоматичне отримання адреси (DHCP).

---

## Теоретичний вступ

### 1. Мережа в Linux

Операційна система Linux має потужну підсистему для роботи з мережею. На відміну від Windows, де більшість мережевих налаштувань виконується через графічний інтерфейс, у Linux традиційно використовуються команди терміналу. Це дає більший контроль, можливість автоматизації через скрипти та є обов'язковим при роботі з серверами без графічного оточення.

В Ubuntu для управління мережевими з'єднаннями використовується **NetworkManager** — служба, яка автоматично налаштовує мережеві інтерфейси, керує підключеннями та забезпечує зручний спосіб зміни мережевих параметрів як через GUI, так і через термінал.

---

### 2. NetworkManager

**NetworkManager** — це фонова служба (демон), яка автоматично керує мережевими з'єднаннями в Linux. Вона стартує разом із системою та виконує такі функції:

- автоматичне виявлення та підключення до дротових і бездротових мереж;
- отримання IP-адреси через DHCP;
- збереження профілів мережевих підключень;
- перемикання між мережами (наприклад, при переході з Wi-Fi на Ethernet);
- надання інтерфейсів для управління: графічний аплет у панелі задач та команда `nmcli` у терміналі.

NetworkManager зберігає конфігурації підключень як **профілі з'єднань** у каталозі `/etc/NetworkManager/system-connections/`. Кожен профіль містить інформацію про тип з'єднання, IP-адресу, шлюз, DNS-сервери та інші параметри.

**Основні компоненти:**

| Компонент | Призначення |
|-----------|-------------|
| `NetworkManager` (демон) | Фонова служба, яка керує мережею |
| `nmcli` | Командний рядок для управління NetworkManager |
| `nmtui` | Текстовий інтерфейс (меню) для налаштування мережі |
| Аплет у панелі задач | Графічний інтерфейс для швидкого доступу до мережевих налаштувань |

---

### 3. Мережеві інтерфейси Linux

**Мережевий інтерфейс** — це програмне представлення фізичного або віртуального мережевого адаптера в системі. Кожен інтерфейс має унікальне ім'я.

| Інтерфейс | Опис |
|------------|------|
| `eth0` | Класична назва першого Ethernet-адаптера (старий стиль іменування) |
| `enp0s3` | Сучасна назва Ethernet-адаптера (Predictable Network Interface Names): **en** = Ethernet, **p0** = шина PCI 0, **s3** = слот 3. Саме така назва використовується в Ubuntu у VirtualBox |
| `wlan0` або `wlp2s0` | Бездротовий (Wi-Fi) адаптер |
| `lo` | Loopback-інтерфейс — віртуальний інтерфейс для зворотного зв'язку. IP-адреса завжди `127.0.0.1`. Використовується для зв'язку програм із самим комп'ютером |

> **У VirtualBox** при стандартному налаштуванні (тип підключення NAT) ваш мережевий адаптер називатиметься `enp0s3`, а його IP-адреса зазвичай буде `10.0.2.15/24` зі шлюзом `10.0.2.2`.

---

### 4. Типи мережевого підключення у VirtualBox

VirtualBox надає кілька режимів мережевого підключення для віртуальних машин:

| Тип | Доступ до Інтернету | Зв'язок хост-ВМ | ВМ видима в мережі | Опис |
|-----|---------------------|------------------|--------------------|------|
| **NAT** | Так | Обмежений | Ні | Стандартний режим. ВМ виходить в Інтернет через віртуальний маршрутизатор VirtualBox. Зовнішні пристрої не бачать ВМ |
| **Мережевий міст (Bridged)** | Так | Так | Так | ВМ отримує IP-адресу від фізичного роутера і виглядає як окремий комп'ютер у мережі |
| **Внутрішня мережа (Host-Only)** | Ні | Так | Ні | Ізольована мережа тільки між хостом та ВМ. Немає доступу до Інтернету |
| **NAT Network** | Так | Обмежений | Ні | Як NAT, але декілька ВМ можуть бачити одна одну |

> **Для цієї практичної роботи** використовуємо режим **NAT** (встановлений за замовчуванням). Він дозволяє ВМ виходити в Інтернет, що потрібно для перевірки з'єднання командами `ping` та `curl`.

---

### 5. Основні мережеві команди Linux

| Команда | Призначення |
|---------|-------------|
| `ip addr show` (або `ip a`) | Показати інформацію про всі мережеві інтерфейси: IP-адреса, маска, MAC-адреса, стан |
| `ip route show` (або `ip r`) | Показати таблицю маршрутизації та шлюз за замовчуванням |
| `ping` | Перевірити з'єднання з іншим хостом (надсилає ICMP-пакети) |
| `nmcli device status` | Показати стан мережевих пристроїв через NetworkManager |
| `nmcli connection show` | Показати список усіх профілів підключень |
| `cat /etc/resolv.conf` | Переглянути налаштовані DNS-сервери |
| `hostname` | Показати ім'я комп'ютера |
| `hostnamectl` | Детальна інформація про ім'я хоста та ОС (зберігається після перезавантаження) |
| `ss -tulnp` | Показати відкриті мережеві порти та програми, що їх слухають |
| `curl` | Завантажити дані з URL-адреси (перевірити доступ до веб-ресурсу) |

---

### 6. Порівняння IP-адресації: DHCP та статична IP

| Параметр | DHCP (автоматично) | Статична IP (вручну) |
|----------|--------------------|----------------------|
| Спосіб отримання IP | Автоматично від DHCP-сервера | Адміністратор вводить вручну |
| Коли використовувати | Домашні мережі, робочі станції | Сервери, мережеві принтери, маршрутизатори |
| Переваги | Простота, немає конфліктів IP | Передбачуваність, стабільність адреси |
| Недоліки | Адреса може змінитися | Потрібно знати параметри мережі, ризик конфлікту |

## Практичні завдання

### Завдання 1. Перегляд мережевих інтерфейсів

1. Запустіть віртуальну машину Ubuntu у VirtualBox.
2. Відкрийте термінал: натисніть `Ctrl + Alt + T` або знайдіть «Термінал» у меню програм.
3. Введіть команду:
   ```
   ip addr show
   ```
   або скорочений варіант:
   ```
   ip a
   ```
4. Знайдіть у виводі два інтерфейси: `lo` (loopback) та `enp0s3` (або інша назва Ethernet-адаптера).
5. Зробіть знімок екрана результату.

**Приклад виводу:**

```
1: lo: <LOOPBACK,UP,LOWER_UP> mtu 65536
    link/loopback 00:00:00:00:00:00
    inet 127.0.0.1/8 scope host lo
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500
    link/ether 08:00:27:a1:b2:c3
    inet 10.0.2.15/24 brd 10.0.2.255 scope global dynamic enp0s3
```

---

### Завдання 2. Запис параметрів мережевого інтерфейсу

На основі результату команди `ip addr show` заповніть таблицю для інтерфейсу `enp0s3`:

| Параметр | Де шукати у виводі | Ваше значення |
|----------|---------------------|---------------|
| Назва інтерфейсу | Перше слово після номера (наприклад, `enp0s3`) | |
| Стан інтерфейсу | `UP` або `DOWN` у кутових дужках `< >` | |
| MAC-адреса | Після `link/ether` (наприклад, `08:00:27:a1:b2:c3`) | |
| IP-адреса | Після `inet` (наприклад, `10.0.2.15`) | |
| Маска підмережі (у CIDR) | Число після `/` (наприклад, `/24` = `255.255.255.0`) | |
| Broadcast-адреса | Після `brd` (наприклад, `10.0.2.255`) | |
| Тип отримання IP | `dynamic` = DHCP, відсутність слова = статична | |

Заповніть аналогічну таблицю для інтерфейсу `lo`.

---

### Завдання 3. Перегляд маршрутів та шлюзу за замовчуванням

1. Введіть команду:
   ```
   ip route show
   ```
   або скорочено:
   ```
   ip r
   ```
2. Знайдіть рядок, що починається з `default via` — це шлюз за замовчуванням (default gateway).
3. Запишіть:

| Параметр | Ваше значення |
|----------|---------------|
| Шлюз за замовчуванням (default gateway) | |
| Мережа підмережі | |
| Інтерфейс, через який йде маршрут | |

**Приклад виводу:**

```
default via 10.0.2.2 dev enp0s3 proto dhcp metric 100
10.0.2.0/24 dev enp0s3 proto kernel scope link src 10.0.2.15 metric 100
```

> **Пояснення:** `default via 10.0.2.2` означає, що весь трафік, який не належить до локальної мережі, надсилається на шлюз `10.0.2.2`. У VirtualBox з NAT це віртуальний маршрутизатор VirtualBox.

---

### Завдання 4. Перегляд DNS-серверів

1. Введіть команду:
   ```
   cat /etc/resolv.conf
   ```
2. Знайдіть рядки, що починаються з `nameserver` — це DNS-сервери, які використовує система.
3. Запишіть усі DNS-сервери:

| DNS-сервер | IP-адреса |
|------------|-----------|
| Первинний (перший рядок nameserver) | |
| Вторинний (другий рядок, якщо є) | |

> **Примітка:** В Ubuntu з NetworkManager файл `/etc/resolv.conf` може бути символічним посиланням і керуватися автоматично службою `systemd-resolved`. Це нормально. Типове значення для VirtualBox NAT: `nameserver 127.0.0.53` (локальний DNS-проксі systemd-resolved).

Щоб побачити реальні DNS-сервери, можна також виконати:
```
resolvectl status
```

---

### Завдання 5. Перевірка підключення до Інтернету (ping за IP)

1. Введіть команду:
   ```
   ping -c 4 8.8.8.8
   ```
   Ця команда надсилає 4 ICMP-пакети на DNS-сервер Google (`8.8.8.8`).
2. Дочекайтеся завершення та запишіть результати:

| Параметр | Ваше значення |
|----------|---------------|
| Кількість відправлених пакетів | |
| Кількість отриманих пакетів | |
| Відсоток втрачених пакетів | |
| Середній час відповіді (avg) | |

**Приклад виводу:**

```
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data.
64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=12.3 ms
64 bytes from 8.8.8.8: icmp_seq=2 ttl=117 time=11.8 ms
64 bytes from 8.8.8.8: icmp_seq=3 ttl=117 time=12.1 ms
64 bytes from 8.8.8.8: icmp_seq=4 ttl=117 time=11.9 ms

--- 8.8.8.8 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3004ms
rtt min/avg/max/mdev = 11.8/12.025/12.3/0.189 ms
```

> **Зверніть увагу:** На відміну від Windows, де `ping` за замовчуванням надсилає лише 4 пакети, у Linux `ping` працює безперервно. Параметр `-c 4` обмежує кількість пакетів до чотирьох. Для зупинки безперервного пінгу натисніть `Ctrl + C`.

### Завдання 6. Перевірка роботи DNS (ping за доменним ім'ям)

1. Введіть команду:
   ```
   ping -c 4 google.com
   ```
2. Якщо команда працює — DNS налаштовано правильно (система змогла перетворити доменне ім'я `google.com` на IP-адресу).
3. Запишіть:

| Параметр | Ваше значення |
|----------|---------------|
| IP-адреса, яку DNS повернув для google.com | |
| Чи успішний ping? (так/ні) | |
| Середній час відповіді | |

> **Якщо `ping 8.8.8.8` працює, а `ping google.com` — ні**, це означає, що є проблема з DNS (система не може перетворити доменне ім'я на IP-адресу). Перевірте файл `/etc/resolv.conf`.

---

### Завдання 7. Стан мережевих пристроїв через NetworkManager

1. Введіть команду:
   ```
   nmcli device status
   ```
2. Запишіть результат у таблицю:

| DEVICE (Пристрій) | TYPE (Тип) | STATE (Стан) | CONNECTION (Підключення) |
|---------------------|------------|--------------|--------------------------|
| | | | |
| | | | |

**Приклад виводу:**

```
DEVICE   TYPE      STATE      CONNECTION
enp0s3   ethernet  connected  Wired connection 1
lo       loopback  unmanaged  --
```

3. Для детальної інформації про пристрій введіть:
   ```
   nmcli device show enp0s3
   ```
4. Знайдіть і запишіть: IP-адресу, шлюз, DNS-сервер, MAC-адресу.

---

### Завдання 8. Активні підключення через NetworkManager

1. Введіть команду:
   ```
   nmcli connection show
   ```
2. Запишіть інформацію про активні підключення:

| NAME (Ім'я) | UUID | TYPE (Тип) | DEVICE (Пристрій) |
|-------------|------|------------|---------------------|
| | | | |

3. Для детальної інформації про конкретне підключення введіть:
   ```
   nmcli connection show "Wired connection 1"
   ```
   (Замініть назву на ту, що відображається у вашій системі.)

4. Знайдіть у виводі рядки:
   - `ipv4.method` — спосіб отримання IP (`auto` = DHCP, `manual` = статична);
   - `ipv4.addresses` — IP-адреса;
   - `ipv4.gateway` — шлюз;
   - `ipv4.dns` — DNS-сервери.

---

### Завдання 9. Перегляд мережевих налаштувань через графічний інтерфейс

1. Натисніть на іконку мережі у правому верхньому куті робочого столу Ubuntu.
2. Оберіть **Параметри з'єднання** (або **Wired Settings** / **Налаштування дротового з'єднання**).
3. Або відкрийте: **Параметри (Settings) → Мережа (Network)**.
4. У розділі **Дротове з'єднання (Wired)** натисніть на значок шестерні (налаштування).
5. Перегляньте вкладки:
   - **Подробиці (Details)** — IP-адреса, маска, шлюз, DNS, MAC-адреса, швидкість з'єднання.
   - **IPv4** — метод отримання адреси (Автоматично/DHCP або Вручну/Manual).
   - **IPv6** — аналогічно для IPv6.
6. Зробіть знімок екрана вкладки з деталями підключення.
7. Порівняйте інформацію з тим, що показали команди `ip addr show` та `nmcli device show`. Запишіть, чи збігаються дані.

---

### Завдання 10. Налаштування статичної IP-адреси та повернення DHCP

> **УВАГА:** Виконуйте це завдання уважно. Після встановлення статичної IP-адреси ви обов'язково повинні повернути DHCP наприкінці!

#### Частина А. Налаштування статичної IP через nmcli

1. Спочатку запишіть поточні мережеві параметри (вони знадобляться для відновлення):
   ```
   ip addr show enp0s3
   ip route show
   ```
2. Дізнайтеся назву вашого підключення:
   ```
   nmcli connection show
   ```
   Запам'ятайте або запишіть значення зі стовпця NAME (наприклад, `"Wired connection 1"`).

3. Встановіть статичну IP-адресу (замініть назву підключення на свою):
   ```
   sudo nmcli connection modify "Wired connection 1" ipv4.method manual ipv4.addresses 10.0.2.100/24 ipv4.gateway 10.0.2.2 ipv4.dns "8.8.8.8 8.8.4.4"
   ```
4. Застосуйте зміни:
   ```
   sudo nmcli connection up "Wired connection 1"
   ```
5. Перевірте нову IP-адресу:
   ```
   ip addr show enp0s3
   ```
6. Переконайтеся, що Інтернет працює:
   ```
   ping -c 2 8.8.8.8
   ```
7. Запишіть нову IP-адресу та підтвердіть, що вона змінилася на `10.0.2.100`.

#### Частина Б. Повернення DHCP

8. Поверніть автоматичне отримання адреси:
   ```
   sudo nmcli connection modify "Wired connection 1" ipv4.method auto ipv4.addresses "" ipv4.gateway "" ipv4.dns ""
   ```
9. Застосуйте зміни:
   ```
   sudo nmcli connection up "Wired connection 1"
   ```
10. Перевірте, що IP-адреса знову отримана автоматично:
    ```
    ip addr show enp0s3
    ```
11. Запишіть IP-адресу після повернення до DHCP. Вона має повернутися до попереднього значення (наприклад, `10.0.2.15`).

#### Альтернатива: через GUI

Ту саму операцію можна виконати через графічний інтерфейс:
1. **Параметри → Мережа → Дротове з'єднання** (значок шестерні) → вкладка **IPv4**.
2. Змінити метод з «Автоматично (DHCP)» на «Вручну (Manual)».
3. Ввести: Адреса: `10.0.2.100`, Маска: `255.255.255.0`, Шлюз: `10.0.2.2`, DNS: `8.8.8.8`.
4. Натиснути **Застосувати (Apply)**.
5. Вимкнути та увімкнути з'єднання (тумблер у налаштуваннях мережі).
6. Для повернення: змінити метод назад на «Автоматично (DHCP)», натиснути **Застосувати**, перепідключити.

## Додаткові команди

### hostname та hostnamectl

Команда `hostname` показує ім'я комп'ютера у мережі:

```
hostname
```

Команда `hostnamectl` виводить детальну інформацію та дозволяє змінити ім'я хоста назавжди (зміна зберігається після перезавантаження):

```
hostnamectl
```

**Приклад виводу:**

```

       Icon name: computer-vm
         Chassis: vm
      Machine ID: a1b2c3d4e5f6...
         Boot ID: f6e5d4c3b2a1...
  Virtualization: oracle
Operating System: Ubuntu 24.04.1 LTS
          Kernel: Linux 6.8.0-41-generic
    Architecture: x86-64

### ss — перегляд відкритих портів

Команда `ss` (Socket Statistics) замінює застарілу команду `netstat` і показує інформацію про мережеві з'єднання та відкриті порти:

```
ss -tulnp
```

**Розшифровка параметрів:**

| Параметр | Значення |
|----------|----------|
| `-t` | Показати TCP-з'єднання |
| `-u` | Показати UDP-з'єднання |
| `-l` | Тільки з'єднання, що прослуховуються (listening) |
| `-n` | Показувати номери портів замість назв служб |
| `-p` | Показати процес, який використовує порт |

> **Примітка:** Для параметра `-p` може знадобитися `sudo`, щоб побачити процеси інших користувачів:
> ```
> sudo ss -tulnp
> ```

### curl — перевірка веб-доступу

Команда `curl` дозволяє завантажити дані з веб-сервера прямо в термінал:

```
curl -I https://google.com
```

Параметр `-I` показує лише заголовки HTTP-відповіді (без тіла сторінки). Якщо у відповіді є рядок `HTTP/... 200` або `HTTP/... 301` — з'єднання з Інтернетом працює.

Простий спосіб перевірити зовнішню IP-адресу вашої машини:
```
curl ifconfig.me
```

## Порівняння мережевих команд: Windows vs Linux

| Дія | Windows | Linux |
|-----|---------|-------|
| Переглянути IP-адресу та мережеві інтерфейси | `ipconfig` | `ip addr show` (або `ip a`) |
| Детальна інформація про всі адаптери | `ipconfig /all` | `ip addr show` + `nmcli device show` |
| Переглянути таблицю маршрутизації | `route print` | `ip route show` (або `ip r`) |
| Переглянути шлюз за замовчуванням | `ipconfig` (Default Gateway) | `ip route show` (рядок `default via`) |
| Оновити IP-адресу (DHCP) | `ipconfig /release` та `ipconfig /renew` | `sudo dhclient -r` та `sudo dhclient` |
| Очистити DNS-кеш | `ipconfig /flushdns` | `sudo resolvectl flush-caches` |
| Перевірити з'єднання (ping) | `ping google.com` (4 пакети) | `ping -c 4 google.com` (безперервний без `-c`) |
| Трасування маршруту | `tracert google.com` | `traceroute google.com` |
| DNS-запит | `nslookup google.com` | `nslookup google.com` або `dig google.com` |
| Відкриті порти та з'єднання | `netstat -ano` | `ss -tulnp` |
| Ім'я комп'ютера | `hostname` | `hostname` або `hostnamectl` |
| Стан мережевих адаптерів | Центр керування мережами | `nmcli device status` |
| Мережеві підключення | `ncpa.cpl` (Мережеві підключення) | `nmcli connection show` |

> **Зверніть увагу на ключові відмінності:**
> - У Windows `ping` надсилає 4 пакети, у Linux — працює безперервно (зупинка `Ctrl + C`).
> - У Windows трасування — `tracert`, у Linux — `traceroute`.
> - У Linux замість `ipconfig` використовується `ip addr` (команда `ifconfig` застаріла).
> - У Linux замість `netstat` рекомендовано використовувати `ss`.
