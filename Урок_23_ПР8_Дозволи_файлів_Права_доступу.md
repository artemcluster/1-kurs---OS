## Мета роботи

Навчитися переглядати, налаштовувати та змінювати права доступу до файлів і папок в операційних системах Windows та Linux. Зрозуміти принцип найменших привілеїв та його застосування для забезпечення безпеки даних.

## Частина 1. Теоретичний вступ

### 1.1. Навіщо потрібні права доступу?

У будь-якій операційній системі може працювати кілька користувачів (навіть на одному комп'ютері). Кожен з них створює свої файли, зберігає особисті дані, встановлює програми. Без системи прав доступу будь-хто міг би:

- **читати** чужі документи (листи, паролі, фотографії);
- **змінювати або видаляти** важливі файли (в тому числі системні);
- **запускати** небезпечні програми від імені адміністратора.

Права доступу вирішують три ключові завдання:

| Завдання | Пояснення |
|---|---|
| **Безпека** | Захист від несанкціонованого доступу та шкідливого ПЗ |
| **Конфіденційність** | Лише уповноважені користувачі бачать певні дані |
| **Захист від помилок** | Користувач не може випадково видалити системний файл |

### 1.2. Принцип найменших привілеїв (Principle of Least Privilege)

**Принцип найменших привілеїв** — це правило інформаційної безпеки, за яким кожен користувач, програма або процес повинен мати лише той мінімальний рівень доступу, який необхідний для виконання його завдань, і нічого більше.

**Приклад з реального життя.** Уявіть, що ви наймаєте садівника. Ви даєте йому ключ від подвір'я, але не від спальні чи домашнього сейфа. Він отримує доступ лише до того, що потрібно для роботи.

**Приклад в ОС.** Обліковий запис, створений лише для резервного копіювання, не повинен мати права встановлювати програми. Він отримує дозвіл лише на запуск програм для бекапу.

**Чому це важливо:**
- Зменшує площу атаки — зловмиснику складніше отримати доступ до всієї системи.
- Обмежує поширення вірусів — якщо шкідлива програма потрапила на комп'ютер через обліковий запис із мінімальними правами, вона не зможе пошкодити системні файли.
- Підвищує стабільність — програми з обмеженими правами менш здатні порушити роботу ОС.

## Частина 2. Права доступу в Windows (NTFS Permissions)

### 2.1. Файлова система NTFS

**NTFS** (New Technology File System) — файлова система за замовчуванням у Windows 10/11. Саме вона забезпечує механізм дозволів на рівні окремих файлів і папок.

> **Важливо:** Дозволи NTFS працюють лише на дисках, відформатованих у NTFS. На флешках із FAT32 або exFAT прав доступу немає.

### 2.2. Базові дозволи NTFS

Windows надає шість базових (стандартних) рівнів дозволів:

| Дозвіл | Позначення | Що дозволяє |
|---|---|---|
| **Read** (Читання) | R | Переглядати вміст файлів і папок, бачити атрибути |
| **Write** (Запис) | W | Створювати нові файли, редагувати існуючі (але не читати і не видаляти) |
| **Read & Execute** (Читання і виконання) | RX | Читання + запуск програм і скриптів |
| **List Folder Contents** (Вміст папки) | — | Перегляд списку файлів у папці (тільки для папок) |
| **Modify** (Змінення) | M | Читання + запис + виконання + видалення (але не зміна дозволів) |
| **Full Control** (Повний контроль) | F | Усі права, включаючи зміну дозволів та отримання права власності |

**Ієрархія дозволів** (від найменшого до найбільшого):

```
Read → Read & Execute → Write → Modify → Full Control
```

**Ключова різниця:** Modify дозволяє майже все, крім зміни самих дозволів та зміни власника. Full Control — це Modify + право змінювати дозволи та ставати власником файлу.

### 2.3. Користувачі та групи в Windows

Windows використовує вбудовані групи для організації доступу:

| Група | Опис |
|---|---|
| **Administrators** | Повний контроль над системою. Можуть змінювати будь-які налаштування |
| **Users** | Звичайні користувачі. Можуть працювати зі своїми файлами, але не можуть змінювати системні |
| **Everyone** | Усі користувачі, включаючи гостей |
| **SYSTEM** | Спеціальний обліковий запис самої ОС для внутрішніх процесів |

### 2.4. Вкладка "Безпека" у властивостях файлу/папки

Щоб переглянути або змінити дозволи:

1. Клацніть правою кнопкою миші на файлі або папці.
2. Оберіть **"Властивості"** (Properties).
3. Перейдіть на вкладку **"Безпека"** (Security).
4. У верхній частині — список користувачів і груп.
5. У нижній частині — дозволи для обраного користувача/групи.
6. Натисніть **"Змінити"** (Edit), щоб модифікувати дозволи.
7. Натисніть **"Додати"** (Add), щоб додати нового користувача або групу.

### 2.5. Спадкування дозволів (Inheritance)

За замовчуванням файли та вкладені папки **успадковують** дозволи від батьківської папки. Це означає:

- Якщо папка `C:\Проєкти` має дозвіл Read для групи Users, то всі файли всередині неї також автоматично отримають цей дозвіл.
- Успадковані дозволи позначені сірим кольором (їх не можна змінити напряму).

**Як вимкнути спадкування:**

1. Властивості файлу/папки → Безпека → **Додатково** (Advanced).
2. Натисніть **"Вимкнути успадкування"** (Disable inheritance).
3. Оберіть:
   - **Перетворити** успадковані дозволи на явні (зберегти поточні дозволи, але зробити їх незалежними).
   - **Видалити** всі успадковані дозволи (почати "з чистого аркуша").

### 2.6. Команда icacls у командному рядку (CMD)

**icacls** — потужна утиліта командного рядка для перегляду та зміни дозволів NTFS.

**Перегляд поточних дозволів:**
```cmd
icacls "C:\Users\Student\Documents\test.txt"
```

**Коди дозволів icacls:**

| Код | Значення |
|---|---|
| F | Full Control (повний контроль) |
| M | Modify (змінення) |
| RX | Read & Execute (читання і виконання) |
| R | Read (читання) |
| W | Write (запис) |
| D | Delete (видалення) |
| N | No access (немає доступу) |

**Надання дозволів:**
```cmd
icacls "C:\TestFolder" /grant Users:(R)
```

**Заборона дозволів:**
```cmd
icacls "C:\TestFolder" /deny Users:(W)
```

**Видалення дозволів для користувача:**
```cmd
icacls "C:\TestFolder" /remove Users
```

**Вимкнення спадкування:**
```cmd
icacls "C:\TestFolder" /inheritance:d
```

**Прапори спадкування в icacls:**

| Прапор | Значення |
|---|---|
| (I) | Дозвіл успадковано від батьківського каталогу |
| (OI) | Об'єкти в каталозі успадковують дозвіл |
| (CI) | Підкаталоги успадковують дозвіл |
| (IO) | Лише успадкування (не застосовується до самого об'єкта) |

## Частина 3. Права доступу в Linux

### 3.1. Система дозволів rwx

У Linux кожен файл і каталог має набір дозволів, що визначають, хто і що може з ним робити. Система побудована на трьох типах дозволів:

| Дозвіл | Літера | Число | Для файлу | Для каталогу |
|---|---|---|---|---|
| **Читання** | r | 4 | Перегляд вмісту файлу | Перегляд списку файлів у каталозі |
| **Запис** | w | 2 | Редагування файлу | Створення/видалення файлів у каталозі |
| **Виконання** | x | 1 | Запуск як програми | Вхід у каталог (команда cd) |

### 3.2. Три категорії користувачів

Дозволи задаються окремо для трьох категорій:

| Категорія | Позначення | Опис |
|---|---|---|
| **Власник** (Owner) | u (user) | Користувач, який створив файл |
| **Група** (Group) | g (group) | Група, до якої належить файл |
| **Інші** (Others) | o (others) | Усі решта користувачів системи |

### 3.3. Як читати вивід ls -la

Команда `ls -la` виводить детальну інформацію про файли:

```
-rwxr-xr-- 1 student students 2048 бер 10 14:30 script.sh
```

Розбір по частинах:

```
-    rwx    r-x    r--    1  student  students  2048  бер 10 14:30  script.sh
|    |      |      |      |    |        |        |       |            |
тип  Owner  Group  Others кіл. власник  група  розмір   дата        ім'я
     (u)    (g)    (o)    пос.                  (байт)
```

- **Перший символ** — тип: `-` (звичайний файл), `d` (каталог), `l` (символічне посилання).
- **Наступні 9 символів** — три трійки дозволів (u, g, o).
- **student** — ім'я власника.
- **students** — ім'я групи.

**Приклади читання дозволів:**

| Рядок дозволів | Власник | Група | Інші |
|---|---|---|---|
| `rwxrwxrwx` | читання, запис, виконання | читання, запис, виконання | читання, запис, виконання |
| `rwxr-xr-x` | читання, запис, виконання | читання, виконання | читання, виконання |
| `rw-r--r--` | читання, запис | тільки читання | тільки читання |
| `rwx------` | читання, запис, виконання | нічого | нічого |

### 3.4. Числова (вісімкова) нотація chmod

Кожен дозвіл має числове значення. Для кожної категорії значення додаються:

```
r = 4,  w = 2,  x = 1

rwx = 4 + 2 + 1 = 7
rw- = 4 + 2 + 0 = 6
r-x = 4 + 0 + 1 = 5
r-- = 4 + 0 + 0 = 4
--- = 0 + 0 + 0 = 0
```

**Три цифри:** перша — власник, друга — група, третя — інші.

**Найпоширеніші комбінації:**

| Числове значення | Літерний запис | Типове використання |
|---|---|---|
| **755** | rwxr-xr-x | Каталоги, скрипти (власник — все; інші — читання і виконання) |
| **644** | rw-r--r-- | Звичайні файли (власник — читання/запис; інші — лише читання) |
| **700** | rwx------ | Приватні каталоги (доступ лише для власника) |
| **600** | rw------- | Приватні файли (читання/запис лише для власника) |
| **777** | rwxrwxrwx | Повний доступ для всіх (НЕБЕЗПЕЧНО! Використовувати лише для тестування) |
| **400** | r-------- | Лише читання для власника (конфіденційні файли, наприклад, SSH-ключі) |

### 3.5. Символьна нотація chmod

Замість чисел можна використовувати літери та оператори:

**Оператори:**
- `+` — додати дозвіл
- `-` — забрати дозвіл
- `=` — встановити точно (скидає попередні)

**Приклади:**

```bash
chmod u+x script.sh        # Додати виконання для власника
chmod g-w file.txt          # Забрати запис у групи
chmod o=r file.txt          # Встановити для інших лише читання
chmod a+r file.txt          # Додати читання для всіх (a = all)
chmod u+rwx,g+rx,o+r file   # Встановити 754
chmod +x script.sh          # Додати виконання для всіх категорій
```

### 3.6. Основні команди Linux для роботи з дозволами

**chmod** — зміна дозволів:
```bash
chmod 755 script.sh              # Числова нотація
chmod u+x script.sh              # Символьна нотація
chmod -R 644 /home/student/docs  # Рекурсивно для всіх файлів у каталозі
```

**chown** — зміна власника (потребує прав root або sudo):
```bash
chown student file.txt               # Змінити власника
chown student:students file.txt      # Змінити власника і групу одночасно
chown -R student:students /home/dir  # Рекурсивно
```

**chgrp** — зміна групи:
```bash
chgrp students file.txt              # Змінити групу файлу
chgrp -R students /home/dir          # Рекурсивно для каталогу
```

**ls -la** — перегляд дозволів:
```bash
ls -la                    # Показати всі файли з дозволами в поточному каталозі
ls -la /etc/passwd        # Переглянути дозволи конкретного файлу
```

## Частина 4. Порівняння Windows та Linux

| Критерій | Windows (NTFS) | Linux (ext4, xfs тощо) |
|---|---|---|
| Базові дозволи | Read, Write, R&E, Modify, Full Control, List Folder | rwx (read, write, execute) |
| Категорії | Окремі користувачі та групи (ACL) | Owner, Group, Others |
| Графічний інтерфейс | Властивості → Безпека | Файловий менеджер (Nautilus, Thunar) |
| Командний рядок | icacls | chmod, chown, chgrp |
| Спадкування | Так, через вкладку "Додатково" | Немає автоматичного спадкування (є umask для нових файлів) |
| Гнучкість налаштування | Дуже висока (ACL — списки контролю доступу) | Базова система rwx + розширені ACL (setfacl) |

## Частина 5. Практичні завдання

> **Примітка:** Для виконання завдань на Linux використовуйте віртуальну машину (VirtualBox / VMware) з Ubuntu або іншим дистрибутивом, або WSL (Windows Subsystem for Linux).

### Завдання 1. Перегляд дозволів у Windows (GUI)

1. Створіть на робочому столі папку `ПР8_Тест`.
2. Всередині неї створіть текстовий файл `secret.txt` і запишіть у нього будь-який текст.
3. Клацніть правою кнопкою на файлі → **Властивості** → вкладка **"Безпека"**.
4. Запишіть у зошит (або в окремий файл-звіт):
   - Які користувачі та групи мають доступ до файлу?
   - Які саме дозволи надано кожному?
5. Зробіть скріншот вкладки "Безпека".

### Завдання 2. Зміна дозволів у Windows — заборона запису

1. Відкрийте властивості папки `ПР8_Тест` → вкладка **"Безпека"** → **"Змінити"**.
2. Оберіть групу **Users**.
3. У стовпці **"Заборонити"** (Deny) поставте галочку навпроти **"Запис"** (Write).
4. Натисніть **"Застосувати"** → **"ОК"**.
5. Спробуйте створити новий файл у цій папці від імені звичайного користувача.
6. Запишіть результат: чи вдалося створити файл? Яке повідомлення з'явилося?
7. **Поверніть дозволи на початкові** (зніміть заборону).

### Завдання 3. Вимкнення спадкування дозволів у Windows

1. Клацніть правою кнопкою на папці `ПР8_Тест` → **Властивості** → **Безпека** → **Додатково**.
2. Натисніть **"Вимкнути успадкування"** (Disable inheritance).
3. Оберіть **"Перетворити успадковані дозволи на явні"**.
4. Тепер видаліть групу **Everyone** зі списку (якщо вона є).
5. Додайте свого користувача з дозволом **Modify**.
6. Перевірте: чи можете ви тепер читати, записувати та видаляти файли в папці?
7. Зробіть скріншот вікна "Додаткові параметри безпеки".

### Завдання 4. Робота з icacls у CMD

1. Відкрийте **Командний рядок** (CMD) від імені адміністратора.
2. Виконайте команду для перегляду дозволів:
   ```cmd
   icacls "C:\Users\%USERNAME%\Desktop\ПР8_Тест"
   ```
3. Скопіюйте вивід команди у файл-звіт.
4. Надайте групі Users дозвіл на читання:
   ```cmd
   icacls "C:\Users\%USERNAME%\Desktop\ПР8_Тест" /grant Users:(R)
   ```
5. Заборонте групі Users запис:
   ```cmd
   icacls "C:\Users\%USERNAME%\Desktop\ПР8_Тест" /deny Users:(W)
   ```
6. Знову перегляньте дозволи та порівняйте з попереднім виводом:
   ```cmd
   icacls "C:\Users\%USERNAME%\Desktop\ПР8_Тест"
   ```
7. Видаліть заборону:
   ```cmd
   icacls "C:\Users\%USERNAME%\Desktop\ПР8_Тест" /remove:d Users
   ```

### Завдання 5. Розбір виводу ls -la в Linux

1. Відкрийте термінал Linux (VM або WSL).
2. Перейдіть до домашнього каталогу і перегляньте його вміст:
   ```bash
   cd ~
   ls -la
   ```
3. Знайдіть у виводі каталог `Desktop` (або `Робочий стіл`) і файл `.bashrc`.
4. Запишіть у звіт для кожного:
   - Тип (файл чи каталог).
   - Дозволи для власника, групи, інших.
   - Ім'я власника та групи.
5. Переведіть дозволи з літерного запису в числовий (наприклад, `rwxr-xr-x` → `755`).

### Завдання 6. Створення файлу та зміна дозволів через chmod

1. Створіть каталог для практичної роботи та перейдіть до нього:
   ```bash
   mkdir ~/pr8_test
   cd ~/pr8_test
   ```
2. Створіть текстовий файл:
   ```bash
   echo "Hello, Linux permissions!" > test_file.txt
   ```
3. Перегляньте поточні дозволи:
   ```bash
   ls -la test_file.txt
   ```
4. Змініть дозволи так, щоб лише власник міг читати та записувати:
   ```bash
   chmod 600 test_file.txt
   ```
5. Перевірте результат:
