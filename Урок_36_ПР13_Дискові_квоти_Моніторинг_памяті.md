## 1. Мета роботи

- Зрозуміти призначення та принцип роботи дискових квот у Windows 10/11.
- Навчитися вмикати, налаштовувати та переглядати дискові квоти через графічний інтерфейс та командний рядок.
- Оволодіти навичками моніторингу оперативної пам'яті за допомогою Диспетчера завдань, Монітора ресурсів та Монітора продуктивності.
- Навчитися створювати лічильники продуктивності для відстеження стану пам'яті.
- Визначати ознаки нестачі RAM та знаходити процеси, що споживають найбільше пам'яті.

---

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11.
- Клавіатура та миша.
- Командний рядок (CMD), запущений від імені адміністратора.
- Диск, відформатований у файлову систему NTFS (зазвичай C:).
- Наявність хоча б двох облікових записів користувачів у системі (для роботи з квотами).

---

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не змінюйте налаштування квот для системних облікових записів** (`NT AUTHORITY\SYSTEM`, `BUILTIN\Administrators`, `NT Service\TrustedInstaller`) — це може порушити роботу Windows.
4. Після завершення практичної роботи **обов'язково вимкніть дискові квоти**, щоб повернути систему до початкового стану.
5. Не завершуйте критичні системні процеси через Диспетчер завдань — це може призвести до збою системи.
6. Записуйте всі початкові налаштування перед їх зміною, щоб мати змогу відновити попередній стан.
7. Після завершення роботи приведіть робоче місце до початкового стану.

## 4. Теоретичні відомості

### 4.1. Дискові квоти — що це і навіщо

**Дискова квота** (Disk Quota) — це механізм обмеження дискового простору, який кожен користувач може зайняти на певному томі (диску). Квоти дозволяють адміністратору контролювати, скільки місця використовує кожен обліковий запис, і запобігати ситуації, коли один користувач займає весь доступний простір.

**Навіщо потрібні дискові квоти:**

| Проблема | Як вирішують квоти |
|----------|-------------------|
| Один користувач завантажив фільми на 100 ГБ і зайняв весь диск | Квота обмежує максимальний обсяг даних для кожного користувача |
| Адміністратор не знає, хто скільки місця займає | Квоти відстежують використання простору кожним користувачем |
| Диск сервера повністю заповнений, система перестає працювати | Квота попередить або заборонить запис даних при досягненні ліміту |
| Потрібно рівномірно розподілити дисковий простір між працівниками | Квоти дозволяють задати однаковий ліміт для всіх або індивідуальний для кожного |

**Два режими роботи квот:**

- **Відстеження (Tracking)** — система лише записує, скільки місця використовує кожен користувач, але не обмежує їх. Корисно для аналізу перед встановленням лімітів.
- **Примусове обмеження (Enforcing)** — система не дозволяє користувачу перевищити встановлений ліміт. При спробі зберегти файл, що перевищує квоту, Windows видає помилку «Недостатньо місця на диску».

**Два порогових значення:**

- **Поріг попередження (Warning Level)** — коли користувач досягає цього рівня, Windows записує подію в журнал подій. Наприклад: поріг 80 МБ із ліміту 100 МБ.
- **Ліміт (Quota Limit)** — максимальний обсяг. Якщо увімкнено примусове обмеження, запис нових даних блокується.

### 4.2. NTFS Disk Quotas — вимоги та обмеження

Дискові квоти у Windows працюють **тільки на томах із файловою системою NTFS**. Файлові системи FAT32 та exFAT не підтримують цей механізм.

**Ключові особливості NTFS Disk Quotas:**

| Характеристика | Опис |
|----------------|------|
| **Рівень застосування** | Квоти працюють на рівні тому (диска), а не папки. Не можна задати квоту для окремої папки |
| **Прив'язка до користувача** | Кожен файл має власника (user). Квота рахує загальний обсяг файлів, що належать конкретному користувачу |
| **Стиснені файли** | Квота враховує **оригінальний** (нестиснутий) розмір файлу, навіть якщо файл стиснуто засобами NTFS |
| **Оновлення даних** | Дані про використання простору оновлюються приблизно щогодини |
| **Хто може налаштовувати** | Тільки адміністратори. Звичайні користувачі не можуть змінювати квоти |
| **Системні акаунти** | Квоти не слід застосовувати до системних акаунтів (SYSTEM, TrustedInstaller), інакше Windows може працювати некоректно |

**Як перевірити файлову систему диска:**

1. Відкрити **Провідник** (Win + E).
2. Натиснути правою кнопкою миші на диску --> **Властивості**.
3. На вкладці **Загальні** (General) буде вказана файлова система.

> **Важливо:** Якщо диск відформатований у FAT32 або exFAT, вкладка «Квота» у властивостях диска буде відсутня.

### 4.3. Моніторинг пам'яті — навіщо відстежувати

**Оперативна пам'ять (RAM)** — це швидкодіюча пам'ять, у якій зберігаються дані програм та процесів під час їх виконання. Коли RAM заповнюється, Windows починає використовувати **файл підкачки** (Page File) — спеціальний файл на жорсткому диску, який імітує додаткову пам'ять. Однак диск працює в десятки разів повільніше за RAM, тому надмірне використання файлу підкачки різко уповільнює систему.

**Навіщо моніторити пам'ять:**

- Визначити, чи вистачає встановленої RAM для повсякденних задач.
- Знайти програми, що «витікають» пам'ять (memory leak) — поступово займають все більше RAM, не звільняючи її.
- Діагностувати причини повільної роботи комп'ютера.
- Обґрунтувати необхідність оновлення (додавання) оперативної пам'яті.
- Контролювати серверні системи, де стабільна робота пам'яті критично важлива.

**Що таке сторінкові помилки (Page Faults):**

Коли програма звертається до даних, яких немає в оперативній пам'яті, виникає **сторінкова помилка**:

- **М'яка помилка (Soft Page Fault)** — дані знаходяться у кеші або в іншій ділянці RAM. Обробка швидка.
- **Жорстка помилка (Hard Page Fault)** — дані потрібно зчитати з диска (файлу підкачки). Обробка повільна, оскільки потрібен доступ до диска.

Велика кількість жорстких сторінкових помилок — ознака нестачі оперативної пам'яті.

### 4.4. Інструменти моніторингу пам'яті у Windows

Windows має кілька вбудованих інструментів для аналізу використання пам'яті:

**1. Диспетчер завдань (Task Manager)**

- **Як відкрити:** Ctrl + Shift + Esc або Ctrl + Alt + Delete --> Диспетчер завдань.
- **Вкладка «Процеси»** — стовпець «Пам'ять» показує, скільки RAM використовує кожна програма.
- **Вкладка «Продуктивність» --> «Пам'ять»** — загальний графік використання RAM у реальному часі.

Інформація на вкладці «Пам'ять» Диспетчера завдань:

| Показник | Опис |
|----------|------|
| **Використовується** | Обсяг RAM, зайнятий програмами та системою |
| **Доступно** | Обсяг RAM, вільний для нових програм |
| **Виділено** | Загальний обсяг віртуальної пам'яті (RAM + файл підкачки), що зарезервований системою |
| **Кешовано** | Дані, завантажені в RAM для пришвидшення повторного доступу |
| **Вивантажувальний пул** | Частина пам'яті ядра Windows, яку можна перемістити у файл підкачки |
| **Невивантажувальний пул** | Частина пам'яті ядра, яка завжди залишається в RAM |

**2. Монітор ресурсів (Resource Monitor)**

- **Як відкрити:** Win + R --> ввести `resmon` --> Enter; або через Диспетчер завдань --> вкладка «Продуктивність» --> «Відкрити монітор ресурсів».
- Надає більш детальну інформацію, ніж Диспетчер завдань.
- **Вкладка «Пам'ять»** показує для кожного процесу чотири ключових стовпці:

| Стовпець | Тип пам'яті | Опис |
|----------|-------------|------|
| **Commit (КБ)** | Віртуальна | Обсяг віртуальної пам'яті, зарезервованої ОС для процесу (RAM + файл підкачки) |
| **Working Set (КБ)** | Фізична (RAM) | Загальний обсяг RAM, що зараз використовується процесом |
| **Shareable (КБ)** | Фізична (RAM) | Частина Working Set, яка може бути спільною з іншими процесами (наприклад, бібліотеки DLL) |
| **Private (КБ)** | Фізична (RAM) | Частина Working Set, яка використовується виключно цим процесом і не може бути спільною |

> **Зв'язок:** Working Set = Shareable + Private. Тобто загальна фізична пам'ять процесу складається зі спільної та приватної частин.

Нижня частина вкладки «Пам'ять» у Моніторі ресурсів містить горизонтальну діаграму, що показує розподіл фізичної пам'яті: зарезервована, використовується, змінена, у режимі очікування, вільна.

**3. Монітор продуктивності (Performance Monitor / perfmon)**

- **Як відкрити:** Win + R --> ввести `perfmon` --> Enter.
- Дозволяє створювати **власні лічильники** для відстеження конкретних показників пам'яті у реальному часі.
- Підтримує запис даних у журнал для подальшого аналізу.
- Корисний для довготривалого моніторингу та виявлення тенденцій.

### 4.5. Лічильники продуктивності (Performance Counters) для пам'яті

**Лічильники продуктивності** (Performance Counters) — це числові показники, які Windows постійно збирає про різні аспекти роботи системи. Для пам'яті найважливіші такі лічильники:

| Лічильник | Що вимірює | Норма | Проблема |
|-----------|------------|-------|----------|
| **Memory \ Available MBytes** | Обсяг вільної фізичної пам'яті (в МБ), доступної для процесів | Більше 25% від загальної RAM | Менше 20–25% від загальної RAM — потрібно додавати пам'ять або оптимізувати |
| **Memory \ Pages/sec** | Кількість сторінок на секунду, зчитаних з диска або записаних на диск через сторінкові помилки | До 20 (середнє значення) | Стабільно більше 20 — надмірне підкачування, вказує на нестачу RAM |
| **Memory \ % Committed Bytes In Use** | Відсоток виділеної віртуальної пам'яті від максимально доступної | До 80% | Більше 80% — система близька до вичерпання віртуальної пам'яті |
| **Memory \ Page Faults/sec** | Загальна кількість сторінкових помилок на секунду (м'яких + жорстких) | Залежить від навантаження | Різкі стрибки можуть вказувати на memory leak |
| **Process \ Working Set** | Обсяг фізичної пам'яті (RAM), що використовується конкретним процесом | Залежить від програми | Постійне зростання без зменшення — можлива «витікання» пам'яті |
| **Paging File \ % Usage** | Відсоток використання файлу підкачки | До 50% | Стабільно більше 70% — потрібно більше RAM |

**Як читати Pages/sec разом з Available MBytes:**

- Якщо Pages/sec високий, але Available MBytes в нормі — це м'які помилки (soft faults), кешовані дані. Проблеми немає.
- Якщо Pages/sec високий **і** Available MBytes низький — це жорсткі помилки (hard faults), система активно підкачує з диска. Потрібно більше RAM.

### 4.6. Ознаки нестачі оперативної пам'яті

Як зрозуміти, що комп'ютеру не вистачає оперативної пам'яті:

| Ознака | Що відбувається |
|--------|----------------|
| **Повільна робота системи** | Програми довго відкриваються, перемикання між вікнами з затримкою, «підвисання» |
| **Високий Disk Usage (100%)** | У Диспетчері завдань стовпець «Диск» показує 100% — система постійно читає/записує файл підкачки |
| **Часті жорсткі сторінкові помилки (Hard Page Faults)** | У Моніторі ресурсів стовпець «Hard Faults/sec» має високі значення |
| **Мало доступної пам'яті** | У Диспетчері завдань показник «Доступно» менше 500 МБ (для 8 ГБ системи) |
| **Повідомлення «Недостатньо пам'яті»** | Windows показує попередження та пропонує закрити програми |
| **Програми аварійно завершуються** | Програми закриваються самі або показують помилки пам'яті |
| **Довге завантаження Windows** | Система завантажується помітно довше, ніж зазвичай |

**Що робити при нестачі RAM:**

1. Закрити непотрібні програми та вкладки браузера (кожна вкладка Chrome може займати 50–300 МБ).
2. Перевірити автозавантаження (Диспетчер завдань --> вкладка «Автозавантаження») та вимкнути непотрібні програми.
3. Збільшити розмір файлу підкачки (тимчасове рішення).
4. Додати фізичну оперативну пам'ять (найкраще рішення).

## 5. Практичні завдання з дискових квот

### Завдання 1. Відкрити Властивості диска та знайти вкладку «Квота»

1. Відкрийте **Провідник** (Win + E) --> «Цей ПК» (This PC).
2. Натисніть правою кнопкою миші на диску **C:** --> **«Властивості»** (Properties).
3. Знайдіть вкладку **«Квота»** (Quota) у верхній частині вікна.
4. Натисніть кнопку **«Показати параметри квоти»** (Show Quota Settings).
5. Запишіть у зошит поточний стан:

| Параметр | Значення |
|----------|----------|
| Чи увімкнене керування квотами? (Enable quota management) | |
| Чи встановлено обмеження? | |
| Стан квот (відстеження / примусове обмеження) | |

6. **Не змінюйте нічого на цьому етапі** — просто вивчіть інтерфейс.

> **Якщо вкладки «Квота» немає:** це означає, що диск відформатований не в NTFS. Перевірте файлову систему на вкладці «Загальні». Для практичної роботи потрібен NTFS-том.

---

### Завдання 2. Увімкнути дискові квоти та задати ліміт

1. У вікні параметрів квоти (з завдання 1) поставте галочку **«Увімкнути керування квотами»** (Enable quota management).
2. Оберіть **«Не допускати використання дискового простору понад обсяг квоти»** (Deny disk space to users exceeding quota limit).
3. Оберіть **«Обмежити місце на диску до»** (Limit disk space to) та встановіть: **100 МБ**.
4. У полі **«Установити рівень попередження»** (Set warning level to) встановіть: **80 МБ**.
5. Заповніть таблицю в зошиті:

| Параметр | Встановлене значення |
|----------|---------------------|
| Ліміт дискового простору | |
| Рівень попередження | |
| Заборона перевищення | Так / Ні |
