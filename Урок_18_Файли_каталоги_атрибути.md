## 1. Поняття файлу

**Файл** -- це іменована область даних на носії інформації (жорсткому диску, SSD, флеш-накопичувачі тощо), яка має унікальне ім'я і зберігається у файловій системі.

Простіше кажучи, файл -- це контейнер для зберігання інформації. Все, з чим ви працюєте на комп'ютері -- документи, фотографії, музика, програми, відео -- зберігається у вигляді файлів.

### Основні характеристики файлу:

- **Має ім'я** -- кожен файл ідентифікується унікальним іменем у межах каталогу.
- **Має розмір** -- вимірюється в байтах (Б), кілобайтах (КБ), мегабайтах (МБ), гігабайтах (ГБ).
- **Має тип** -- визначається розширенням (наприклад, `.txt`, `.jpg`, `.exe`).
- **Має атрибути** -- додаткова інформація про файл (дата створення, права доступу тощо).
- **Займає місце на носії** -- фізично записаний на диску у вигляді послідовності байтів.

### Аналогія для розуміння:

Уявіть файлову систему як бібліотеку. Файл -- це книга. Ім'я файлу -- назва книги. Каталог (папка) -- це полиця або шафа, де зберігається книга. Розширення файлу -- жанр книги (роман, підручник, енциклопедія). Атрибути -- додаткова інформація (автор, рік видання, кількість сторінок).

## 2. Ім'я файлу та правила іменування

Ім'я файлу складається з двох частин, розділених крапкою:

```
назва_файлу.розширення
```

Наприклад: `реферат.docx`, `фото_01.jpg`, `програма.py`

### 2.1. Правила іменування у Windows

| Параметр | Значення |
|----------|----------|
| Максимальна довжина імені файлу | 255 символів |
| Максимальна довжина повного шляху | 260 символів (за замовчуванням) |
| Регістрозалежність | **Ні** -- `Файл.txt` і `файл.txt` -- це один і той самий файл |
| Дозволені символи | Літери (будь-якого алфавіту), цифри, пробіли, символи `_ - . , @ # $ % & ( ) { } [ ] ! ~ +` |
| **Заборонені символи** | `\ / : * ? " < > \|` |

**Заборонені імена** у Windows (зарезервовані системою):
`CON`, `PRN`, `AUX`, `NUL`, `COM1`--`COM9`, `LPT1`--`LPT9`

> **Приклад допустимого імені:** `Мій реферат (версія 2).docx`
>
> **Приклад НЕдопустимого імені:** `Реферат:фінал?.docx` (містить заборонені символи `:` та `?`)

### 2.2. Правила іменування у Linux

| Параметр | Значення |
|----------|----------|
| Максимальна довжина імені файлу | 255 байтів |
| Регістрозалежність | **Так** -- `Файл.txt`, `файл.txt` і `ФАЙЛ.TXT` -- це три різні файли |
| Заборонені символи | Лише два: `/` (слеш) та `\0` (нульовий символ) |
| Рекомендовано уникати | Пробіли, `* ? < > \| \ : ; & ( )` -- вони мають спеціальне значення у терміналі |

> **Важлива відмінність:** У Linux ім'я файлу може починатися з крапки (наприклад, `.bashrc`). Такі файли вважаються **прихованими** і не відображаються командою `ls` без прапорця `-a`.

### 2.3. Порівняльна таблиця

| Критерій | Windows | Linux |
|----------|---------|-------|
| Регістрозалежність | Ні | Так |
| Максимальна довжина імені | 255 символів | 255 байтів |
| Заборонені символи | `\ / : * ? " < > \|` | `/` та `\0` |
| Пробіли в іменах | Дозволено | Дозволено, але не рекомендовано |
| Приховані файли | Через атрибут Hidden | Ім'я починається з крапки |

## 3. Розширення файлу

**Розширення файлу** -- це частина імені файлу після останньої крапки, яка вказує на тип (формат) файлу та визначає, яка програма буде використана для його відкриття.

```
доповідь.docx
^^^^^^^^ ^^^^
  ім'я   розширення
```

### Навіщо потрібне розширення?

1. **Визначає тип файлу** -- операційна система "розуміє", який це файл: текст, зображення, програма тощо.
2. **Пов'язує файл із програмою** -- при подвійному натисканні на файл ОС відкриває його у відповідній програмі (наприклад, `.docx` -- у Microsoft Word).
3. **Допомагає користувачу** -- за розширенням можна швидко визначити вміст файлу.

### Важливо знати:

- У Windows розширення за замовчуванням **приховане** (його можна увімкнути у Провіднику: Вигляд -> Розширення імен файлів).
- У Linux розширення -- це лише **угода**, система визначає тип файлу за його вмістом (заголовком), а не за розширенням.
- Зміна розширення **не змінює** вміст файлу. Якщо перейменувати `фото.jpg` на `фото.txt`, файл не стане текстовим -- він просто перестане правильно відкриватися.

### Приклади поширених розширень:

| Розширення | Тип файлу | Програма для відкриття |
|------------|-----------|----------------------|
| `.txt` | Текстовий файл | Блокнот, будь-який текстовий редактор |
| `.docx` | Документ Word | Microsoft Word, LibreOffice Writer |
| `.pdf` | Портативний документ | Adobe Acrobat Reader, браузер |
| `.jpg` / `.png` | Зображення | Переглядач фото, Photoshop, GIMP |
| `.mp3` | Аудіофайл | Музичний програвач |
| `.mp4` | Відеофайл | Відеоплеєр (VLC, Windows Media Player) |
| `.exe` | Виконуваний файл (Windows) | Операційна система |
| `.zip` | Архів | 7-Zip, WinRAR |
| `.html` | Веб-сторінка | Браузер (Chrome, Firefox) |
| `.py` | Скрипт Python | Інтерпретатор Python, IDE |

## 5. Типи файлів

Файли можна класифікувати не тільки за розширенням, а й за їхньою природою у файловій системі.

### 5.1. Звичайні файли (Regular Files)

Це найпоширеніший тип файлів. До них належать:

- Текстові документи (`.txt`, `.docx`)
- Зображення (`.jpg`, `.png`)
- Відео та аудіо (`.mp4`, `.mp3`)
- Виконувані програми (`.exe` у Windows, ELF-файли у Linux)
- Бібліотеки, конфігураційні файли тощо

У Linux при виконанні команди `ls -l` звичайні файли позначаються символом `-` на початку рядка:

```
-rw-r--r--  1  user  group  1024  Jan 15 10:30  document.txt
^
звичайний файл
```

### 5.2. Каталоги (Directories)

Каталог -- це спеціальний тип файлу, який містить список інших файлів та підкаталогів. Докладніше -- у розділі 6.

У Linux позначаються символом `d`:

```
drwxr-xr-x  2  user  group  4096  Jan 15 10:30  Documents
^
каталог
```

### 5.3. Спеціальні файли пристроїв (Device Files) -- Linux

У Linux кожен фізичний пристрій представлений у вигляді файлу у каталозі `/dev`. Це одна з ключових концепцій Unix/Linux: **"все є файлом"**.

Бувають двох видів:

| Тип | Символ у `ls -l` | Опис | Приклад |
|-----|-------------------|------|---------|
| **Блокові** | `b` | Забезпечують буферизований доступ до пристроїв (диски, флешки) | `/dev/sda` (жорсткий диск) |
| **Символьні** | `c` | Передають дані посимвольно (клавіатура, мишка, термінал) | `/dev/tty` (термінал) |

### 5.4. Символічні посилання (Symbolic Links)

**Символічне посилання** (або м'яке посилання, symlink) -- це файл, який містить шлях до іншого файлу або каталогу. Це аналог **ярлика** у Windows.

У Linux позначаються символом `l`:

```
lrwxrwxrwx  1  user  group  11  Jan 15 10:30  link.txt -> original.txt
^
символічне посилання
```

**Особливості:**
- Якщо видалити оригінальний файл, символічне посилання стає "битим" (вказує в нікуди).
- Може вказувати на файли в інших файлових системах та розділах.
- Відрізняється від **жорсткого посилання** (hard link), яке є ще одним іменем для того самого файлу.

### 5.5. Зведена таблиця типів файлів у Linux

| Символ у `ls -l` | Тип файлу |
|-------------------|-----------|
| `-` | Звичайний файл |
| `d` | Каталог |
| `l` | Символічне посилання |
| `b` | Блоковий файл пристрою |
| `c` | Символьний файл пристрою |
| `p` | Іменований канал (FIFO) |
| `s` | Сокет |

## 6. Поняття каталогу (папки)

**Каталог** (папка, директорія, directory) -- це спеціальний файл, який містить список інших файлів та підкаталогів. Каталоги забезпечують ієрархічну організацію файлової системи.

### 6.1. Ієрархія каталогів

Файлова система має деревоподібну структуру:

```
Кореневий каталог
├── Каталог_1
│   ├── Підкаталог_1a
│   │   └── файл_1.txt
│   └── файл_2.txt
├── Каталог_2
│   └── файл_3.jpg
└── файл_4.exe
```

### 6.2. Кореневий каталог

**Кореневий каталог** -- це найвищий рівень ієрархії файлової системи, "вершина дерева".

| ОС | Кореневий каталог | Приклад |
|----|-------------------|---------|
| Windows | Літера диска з двокрапкою та зворотним слешем | `C:\` |
| Linux | Один слеш | `/` |

У Windows кожен диск має свій кореневий каталог (`C:\`, `D:\`, `E:\`), тоді як у Linux є лише один кореневий каталог `/`, до якого підключаються всі пристрої.

### 6.3. Підкаталоги та батьківський каталог

- **Підкаталог** (subdirectory) -- каталог, що знаходиться всередині іншого каталогу.
- **Батьківський каталог** (parent directory) -- каталог, у якому знаходиться поточний каталог. Позначається як `..` (дві крапки).
- **Поточний каталог** позначається як `.` (одна крапка).

### 6.4. Шлях до файлу

**Шлях** (path) -- це адреса файлу у файловій системі.

**Абсолютний шлях** -- повний шлях від кореневого каталогу:
- Windows: `C:\Users\Student\Documents\реферат.docx`
- Linux: `/home/student/Documents/реферат.docx`

**Відносний шлях** -- шлях відносно поточного каталогу:
- `Documents\реферат.docx` (Windows)
- `Documents/реферат.docx` (Linux)

> **Зверніть увагу на роздільник:** Windows використовує зворотний слеш `\`, а Linux -- прямий слеш `/`.

### 6.5. Стандартні каталоги

**Windows:**
- `C:\Windows` -- системні файли ОС
- `C:\Program Files` -- встановлені програми
- `C:\Users\Ім'я` -- домашній каталог користувача
- `C:\Users\Ім'я\Desktop` -- Робочий стіл
- `C:\Users\Ім'я\Documents` -- Документи

**Linux:**
- `/` -- кореневий каталог
- `/home/ім'я` -- домашній каталог користувача
- `/bin`, `/usr/bin` -- виконувані файли (програми)
- `/etc` -- конфігураційні файли
- `/dev` -- файли пристроїв
- `/tmp` -- тимчасові файли

## 7. Атрибути файлів

**Атрибути файлів** -- це додаткова інформація (метадані), яка описує властивості файлу: хто може його читати, чи є він прихованим, коли був створений тощо.

### 7.1. Атрибути файлів у Windows

У Windows кожен файл має чотири основні атрибути:

| Атрибут | Літера | Опис |
|---------|--------|------|
| **Read-only** (Тільки читання) | `R` | Файл захищений від випадкових змін. Його можна читати, але не можна змінити чи видалити без зняття цього атрибуту |
| **Hidden** (Прихований) | `H` | Файл не відображається у Провіднику та при виконанні команди `dir` за замовчуванням |
| **System** (Системний) | `S` | Файл є критично важливим для роботи ОС. Зазвичай поєднується з атрибутом Hidden. Приклад: файли ядра Windows |
| **Archive** (Архівний) | `A` | Вказує, що файл був змінений після останнього резервного копіювання. Використовується програмами для створення бекапів |

**Додаткові атрибути NTFS:** стиснений (Compressed), зашифрований (Encrypted), тимчасовий (Temporary), індексований/неіндексований.

### 7.2. Права доступу у Linux (rwx)

У Linux замість простих атрибутів використовується потужна система **прав доступу**. Кожен файл має права для трьох категорій користувачів:

| Категорія | Англійська назва | Хто це |
|-----------|-----------------|--------|
| **Власник** | Owner (User) | Користувач, який створив файл |
| **Група** | Group | Група користувачів, до якої належить файл |
| **Інші** | Others | Всі решта користувачів системи |

Для кожної категорії визначаються три типи дозволів:

| Дозвіл | Літера | Числове значення | Для файлу | Для каталогу |
|--------|--------|-----------------|-----------|--------------|
| **Читання** | `r` (read) | 4 | Переглядати вміст файлу | Переглядати список файлів у каталозі |
| **Запис** | `w` (write) | 2 | Змінювати вміст файлу | Створювати/видаляти файли у каталозі |
| **Виконання** | `x` (execute) | 1 | Запускати файл як програму | Входити у каталог (використовувати `cd`) |

### Як читати рядок прав доступу

Приклад виводу команди `ls -l`:

```
-rwxr-xr--  1  student  students  2048  Jan 15 10:30  script.sh
```

Розшифровка рядка `-rwxr-xr--`:

```
-    rwx    r-x    r--
|     |      |      |
|     |      |      +-- Інші (Others): читання
|     |      +--------- Група (Group): читання + виконання
|     +---------------- Власник (Owner): читання + запис + виконання
+---------------------- Тип файлу: звичайний файл
```

### Числове (вісімкове) представлення прав

Кожен дозвіл має числове значення: `r=4`, `w=2`, `x=1`. Сума цих значень дає числовий код для кожної категорії:

| Числовий код | Права | Опис |
|--------------|-------|------|
| `7` | `rwx` | Читання + запис + виконання |
| `6` | `rw-` | Читання + запис |
| `5` | `r-x` | Читання + виконання |
| `4` | `r--` | Тільки читання |
| `3` | `-wx` | Запис + виконання |
| `2` | `-w-` | Тільки запис |
| `1` | `--x` | Тільки виконання |
| `0` | `---` | Немає прав |

**Приклади:**
- `chmod 755 script.sh` -- власник: все дозволено (7), група та інші: читання і виконання (5).
- `chmod 644 document.txt` -- власник: читання і запис (6), група та інші: тільки читання (4).

### 7.3. Метадані файлу

Крім основних атрибутів, кожен файл має **метадані** -- службову інформацію:

| Метадані | Опис |
|----------|------|
| **Розмір** | Обсяг даних у файлі (в байтах, КБ, МБ тощо) |
| **Дата створення** | Коли файл був створений |
| **Дата зміни** | Коли файл був востаннє змінений |
| **Дата доступу** | Коли файл був востаннє відкритий (прочитаний) |
| **Власник** | Хто створив файл (ім'я користувача) |
| **Розташування** | Повний шлях до файлу у файловій системі |

## 8. Перегляд атрибутів файлів

### 8.1. У Windows

**Спосіб 1: Через Провідник (графічний інтерфейс)**

1. Натисніть правою кнопкою миші на файлі.
2. Оберіть "Властивості" (Properties).
3. На вкладці "Загальні" (General) ви побачите:
   - Тип файлу та розширення
   - Розташування (шлях)
   - Розмір файлу
   - Дати створення, зміни, доступу
   - Атрибути "Тільки читання" та "Прихований" (прапорці внизу вікна)

**Спосіб 2: Команда `attrib` у командному рядку (CMD)**

```
attrib C:\Users\Student\document.txt
```

Приклад результату:

```
A    R        C:\Users\Student\document.txt
```

Це означає: файл має атрибути **A**rchive (архівний) та **R**ead-only (тільки читання).

**Зміна атрибутів через `attrib`:**

| Команда | Дія |
|---------|-----|
| `attrib +r файл.txt` | Встановити атрибут "тільки читання" |
| `attrib -r файл.txt` | Зняти атрибут "тільки читання" |
| `attrib +h файл.txt` | Зробити файл прихованим |
| `attrib -h файл.txt` | Показати прихований файл |
| `attrib +s файл.txt` | Встановити атрибут "системний" |
| `attrib -s -h файл.txt` | Зняти атрибути "системний" та "прихований" одночасно |

### 8.2. У Linux

**Основна команда: `ls -la`**

```bash
ls -la /home/student/Documents/
```

Приклад результату:

```
drwxr-xr-x  3  student  students  4096  Jan 15 10:30  .
drwxr-xr-x  5  student  students  4096  Jan 10 09:00  ..
-rw-r--r--  1  student  students  2048  Jan 15 10:30  document.txt
-rwxr-xr-x  1  student  students   512  Jan 14 15:45  script.sh
lrwxrwxrwx  1  student  students    12  Jan 13 08:20  link.txt -> original.txt
drwxr-xr-x  2  student  students  4096  Jan 12 14:00  Photos
```

**Розшифровка стовпців:**

| Стовпець | Значення |
|----------|----------|
| `drwxr-xr-x` | Тип файлу та права доступу |
| `3` | Кількість жорстких посилань |
| `student` | Власник файлу |
| `students` | Група |
| `4096` | Розмір у байтах |
| `Jan 15 10:30` | Дата та час останньої зміни |
| `.` | Ім'я файлу/каталогу |

**Корисні варіації команди `ls`:**

| Команда | Що показує |
|---------|-----------|
| `ls -l` | Детальний список (без прихованих файлів) |
| `ls -la` | Детальний список з прихованими файлами |
| `ls -lh` | Детальний список з "людським" форматом розміру (KB, MB, GB) |
| `ls -lS` | Сортування за розміром |
| `ls -lt` | Сортування за датою зміни |

**Зміна прав доступу -- команда `chmod`:**

```bash
chmod 755 script.sh       # Числовий формат
chmod u+x script.sh       # Символьний формат: додати виконання для власника
chmod go-w document.txt   # Забрати запис у групи та інших
```

## 9. Маски файлів (wildcards)

**Маски файлів** (шаблони, wildcards, підстановочні символи) -- це спеціальні символи, які використовуються для пошуку та фільтрації файлів за певним шаблоном.

### Два основні символи:

| Символ | Назва | Значення | Приклад |
|--------|-------|----------|---------|
| `*` | Зірочка (asterisk) | Замінює **будь-яку кількість** будь-яких символів (у тому числі жоден) | `*.txt` -- всі файли з розширенням `.txt` |
| `?` | Знак питання (question mark) | Замінює рівно **один** будь-який символ | `file?.txt` -- `file1.txt`, `fileA.txt`, але не `file10.txt` |

### Приклади використання масок:

| Маска | Що знайде |
|-------|-----------|
| `*` | Абсолютно всі файли |
| `*.jpg` | Всі файли з розширенням `.jpg` |
| `*.mp*` | Файли `.mp3`, `.mp4` тощо |
| `report*` | Всі файли, ім'я яких починається з "report" |
| `*.???` | Всі файли з розширенням рівно з 3 символів |
| `photo_?.png` | `photo_1.png`, `photo_A.png` тощо |
| `doc*.pdf` | `doc.pdf`, `document.pdf`, `doc2024.pdf` тощо |
| `*2024*` | Всі файли, що містять "2024" у будь-якій частині імені |

### Де використовуються маски:

- **Пошук файлів** у Провіднику Windows та у терміналі Linux
- **Командний рядок Windows:** `dir *.txt` -- показати всі текстові файли
- **Термінал Linux:** `ls *.jpg` -- показати всі JPEG-файли
- **Команди копіювання/переміщення/видалення:**
  - Windows: `copy *.docx D:\Backup\`
  - Linux: `cp *.jpg ~/Photos/`
- **Файлові менеджери** (Total Commander, Midnight Commander тощо)

### Практичний приклад:

Потрібно знайти всі файли з розширенням `.pdf` у каталозі `Documents`:

**Windows (CMD):**
```
dir C:\Users\Student\Documents\*.pdf
```

**Linux (термінал):**
```bash
ls ~/Documents/*.pdf
```

## 11. Глосарій

| Термін | Визначення |
|--------|-----------|
| **Файл (File)** | Іменована область даних на носії інформації, яка зберігається у файловій системі |
| **Каталог (Directory, папка)** | Спеціальний файл, який містить список інших файлів та підкаталогів; забезпечує ієрархічну організацію файлової системи |
| **Ім'я файлу (Filename)** | Унікальний ідентифікатор файлу в межах каталогу, що складається з назви та (необов'язково) розширення |
| **Розширення файлу (File extension)** | Частина імені файлу після останньої крапки, що вказує на тип (формат) файлу |
| **Шлях (Path)** | Адреса файлу або каталогу у файловій системі; буває абсолютним (від кореня) та відносним (від поточного каталогу) |
| **Абсолютний шлях (Absolute path)** | Повний шлях до файлу від кореневого каталогу (наприклад, `C:\Users\file.txt` або `/home/user/file.txt`) |
| **Відносний шлях (Relative path)** | Шлях до файлу відносно поточного робочого каталогу |
| **Кореневий каталог (Root directory)** | Найвищий рівень файлової системи (`C:\` у Windows, `/` у Linux) |
| **Батьківський каталог (Parent directory)** | Каталог, у якому знаходиться поточний каталог; позначається `..` |
| **Підкаталог (Subdirectory)** | Каталог, що знаходиться всередині іншого каталогу |
| **Атрибути файлу (File attributes)** | Метадані, що описують властивості файлу (права доступу, видимість, системність тощо) |
| **Read-only (Тільки читання)** | Атрибут Windows, що забороняє зміну та видалення файлу |
| **Hidden (Прихований)** | Атрибут Windows, що робить файл невидимим у стандартному перегляді |
| **System (Системний)** | Атрибут Windows, що позначає файл як критично важливий для ОС |
| **Archive (Архівний)** | Атрибут Windows, що вказує на необхідність резервного копіювання файлу |
| **Права доступу (Permissions)** | Система контролю доступу в Linux, що визначає, хто може читати, змінювати та виконувати файл |
| **rwx** | Позначення прав доступу в Linux: r -- читання (read), w -- запис (write), x -- виконання (execute) |
| **chmod** | Команда Linux для зміни прав доступу до файлу |
| **Власник (Owner)** | Користувач, якому належить файл у Linux |
| **Група (Group)** | Набір користувачів, що мають спільні права доступу до файлу |
| **Символічне посилання (Symbolic link, symlink)** | Файл, що містить шлях до іншого файлу або каталогу; аналог ярлика у Windows |
| **Жорстке посилання (Hard link)** | Додаткове ім'я для існуючого файлу, що вказує на ті самі дані на диску |
| **Файл пристрою (Device file)** | Спеціальний файл у Linux (каталог `/dev`), що представляє фізичний або віртуальний пристрій |
| **Маска файлу (Wildcard)** | Шаблон для пошуку та фільтрації файлів за допомогою символів `*` (будь-яка кількість символів) та `?` (один символ) |
| **Метадані (Metadata)** | Службова інформація про файл: розмір, дати, власник, розташування |
| **attrib** | Команда Windows для перегляду та зміни атрибутів файлу |
| **ls -la** | Команда Linux для перегляду детальної інформації про файли, включаючи приховані |
| **Файлова система (File system)** | Спосіб організації, зберігання та доступу до файлів на носії інформації (NTFS, ext4, FAT32 тощо) |
