## 1. Мета роботи

- Закріпити знання про облікові записи користувачів, типи облікових записів та групи в операційних системах Windows і Linux.
- Навчитися переглядати інформацію про поточного користувача засобами командного рядка.
- Оволодіти практичними навичками створення, зміни та видалення локальних користувачів у Windows 10/11 через графічний інтерфейс та CMD.
- Навчитися працювати з оснасткою «Локальні користувачі та групи» (lusrmgr.msc).
- Оволодіти основними командами управління користувачами в Linux: `useradd`, `passwd`, `usermod`, `userdel`.
- Навчитися аналізувати файл `/etc/passwd` та розуміти формат його записів.
- Повернути систему до початкового стану після виконання практичної роботи.

## 2. Обладнання та програмне забезпечення

- Персональний комп'ютер з операційною системою Windows 10 або Windows 11 (редакція Pro, Enterprise або Education).
- Віртуальна машина з Linux (Ubuntu, Debian або аналогічний дистрибутив) або окремий ПК з Linux.
- Командний рядок (CMD), запущений від імені адміністратора.
- Термінал Linux із доступом до `sudo`.
- Клавіатура та миша.

## 3. Техніка безпеки

1. Перед початком роботи перевірте справність обладнання.
2. Не вмикайте та не вимикайте комп'ютер без дозволу викладача.
3. **Не видаляйте та не змінюйте існуючі облікові записи** (свій обліковий запис, обліковий запис адміністратора навчального ПК, обліковий запис `root` у Linux) — працюйте лише з тестовими користувачами, яких ви створюєте самі.
4. **Не змінюйте паролі** існуючих облікових записів.
5. **Не видаляйте вбудовані групи Windows** (Administrators, Users, Guests тощо) — це може порушити роботу системи.
6. Усі створені тестові облікові записи повинні бути видалені наприкінці практичної роботи.
7. Записуйте у зошит імена створених користувачів та паролі, щоб нічого не забути при очищенні.
8. Після завершення роботи переконайтеся, що система повернута до початкового стану, та приведіть робоче місце до порядку.

## 4. Теоретичний вступ

### 4.1. Облікові записи користувачів — нагадування

**Обліковий запис (акаунт)** — це набір даних, що ідентифікує користувача в операційній системі. Кожен обліковий запис містить:

- **Ім'я користувача (логін)** — унікальне ім'я для входу в систему.
- **Пароль** — секретна послідовність символів для автентифікації.
- **Ідентифікатор користувача** — унікальний числовий ідентифікатор (SID у Windows, UID у Linux).
- **Профіль** — каталог з особистими файлами та налаштуваннями користувача.
- **Належність до групи (або груп)** — визначає, які дозволи та привілеї має користувач.

Операційна система використовує облікові записи для:
- **Автентифікації** — перевірка, чи дійсно це та особа, за яку себе видає.
- **Авторизації** — визначення, до яких ресурсів користувач має доступ.
- **Аудиту** — ведення журналів дій кожного користувача.

---

### 4.2. Типи облікових записів у Windows

| Тип | Опис | Приклад |
|-----|------|---------|
| **Адміністратор (Administrator)** | Повний контроль над системою: встановлення програм, зміна налаштувань безпеки, управління іншими користувачами. | Вбудований обліковий запис `Administrator` (за замовчуванням вимкнений). |
| **Стандартний користувач (Standard User)** | Може працювати з програмами, змінювати власні налаштування, але не може встановлювати програми для всіх користувачів та змінювати системні параметри. | Обліковий запис, який створюється при встановленні Windows. |
| **Гість (Guest)** | Обмежений тимчасовий доступ без можливості встановлення програм та зміни налаштувань. За замовчуванням вимкнений у Windows 10/11. | Вбудований обліковий запис `Guest`. |
| **Обліковий запис Microsoft** | Онлайн-акаунт Microsoft, що синхронізує налаштування, файли OneDrive, додатки з Microsoft Store між пристроями. | email@outlook.com |
| **Локальний обліковий запис** | Існує лише на конкретному комп'ютері, без прив'язки до Інтернету. | Будь-яке ім'я, наприклад `Student1`. |

> **Важливо:** У Windows 10/11 при першому налаштуванні система пропонує створити обліковий запис Microsoft. Проте завжди можна створити локальний обліковий запис.

---

### 4.3. Групи користувачів

**Група** — це набір облікових записів, об'єднаних для зручного управління дозволами. Замість того щоб налаштовувати дозволи кожному користувачеві окремо, адміністратор додає користувача до групи, і той автоматично отримує всі дозволи цієї групи.

**Основні вбудовані групи Windows:**

| Група | Опис |
|-------|------|
| **Administrators** | Повний контроль над системою. Члени цієї групи можуть виконувати будь-які дії. |
| **Users** | Стандартні користувачі. Можуть запускати програми, працювати з файлами у своєму профілі. |
| **Guests** | Мінімальні права. Тимчасовий доступ. |
| **Remote Desktop Users** | Дозволяє підключатися до ПК через віддалений робочий стіл. |
| **Power Users** | Розширені права (у сучасних версіях Windows ця група існує для зворотної сумісності, але практично дорівнює Users). |
| **Backup Operators** | Право створювати резервні копії та відновлювати файли, навіть без дозволу на читання цих файлів. |

> **Принцип:** один користувач може бути членом кількох груп одночасно. Його фактичні дозволи — це сукупність дозволів усіх груп, до яких він належить.

---

### 4.4. Користувачі та групи в Linux

У Linux кожен користувач має:

- **UID (User ID)** — унікальний числовий ідентифікатор. `root` має UID = 0. Системні облікові записи зазвичай мають UID від 1 до 999. Звичайні користувачі — від 1000.
- **GID (Group ID)** — ідентифікатор основної групи користувача.
- **Домашній каталог** — зазвичай `/home/ім'я_користувача`.
- **Оболонка (shell)** — програма-інтерпретатор команд, наприклад `/bin/bash`.

**Ключові файли:**

| Файл | Призначення |
|------|-------------|
| `/etc/passwd` | Інформація про облікові записи (ім'я, UID, GID, домашній каталог, shell). |
| `/etc/shadow` | Зашифровані паролі (доступний лише для root). |
| `/etc/group` | Інформація про групи та їх членів. |

**Формат рядка у `/etc/passwd`:**

```
ім'я:пароль:UID:GID:коментар:домашній_каталог:shell
```

Приклад:

```
student1:x:1001:1001:Student One:/home/student1:/bin/bash
```

| Поле | Значення | Пояснення |
|------|----------|-----------|
| `student1` | Ім'я користувача | Логін для входу в систему |
| `x` | Пароль | Символ `x` означає, що пароль зберігається у `/etc/shadow` |
| `1001` | UID | Унікальний ідентифікатор користувача |
| `1001` | GID | Ідентифікатор основної групи |
| `Student One` | Коментар (GECOS) | Повне ім'я або опис |
| `/home/student1` | Домашній каталог | Особиста папка користувача |
| `/bin/bash` | Shell | Оболонка командного рядка |

**Група `sudo`** (або `wheel` у деяких дистрибутивах) — це аналог групи Administrators у Windows. Члени цієї групи можуть виконувати команди з правами суперкористувача через команду `sudo`.

## 5. Практичні завдання — Windows

> **Передумова:** Усі завдання цього розділу виконуються на ПК з Windows 10 або Windows 11. Командний рядок (CMD) потрібно запустити **від імені адміністратора**: у пошуку Windows введіть `cmd`, натисніть правою кнопкою миші на «Командний рядок» та оберіть «Запуск від імені адміністратора».

---

### Завдання 1. Переглянути інформацію про поточного користувача

**Крок 1.** Відкрийте командний рядок (CMD) від імені адміністратора.

**Крок 2.** Введіть команду `whoami` та натисніть Enter.

```
C:\Windows\System32> whoami
```

Ця команда виводить ім'я комп'ютера та ім'я поточного користувача у форматі `ІМ'Я_ПК\ім'я_користувача`.

**Крок 3.** Введіть команду `whoami /user` — щоб побачити ще й SID (Security Identifier) вашого облікового запису.

```
C:\Windows\System32> whoami /user
```

**Крок 4.** Введіть команду `net user %username%` — щоб побачити детальну інформацію про ваш обліковий запис.

```
C:\Windows\System32> net user %username%
```

Зверніть увагу на такі поля у виводі:

| Поле | Що означає |
|------|------------|
| User name | Ім'я облікового запису |
| Full Name | Повне ім'я (якщо задано) |
| Account active | Чи активний обліковий запис (Yes / No) |
| Account expires | Дата закінчення дії облікового запису |
| Password last set | Коли востаннє змінювався пароль |
| Local Group Memberships | До яких локальних груп належить користувач |

**Крок 5.** Введіть команду `whoami /groups` — щоб побачити список усіх груп, до яких належить поточний користувач.

```
C:\Windows\System32> whoami /groups
```

**Запишіть у зошит:**
- Ваше ім'я користувача та ім'я комп'ютера.
- SID вашого облікового запису.
- До яких груп ви належите.
- Коли було встановлено пароль.

---

### Завдання 2. Створити нового локального користувача через графічний інтерфейс (Параметри)

**Крок 1.** Відкрийте **Параметри** Windows:
- Натисніть `Win + I`, або
- Натисніть «Пуск» та оберіть значок шестерні.

**Крок 2.** Перейдіть до розділу **Облікові записи** (Accounts).

**Крок 3.** У лівому меню (Windows 10) або на головній сторінці розділу (Windows 11) оберіть **Сім'я та інші користувачі** (Family & other users) або **Інші користувачі** (Other users).

**Крок 4.** Натисніть **Додати обліковий запис** (Add account) або **Додати когось іншого на цей ПК** (Add someone else to this PC).

**Крок 5.** У вікні, що з'явиться, натисніть **У мене немає даних для входу цієї особи** (I don't have this person's sign-in information).

**Крок 6.** Натисніть **Додати користувача без облікового запису Microsoft** (Add a user without a Microsoft account).

**Крок 7.** Заповніть поля:
- **Ім'я користувача:** `TestGUI`
- **Пароль:** `Test@2024`
- **Підказка пароля** або **Контрольні запитання:** оберіть будь-які відповіді.

**Крок 8.** Натисніть **Далі** (Next). Обліковий запис створено.

**Крок 9.** Переконайтесь, що новий користувач `TestGUI` з'явився у списку «Інші користувачі».

**Запишіть у зошит:**
- Послідовність кроків створення користувача через Параметри.
- Який тип облікового запису створюється за замовчуванням (Стандартний чи Адміністратор)?

---

### Завдання 3. Створити користувача через командний рядок (CMD)

**Крок 1.** У командному рядку (від імені адміністратора) введіть команду:

```
net user Student1 P@ssw0rd /add
```

Розберемо синтаксис:

| Частина команди | Значення |
|-----------------|----------|
| `net user` | Утиліта управління обліковими записами |
| `Student1` | Ім'я нового користувача |
| `P@ssw0rd` | Пароль (має відповідати вимогам складності, якщо вони увімкнені) |
| `/add` | Ключ, що вказує на створення нового облікового запису |

**Крок 2.** Ви повинні побачити повідомлення:

```
The command completed successfully.
```

**Крок 3.** Перевірте, що користувач створений, командою:

```
net user Student1
```

Ця команда виведе детальну інформацію про щойно створений обліковий запис.

**Крок 4.** Перегляньте список усіх користувачів системи:

```
net user
```

**Крок 5.** Створіть ще одного користувача з додатковими параметрами:

```
net user Student2 P@ssw0rd2 /add /fullname:"Петренко Іван" /comment:"Тестовий обліковий запис"
```

| Параметр | Значення |
|----------|----------|
| `/fullname:"Петренко Іван"` | Повне ім'я користувача |
| `/comment:"Тестовий обліковий запис"` | Коментар до облікового запису |

**Крок 6.** Перевірте інформацію про Student2:

```
net user Student2
```

**Запишіть у зошит:**
- Повний синтаксис команди `net user ... /add`.
- Повне виведення команди `net user Student1` (усі поля).
- До якої групи автоматично додається новий користувач?

### Завдання 4. Додати користувача до групи Administrators

**Крок 1.** Перегляньте поточних членів групи Administrators:

```
net localgroup Administrators
```

**Запишіть у зошит**, хто наразі входить до цієї групи.

**Крок 2.** Додайте користувача Student1 до групи Administrators:

```
net localgroup Administrators Student1 /add
```

Ви повинні побачити:

```
The command completed successfully.
```

**Крок 3.** Переконайтесь, що Student1 тепер є членом групи Administrators:

```
net localgroup Administrators
```

Знайдіть `Student1` у списку.

**Крок 4.** Перевірте також через інформацію про користувача:

```
net user Student1
```

У рядку `Local Group Memberships` тепер повинні бути вказані дві групи: `*Administrators` та `*Users`.

**Крок 5.** Перегляньте список усіх локальних груп на комп'ютері:

```
net localgroup
```

**Запишіть у зошит:**
- Команду додавання до групи та результат.
- Повний список груп, до яких тепер належить Student1.
- Повний список локальних груп на вашому комп'ютері (основні 5–7).

### Завдання 5. Використання оснастки lusrmgr.msc — перегляд користувачів та груп

> **Увага:** Оснастка lusrmgr.msc доступна лише у Windows 10/11 редакцій **Pro, Enterprise та Education**. У редакції Home вона недоступна — у такому разі використовуйте CMD-команди з попередніх завдань.

**Крок 1.** Натисніть `Win + R`, введіть `lusrmgr.msc` та натисніть Enter.

**Крок 2.** У вікні, що відкрилося, зліва є два розділи:
- **Користувачі (Users)** — список усіх локальних облікових записів.
- **Групи (Groups)** — список усіх локальних груп.

**Крок 3.** Натисніть на розділ **Користувачі**. Ви побачите всі облікові записи, включаючи:
- Вашого поточного користувача.
- Вбудовані облікові записи: `Administrator` (зазвичай вимкнений — позначений стрілкою вниз), `Guest` (вимкнений), `DefaultAccount`, `WDAGUtilityAccount`.
- Створені вами: `TestGUI`, `Student1`, `Student2`.

**Крок 4.** Подвійним кліком відкрийте властивості облікового запису `Student1`. Перегляньте вкладки:

| Вкладка | Що містить |
|---------|------------|
| **Загальне (General)** | Повне ім'я, опис, чи вимкнений обліковий запис, чи заблокований, параметри пароля. |
| **Членство в групах (Member Of)** | Список груп, до яких належить цей користувач. |
| **Профіль (Profile)** | Шлях до профілю, сценарій входу, домашній каталог. |

**Крок 5.** Перейдіть до розділу **Групи**. Знайдіть групу **Administrators** та подвійним кліком відкрийте її властивості. Ви побачите список членів групи — серед них повинен бути `Student1`.

**Крок 6.** Знайдіть групу **Users** та перегляньте її членів.

**Запишіть у зошит:**

### Завдання 7. Переглянути інформацію про поточного користувача

### Завдання 8. Створити нового користувача в Linux

**Крок 1.** Створіть нового користувача `student1` з домашнім каталогом:

```bash
sudo useradd -m -s /bin/bash student1
```

Розберемо параметри:

| Параметр | Значення |
|----------|----------|
| `sudo` | Виконання з правами суперкористувача |
| `useradd` | Команда створення користувача |
| `-m` | Створити домашній каталог `/home/student1` |
| `-s /bin/bash` | Призначити оболонку Bash |
| `student1` | Ім'я нового користувача |

**Крок 2.** Встановіть пароль для нового користувача:

```bash
sudo passwd student1
```

Система попросить ввести новий пароль двічі. Введіть, наприклад, `Test1234`. Символи пароля **не відображаються** на екрані — це нормальна поведінка Linux.

**Крок 3.** Перевірте, що користувач створений:

```bash
id student1
```

Ви побачите щось на зразок:

```
uid=1001(student1) gid=1001(student1) groups=1001(student1)
```

**Крок 4.** Переконайтесь, що домашній каталог створено:

```bash
ls -la /home/student1
```

**Крок 5.** Перегляньте запис нового користувача у файлі `/etc/passwd`:

```bash
grep student1 /etc/passwd
```

Результат буде подібний до:

```
student1:x:1001:1001::/home/student1:/bin/bash
```

**Запишіть у зошит:**
- Повний синтаксис команди `useradd` з усіма використаними параметрами.
- UID та GID створеного користувача.
- Повний рядок із `/etc/passwd` для student1 та пояснення кожного поля.

### Завдання 9. Додати користувача до групи sudo (sudoers)

**Крок 1.** Перевірте, до яких груп наразі належить `student1`:

```bash
groups student1
```

Результат: `student1 : student1` — лише власна група.

**Крок 2.** Додайте `student1` до групи `sudo`:

```bash
sudo usermod -aG sudo student1
```

Розберемо параметри:

| Параметр | Значення |
|----------|----------|
| `usermod` | Команда модифікації існуючого користувача |
| `-a` | Append — додати до групи, **не видаляючи** з інших груп |
| `-G sudo` | Група, до якої додаємо |
| `student1` | Ім'я користувача |

> **Увага!** Параметр `-a` (append) є критично важливим. Якщо використати лише `-G sudo` без `-a`, користувач буде **видалений з усіх інших додаткових груп** і залишиться тільки в групі `sudo`. Завжди використовуйте `-aG` разом!

**Крок 3.** Перевірте, що `student1` тепер є членом групи `sudo`:

```bash
groups student1
```

Результат: `student1 : student1 sudo`.

**Крок 4.** Перевірте також через `id`:

```bash
id student1
```

У списку `groups` повинна з'явитися група `sudo` з її GID.

**Крок 5.** Перегляньте список членів групи `sudo`:

```bash
grep sudo /etc/group
```

Результат буде подібний до:

```
sudo:x:27:student,student1
```

**Запишіть у зошит:**
- Команду додавання до групи sudo.
- Чому параметр `-a` є обов'язковим при використанні `-G`.
- Як перевірити належність користувача до групи (дві команди: `groups` та `id`).

### Завдання 10. Переглянути файл /etc/passwd та розібрати його формат

**Крок 1.** Перегляньте повний вміст файлу `/etc/passwd`:

```bash
cat /etc/passwd
```

**Крок 2.** Зверніть увагу на різні типи записів:

```
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
...
nobody:x:65534:65534:nobody:/nonexistent:/usr/sbin/nologin
student:x:1000:1000:Student:/home/student:/bin/bash
student1:x:1001:1001::/home/student1:/bin/bash
```

**Крок 3.** Заповніть таблицю у зошиті, розібравши 4 записи:

| Ім'я | Пароль | UID | GID | Коментар | Домашній каталог | Shell |
|------|--------|-----|-----|----------|------------------|-------|
| root | x | 0 | 0 | root | /root | /bin/bash |
| daemon | x | 1 | 1 | daemon | /usr/sbin | /usr/sbin/nologin |
| student | x | 1000 | 1000 | Student | /home/student | /bin/bash |
| student1 | x | 1001 | 1001 | *(порожньо)* | /home/student1 | /bin/bash |

**Крок 4.** Зверніть увагу на відмінності:
- **root** має UID = 0 — це суперкористувач.
- **daemon** має shell `/usr/sbin/nologin` — це системний обліковий запис, від імені якого запускаються служби; вхід у систему для нього заборонений.
- **student** (UID 1000) — перший звичайний користувач, створений при встановленні Linux.
- **student1** (UID 1001) — створений вами щойно.

**Крок 5.** Порахуйте кількість усіх записів у файлі:

```bash
wc -l /etc/passwd
```

**Крок 6.** Порахуйте кількість «реальних» користувачів (з shell bash):

```bash
grep '/bin/bash' /etc/passwd | wc -l
```

**Запишіть у зошит:**
- Таблицю з розбором 4 записів (як у кроці 3).
- Загальну кількість записів у `/etc/passwd`.
- Кількість записів з оболонкою `/bin/bash`.
- Що означає shell `/usr/sbin/nologin` та для чого він використовується.
- Чому пароль зберігається не в `/etc/passwd`, а у `/etc/shadow`.
