## 1. Що таке файлова система — нагадування та поглиблення

### Визначення

**Файлова система (file system)** — це спосіб організації, зберігання та доступу до даних на носії інформації (жорсткому диску, SSD, флешці, SD-карті). Файлова система визначає, як саме файли та папки розміщуються на диску, як записується їхнє ім'я, розмір, дата створення та де фізично на диску знаходяться дані.

### Аналогія для розуміння

Уявіть бібліотеку. Книги — це файли, полиці — це сектори диска, а каталог бібліотеки — це файлова система. Без каталогу ви не зможете знайти потрібну книгу серед тисяч інших. Так само без файлової системи операційна система не зможе знайти потрібний файл на диску.

### Що робить файлова система?

1. **Організовує простір** — розбиває диск на блоки (кластери) фіксованого розміру та відстежує, які з них зайняті, а які — вільні.
2. **Зберігає метадані** — записує інформацію про кожен файл: ім'я, розмір, дату створення та модифікації, атрибути, права доступу.
3. **Забезпечує пошук** — створює структуру каталогів (папок), що дозволяє швидко знаходити файли.
4. **Контролює цілісність** — деякі файлові системи мають механізми захисту від пошкодження даних (журналювання, контрольні суми).

### Що таке кластер?

**Кластер** — це мінімальна одиниця зберігання даних на диску з точки зору файлової системи. Навіть якщо файл займає лише 1 байт, він все одно займатиме на диску цілий кластер.

Типові розміри кластерів:
- **4 КБ** — стандартний розмір для NTFS та багатьох інших ФС.
- **32 КБ** — часто використовується на великих FAT32-розділах.
- **128 КБ** — може використовуватися для exFAT на великих накопичувачах.

> **Важливо!** Чим більший розмір кластера — тим більше простору може витрачатися «даремно» при зберіганні маленьких файлів (це називається **внутрішня фрагментація**). Наприклад, файл розміром 1 КБ на диску з кластером 32 КБ «з'їсть» усі 32 КБ.

### Як файлова система пов'язана з форматуванням?

Коли ви **форматуєте** диск або флешку, ви створюєте на ньому нову файлову систему. При цьому:
- Створюється порожня структура (таблиця розміщення файлів, каталог тощо).
- Усі попередні дані стають недоступними (хоча фізично можуть залишатися на диску до перезапису).
- Ви обираєте тип файлової системи: FAT32, NTFS, exFAT або іншу.

## 2. Сімейство FAT (File Allocation Table)

### Історія та еволюція FAT

Сімейство файлових систем **FAT** — одне з найстаріших і найпоширеніших у світі. Його назва дослівно перекладається як «Таблиця розміщення файлів».

#### FAT12 (1977–1980)

- Створена **Марком Макдональдом** (Marc McDonald) при обговоренні з **Біллом Гейтсом** (Bill Gates) у **1977 році**.
- Спочатку розроблена для **8-дюймових дискет** у рамках Microsoft Standalone Disk BASIC.
- Використовувала **12-бітні** адреси кластерів, що дозволяло адресувати до **4 078 кластерів**.
- Максимальний розмір розділу — **32 МБ**.
- Використовується й досі на стандартних дискетах 1,44 МБ.

#### FAT16 (1984)

- З'явилася у **1984 році** разом із IBM PC AT та MS-DOS 3.0.
- Використовувала **16-бітні** адреси кластерів — до **65 526 кластерів**.
- Максимальний розмір розділу — **2 ГБ**.
- Була основною файловою системою для MS-DOS та ранніх версій Windows.

#### FAT32 (1996)

- Випущена у **1996 році** разом з **Windows 95 OSR2**.
- Використовує **32-бітні** адреси кластерів (фактично 28 біт).
- Стала великим проривом — підтримка значно більших дисків та ефективніше використання простору.

### Як працює таблиця розміщення файлів (FAT Table)

Принцип роботи FAT досить простий і елегантний:

1. **Диск розбивається на кластери** — пронумеровані блоки фіксованого розміру.
2. **Створюється таблиця FAT** — масив, де кожен елемент відповідає одному кластеру на диску.
3. **Кожен елемент таблиці** містить номер наступного кластера, в якому продовжуються дані файлу, або спеціальне значення (наприклад, «кінець файлу» або «вільний кластер»).

```
Приклад: файл «photo.jpg» займає кластери 5, 8, 12

Таблиця FAT:
Кластер 5  → значення: 8    (наступний кластер — 8)
Кластер 8  → значення: 12   (наступний кластер — 12)
Кластер 12 → значення: EOF  (кінець файлу)
```

4. **Кореневий каталог** зберігає інформацію про файли: ім'я, розмір, дату та номер першого кластера.
5. При читанні файлу ОС знаходить перший кластер у каталозі, потім «проходить ланцюжок» за таблицею FAT до кінця файлу.

> **Надійність:** На диску зберігаються **дві копії** таблиці FAT. Якщо одна пошкоджується, ОС може використати другу.

### Обмеження FAT32

| Обмеження | Значення |
|---|---|
| **Максимальний розмір файлу** | **4 ГБ** (4 294 967 295 байт) — це 2^32 − 1 байт |
| **Максимальний розмір розділу** | **2 ТБ** при стандартних секторах 512 байт; до **16 ТБ** при секторах 4 КБ (теоретично) |
| **Максимальна кількість файлів** | ~268 млн (при кластері 32 КБ) |
| **Максимальна довжина імені файлу** | 255 символів (з підтримкою довгих імен — VFAT) |
| **Форматування в Windows** | Windows не дозволяє форматувати розділи > 32 ГБ у FAT32 стандартними засобами |

> **Практичний наслідок обмеження 4 ГБ:** Ви не зможете записати на флешку з FAT32 файл розміром, наприклад, 4,5 ГБ — навіть якщо на флешці є 16 ГБ вільного місця. Це часто трапляється з фільмами високої якості, ISO-образами та великими архівами.

### Де FAT32 використовується сьогодні

Незважаючи на свій вік, FAT32 все ще широко використовується:

- **USB-флешки** ємністю до 32 ГБ — часто відформатовані у FAT32 з виробництва.
- **SD-карти** (до 32 ГБ) — стандарт SD та SDHC використовує FAT32 за замовчуванням.
- **Ігрові консолі** старшого покоління (PlayStation 3, Xbox 360, деякі портативні консолі).
- **Автомобільні мультимедійні системи** — багато з них підтримують лише FAT32.
- **Розділ EFI System Partition (ESP)** — спеціальний завантажувальний розділ на сучасних комп'ютерах з UEFI завжди використовує FAT32.
- **Цифрові фоторамки, MP3-плеєри** та інші прості пристрої.

## 3. NTFS (New Technology File System)

### Історія створення

**NTFS** (New Technology File System) — файлова система, розроблена компанією **Microsoft** для операційної системи **Windows NT**, яка вийшла у **1993 році**. NTFS була створена для подолання обмежень FAT та задоволення потреб професійних і корпоративних користувачів.

Ключові цілі при розробці NTFS:
- Підтримка великих дисків та файлів.
- Надійність та відновлення після збоїв.
- Безпека та розмежування доступу.
- Ефективне використання дискового простору.

Сьогодні NTFS — **основна файлова система для всіх версій Windows** (Windows 10, Windows 11, Windows Server).

### MFT (Master File Table) — основна структура NTFS

На відміну від FAT, де використовується проста таблиця, в NTFS центральною структурою є **MFT (Master File Table)** — «Головна файлова таблиця».

#### Що таке MFT?

MFT — це реляційна база даних, у якій **кожен файл і кожна папка** на диску мають свій запис. Кожен запис MFT має фіксований розмір — **1024 байти (1 КБ)**.

#### Що зберігається в записі MFT?

Кожен запис містить набір **атрибутів**:

| Атрибут | Опис |
|---|---|
| **$STANDARD_INFORMATION** | Часові мітки (створення, модифікація, доступ), атрибути файлу (прихований, системний, лише для читання) |
| **$FILE_NAME** | Ім'я файлу та ім'я батьківського каталогу |
| **$DATA** | Дані файлу (або посилання на кластери, де вони зберігаються) |
| **$INDEX_ROOT / $INDEX_ALLOCATION** | Індекси для каталогів (список файлів у папці) |
| **$SECURITY_DESCRIPTOR** | Права доступу до файлу |

#### Резидентне зберігання

Якщо файл **дуже маленький** (менше ~700 байт), його дані зберігаються **безпосередньо в записі MFT** — це називається **резидентне зберігання**. Це значно пришвидшує доступ до маленьких файлів, бо не потрібно шукати окремі кластери.

#### Дзеркало MFT

NTFS зберігає **резервну копію перших записів MFT** (MFT Mirror) в іншій частині диска. Якщо основна MFT пошкоджується, система може відновити критичну інформацію з дзеркала.

### Можливості NTFS

NTFS має набагато більше можливостей, ніж FAT:

#### 1. Журналювання (Journaling)

**Журналювання** — це механізм захисту цілісності даних. Перед кожною операцією запису NTFS спочатку записує **намір** виконати цю операцію в спеціальний журнал (`$LogFile`). Якщо під час запису стається збій (наприклад, вимкнулося живлення), при наступному завантаженні NTFS перевіряє журнал і:
- **Завершує** незавершені операції (redo), або
- **Скасовує** частково виконані операції (undo).

Це значно знижує ризик пошкодження файлової системи та прискорює відновлення після збоїв — замість перевірки всього диска (як у FAT), NTFS перевіряє лише журнал.

#### 2. Права доступу (ACL — Access Control List)

NTFS підтримує **гнучку систему прав доступу** до файлів і папок. Для кожного файлу можна вказати, який користувач або група має право на:
- **Читання (Read)** — переглядати вміст файлу.
- **Запис (Write)** — змінювати файл.
- **Виконання (Execute)** — запускати програму.
- **Повний контроль (Full Control)** — все вищезазначене + зміна прав.

> **Приклад:** Адміністратор може налаштувати права так, що студенти можуть лише читати файли у спільній папці, але не можуть їх видаляти або змінювати.

#### 3. Шифрування (EFS — Encrypting File System)

NTFS підтримує вбудоване **шифрування файлів і папок** на рівні файлової системи. Зашифровані файли може прочитати лише той користувач, який їх зашифрував (або адміністратор з відповідними правами). Навіть якщо хтось отримає фізичний доступ до диска, він не зможе прочитати зашифровані дані.

#### 4. Стиснення (Compression)

NTFS може **стискати** файли та папки «на льоту» — при записі дані стискаються, при читанні — розпаковуються. Це відбувається прозоро для користувача. Стиснення дозволяє заощадити дисковий простір, але може трохи сповільнити роботу з файлами.

#### 5. Дискові квоти (Disk Quotas)

Адміністратор може встановити **обмеження** на обсяг дискового простору, який кожен користувач може використовувати. Наприклад, у комп'ютерному класі можна обмежити кожного студента 500 МБ.

#### 6. Жорсткі та символічні посилання (Hard Links, Symbolic Links)

- **Жорстке посилання** — кілька імен файлів можуть вказувати на одні й ті самі дані на диску. Це дозволяє мати «один файл під кількома іменами» без дублювання даних.
- **Символічне посилання** — «ярлик» на файл або папку, подібний до ярлика Windows, але працює на рівні файлової системи.

### Технічні обмеження NTFS

| Параметр | Значення |
|---|---|
| **Максимальний розмір файлу** | **16 ЕБ** (екзабайт) теоретично; **256 ТБ** у реалізації Windows 8 і новіших |
| **Максимальний розмір розділу** | **256 ТБ** (з кластером 64 КБ); **16 ТБ** з кластером 4 КБ за замовчуванням |
| **Максимальна довжина імені файлу** | 255 символів Unicode |
| **Максимальна довжина шляху** | 32 767 символів Unicode |

### Недоліки NTFS

- **Обмежена кросплатформність:** macOS може лише читати NTFS-диски, але не записувати на них (без стороннього ПЗ). Linux підтримує читання і запис через драйвер NTFS-3G / ntfs3.
- **Не оптимальна для флешок:** NTFS веде активний журнал, що збільшує кількість операцій запису — це може скоротити термін служби флеш-пам'яті.
- **Складність структури:** NTFS значно складніша за FAT, що ускладнює реалізацію підтримки на простих пристроях.

## 4. exFAT (Extended File Allocation Table)

### Історія та призначення

**exFAT** (Extended File Allocation Table) — файлова система, створена компанією **Microsoft у 2006 році**. Вона була розроблена спеціально як «золота середина» між FAT32 та NTFS:
- **Від FAT32** exFAT успадкувала простоту та широку сумісність.
- **Від NTFS** — підтримку великих файлів.

Головне призначення exFAT — **великі зовнішні накопичувачі**: флешки ємністю понад 32 ГБ та SD-карти формату SDXC (64 ГБ і більше).

### Технічні характеристики exFAT

| Параметр | Значення |
|---|---|
| **Максимальний розмір файлу** | **16 ЕБ** (теоретично) |
| **Максимальний розмір розділу** | **128 ПБ** (петабайт) теоретично; **512 ТБ** — рекомендований максимум |
| **Максимальний розмір кластера** | **32 МБ** |
| **Розрядність адрес кластерів** | **32 біти** (що забезпечує до 4 млрд кластерів) |
| **Максимальна довжина імені файлу** | 255 символів Unicode |

### Особливості exFAT

#### Переваги

- **Немає обмеження 4 ГБ на файл** — головна перевага перед FAT32. Можна зберігати файли будь-якого розміру.
- **Кросплатформність** — підтримується «з коробки» на:
  - **Windows** (XP SP2 і новіші)
  - **macOS** (10.6.5 Snow Leopard і новіші)
  - **Linux** (ядро 5.4+ з 2019 року, повна підтримка в ядрі)
  - **Android**, **iOS**, сучасні ігрові консолі
- **Легковага** — не має складних структур, як MFT в NTFS. Добре підходить для пристроїв з обмеженими ресурсами.
- **Менше зайвих записів на флешку** — на відміну від NTFS, немає журналювання, що зменшує знос флеш-пам'яті.
- **Відкрита специфікація** — у 2019 році Microsoft опублікувала специфікацію exFAT та передала патенти в Open Invention Network, що дозволило вільну реалізацію в Linux.

#### Недоліки

- **Немає журналювання** — якщо під час запису вимкнути живлення або неправильно витягнути флешку, дані можуть пошкодитися. Відновлення складніше, ніж у NTFS.
- **Немає прав доступу** — exFAT не підтримує ACL, шифрування чи дискові квоти. Будь-хто з фізичним доступом до носія може прочитати всі файли.
- **Немає стиснення** — файли зберігаються без стиснення.
- **Не підтримується старими пристроями** — деякі старі ігрові консолі, автомагнітоли та MP3-плеєри не розуміють exFAT.

### Стандарт SD-карт і exFAT

Асоціація SD Card Association визначила:
- **SD та SDHC** (до 32 ГБ) — форматуються у **FAT32**.
- **SDXC** (64 ГБ — 2 ТБ) — форматуються у **exFAT**.
- **SDUC** (до 128 ТБ) — також використовують **exFAT**.

Тому, якщо ви купуєте SD-карту на 64 ГБ або більше, вона майже напевно буде відформатована в exFAT.

## 5. Інші файлові системи (коротко)

Окрім FAT, NTFS та exFAT, існує багато інших файлових систем, створених для різних операційних систем та завдань.

### ext4 (Fourth Extended File System)

- **Операційна система:** Linux.
- **Рік появи:** 2008.
- **Основна ФС** для більшості дистрибутивів Linux (Ubuntu, Fedora, Debian тощо).
- **Можливості:** журналювання, підтримка великих файлів (до 16 ТБ), розділів до 1 ЕБ.
- **Особливості:** висока стабільність, перевірена десятиріччями роботи. Не підтримує знімки (snapshots) і контрольні суми даних.

### APFS (Apple File System)

- **Операційна система:** macOS, iOS, iPadOS, watchOS, tvOS.
- **Рік появи:** 2017 (стала основною ФС macOS з версії 10.13 High Sierra).
- **Розробник:** Apple.
- **Можливості:** шифрування, знімки (snapshots), клонування файлів, технологія Copy-on-Write (CoW), оптимізація для SSD.
- **Особливості:** прийшла на заміну HFS+. Глибоко інтегрована з екосистемою Apple, не підтримується Windows.

### Btrfs (B-Tree File System)

- **Операційна система:** Linux.
- **Рік появи:** 2009 (стабільна версія).
- **Можливості:** Copy-on-Write, знімки, контрольні суми даних і метаданих (захист від «тихого» пошкодження даних), прозоре стиснення, вбудоване управління RAID.
- **Особливості:** є стандартною ФС у Fedora та openSUSE. Вважається «наступником» ext4 для Linux.

### ZFS (Zettabyte File System)

- **Операційна система:** розроблена Sun Microsystems (тепер Oracle) для Solaris; доступна для Linux через OpenZFS та для FreeBSD.
- **Рік появи:** 2005.
- **Можливості:** поєднує файлову систему та менеджер томів. Контрольні суми всіх даних, «самозцілення» при використанні RAID-Z, знімки, клони, дедуплікація, стиснення.
- **Особливості:** вважається найнадійнішою ФС. Потребує багато оперативної пам'яті. Використовується в серверах та системах зберігання даних.

## 6. Порівняльна таблиця FAT32 vs NTFS vs exFAT

| Критерій | FAT32 | NTFS | exFAT |
|---|---|---|---|
| **Рік створення** | 1996 | 1993 | 2006 |
| **Максимальний розмір файлу** | 4 ГБ | 256 ТБ (Windows 8+) | 16 ЕБ (теоретично) |
| **Максимальний розмір розділу** | 2 ТБ (8 ТБ з великими секторами) | 256 ТБ | 128 ПБ (теоретично) |
| **Журналювання** | Ні | Так | Ні |
| **Права доступу (ACL)** | Ні | Так (повноцінні) | Ні |
| **Шифрування (EFS)** | Ні | Так | Ні |
| **Стиснення файлів** | Ні | Так | Ні |
| **Дискові квоти** | Ні | Так | Ні |
| **Сумісність з Windows** | Всі версії | Windows NT і новіші | Windows XP SP2 і новіші |
| **Сумісність з macOS** | Читання + запис | Тільки читання (без стороннього ПЗ) | Читання + запис |
| **Сумісність з Linux** | Читання + запис | Читання + запис (через ntfs3/NTFS-3G) | Читання + запис (ядро 5.4+) |
| **Оптимальність для флешок** | Так (але обмеження 4 ГБ) | Ні (зайвий знос від журналу) | Так (найкращий вибір) |
| **Надійність при збоях** | Низька | Висока (журнал) | Середня |

## 7. Яку файлову систему обрати? — практичні рекомендації

### Покрокове рішення

```
Що ви форматуєте?
│
├── Внутрішній диск Windows (HDD/SSD) ──────────► NTFS
│
├── Зовнішній жорсткий диск
│   ├── Лише для Windows ───────────────────────► NTFS
│   └── Для Windows + Mac + Linux ──────────────► exFAT
│
├── USB-флешка
│   ├── Файли більше 4 ГБ? ─── Так ────────────► exFAT
│   ├── Файли більше 4 ГБ? ─── Ні
│   │   └── Потрібна максимальна сумісність? ──► FAT32
│   │   └── Сучасні пристрої? ─────────────────► exFAT
│
├── SD-карта
│   ├── До 32 ГБ (SD/SDHC) ────────────────────► FAT32
│   └── 64 ГБ і більше (SDXC) ─────────────────► exFAT
│
└── Диск для Linux ─────────────────────────────► ext4
```

### Практичні приклади

| Ситуація | Рекомендована ФС | Чому |
|---|---|---|
| Системний диск Windows 10/11 | **NTFS** | Обов'язкова вимога Windows |
| Флешка 16 ГБ для обміну документами | **FAT32** | Максимальна сумісність, документи < 4 ГБ |
| Флешка 64 ГБ для фільмів та ISO-образів | **exFAT** | Файли можуть бути > 4 ГБ, працює скрізь |
| Зовнішній HDD 1 ТБ для резервних копій Windows | **NTFS** | Права доступу, журналювання, великі файли |
| SD-карта 128 ГБ для камери/дрона | **exFAT** | Стандарт SDXC, великі відеофайли |
| Зовнішній SSD для роботи на Mac і Windows | **exFAT** | Повна підтримка обох ОС |
| Диск для встановлення Ubuntu | **ext4** | Стандартна ФС Linux |

## 8. Як дізнатися файлову систему диска або флешки

### Спосіб 1: Провідник Windows (найпростіший)

1. Відкрийте **Провідник** (Файловий менеджер) — натисніть `Win + E`.
2. Перейдіть до **«Цей ПК»** (This PC).
3. Натисніть **правою кнопкою миші** на потрібний диск або флешку.
4. Виберіть **«Властивості»** (Properties).
5. На вкладці **«Загальне»** (General) побачите рядок **«Файлова система:»** — там буде вказано FAT32, NTFS або exFAT.

### Спосіб 2: Командний рядок (CMD)

Відкрийте командний рядок і введіть:

```
wmic logicaldisk get name, filesystem
```

Результат буде виглядати приблизно так:

```
FileSystem    Name
NTFS          C:
exFAT         D:
FAT32         E:
```

### Спосіб 3: PowerShell

Відкрийте PowerShell і введіть:

```powershell
Get-Volume | Format-Table DriveLetter, FileSystemType, Size
```

### Спосіб 4: Керування дисками (Disk Management)

1. Натисніть `Win + X` і виберіть **«Керування дисками»** (Disk Management).
2. У списку розділів побачите стовпець **«Файлова система»** для кожного розділу.

### Спосіб 5: На macOS

Відкрийте програму **«Дискова утиліта»** (Disk Utility) через Spotlight (`Cmd + Space` → «Disk Utility»). Виберіть диск або розділ — файлова система буде вказана в інформаційній панелі.

### Спосіб 6: На Linux

Відкрийте термінал і введіть:

```bash
lsblk -f
```

Команда покаже всі підключені диски та їхні файлові системи.

## 10. Глосарій

| Термін | Визначення |
|---|---|
| **Файлова система (File System)** | Спосіб організації, зберігання та доступу до даних на носії інформації, що визначає структуру розміщення файлів, каталогів та метаданих на диску |
| **Кластер (Cluster)** | Мінімальна одиниця зберігання даних на диску з точки зору файлової системи; група суміжних секторів, яка виділяється як єдиний блок |
| **FAT (File Allocation Table)** | Таблиця розміщення файлів — структура даних, яка зберігає інформацію про те, які кластери належать кожному файлу та в якому порядку вони з'єднані |
| **FAT12 / FAT16 / FAT32** | Версії файлової системи FAT, що відрізняються розрядністю адрес кластерів (12, 16 та 32 біти відповідно), а отже — максимальним розміром розділу та файлу |
| **NTFS (New Technology File System)** | Журнальована файлова система Microsoft, стандартна для Windows. Підтримує права доступу, шифрування, стиснення, дискові квоти та великі файли |
| **MFT (Master File Table)** | Головна файлова таблиця NTFS — центральна база даних, що містить записи (по 1 КБ кожен) з метаданими про кожен файл і каталог на розділі |
| **Журналювання (Journaling)** | Механізм файлової системи, що записує намір виконати операцію у спеціальний журнал перед її виконанням, забезпечуючи можливість відновлення після збоїв |
| **ACL (Access Control List)** | Список контролю доступу — механізм NTFS, що дозволяє задавати права (читання, запис, виконання) для кожного файлу окремо для різних користувачів та груп |
| **EFS (Encrypting File System)** | Система шифрування файлів, вбудована в NTFS, що дозволяє шифрувати файли та папки для захисту від несанкціонованого доступу |
| **exFAT (Extended FAT)** | Розширена версія FAT, створена Microsoft у 2006 році для великих зовнішніх накопичувачів. Знімає обмеження FAT32 на розмір файлу, зберігаючи кросплатформну сумісність |
| **Форматування (Formatting)** | Процес створення файлової системи на носії інформації. При форматуванні створюється порожня структура ФС, а попередні дані стають недоступними |
| **Внутрішня фрагментація** | Втрата дискового простору через те, що файл, менший за розмір кластера, все одно займає цілий кластер |
| **Резидентне зберігання** | Особливість NTFS, при якій дані дуже маленьких файлів зберігаються безпосередньо в записі MFT, без виділення окремих кластерів |
| **ext4 (Fourth Extended FS)** | Стандартна файлова система Linux з журналюванням. Підтримує файли до 16 ТБ та розділи до 1 ЕБ |
| **APFS (Apple File System)** | Файлова система Apple, що замінила HFS+. Оптимізована для SSD, підтримує шифрування, знімки та технологію Copy-on-Write |
| **Btrfs (B-Tree File System)** | Сучасна файлова система Linux з підтримкою знімків, контрольних сум, стиснення та вбудованого RAID |
| **ZFS (Zettabyte File System)** | Високонадійна файлова система, що поєднує ФС і менеджер томів. Підтримує контрольні суми, самозцілення, знімки та дедуплікацію |
| **Copy-on-Write (CoW)** | Технологія, при якій замість перезапису даних на місці створюється нова копія зміненого блоку, а старий блок зберігається до звільнення. Забезпечує атомарність операцій |
| **SDXC (Secure Digital eXtended Capacity)** | Стандарт SD-карт ємністю від 64 ГБ до 2 ТБ, що використовує файлову систему exFAT за замовчуванням |
| **EFI System Partition (ESP)** | Спеціальний завантажувальний розділ на сучасних комп'ютерах з UEFI, який завжди форматується у FAT32 |
